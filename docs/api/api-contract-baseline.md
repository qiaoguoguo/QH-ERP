# 接口契约基线

## 目标

统一前后端接口协作口径，减少联调返工。

## 基础约定

- 接口路径应体现业务资源和操作含义。
- 请求参数、响应字段、错误码和分页格式必须文档化。
- 时间、金额、数量、单位、状态、字典值必须统一格式。
- 后端错误应返回可定位的问题信息，前端负责转成用户可理解提示。

## 统一成功响应

非分页接口返回结构：

```json
{
  "success": true,
  "code": "OK",
  "message": "成功",
  "data": {},
  "traceId": "请求追踪标识",
  "timestamp": "2026-07-02T10:00:00+08:00"
}
```

约定：

- `success` 表示业务处理是否成功。
- `code` 为稳定机器码，前端不得依赖中文文案判断逻辑。
- `message` 为可读提示，可由前端按场景转换。
- `data` 承载业务数据；无业务数据时返回空对象或 `null`，由接口文档明确。
- `traceId` 用于定位日志。
- `timestamp` 使用明确时区的时间格式。

## 统一错误响应

错误接口返回结构：

```json
{
  "success": false,
  "code": "AUTH_UNAUTHORIZED",
  "message": "未登录或登录已失效",
  "details": [],
  "traceId": "请求追踪标识",
  "timestamp": "2026-07-02T10:00:00+08:00"
}
```

约定：

- 认证失败、权限不足、参数错误、资源不存在、业务冲突和系统异常必须使用不同错误码。
- `details` 用于返回字段级校验或批量错误信息。
- 前端必须根据错误类型给出明确反馈，不能只显示空白页或静默失败。

## 分页响应

分页接口的 `data` 结构：

```json
{
  "items": [],
  "page": 1,
  "pageSize": 20,
  "total": 0,
  "totalPages": 0
}
```

约定：

- `page` 从 1 开始。
- `pageSize` 由接口限制最大值。
- 空页返回空 `items`，不返回错误。
- 排序字段和排序方向必须进入接口文档，后端只允许白名单字段排序。

## 错误码分段

| 分类 | 示例 | 含义 |
|---|---|---|
| 通用 | `OK` | 成功 |
| 认证 | `AUTH_UNAUTHORIZED` | 未登录或登录失效 |
| 权限 | `AUTH_FORBIDDEN` | 已登录但无权限 |
| 参数 | `VALIDATION_ERROR` | 请求参数或字段校验失败 |
| 数据 | `DATA_NOT_FOUND` | 数据不存在 |
| 冲突 | `BUSINESS_CONFLICT` | 业务状态或唯一性冲突 |
| 系统 | `SYSTEM_ERROR` | 非预期系统异常 |

具体模块错误码在模块任务文档中扩展，不能破坏上述分类。

## 字段格式

- 时间：接口使用带时区的 ISO 8601 格式。
- 金额：接口使用字符串或定点数值口径，禁止浮点误差进入业务计算。
- 数量：按模块明确精度，后端负责校验。
- 枚举：以后端定义为准，OpenAPI 必须暴露枚举值和含义。
- 布尔值：只用于真实二值状态，不用于表达复杂业务流转。

## OpenAPI 要求

- 每个接口必须包含请求参数、响应字段、正常示例、异常示例和权限要求。
- 账号权限模块开始前，OpenAPI 至少覆盖健康检查、登录、退出、当前用户、用户列表、角色列表和权限校验相关接口。
- 前后端联调以 OpenAPI 和任务文档为准，不以临时口头约定为准。

## 联调验收

每个接口应至少具备：

- 字段说明。
- 正常示例。
- 异常示例。
- 权限要求。
- 关联页面或业务流程。
