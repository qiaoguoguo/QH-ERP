# 接口契约基线

## 目标

统一前后端接口协作口径，减少联调返工。

## 基础约定

- 接口路径应体现业务资源和操作含义。
- 请求参数、响应字段、错误码和分页格式必须文档化。
- 时间、金额、数量、单位、状态、字典值必须统一格式。
- 后端错误应返回可定位的问题信息，前端负责转成用户可理解提示。

## 响应结构后续确认项

后续技术选型时需要确认：

- 成功响应结构。
- 失败响应结构。
- 分页响应结构。
- 字典和枚举返回方式。
- 文件上传和下载方式。

## 联调验收

每个接口应至少具备：

- 字段说明。
- 正常示例。
- 异常示例。
- 权限要求。
- 关联页面或业务流程。
