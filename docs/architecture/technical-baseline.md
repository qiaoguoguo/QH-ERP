# 技术基线

## 目标

在正式编码前建立技术方案评审框架。本文件记录技术关注点和质量门槛，具体选型结论由技术选型决策记录承载。

当前技术选型结论见：[技术选型决策记录](technology-decision.md)。

## 架构关注点

- 单公司 Web ERP，优先保证维护性、数据可信度和部署可控。
- 前后端职责边界清晰。
- 核心业务逻辑以后端可信数据和事务为准。
- 库存、生产、成本、财务相关操作必须考虑一致性和审计。

## 工程初始化评审事项

- 前端框架、组件库和状态管理方案。
- 后端框架、认证鉴权和接口风格。
- 数据库类型、迁移方案和备份恢复方案。
- 本地开发、测试和阶段验收部署方式。
- 日志、异常、监控和审计策略。

## 质量门槛

- 有清晰目录结构。
- 有统一接口契约。
- 有可重复启动和部署方式。
- 有基本测试和验收命令。
- 有本地浏览器验收路径。
