# 核心数据口径

## 目标

统一核心业务对象的含义，避免产品、前端、后端、测试使用不同口径。

主数据、编码、状态、数量、金额、时间和初始化数据规则见：[主数据与业务规则基线](master-data-rules.md)。

## 核心对象

- 物料：生产和库存管理中的基础对象，包括原材料、半成品、成品、辅料等。
- BOM：物料清单，描述一个产出物料需要哪些子项物料、数量、损耗和版本。
- 仓库：库存存放地点，可后续扩展库区、库位。
- 库存：某物料在仓库、质量状态、批次/序列、所有权和项目维度下的数量、可用性与估值状态汇总，不是来源事实本身。
- 库存变动：任何导致库存数量变化的不可变业务记录；023 后与对应价值流水同事务形成。
- 销售项目：项目型销售的一级管理对象，承载项目编号、客户、负责人、计划周期、目标收入、目标成本和状态；当前已贯穿合同、报价、销售订单、交付计划、项目库存所有权/来源成本层和销售履约关闭，但不代表已经形成正式收入确认、正式项目成本或项目利润。
- 销售合同：销售项目下的商业事实，分为主合同和补充合同。一个项目最多有一个非取消主合同，可以有多个引用该主合同的补充合同。
- 销售报价：客户商业意向事实，审批通过且在有效期内时可幂等转换一次为合同草稿或销售订单草稿，并保留头、行来源。
- 交付计划：销售订单行按承诺日期拆分的履约计划，保存计划数量、已发数量、状态和出库来源，不等同实际出库。
- 销售订单变更：确认后对未交付数量、税价或交期的受控变更事实，保存变更前后快照并经固定审批应用，不覆盖已过账出库。
- 商业信用档案：客户层面的信用额度、冻结、逾期阻断和复核资料；信用占用读取订单承诺、待建应收出库和基础应收未收三段业务事实，不是正式财务授信或会计应收。
- 有效销售需求：供后续缺料与供给建议只读消费的开放订单/交付计划需求结果，包含来源、数量、日期、状态和稳定的不计入原因，本身不执行净算或生成建议。
- 物料需求运行：026 对一个确定范围执行一次订单驱动净算形成的运行事实，保存请求键、范围、状态、来源指纹、过期状态和版本；重算产生新运行，不覆盖旧运行。
- 物料需求行：运行快照中的需求结果，保存销售来源、项目/公共范围、成品、BOM 快照、需求物料、基本单位数量、需求日期、覆盖量、短缺量、预计可用日期和稳定异常，不等同库存预留或生产任务。
- 供给分配：某一需求行对合格库存、有效采购供给或既有工单供给的只读覆盖明细，保存来源、所有权/项目边界、分配数量、可用日期和排除原因；不会修改被读取的库存、采购或工单事实。
- 缺料建议：由短缺快照派生的采购请购、生产、外协、公共库存使用或既有供给使用建议。建议可确认或驳回；已确认采购建议可幂等转为 024 请购草稿，已确认且来源仍有效的生产/外协建议可在 027 分别幂等转为一个生产工单或外协订单草稿。
- 替代料提示：按 021 受控替代关系生成的只读说明，不参与自动净算抵扣，不自动改变领料、库存或建议数量。
- 生产工单：内部生产任务的核心单据，关联产出物料、数量、BOM、项目/公共所有权和状态；027 起可保留 026 建议与运行来源，历史工单默认公共且项目为空。
- 领料单：生产工单消耗物料的出库依据。
- 报工记录：生产执行过程和产出情况记录。
- 完工入库单：生产完成后成品或半成品入库依据。
- 外协订单：委托供应商完成制造或加工的独立执行对象，保存项目/公共归属、产出物料、数量、供应商、来源建议和状态；不是采购订单、内部工单、对账单或应付单。
- 外协发料单：按外协订单把材料发给供应商的库存出库事实，保留项目/公共所有权、成本层和批次/序列来源。
- 外协收货单：按外协订单接收外协产出的库存入库事实，可形成项目或公共库存及暂估价值状态；不等同采购入库、供应商发票或正式外协成本。
- 成本归集：围绕工单或产品归集材料、人工、制造费用等成本信息。
- 公共计价池：单公司下按物料汇总的公共库存数量、金额和移动平均价，不按仓库、质量、批次或序列分别定价。
- 项目成本层：按项目、物料和真实来源保存原始/剩余数量金额、来源单据和父层关系的项目库存价值事实，不等同正式项目成本核算。
- 价值流水：库存数量动作对应的不可变金额变化，记录估值方法、单位成本、金额、前后余额和业务来源。
- 固定审批实例：022 预定义业务场景的一次审批事实，审批状态与业务单据状态分开，不是通用工作流。
- 文档任务：导入、导出或固定打印的异步执行事实，包含租约、进度、错误、结果、过期和重新鉴权。

## 销售项目与合同口径

- `projectId`：销售项目的内部标识，不等同项目编号。销售订单可选保存；023 库存余额、流水和项目成本层可显式引用，024 项目专采、027 项目工单和外协执行继续沿用同一标识。历史生产、原有成本和财务数据允许为空，不得按客户、日期或物料推导回填。
- 项目状态：`DRAFT`（草稿）、`ACTIVE`（已激活）、`CLOSED`（正常关闭）、`CANCELLED`（草稿取消）。
- 合同类型：`MAIN` 表示主合同，`SUPPLEMENT` 表示补充合同；补充合同必须引用同项目主合同。
- 合同状态：`DRAFT`（草稿）、`EFFECTIVE`（已生效）、`CLOSED`（正常履约结束）、`TERMINATED`（提前终止）、`CANCELLED`（草稿取消）。
- 商业金额：合同层面的商业约定金额。主合同金额必须大于 0；补充合同调整金额可正可负但不得为 0。该金额不拆税，不作为收入确认、开票、回款、实际成本或利润口径。
- 目标收入、目标成本：销售项目上的非负管理目标，用于计划与管理展示，不等同实际收入、合同商业金额、实际成本或财务核算结果。
- 销售订单项目合同关联：`projectId` 与 `contractId` 同时为空表示不关联，同时有值表示订单归属该项目及其合同。历史订单允许保持空关联。
- 价格来源：直接新建订单行为 `MANUAL`；报价转单行为 `QUOTE` 并保留 `sourceQuoteLineId`。只调整履约字段不得丢失报价来源，修改报价来源行的物料、单位、价格、税率或币种必须拒绝。
- 销售商业快照：订单确认时冻结客户结算、项目合同、报价来源、税价、币种和信用检查引用；已过账出库行再保存自己的税价与来源快照，后续订单变更不得反向改写。
- 销售预留：订单确认按订单行预留仓库形成 `SALES_ORDER`/`RESERVATION` 库存预留；出库过账消耗，取消、关闭或减量释放，增量变更补足。
- 项目销售履约关闭：项目的销售履约终态检查，要求关联报价、订单、计划和变更没有开放事项；它是项目正式 `CLOSED` 前置条件，但不替代项目状态。

## 项目生产与外协口径

- 工单所有权：`PROJECT` 工单必须有 `projectId`，`PUBLIC` 工单必须为空；同一工单的领退补料、报工和完工继承该边界，不能从其他项目无控制取料。
- 规划来源：由 026 建议转换的工单或外协订单保存运行、建议和需求来源；同一建议只能对应一个目标。手工工单允许没有规划来源，但不能伪造建议引用。
- 项目完工：项目工单完工入库进入同项目库存；公共工单进入公共库存。没有可靠单位成本时使用明确暂估或未估值状态，不能把未知成本写成零。
- 生产退料：必须引用原已过账领料并回到原所有权、项目、成本层和追踪来源，不按退料时公共平均价重新估值。
- 外协状态：订单以草稿、已发布、执行中、已完成/已关闭或已取消表达生命周期；发料和收货各自具有草稿、已过账、已取消等单据状态，订单状态不能替代执行单据状态。
- 当前边界：027 业务事实可供后续成本归集使用，但不等同在制成本、完工成本、制造费用、外协结算、项目毛利或会计凭证。

## 数量与金额原则

- 数量、单位、金额和成本字段必须定义精度。
- 不同单位之间的换算必须有明确来源。
- 成本和库存金额不得只以前端展示值作为依据。
- 数量、单价和金额 API 使用十进制字符串；公共平均价、项目层金额和尾差由后端统一计算。
- 公共库存移动平均与项目实际来源层是两套不同计价口径，不得互相借用或合并。
- 仓库调拨不改变企业总价值；估值调整只改变当前剩余库存金额，不回写历史出库。

## 状态原则

核心单据必须具备明确状态，不允许仅依赖备注、布尔字段或前端临时判断表达业务流转。
