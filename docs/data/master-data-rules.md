# 主数据与业务规则基线

## 目标

统一一期核心主数据、状态、编码、数量、金额、时间和初始化数据口径，为账号权限、物料、BOM、库存和生产模块提供共同语言。

## 一期优先主数据

### 账号权限

- 用户。
- 角色。
- 权限点。
- 菜单。
- 按钮或操作。

### 基础资料

- 计量单位。
- 仓库。
- 供应商。
- 客户。
- 物料分类。
- 物料。

### 生产主链路

- BOM。
- BOM 版本。
- 生产工单。
- 领料单。
- 报工记录。
- 完工入库单。
- 成本记录。

## 编码规则

一期必须先定义以下编码，具体格式在对应模块任务文档中落定：

- 用户账号：企业内部唯一。
- 角色编码：系统内唯一，建议稳定不随名称变更。
- 权限编码：按模块和动作分段，例如账号、角色、物料、BOM、工单等。
- 物料编码：系统内唯一，不依赖物料名称。
- 仓库编码：系统内唯一。
- BOM 版本：同一产出物料下唯一。
- 业务单据编号：按单据类型和生成规则唯一。

编码一旦被业务数据引用，不得通过简单改名破坏历史追溯。

## 状态规则

核心状态必须由后端定义，前端只展示和提交允许操作。

一期优先统一：

- 用户状态：启用、停用。
- 角色状态：启用、停用。
- 物料状态：启用、停用。
- BOM 状态：草稿、启用、停用。
- 工单状态：草稿、已下达、生产中、已完工、已关闭。
- 领料状态：草稿、已出库、已作废。
- 报工状态：已提交、已作废。
- 完工入库状态：草稿、已入库、已作废。
- 成本记录状态：已记录、已作废。

## 数量、金额和时间

- 数量和金额禁止使用浮点数承载业务计算。
- 数量精度、金额精度和舍入规则必须在模块任务文档中明确。
- 计量单位换算必须有来源，不允许前端临时换算作为业务依据。
- 接口时间格式必须统一，并明确时区口径。
- 创建时间、更新时间和关键操作时间以后端生成或校验为准。

## 数据追溯

库存、生产、成本和财务相关数据必须保留来源：

- 来源单据类型。
- 来源单据编号或标识。
- 操作人。
- 操作时间。
- 业务状态。
- 必要的变更前后信息。

## 数据库迁移与初始化

- 数据库结构必须由 Flyway 迁移脚本管理。
- 初始化数据也应纳入可重复执行的机制。
- 超级管理员、基础角色、基础权限和基础菜单必须幂等初始化。
- 本地空 PostgreSQL 数据库必须能通过迁移和初始化得到可登录、可验收的基础环境。
- 禁止把手工改库作为阶段成果交付依据。
