# 前端工程基线

## 目标

建立前端开发前的协作和质量基线，后续技术选型完成后再补充具体框架命令。

## 页面类型

- 登录页。
- 首页或工作台。
- 列表页。
- 详情页。
- 表单页。
- 弹窗与确认操作。
- 导入导出页面。
- 报表与图表页面。

## 通用交互

- 加载状态。
- 空状态。
- 成功提示。
- 错误提示。
- 表单校验。
- 删除、作废、提交等危险操作确认。
- 权限不足提示。

## 业务展示规则

物料编码、BOM 版本、库存数量、单位、金额、工单状态、领料状态、报工结果等展示格式必须统一。

## 模块开发规范

- 每个业务模块必须按页面、路由、接口、状态管理和测试拆分，不把多个模块混入同一页面文件。
- 路由命名、菜单标识、权限编码和后端权限点必须可对应。
- 列表页必须统一查询、重置、分页、加载、空状态、错误状态和行操作。
- 表单页必须统一必填、长度、格式、重复提交、保存中禁用和提交失败反馈。
- 详情页必须统一基础信息、状态信息、关联明细和操作记录。
- 危险操作必须有确认，不得只依赖按钮文案提示。

## 权限表现

- 前端负责菜单、路由、按钮和字段表现，但不能作为最终安全边界。
- 无权限菜单默认不可见。
- 无权限按钮可按场景不可见或禁用；禁用时必须能说明原因。
- 直接访问无权限路由必须进入无权限提示或被重定向。
- 后端返回无权限错误时，前端必须给出明确反馈。

## 接口联调要求

- 所有请求必须使用统一请求封装。
- 登录失效、权限不足、参数错误、业务冲突、网络异常必须有统一处理规则。
- 分页参数和分页响应必须符合接口契约。
- 枚举和字典以后端返回或共享契约为准，前端不得自行扩展业务枚举。

## 基础验证命令

当前前端工程使用 npm，基础命令以 `apps/web/package.json` 为准：

- `npm test`
- `npm run typecheck`
- `npm run build`

后续引入端到端测试后，必须补充浏览器验收命令。

## 验收要求

页面不能只以“能打开”为完成标准，必须同时验证数据展示、交互状态、权限表现、异常反馈和浏览器实际操作路径。
