## 项目定位

QH ERP 是面向制造业单公司内部使用的 Web ERP 系统，不是零售、门店收银、电商、会员营销、轻量通用进销存或多租户平台。

长期方向覆盖基础资料、采购、销售、库存、生产、财务、报表等常规 ERP 能力。当前产品重心是制造业生产管理，尤其重视物料、BOM、库存、生产工单、领料、报工、完工入库、成本归集、采购入库、销售出库、财务往来、经营报表、质量状态、库存可用量和批次/序列追溯。

## 当前核心业务链路

生产主链路：

```text
基础资料 -> 物料 -> BOM -> 库存 -> 生产工单 -> 领料 -> 报工 -> 完工入库 -> 成本归集
```

采购链路：

```text
供应商 -> 采购订单 -> 采购入库 -> 库存增加 -> 应付台账 -> 付款记录 -> 来源追溯
```

销售链路：

```text
客户 -> 销售订单 -> 销售出库 -> 库存扣减 -> 应收台账 -> 收款记录 -> 来源追溯
```

质量库存状态链路：

```text
采购入库、生产完工、销售退货、生产退料 -> 默认待检库存 -> 质量确认 -> 合格、不合格、冻结库存
合格库存 -> 销售出库、生产领料、生产补料可用
待检、不合格、冻结库存 -> 不参与普通出库和领料可用量
冻结库存 -> 解冻后才能参与后续允许动作
```

库存可用量与预留链路：

```text
合格现存 -> 销售订单预留仓库确认 -> 销售预留 -> 销售出库按同仓消费预留 -> 合格库存扣减
合格现存 -> 生产工单发布 -> BOM 用料预留 -> 生产领料按工单领料仓库消费预留 -> 合格库存扣减
采购订单确认/部分入库 -> 物料级采购在途参考 -> 采购入库过账 -> 具体仓库待检库存
仓库级合格现存 - 占用 - 预留 -> 现货净可用/可承诺量
```

019 追踪链路：

```text
物料追踪方式 -> 采购入库/完工入库追踪分配 -> 批次/序列库存底账
销售出库/生产领料/采购退货/生产补料 -> 后端候选追踪库存选择 -> 来源去向串链
销售退货/生产退料 -> 继承原出库/领料追踪身份 -> 反向追溯
质量确认/冻结/解冻 -> 保留批次/序列身份 -> 库存余额/流水/追溯抽屉可查
```
