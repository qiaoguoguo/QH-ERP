## 项目定位与优先级调整

QH ERP 仍是一套面向制造业单公司内部使用的 Web ERP，不是零售、门店收银、电商、通用进销存或多租户平台。

长期仍覆盖基础资料、采购、销售、库存、生产、成本、财务和报表。但结合用户实际业务，本轮对第一版优先级作出调整：

- 主要产品是配电柜等项目型、大件、低频次制造产品。
- 第一版不需要工艺路线、工作中心、精细工序流转、复杂派工和产能排程。
- 第一版不需要完整 QMS、复杂抽样、SPC 或精细供应商质量分析。
- 第一版更重视销售项目收入、项目专采、公共库存领料、项目实际成本、在制完工结转、发票、往来、月结和正式财务。
- 生产模块保持工单、BOM 快照、领退补料、简单报工、完工入库、外协和序列追溯，作为项目成本的可信业务来源。

该调整不是放弃制造业定位，而是把第一版从“精细制造执行优先”调整为“项目型制造的业财成本闭环优先”。

## 已确认的产品决策

### 一级成本和利润对象

- 销售项目是一级成本、收入和利润对象。
- 一个销售项目允许包含一个主合同、多个补充合同和多次追加订单。
- 合同负责商业收入、开票、回款和变更归属。
- 销售订单、采购、生产工单、费用、发票、收付款和成本都应能归集到项目。
- 生产工单是成本归集明细对象，但不是一级利润对象。
- 柜体序列号用于单台追溯和必要的成本下钻，不作为第一版一级核算对象。

目标关系：

```text
销售项目
  -> 主合同 / 补充合同
  -> 销售订单
  -> 项目专采 + 公共库存领料
  -> 生产工单 / 外协 / 人工 / 费用
  -> 在制成本
  -> 完工成本
  -> 发货 / 开票 / 回款
  -> 项目收入 / 成本 / 毛利 / 现金回收分析
```

### 材料来源

- 项目专门采购与公共库存领用并存。
- 项目专采库存必须保留项目归属，不得无控制地被其他项目消耗。
- 公共库存可以按受控领料分配给不同项目。
- 项目专属库存转为公共库存或转给其他项目必须通过受控业务动作，并保留原因、审批和价值变化追溯。

### 库存计价

- 项目专采材料按项目和实际采购批次成本计价。
- 公共库存按移动加权平均计价。
- 退料应回到原项目归属和原成本口径，不能无依据地按当前平均价重估。
- 调拨不改变企业总库存价值；项目归属转换必须明确价值承接规则。
- 库存数量、成本层、库存金额和总账之间必须可核对。

### 第一版生产深度

- 保留工单级 BOM、领退补料、简单报工、完工入库、外协和序列追溯。
- 不建设完整主生产计划、有限产能排程和 APS。
- 不建设工艺路线、工作中心、工序派工和工序转移。
- 不建设完整 QMS、复杂抽样、SPC 和复杂供应商质量体系。
- 质量只保留来料或完工结果、让步、返工、报废和附件等成本与库存所需的最低闭环。

- 026 采用订单和项目驱动的轻量缺料分析，不进入完整 MPS、APS 或产能优化。

- 028 先补发票、费用和往来，再由 029 形成完整项目实际成本。
- 030 先固化业务月结和期间快照，再进入 031 至 032 正式财务，避免总账建立在可变化的业务期间之上。
- 033 只交付固定、可追溯的经营财务分析，不建设可配置 BI。
- 034 至 036 完成平台治理、上线准备和最终交付，不允许用最终验收代替各阶段自己的定向验证。

## 下一阶段 021 的启动输入

`021 成本财务主数据与 BOM 治理`

### 阶段目标

### 建议纳入

- 单位换算、基本单位与业务单位关系、精度和舍入规则。
- 现有 BOM 版本唯一性、有效期、复制、发布、停用和历史有效性治理。

- 替代料关系、优先级、适用范围和用量换算基础。
- 物料成本属性、库存计价分类和后续成本核算需要的稳定枚举。
- 供应商和客户的结算、开票、税务和付款条件等必要资料增强。

- 项目专属库存、移动加权平均、批次实际成本、盘点和盘差，属于 023。
- 请购、询价、价格协议和项目专采，属于 024。
- 销售报价、信用和项目履约深化，属于 025。
- 缺料净算和供给建议，属于 026。
- 工单项目归集、外协和正式项目成本，属于 027 和 029。
- 发票、收入确认、凭证、总账和财务结账，属于 028 至 032。
- 泛化产品配置、复杂多属性平台、工艺路线、精细工序、复杂质量和 APS。

### 021 启动前必须确定

- 工程变更是生成新版本还是允许草稿内修改，以及变更影响对象和追溯字段。

- 编码规则的受控对象、并发唯一性和历史编码兼容。
- 021 的权限、审计、导入影响、异常码和浏览器验收矩阵。

### 021 完成结果
