### 2026-07-13 固定五角色 021 目标讨论结论

五个固定角色一致建议将 021 定位为“成本财务主数据与 BOM 治理”。阶段目标是：在 020 销售项目维度之上，补齐后续库存计价、项目专采、发票往来、项目成本和财务闭环共同依赖的主数据契约，并把现有 BOM 从基础版本字段提升为可治理、可追溯、可历史保护的版本体系。

讨论结果确认 021 必须纳入：单位换算、BOM 版本治理、工程变更记录、替代料关系、物料成本属性、客户/供应商结算税务资料、编码规则、权限、审计和历史兼容。021 明确不纳入：审批平台、消息、附件、打印、库存计价、项目专采、缺料净算、生产工单深化、自动替代领料、发票、凭证、总账、月结、正式项目成本、工艺路线、APS、复杂配置器、多组织多账套和 BI。

关键决策口径：物料基本单位仍是库存与成本数量基准，业务单位必须通过后端换算落到基本单位；已发布 BOM 不直接改明细，变更通过新版本或工程变更承接；已发布工单继续使用当时 BOM 快照；替代料在 021 只维护受控关系，不自动替代业务单据；成本属性只定义后续可复用枚举，不计算成本；编码规则由后端并发唯一控制；新增 `/api/admin` 后端资源必须同步权限种子和权限映射。

固定协作约束：

- 主代理只负责总体规划、范围控制、工作包划分、并行协调、风险判断、审查汇总、交付决策和向用户汇报。

- 每个角色每阶段原则上只接收一个主 `/goal`；后续补充、审查和返工均通过同一会话增量派发。
- 工作包没有固定数量上限，也不受最多 4 个子任务限制，但必须按完整业务能力、所有权和真实依赖划分，避免机械细拆。

## 项目定位与优先级调整

QH ERP 仍是一套面向制造业单公司内部使用的 Web ERP，不是零售、门店收银、电商、通用进销存或多租户平台。

长期仍覆盖基础资料、采购、销售、库存、生产、成本、财务和报表。但结合用户实际业务，本轮对第一版优先级作出调整：

- 主要产品是配电柜等项目型、大件、低频次制造产品。
- 第一版不需要工艺路线、工作中心、精细工序流转、复杂派工和产能排程。
- 第一版不需要完整 QMS、复杂抽样、SPC 或精细供应商质量分析。
- 第一版更重视销售项目收入、项目专采、公共库存领料、项目实际成本、在制完工结转、发票、往来、月结和正式财务。
- 生产模块保持工单、BOM 快照、领退补料、简单报工、完工入库、外协和序列追溯，作为项目成本的可信业务来源。

该调整不是放弃制造业定位，而是把第一版从“精细制造执行优先”调整为“项目型制造的业财成本闭环优先”。

当前没有请购、询价比价、价格协议、项目专采、采购发票或销售发票；项目尚未贯穿采购、库存、生产、费用和正式成本。

### 成本与财务

- 工单关联的材料数量、人工数量、产出数量和手工成本记录。
- 应收、应付、收款、付款、余额核销和往来调整。
- 业务金额和来源追溯。

当前成本只是业务记录，不是库存计价、项目实际成本或正式财务成本。当前往来只是业务台账，不是发票、凭证、总账或财务结账。

## 已确认的产品决策

### 一级成本和利润对象

- 销售项目是一级成本、收入和利润对象。
- 一个销售项目允许包含一个主合同、多个补充合同和多次追加订单。
- 合同负责商业收入、开票、回款和变更归属。
- 销售订单、采购、生产工单、费用、发票、收付款和成本都应能归集到项目。
- 生产工单是成本归集明细对象，但不是一级利润对象。
- 柜体序列号用于单台追溯和必要的成本下钻，不作为第一版一级核算对象。

目标关系：

```text
销售项目
  -> 主合同 / 补充合同
  -> 销售订单
  -> 项目专采 + 公共库存领料
  -> 生产工单 / 外协 / 人工 / 费用
  -> 在制成本
  -> 完工成本
  -> 发货 / 开票 / 回款
  -> 项目收入 / 成本 / 毛利 / 现金回收分析
```

### 材料来源

- 项目专门采购与公共库存领用并存。
- 项目专采库存必须保留项目归属，不得无控制地被其他项目消耗。
- 公共库存可以按受控领料分配给不同项目。
- 项目专属库存转为公共库存或转给其他项目必须通过受控业务动作，并保留原因、审批和价值变化追溯。

### 库存计价

- 项目专采材料按项目和实际采购批次成本计价。
- 公共库存按移动加权平均计价。
- 退料应回到原项目归属和原成本口径，不能无依据地按当前平均价重估。
- 调拨不改变企业总库存价值；项目归属转换必须明确价值承接规则。
- 库存数量、成本层、库存金额和总账之间必须可核对。

### 第一版生产深度

- 保留工单级 BOM、领退补料、简单报工、完工入库、外协和序列追溯。
- 不建设完整主生产计划、有限产能排程和 APS。
- 不建设工艺路线、工作中心、工序派工和工序转移。
- 不建设完整 QMS、复杂抽样、SPC 和复杂供应商质量体系。
- 质量只保留来料或完工结果、让步、返工、报废和附件等成本与库存所需的最低闭环。

## 给下一任负责人

当前系统已经稳定完成到 021：020 提供销售项目与合同主维度，021 提供单位换算、成本财务主数据、编码规则和 BOM/ECO/替代料治理基础。下一步是 022 固定审批与业务单据平台基础，为后续价格、盘差、期间和财务场景建立受控公共能力。

不要直接把路线表当作 022 实现规格，也不要把“公共平台”扩展成通用工作流设计器。应先由固定五角色确定首批业务对象、固定审批状态、待办/消息、附件、打印、导入导出和审计增强的最小闭环，再进入业务实现。任何阶段不得把规划能力写成已交付能力，也不得为了加快财务页面出现而绕过库存计价、项目成本和期间快照。
