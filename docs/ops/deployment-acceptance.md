# 部署与阶段验收

## 目标

保证每次阶段成果都能在本地完成部署或启动，并通过浏览器验收。

## 部署验收清单

- 依赖安装完成。
- 环境配置明确。
- 数据库或初始化数据可用。
- 服务能够启动。
- 浏览器能够访问验收地址。
- 核心页面能够打开。
- 核心接口能够返回预期结果。
- 测试账号可登录。
- 未登录和无权限访问能够返回预期拦截结果。
- 浏览器验收路径能够按文档重复执行。
- 浏览器验收已完成关键页面截图和视觉分析记录。
- 视觉分析记录已经保存到对应的 `docs/testing/<模块标识>-visual-audit/` 目录，并能追溯截图、视口、页面和处理结论。
- 视觉分析未发现影响核心操作、关键数据识别、权限状态判断或工作台扫描效率的阻断问题。

## 验收通知内容

通知用户验收时必须提供：

- 浏览器访问地址。
- 验收模块和范围。
- 已完成内容。
- 已执行验证。
- 视觉分析记录和截图保存位置。
- 视觉分析发现的问题、修复结果和重新截图验证结论。
- 已知问题和风险。
- 建议验收路径。

## 故障处理

如部署、启动或浏览器访问失败，必须优先定位并修复。故障未解决前，不得声称阶段成果可验收。

## 主分支阶段验收前置条件

- 当前任务分支已经完成任务文档列出的功能、测试和部署验收。
- 无阻断缺陷。
- 严重缺陷已有处理结论且不影响本次验收范围。
- 本地服务已启动并提供浏览器访问地址。
- 浏览器功能验收、视觉截图和视觉分析结论已经完成并可追溯。
- 验收通知已说明范围、路径、验证结果、已知问题和建议查验步骤。
