# 部署与阶段验收

## 目标

保证每次阶段成果都能在本地完成部署或启动，并通过浏览器验收。

## 部署验收清单

- 依赖安装完成。
- 环境配置明确。
- 数据库或初始化数据可用。
- 服务能够启动。
- 浏览器能够访问验收地址。
- 核心页面能够打开。
- 核心接口能够返回预期结果。
- 测试账号可登录。

## 验收通知内容

通知用户验收时必须提供：

- 浏览器访问地址。
- 验收模块和范围。
- 已完成内容。
- 已执行验证。
- 已知问题和风险。
- 建议验收路径。

## 故障处理

如部署、启动或浏览器访问失败，必须优先定位并修复。故障未解决前，不得声称阶段成果可验收。
