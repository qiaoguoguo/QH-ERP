# 部署与阶段验收

## 目标

保证每次阶段成果都能在本地完成部署或启动，并通过浏览器验收。

## 部署验收清单

- 依赖安装完成。
- 环境配置明确。
- 数据库或初始化数据可用；需要共享验收库时，已按全量演示数据基线完成备份、受控重建和独立验证。
- 服务能够启动。
- 浏览器能够访问验收地址。
- 核心页面能够打开。
- 核心接口能够返回预期结果。
- 测试账号可登录。
- 未登录和无权限访问能够返回预期拦截结果。
- 浏览器验收路径能够按文档重复执行。
- 浏览器验收已完成本阶段相关页面改动的桌面端视觉检验，不检查未变化页面或移动端适配。
- 视觉检验不要求采集、保存或归档截图，也不要求建立阶段视觉截图目录。
- 视觉检验未发现影响核心操作、关键数据识别、权限状态判断或工作台扫描效率的阻断问题。

## 验收通知内容

通知用户验收时必须提供：

- 浏览器访问地址。
- 验收模块和范围。
- 已完成内容。
- 已执行验证。
- 桌面端视觉检验范围和结论。
- 视觉检验发现的问题、修复结果和受影响路径复验结论。
- 已知问题和风险。
- 建议验收路径。

## 故障处理

如部署、启动或浏览器访问失败，必须优先定位并修复。故障未解决前，不得声称阶段成果可验收。

## 主分支阶段验收前置条件

- 当前任务分支已经完成任务文档列出的功能、测试和部署验收。
- 无阻断缺陷。
- 严重缺陷已有处理结论且不影响本次验收范围。
- 本地服务已启动并提供浏览器访问地址。
- 浏览器功能验收和本阶段相关页面改动的桌面端视觉检验已经完成，检查范围和结论可追溯。
- 验收通知已说明范围、路径、验证结果、已知问题和建议查验步骤。
- 当前提交已合入 `main` 并推送 `origin/main`，本地与远端哈希一致，工作树干净。
- 验收服务从主分支工作区启动；旧工作树或未推送分支不得作为已交付入口。

固定五角色协作、集中审查、唯一全量验证和接管门禁见 `stage-collaboration-delivery-process.md`。

`000` 至 `023` 共享验收环境的数据范围、安全重建和验证规则见 `../data/electrical-manufacturing-demo-data.md`。自动化测试夹具不得替代该验收数据，也不得直接写入共享验收库。
