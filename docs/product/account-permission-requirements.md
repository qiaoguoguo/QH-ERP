# 账号与权限需求说明

## 需求名称

账号与权限基础模块。

## 业务背景

QH ERP 后续将覆盖物料、BOM、库存、生产、采购、销售、财务和报表。所有模块都需要明确“谁可以登录、看到哪些入口、执行哪些操作、是否能绕过前端直接访问接口”。因此账号与权限模块必须作为首个业务模块先完成。

## 用户角色

- 初始超级管理员：系统初始化后可登录，负责创建用户、角色和权限配置。
- 系统管理员：日常维护账号、角色、权限。
- 业务用户：按角色访问授权功能。
- 停用用户：不可登录，不再拥有访问权限，仅保留历史归属记录。

## 功能范围

### 必须实现

- 登录、退出、当前用户信息。
- 初始超级管理员。
- 用户查询、新增、编辑、启用、停用、重置密码。
- 角色查询、新增、编辑、启用、停用。
- 用户分配角色。
- 角色配置菜单权限和按钮权限。
- 后端接口权限校验。
- 未登录、登录失败、账号停用、无权限访问的明确反馈。
- 用户、角色、权限关键变更审计。

### 保留模型但本期不做完整配置界面

- 字段级权限。
- 数据范围权限。

### 明确不做

- 外部客户、供应商或会员账号。
- 多租户、多公司、多组织隔离。
- 单点登录和第三方登录。
- 复杂审批流。
- 完整审计报表。

## 业务规则

- 用户规则：
  - 登录账号全系统唯一。
  - 用户状态为启用、停用。
  - 停用用户不可登录；已登录用户被停用后，后端受保护接口必须拒绝访问。
  - 用户不物理删除。
- 密码规则：
  - 密码只存储哈希，不保存明文。
  - 重置密码必须写入审计。
  - 初始管理员密码必须支持通过环境变量配置；本地默认值仅用于开发验收。
- 角色规则：
  - 角色编码全系统唯一。
  - 停用角色不可再分配。
  - 已分配角色停用后，用户不再获得该角色权限。
  - 角色不物理删除。
- 权限规则：
  - 权限编码是后端鉴权稳定标识。
  - 菜单权限控制导航和页面入口。
  - 按钮权限控制页面操作入口。
  - 接口权限必须由后端校验。
  - 超级管理员拥有全部系统权限，但仍经过统一鉴权链路。
- 会话规则：
  - 推荐 HttpOnly Cookie + 服务端 Session。
  - 退出登录后会话失效。
  - 未登录访问受保护资源返回认证错误。
- 权限变更生效规则：
  - 后端接口鉴权实时按最新用户、角色和权限状态判断。
  - 前端菜单和按钮通过刷新当前用户信息或重新登录更新。

## 字段说明

### 用户

| 字段 | 含义 | 是否必填 | 校验规则 | 备注 |
|---|---|---|---|---|
| `id` | 用户标识 | 是 | 后端生成 | 不暴露数据库自增假设 |
| `username` | 登录账号 | 是 | 唯一，长度 4-50 | 不允许重复 |
| `displayName` | 显示姓名 | 是 | 长度 1-50 | 用于页面展示 |
| `phone` | 手机号 | 否 | 长度不超过 30 | 本期不做短信验证 |
| `email` | 邮箱 | 否 | 邮箱格式 | 本期不做邮件验证 |
| `status` | 用户状态 | 是 | `ENABLED`、`DISABLED` | 后端枚举 |
| `roles` | 角色列表 | 否 | 角色必须存在且启用 | 用户可无业务角色但不能越权 |
| `lastLoginAt` | 最后登录时间 | 否 | 后端生成 | 登录成功后更新 |
| `createdAt` | 创建时间 | 是 | 后端生成 | 审计字段 |
| `updatedAt` | 更新时间 | 是 | 后端生成 | 审计字段 |

## 流程说明

1. 系统从空库启动，Flyway 建表，初始化超级管理员、基础角色、基础权限。
2. 超级管理员登录系统。
3. 超级管理员创建角色并配置菜单、按钮权限。
4. 超级管理员创建用户并分配角色。
5. 用户登录后获取当前用户、菜单树和权限编码。
6. 前端按菜单和按钮权限展示入口，后端按接口权限强制校验。
7. 用户、角色、权限发生变更后写入审计；后端鉴权按最新状态生效。

## 验收标准

- 正常流程：管理员可登录、创建用户、创建角色、配置权限、分配角色，不同账号看到不同菜单和按钮。
- 异常流程：登录失败、账号停用、无权限访问、参数错误都有明确反馈。
- 权限验证：直接访问越权接口会被后端拒绝。
- 数据验证：初始化数据幂等；用户名、角色编码、权限编码唯一；关键变更可追溯。
- 页面验证：登录页、后台布局、用户列表、角色列表、权限配置、无权限页均可浏览器操作。
- 接口验证：成功、未登录、无权限、账号停用、参数错误返回统一结构。
- 部署验证：本地数据库、后端、前端可启动并完成浏览器验收路径。
