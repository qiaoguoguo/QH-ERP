# 核心业务流程

## 016 业务期间与历史数据保护流程

### 期间锁定

管理员维护不重叠的业务期间 -> 选择开放期间并填写锁定原因 -> 后端校验权限与状态 -> 更新为已锁定 -> 写入审计记录 -> 列表展示锁定人、时间和原因。

### 业务写入保护

用户提交库存、采购、销售、生产、成本、往来或反向业务动作 -> 业务服务取得业务日期 -> `BusinessPeriodGuard` 校验期间 -> 开放或未配置期间继续原业务逻辑 -> 已锁定期间返回 `BUSINESS_PERIOD_LOCKED` 并保留失败审计。

### 跨期反向与查询

锁定期间历史原单 -> 开放期间创建反向业务并引用来源 -> 校验反向单自身业务日期 -> 写入开放期间影响并保留来源追溯。用户查询锁定期间时，列表、详情、经营报表和来源追溯继续可用。

## 017 质量检验与库存质量状态基础流程

### 库存质量状态主链路

```text
采购入库/完工入库/销售退货/生产退料 -> 待检库存 -> 质量确认 -> 合格/不合格/冻结库存 -> 合格库存参与销售出库和生产领料
```

### 可用量与消耗控制

库存余额按仓库、物料和质量状态分层。`availableQuantity` 只代表合格可用库存，不包含待检、不合格或冻结库存。销售出库、生产领料和生产补料只能消耗合格库存；采购退货必须显式选择质量状态，冻结库存需先解冻。

### 期间保护

质量确认、冻结、解冻、入库质量状态落账和按质量状态扣减库存均按业务日期受 016 业务期间保护。锁定期间内历史质量记录、库存余额、库存流水和来源追溯仍可查询。

## 020 销售项目与合同管理基础流程

### 销售项目主链路

```text
客户 -> 销售项目草稿 -> 主合同草稿 -> 主合同激活（EFFECTIVE） -> 项目激活（ACTIVE） -> 销售订单（可选关联项目与合同） -> 既有销售履约
ACTIVE 销售项目 + EFFECTIVE 主合同 -> 补充合同 -> 合同激活 -> 商业增减留痕
```

销售项目是一级管理对象，一个项目最多有一个非取消主合同，可以有多个补充合同。补充合同必须引用同项目主合同，并且只允许在项目为 `ACTIVE`、主合同为 `EFFECTIVE` 时创建和激活。合同生效后关键商业字段不得直接覆盖，商业增减通过补充合同保留。

销售订单关联是可选的。`projectId` 与 `contractId` 必须同时为空或同时有值；仅草稿订单可关联、切换或解除，保存和确认时均校验项目为 `ACTIVE`、合同为 `EFFECTIVE`、合同属于项目且客户一致。历史订单允许两个字段继续为空，不强制补录，也不伪造项目归属。

### 状态流转

- 销售项目：`DRAFT -> ACTIVE -> CLOSED`，或 `DRAFT -> CANCELLED`。激活要求存在已生效主合同；关闭要求全部合同到达终态，且关联销售订单均为 `SHIPPED`、`CLOSED` 或 `CANCELLED`；取消仅适用于未形成有效合同且没有关联销售订单的草稿项目。
- 项目在 `ACTIVE` 状态仍可更新负责人、计划日期、目标收入、目标成本和备注，项目编号与客户保持锁定；`CLOSED`、`CANCELLED` 不可更新。
- 合同：`DRAFT -> EFFECTIVE -> CLOSED` 或 `TERMINATED`，也可由 `DRAFT -> CANCELLED`。`CLOSED` 表示正常履约结束，`TERMINATED` 表示提前终止，三种结束状态均为终态。
- 项目关闭、项目取消、合同关闭、合同终止和合同取消均须记录 1 至 200 字原因。

### 权限、审计与当前边界

项目、合同和销售订单权限相互独立。查看项目不自动获得合同或销售订单查看权；无合同查看权限时合同编号、金额、状态和数量不返回，无销售订单查看权限时订单数量与摘要不返回且不得显示为 0。项目操作记录按合同和订单权限过滤或脱敏，不得成为受限信息的旁路。

销售项目和合同属于商业主数据与管理事实，不使用业务期间保护。项目、合同的所有写入与状态动作均写入系统审计；销售订单关联、解除或切换项目合同时也写入销售项目审计。审计保留动作、目标、操作人、时间以及原因或关键变更字段摘要，不要求结构化变更前后值。

020 只让销售订单接入 `projectId` 与 `contractId`，不改变销售出库、库存预留、库存扣减或来源追溯语义。023 后库存余额、流水和项目实际来源成本层可以通过受审批所有权转换显式引用销售项目，但不得从销售订单关联自动推导项目库存；024/027/028 后，采购、生产/外协、业务发票、费用和核销可以显式继承项目或公共归属。原有历史记录仍不回填，正式收入确认和正式项目成本尚未接入。

## 025 销售深化与项目履约流程

### 报价、订单与商业快照

```text
销售报价草稿 -> 固定审批通过 -> 有效期内幂等转换一次 -> 合同草稿或销售订单草稿
销售订单草稿 -> 税价/来源/项目合同/交付计划/信用校验 -> 确认 -> 商业快照 + 库存预留
```

直接新建订单行使用 `MANUAL` 来源；报价转单行使用 `QUOTE` 并保留报价行来源。报价来源行只允许在草稿期调整履约字段，物料、单位、价格、税率和币种不可通过公共更新接口改写或静默降级为手工来源。订单确认后冻结客户结算、报价、项目合同、币种、税价和信用检查引用；订单头金额严格等于本单行汇总。

### 交付、变更、信用与关闭

```text
订单确认 -> 行级交付计划 -> 销售出库引用开放计划 -> 统一库存过账 -> 计划/订单履约推进
确认后变更 -> 固定审批 -> 仅作用于未交付数量、税价或交期 -> 已发快照保持不变
订单承诺 + 待建应收出库 + 基础应收未收 -> 商业信用占用 -> 确认/增量变更门禁
订单正常或短交关闭 -> 项目销售履约关闭 -> 项目正式关闭门禁
```

订单确认按行指定仓库形成销售库存预留；出库过账消耗预留，取消、关闭或减量释放，增量变更补足。新出库必须引用开放交付计划，提前交付必须填写原因，超计划和超订单数量固定拒绝。销售退货减少净履约并沿用既有反向链路，但不自动重开有效需求。

商业信用是业务控制而非正式财务。无信用档案、冻结、超限或逾期按规则阻断，允许的例外必须经固定审批并在应用前重新计算。有效销售需求仅输出开放订单/计划的只读来源、数量、日期和不计入原因，026 才负责缺料净算和供给建议。

## 026 订单驱动缺料与供给建议流程

### 需求净算与快照

```text
025 有效销售需求 -> 按项目/公共范围读取有效 BOM -> 逐层展开并换算基本单位
-> 按需求日期与稳定键分配项目库存/项目采购/公共库存/公共采购/既有工单
-> 固化需求、分配、短缺、异常、建议、来源指纹和 BOM 快照
```

026 只把 `countedOnly=true` 的有效销售需求纳入正式净算；被排除的销售需求只用于解释原因。项目需求不得自动使用其他项目资源，公共需求不得使用项目专属资源；合格可用库存、有效采购供给和既有工单按冻结优先级与日期确定性分配，同一供给不得重复覆盖。计算读取当前 021/023/024/025 事实，查询只读已持久化快照；重算产生新快照，旧快照保留并在来源变化或时间围栏到期后标记过期。

### 建议、追溯与受控转换

```text
短缺 -> 采购请购建议 / 生产建议 / 公共库存使用建议 / 既有供给使用建议
采购请购建议 -> 人工确认 -> 幂等转 024 采购请购草稿 -> 保留双向来源
已确认生产建议 -> 027 幂等转项目/公共生产工单 -> 保留建议与运行来源
已确认外协建议 -> 027 幂等转独立外协订单 -> 保留建议与运行来源
```

建议可以人工确认或驳回；只有具备 026 转换权限和 024 请购创建权限的已确认采购建议，才允许幂等生成一个采购请购草稿。转换不自动提交审批、不选价、不生成采购订单。替代料只显示受控提示，不自动抵扣、替代领料或改写库存；公共库存使用只形成建议，不自动执行所有权转换。列表、详情、导出和追溯按项目、订单、BOM、物料、仓库、采购、工单与替代料来源权限分别脱敏。

## 027 项目生产执行与外协基础流程

### 项目生产执行

```text
026 已确认生产建议 -> 幂等创建项目/公共工单草稿 -> 发布并固化 BOM/来源/项目快照
-> 按项目或公共所有权领料/补料 -> 简单报工 -> 完工入库项目或公共库存
原领料 -> 退料 -> 原所有权/原项目/原成本层/原批次序列来源
```

建议转换要求父运行仍有效、建议为 `CONFIRMED` 且来源版本一致；同一建议只允许一个目标，重复请求返回原目标。项目工单不消费其他项目库存；公共库存的使用必须遵守明确来源和统一库存过账。生产执行保留工单、建议、销售需求、项目、BOM、领退补料、报工、完工及批次/序列来源，但不形成正式在制或项目成本。

### 外协执行

```text
026 已确认外协建议 -> 幂等创建外协订单草稿 -> 发布
-> 外协发料过账 -> 供应商在外材料事实 -> 外协收货过账 -> 项目或公共库存
-> 全部执行完成后关闭外协订单
```

外协订单、发料和收货是独立对象，不复用采购订单或内部生产工单冒充。发料/收货草稿可编辑并保留备注，过账后不可直接改写；库存数量、价值、所有权、项目、批次/序列和审计与 023 统一过账保持一致。028 已承接外协采购侧发票、应付与结算来源，但不回写 027 库存数量价值；正式外协成本仍属于 029。

## 028 发票、费用与往来深化流程

### 销售发票与回款

```text
已过账销售出库 -> 销售发票草稿 -> 确认 -> 绑定既有应收或唯一生成应收
真实收款 -> 分配到应收和/或保留预收余额 -> 过账
预收余额 -> 同客户、同人民币、同项目/公共归属核销应收 -> 不新增现金
```

销售发票按客户、项目/公共归属和出库来源建立业务开票事实；确认不能重复生成应收，也不能改写 V10/V11 历史余额。应收编号提供显式返回链路。收款与核销分开表达现金发生和余额分配，多目标分配必须原子完成且不得超额、跨主体、跨项目或跨公共归属。

### 采购、外协发票与付款

```text
采购订单 + 已过账采购入库 + 采购发票 -> 零容差三单匹配 -> 确认 -> 应付
已过账外协收货 -> 外协采购发票 -> 不适用标准三单匹配 -> 确认 -> 应付
真实付款 -> 分配到应付和/或保留预付余额 -> 过账
预付余额 -> 同供应商、同人民币、同项目/公共归属核销应付 -> 不新增现金
```

标准采购的数量、未税/含税单价及金额任一差异都会登记结构化差异并阻止确认。外协发票保留外协订单/收货、供应商、项目/公共归属和金额来源，不伪装成标准采购订单三单匹配，也不回写外协收货形成的库存数量价值。

### 费用、核销与凭证草稿

```text
合法费用来源或手工非采购支出 -> 费用单草稿 -> 确认 -> 供应商应付
收款/付款/预收/预付 -> 核销工作台 -> 多目标原子分配与余额追溯
确认发票/费用、过账资金与核销事实 -> 凭证草稿 -> 业务分类建议
```

费用只形成供应商侧应付和 029 可消费的项目/公共归属事实，不直接写项目正式成本。凭证草稿不产生正式科目、期间、凭证号、记账或总账；同一来源和版本幂等生成，后续正式会计处理由 031 承接。

## 029 项目成本核算流程

### 核算运行与来源追溯

```text
销售项目 + 023 库存价值 + 027 生产/外协 + 028 费用/外协结算 + 销售履约
-> 创建截至指定业务日期的核算运行 -> 固化来源快照与完整来源指纹
-> 形成材料/人工/外协/制造费用/项目费用分类和在制/完工/交付/调整阶段分录
-> 展示差异、完整性以及目标/开票/发货三种经营毛利
```

核算运行不覆盖历史运行。来源不变时同一请求幂等返回；上游数量、金额、版本、发货、退货、发票或目标收入变化会使旧结果标记为 `STALE`。确认前重新校验来源指纹；已取消运行仍保留历史，但不能成为有效当前结果。

### 调整、确认与业务边界

```text
项目成本调整或公共制造费用分配草稿 -> 提交 022 固定审批
-> 审批通过 -> 确认 -> 形成 029 独立调整分录 -> 后续核算读取
```

调整不得修改库存余额、价值流水、项目成本层、工单、外协、发票、费用或往来余额。029 的在制、完工、交付阶段只是项目成本子账分布，阶段转移不改变项目成本总额；经营毛利不是正式收入确认、会计期间、凭证、总账或月结结果，后续由 030 至 032 承接。

## 030 业务月结与期间快照流程

### 检查、关闭与重开

```text
选择开放业务期间 -> 同步月末检查 -> BLOCKED 或 READY
READY -> 原子复检 -> 生成不可变快照 -> 期间 LOCKED -> CLOSED
CLOSED -> 填写原因重开 -> 期间 OPEN -> REOPENED -> 后续重检并生成新版本
```

阻断项包括未估值库存、库存/项目成本来源断链、开放阻断差异和跨期反向等；合法在制等只作为警告。关闭与经营写入使用同一期间级互斥锁，失败时不得留下半快照、半锁期或半审计。历史手工锁定不自动补认为月结关闭，重开不覆盖或删除旧快照。

### 快照与后续边界

库存、在制/生产、项目成本及八类经营报表基线按月结版本只读冻结，来源和金额继续受原领域权限控制。实时 014 报表仍表示当前事实，不能冒充历史快照；030 不创建正式会计期间关闭、会计凭证、总账、试算平衡或财务关账。031 已在功能分支承接正式凭证与总账基础，032 承接财务结账与资金税务基础。

## 031 会计科目、凭证与总账基础流程

### 基础设置与规则

```text
初始化 MAIN/CNY 总账启用月 -> 顺序新增 OPEN 会计期间
-> 维护制造业基础科目和辅助核算 -> 校验并激活固定过账规则
```

031 固定单公司、单账簿 `MAIN`、人民币 `CNY`。会计期间只提供启用首月和顺序新增 `OPEN` 期间，不提供关闭、反关闭或损益结转。科目模板允许扩展下级科目，只有启用、末级且 `postable=true` 的科目可入分录；已被规则、凭证、期初或账簿引用的关键属性不可直接改写。辅助核算内置客户、供应商、项目，并允许自定义维度和自定义项目。

固定过账规则按 `sourceType + sourceVariant + normalizedFactCode + direction` 映射到科目和辅助来源。V33 预置 7 条 `ACTIVE` 规则、17 行和 9 个辅助映射；同一来源类型和变体只能有一个 `ACTIVE`。规则预览只返回分录建议，不占用来源、不创建凭证。

### 凭证、审批与记账

```text
手工 GENERAL/OPENING 草稿或 028 READY 来源转换 -> DRAFT
-> 提交 022 固定审批 -> 非提交人最终审批通过
-> 同事务分配连续正式号、POSTED、写总账/明细账/余额/审计
```

手工凭证必须至少两行、单边正数、借贷平衡，凭证日期落入唯一 `OPEN` 会计期间。`OPENING` 只能用于启用首月第一天，走同一审批记账链路，只计入期初；任一普通凭证已记账后不得再新增或提交有效期初。028 READY 六类来源转换时，后端重新读取原销售发票、采购发票、费用、收款、付款或核销事实，校验状态、版本和指纹后生成正式凭证草稿；不复制 028 凭证草稿两行建议，不回写 028。

最终审批必须由非提交人完成。审批通过与来源复检、连续编号、凭证 `POSTED`、不可变账簿、期间汇总、来源占用和审计同事务完成；任一环节失败不得消耗号段或留下半账。POSTED 凭证、分录、辅助快照和账簿分录不可更新、删除或补写。

### 账簿查询、冲销与边界

总账、明细账、科目余额和试算平衡只读取 POSTED 账簿事实。试算平衡必须分别展示期初、本期发生和期末三组借贷；若汇总缓存与不可变分录不一致，必须失败关闭并给出差异科目。已记账凭证更正通过新凭证或冲销表达；冲销创建反向正金额 `GENERAL` 草稿，重新审批并分配独立正式号，原凭证状态和内容保持不变，重复冲销被阻断或幂等返回既有草稿。

031 不补造启用月之前的历史总账，不把 029 项目成本运行或 030 月结快照自动制证，也不让 030 状态控制会计期间。032 财务结账与资金税务基础、期间关闭/反关闭、银行对账、税务基础、期末损益结转和正式财务报表仍未交付。

## 022 固定审批与业务单据平台流程

### 固定审批主链路

```text
合同草稿/工程变更草稿 -> 提交固定审批 -> 发起人不可自批 -> 授权审批人待办 -> 通过/驳回
通过 -> 同一事务执行业务动作 -> 合同生效/工程变更应用 -> 消息与审计
撤回/驳回/取消 -> 业务对象保持原业务状态 -> 附件恢复可维护
```

审批状态与合同、ECO 的业务状态分开保存。待办只返回本人真实候选任务，已处理只返回本人实际处理记录，我发起的只返回本人提交记录；列表、详情、数量、附件快照和消息同时受平台权限、业务对象权限及本人关系约束。动作携带任务或实例版本与幂等键，陈旧、重复或自批请求不能改变业务状态。

### 附件与固定打印

合同和 ECO 附件上传到私有 MinIO，数据库只保存受控文件元数据和业务关系。上传、下载、删除继承业务对象权限；提交审批时固化附件快照，审批开放期间禁止增删。固定打印模板只覆盖合同生效审批单和 ECO 应用审批单，预览和 PDF 同时记录模板代码与版本，下载时重新校验业务权限，不提供模板设计器。

### 导入导出与文档任务

```text
下载固定模板 -> 上传 XLSX -> 全文件校验与暂存 -> 查看预览/错误 -> 携带版本和幂等键确认 -> 业务服务原子提交 -> 消息/审计/结果
```

物料导入仅新增物料，筛选导出使用页面当前条件；BOM 导入只创建或更新一个草稿版本，保留仓库兼容空列，不发布、不应用 ECO、不改写已发布工单快照。任务由数据库 worker 通过租约执行，支持恢复、取消、错误明细、结果下载和过期。任一校验或提交错误不得形成部分业务数据，导入不能绕过现有业务服务直接写事实表。

## 023 库存计价与仓储账实一致流程

### 公共库存与项目库存价值

```text
公共采购/期初/普通完工 -> 公共库存 -> 单公司+物料移动加权平均池 -> 公共出库按当时平均价
公共库存 -> 所有权转换审批 -> 项目库存 -> 项目+物料+真实来源成本层 -> 指定层消费/退回
```

库存数量按仓库、物料、质量、批次/序列、所有权和项目分层；公共平均价跨仓库、质量和追踪身份共享，项目库存保留真实来源层。数量过账与价值流水在同一事务完成，历史未估值库存不能被静默写成零成本，也不能绕过期初估值执行价值型出库、盘亏或所有权转换。

### 仓库调拨、所有权转换、盘点和估值

- 仓库调拨只改变仓库数量分布，承接原所有权、项目、质量、批次/序列和成本依据，企业总数量与总价值保持不变。
- 公共转项目、项目转公共、项目转项目必须走固定审批；审批通过后原子扣减来源并建立或并入目标价值载体。
- 盘点开始时固化账面快照并锁定范围；未盘、实盘 `0` 和实盘非零是三个不同状态。零差异可以直接结束，有差异必须审批后过账。
- 期初估值只处理历史未估值公共库存；暂估调整只调整当前剩余库存金额，不改数量、不回写历史出库。
- 单价、金额、平均价和成本层金额受 `inventory:valuation:view` 后端强鉴权；无权限用户仍可按原业务权限查看数量，但不能从页面、接口、导出、审批、附件或来源跳转获得成本数据。

```

```

```

```

## 退货退款与业务反冲链路

```text
销售出库 -> 销售退货 -> 待检库存 -> 质量确认 -> 应收冲减 -> 销售净额报表 -> 来源追溯
采购入库 -> 采购退货 -> 按显式质量状态退货出库流水 -> 应付冲减 -> 采购净额报表 -> 来源追溯
生产领料 -> 生产退料/补料 -> 退料进入待检或补料消耗合格库存 -> 成本净额 -> 生产与成本报表 -> 来源追溯
```

## 关键节点

- 基础资料：维护单位、仓库、供应商、客户、岗位、角色等基础数据。
- 物料：维护原材料、半成品、成品、辅料等物料档案。
- BOM：定义成品或半成品的用料结构和版本。
- 库存：记录公共/项目所有权下的总现存、合格可用、待检、不合格、冻结、预留、批次/序列、估值状态、成本层和数量/价值变动来源。
- 质量确认：将待检库存转为合格、不合格或冻结，并记录业务日期、原因、操作人、来源单据和审计。
- 采购订单：记录供应商采购物料、数量、采购单价、预计到货日期和入库进度。
- 采购入库：基于采购订单完成原材料入库，过账后增加库存并生成采购来源库存流水。
- 销售项目：记录项目编号、客户、负责人、计划周期、目标收入、目标成本和状态，是合同与可选销售订单关联的一级管理对象。
- 销售合同：记录项目下的主合同和补充合同、商业金额、日期与状态；合同商业金额不等同实际收入、开票金额或项目实际成本。
- 销售订单：记录客户销售成品或半成品、数量、销售单价、预计交付日期和出库进度；可在草稿阶段成对关联销售项目与已生效合同，本阶段不新增物料可销售字段。
- 销售出库：基于销售订单完成销售发出，过账后扣减库存并生成销售来源库存流水。
- 应收台账：基于已过账销售出库生成业务应收金额快照，记录应收金额、已收金额、累计调整额、未收金额和状态。
- 收款记录：基于应收台账登记收款并过账，过账后更新应收余额和状态。
- 应付台账：基于已过账采购入库生成业务应付金额快照，记录应付金额、已付金额、累计调整额、未付金额和状态。
- 付款记录：基于应付台账登记付款并过账，过账后更新应付余额和状态。
- 经营报表：基于已有业务单据和业务状态形成固定经营汇总、来源追溯和异常清单，不作为正式财务报表或 BI 平台。
- 退货退款与业务反冲：基于已过账原单建立反向业务记录，表达退货、退款、退料、补料、冲减和净额影响，不直接改写历史原单。
- 生产工单：承载生产任务、计划数量、状态流转和关联 BOM。
- 领料：根据工单和 BOM 从库存领取物料。
- 报工：记录生产进度、合格数量、不良数量和工序结果。
- 完工入库：将完工产品入库，并形成库存变动。
- 成本归集：归集材料、人工、制造费用或其他一期可记录成本；与 023 库存计价和项目库存来源成本层分开，尚不是正式项目成本核算。

## 数据追溯要求

任何库存变动都应能追溯到来源单据。任何生产结果都应能追溯到工单、物料、BOM 和相关操作记录。

采购入库形成的库存变动必须能追溯到采购入库单和采购订单。采购订单必须能查看已入库数量、未入库数量和关联入库记录。

销售出库形成的库存变动必须能追溯到销售出库单和销售订单。销售订单必须能查看已出库数量、未出库数量和关联出库记录。

销售项目、合同及销售订单项目关联的写入和状态动作必须保留系统审计。历史销售订单可不关联项目与合同；库存只能通过 023 受控所有权转换显式建立项目归属，采购、生产、原有成本和财务对象不得据销售订单补写或推导项目归属。

库存数量和价值必须能从业务单据追溯到不可变数量流水、价值流水、公共计价池或项目成本层。仓库调拨必须有成对出入流水且总价值不变；项目层消费、退回、冲销和所有权转换必须保留原层或父层引用。

固定审批、消息、附件、打印和文档任务必须记录场景、业务对象、操作人、动作、状态变化、模板版本、任务结果和权限拒绝等必要审计；历史合同、ECO、物料和 BOM 不补造审批、附件或任务记录。

应收台账必须能追溯到销售发票、销售出库和销售订单；历史无发票台账继续允许直接追溯原出库。收款、预收核销及往来调整必须能追溯到对应应收。应收金额必须等于已收金额、累计调整额与未收金额之和，且状态与余额一致；发票确认不得重复增加同一余额。

应付台账必须能追溯到采购发票及其标准采购入库、外协收货或费用来源；历史无发票台账继续允许直接追溯原入库。付款、预付核销及往来调整必须能追溯到对应应付。应付金额必须等于已付金额、累计调整额与未付金额之和，且状态与余额一致；外协发票不得回写库存价值。

一笔真实收款或付款可以原子分配多个目标并保留未核销预收/预付余额；预收预付核销只能改变余额分配，不能再次增加现金。发票、费用、收付款、核销及凭证草稿必须保留来源、往来方、项目/公共归属、金额、版本和幂等审计。

成本归集一期必须能追溯到生产工单、产品和来源单据。生产领料形成材料数量口径记录，生产报工形成人工数量口径记录，完工入库作为产出数量追溯。手工成本记录必须关联有效生产工单，产品从工单带出，并说明金额或数量口径。

成本归集一期只作为业务记录与追溯，不作为正式财务核算结果。

028 财务往来深化只作为业务发票、费用、收付款、预收预付、核销、应收应付链接和凭证草稿来源追溯，不作为正式会计凭证、总账、税务、正式收入确认或正式成本结转结果。

031 会计核算只把合法手工凭证和 028 READY 来源转换后的正式凭证写入总账；029 项目成本、030 业务月结快照和历史业务事实不会被回填、回写或自动制证。总账、明细账、科目余额和试算平衡必须能追溯到 POSTED 凭证、分录、辅助快照、规则版本和来源版本/指纹。

经营报表基础只作为固定经营汇总和来源追溯，不作为可配置 BI、总账、凭证、正式财务报表、结账或税务申报结果。经营报表中的经营金额必须能追溯来源业务单据，并明确不等同正式财务入账金额。

退货退款与业务反冲基础只作为业务反向闭环，不作为审批、红字发票、总账凭证、税务、银行退款支付或正式财务核算结果。所有反向业务必须能追溯原单、原行、库存流水、往来冲减、成本影响和报表净额。

质量检验与库存质量状态基础只作为库存可用性控制，不作为完整 QMS、SPC、批次、库位、MRP 或正式财务能力。质量状态必须能追溯来源单据、质量确认、状态转换、库存流水和后续消耗结果。

## 异常场景

- BOM 缺失或版本不可用。
- 库存不足。
- 领料数量超过规则允许范围。
- 报工数量异常。
- 完工入库与工单数量不一致。
- 成本未归集或归集数据异常。
- 成本来源单据不存在、未过账或重复归集。
- 成本金额、数量或口径不合法。
- 采购订单未确认、已取消或已关闭时创建入库。
- 采购入库数量超过采购订单未入库数量。
- 采购入库重复过账。
- 销售订单未确认、已取消或已关闭时创建出库。
- 销售出库数量超过销售订单未出库数量。
- 销售出库库存不足。
- 销售出库重复过账。
- 未过账销售出库生成应收。
- 未过账采购入库生成应付。
- 销售出库或采购入库来源重复生成往来台账。
- 收款金额超过应收未收金额。
- 付款金额超过应付未付金额。
- 已收清应收继续登记收款。
- 已付清应付继续登记付款。
- 收付款失败后金额余额或状态产生部分写入。
- 经营报表汇总值与来源明细不一致。
- 经营报表来源追溯断链或跳转到错误业务单据。
- 无报表权限用户查看经营报表页面或接口数据。
- 经营报表把业务经营金额误表达为总账、凭证或正式财务报表。
- 销售退货数量超过可退销售出库数量。
- 采购退货数量超过可退采购入库数量，或退货出库库存不足。
- 生产退料数量超过原领料数量，或补料出库库存不足。
- 退款或往来冲减金额超过可冲金额。
- 已过账原单被退货、退款或反冲直接改写。
- 反向业务未同步影响库存、往来、成本或经营报表净额。
- 销售出库、生产领料或生产补料尝试消耗待检、不合格或冻结库存。
- 总库存充足但合格库存不足，错误提示未明确质量状态不可用原因。
- 质量确认结果数量不等于本次处理待检数量。
- 质量状态转换造成任一状态库存为负数或总库存不一致。
- 采购退货尝试直接消耗冻结库存。
- 锁定业务期间内提交质量确认、冻结、解冻或质量状态库存过账。
- 项目缺少已生效主合同时尝试激活，或合同、关联销售订单未到终态时尝试关闭项目。
- 补充合同创建或激活时项目不是 `ACTIVE`、主合同不是 `EFFECTIVE`，或主合同不属于同一项目。
- 销售订单只提交 `projectId` 或 `contractId`、客户不一致、合同不属于项目，或确认时项目合同状态已失效。
- 无合同或销售订单查看权限的用户从项目摘要或操作记录获得受限信息。
- 历史未估值库存被写成零成本，或未完成期初估值即可执行价值型出库、盘亏和所有权转换。
- 公共库存、项目库存、公共计价池、项目成本层和价值流水数量或金额不一致。
- 仓库调拨改变企业总库存价值，或公共/项目库存被其他所有权无控制消费。
- 盘点未盘与实盘零混淆、范围锁泄漏、差异未审批即过账或多行半提交。
- 无库存价值权限用户从接口、页面、审批、附件、导出或来源追溯获得单价、金额、平均价或成本层金额。
