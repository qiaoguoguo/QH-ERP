# 核心业务流程

## 016 业务期间与历史数据保护流程

### 期间锁定

管理员维护不重叠的业务期间 -> 选择开放期间并填写锁定原因 -> 后端校验权限与状态 -> 更新为已锁定 -> 写入审计记录 -> 列表展示锁定人、时间和原因。

### 业务写入保护

用户提交库存、采购、销售、生产、成本、往来或反向业务动作 -> 业务服务取得业务日期 -> `BusinessPeriodGuard` 校验期间 -> 开放或未配置期间继续原业务逻辑 -> 已锁定期间返回 `BUSINESS_PERIOD_LOCKED` 并保留失败审计。

### 跨期反向与查询

锁定期间历史原单 -> 开放期间创建反向业务并引用来源 -> 校验反向单自身业务日期 -> 写入开放期间影响并保留来源追溯。用户查询锁定期间时，列表、详情、经营报表和来源追溯继续可用。

## 017 质量检验与库存质量状态基础流程

### 库存质量状态主链路

```text
采购入库/完工入库/销售退货/生产退料 -> 待检库存 -> 质量确认 -> 合格/不合格/冻结库存 -> 合格库存参与销售出库和生产领料
```

### 可用量与消耗控制

库存余额按仓库、物料和质量状态分层。`availableQuantity` 只代表合格可用库存，不包含待检、不合格或冻结库存。销售出库、生产领料和生产补料只能消耗合格库存；采购退货必须显式选择质量状态，冻结库存需先解冻。

### 期间保护

质量确认、冻结、解冻、入库质量状态落账和按质量状态扣减库存均按业务日期受 016 业务期间保护。锁定期间内历史质量记录、库存余额、库存流水和来源追溯仍可查询。

## 生产主链路

```text
基础资料 -> 物料 -> BOM -> 库存 -> 生产工单 -> 领料 -> 报工 -> 完工入库 -> 成本归集
```

## 采购入库链路

```text
供应商 -> 采购订单 -> 采购入库 -> 待检库存 -> 质量确认 -> 合格库存 -> 生产领料 -> 报工 -> 完工入库 -> 成本归集
```

## 销售出库链路

```text
客户 -> 销售订单 -> 销售出库 -> 库存扣减 -> 销售来源库存流水 -> 来源追溯
```

## 财务往来链路

```text
销售出库 -> 应收台账 -> 收款记录 -> 应收余额 -> 销售来源追溯
采购入库 -> 应付台账 -> 付款记录 -> 应付余额 -> 采购来源追溯
```

## 经营报表链路

```text
采购、销售、库存、生产、成本、财务往来 -> 固定经营报表 -> 来源明细 -> 来源业务详情
```

## 退货退款与业务反冲链路

```text
销售出库 -> 销售退货 -> 待检库存 -> 质量确认 -> 应收冲减 -> 销售净额报表 -> 来源追溯
采购入库 -> 采购退货 -> 按显式质量状态退货出库流水 -> 应付冲减 -> 采购净额报表 -> 来源追溯
生产领料 -> 生产退料/补料 -> 退料进入待检或补料消耗合格库存 -> 成本净额 -> 生产与成本报表 -> 来源追溯
```

## 关键节点

- 基础资料：维护单位、仓库、供应商、客户、岗位、角色等基础数据。
- 物料：维护原材料、半成品、成品、辅料等物料档案。
- BOM：定义成品或半成品的用料结构和版本。
- 库存：记录总现存、合格可用库存、待检库存、不合格库存、冻结库存和库存变动来源。
- 质量确认：将待检库存转为合格、不合格或冻结，并记录业务日期、原因、操作人、来源单据和审计。
- 采购订单：记录供应商采购物料、数量、采购单价、预计到货日期和入库进度。
- 采购入库：基于采购订单完成原材料入库，过账后增加库存并生成采购来源库存流水。
- 销售订单：记录客户销售成品或半成品、数量、销售单价、预计交付日期和出库进度；本阶段不新增物料可销售字段。
- 销售出库：基于销售订单完成销售发出，过账后扣减库存并生成销售来源库存流水。
- 应收台账：基于已过账销售出库生成业务应收金额快照，记录应收金额、已收金额、未收金额和状态。
- 收款记录：基于应收台账登记收款并过账，过账后更新应收余额和状态。
- 应付台账：基于已过账采购入库生成业务应付金额快照，记录应付金额、已付金额、未付金额和状态。
- 付款记录：基于应付台账登记付款并过账，过账后更新应付余额和状态。
- 经营报表：基于已有业务单据和业务状态形成固定经营汇总、来源追溯和异常清单，不作为正式财务报表或 BI 平台。
- 退货退款与业务反冲：基于已过账原单建立反向业务记录，表达退货、退款、退料、补料、冲减和净额影响，不直接改写历史原单。
- 生产工单：承载生产任务、计划数量、状态流转和关联 BOM。
- 领料：根据工单和 BOM 从库存领取物料。
- 报工：记录生产进度、合格数量、不良数量和工序结果。
- 完工入库：将完工产品入库，并形成库存变动。
- 成本归集：归集材料、人工、制造费用或其他一期可记录成本。

## 数据追溯要求

任何库存变动都应能追溯到来源单据。任何生产结果都应能追溯到工单、物料、BOM 和相关操作记录。

采购入库形成的库存变动必须能追溯到采购入库单和采购订单。采购订单必须能查看已入库数量、未入库数量和关联入库记录。

销售出库形成的库存变动必须能追溯到销售出库单和销售订单。销售订单必须能查看已出库数量、未出库数量和关联出库记录。

应收台账必须能追溯到销售出库和销售订单。收款记录必须能追溯到对应应收。应收金额、已收金额、未收金额和状态必须一致。

应付台账必须能追溯到采购入库和采购订单。付款记录必须能追溯到对应应付。应付金额、已付金额、未付金额和状态必须一致。

成本归集一期必须能追溯到生产工单、产品和来源单据。生产领料形成材料数量口径记录，生产报工形成人工数量口径记录，完工入库作为产出数量追溯。手工成本记录必须关联有效生产工单，产品从工单带出，并说明金额或数量口径。

成本归集一期只作为业务记录与追溯，不作为正式财务核算结果。

财务往来基础只作为业务台账、收付款登记和来源追溯，不作为总账、凭证、税务、完整发票、正式收入确认或正式成本结转结果。

经营报表基础只作为固定经营汇总和来源追溯，不作为可配置 BI、总账、凭证、正式财务报表、结账或税务申报结果。经营报表中的经营金额必须能追溯来源业务单据，并明确不等同正式财务入账金额。

退货退款与业务反冲基础只作为业务反向闭环，不作为审批、红字发票、总账凭证、税务、银行退款支付或正式财务核算结果。所有反向业务必须能追溯原单、原行、库存流水、往来冲减、成本影响和报表净额。

质量检验与库存质量状态基础只作为库存可用性控制，不作为完整 QMS、SPC、批次、库位、MRP 或正式财务能力。质量状态必须能追溯来源单据、质量确认、状态转换、库存流水和后续消耗结果。

## 异常场景

- BOM 缺失或版本不可用。
- 库存不足。
- 领料数量超过规则允许范围。
- 报工数量异常。
- 完工入库与工单数量不一致。
- 成本未归集或归集数据异常。
- 成本来源单据不存在、未过账或重复归集。
- 成本金额、数量或口径不合法。
- 采购订单未确认、已取消或已关闭时创建入库。
- 采购入库数量超过采购订单未入库数量。
- 采购入库重复过账。
- 销售订单未确认、已取消或已关闭时创建出库。
- 销售出库数量超过销售订单未出库数量。
- 销售出库库存不足。
- 销售出库重复过账。
- 未过账销售出库生成应收。
- 未过账采购入库生成应付。
- 销售出库或采购入库来源重复生成往来台账。
- 收款金额超过应收未收金额。
- 付款金额超过应付未付金额。
- 已收清应收继续登记收款。
- 已付清应付继续登记付款。
- 收付款失败后金额余额或状态产生部分写入。
- 经营报表汇总值与来源明细不一致。
- 经营报表来源追溯断链或跳转到错误业务单据。
- 无报表权限用户查看经营报表页面或接口数据。
- 经营报表把业务经营金额误表达为总账、凭证或正式财务报表。
- 销售退货数量超过可退销售出库数量。
- 采购退货数量超过可退采购入库数量，或退货出库库存不足。
- 生产退料数量超过原领料数量，或补料出库库存不足。
- 退款或往来冲减金额超过可冲金额。
- 已过账原单被退货、退款或反冲直接改写。
- 反向业务未同步影响库存、往来、成本或经营报表净额。
- 销售出库、生产领料或生产补料尝试消耗待检、不合格或冻结库存。
- 总库存充足但合格库存不足，错误提示未明确质量状态不可用原因。
- 质量确认结果数量不等于本次处理待检数量。
- 质量状态转换造成任一状态库存为负数或总库存不一致。
- 采购退货尝试直接消耗冻结库存。
- 锁定业务期间内提交质量确认、冻结、解冻或质量状态库存过账。
