# 产品决策记录

## 目标

记录已经确认的产品方向和阶段性决策，避免后续模块开发反复讨论同一问题。

## 已确认决策

### 项目定位

本项目是制造业单公司内部使用的 Web ERP 系统，以生产管理为核心，不做零售、门店收银、电商或多租户平台方向。

### 首个业务模块

首个业务模块为账号与权限基础。

原因：

- 所有后续模块都依赖登录态、用户身份、角色权限、菜单入口和接口鉴权。
- 先完成账号权限可以让后续模块从第一天起按真实权限场景验收。
- 避免物料、BOM、库存和生产模块完成后再反向补权限，造成返工。

### 一期验收方式

一期采用模块逐步验收，并在生产主链路完成后追加端到端闭环验收。

模块逐步验收用于控制交付节奏；端到端闭环验收用于验证物料、BOM、库存、工单、领料、报工、完工入库和成本记录之间的数据贯通。

### 成本归集一期深度

一期成本归集做到业务记录，不承诺正式财务核算结果。

业务记录应至少能说明：

- 成本来源。
- 关联工单或产品。
- 成本类型。
- 成本金额或数量口径。
- 记录人和记录时间。
- 后续可追溯的来源单据。

正式核算结果需要稳定的物料、库存、工单、领料、报工、入库和财务规则支撑，后续阶段再展开。

## 待后续模块确认

- 审批流程是否进入账号权限模块一期。
- 数据范围是否按仓库、部门或业务线限制。
- 财务模块一期是否只做应收、应付和成本查询，还是引入更完整凭证规则。
- 报表一期按固定看板还是可配置报表推进。

## 决策变更规则

任何产品决策变更必须记录：

- 变更原因。
- 影响模块。
- 影响任务文档和验收标准。
- 是否影响当前阶段成果。
- 主代理决策结论或用户确认结果。
