# 项目启动基线

## 目标

建立项目正式开发前的统一基线，确保后续产品、设计、前端、后端、测试和验收工作围绕同一目标推进。

## 项目定位

本项目是制造业单公司 Web ERP 系统。系统以生产管理为核心，不做零售、门店收银或电商导向设计。

## 启动原则

- 先建立任务文档，再执行开发、设计、测试或验收。
- 所有阶段成果必须有明确目标、范围、输入、输出和验收标准。
- 涉及物料、BOM、库存、工单、成本、财务金额的数据设计必须可追溯。
- 主分支只承载可阶段验收的稳定成果。
- 阶段验收以本地可部署、浏览器可访问、用户可操作为准。

## 当前已完成

- 建立仓库规范。
- 建立 5 个子代理角色与职责分工。
- 建立启动基线文档结构。
- 确认首个业务模块为账号与权限基础。
- 确认一期采用模块逐步验收，并在生产主链路完成后追加端到端闭环验收。
- 确认成本归集一期先做到业务记录，不承诺正式财务核算结果。

## 下一步

1. 完成工程骨架初始化和基础验证。
2. 为账号与权限基础模块创建任务文档、需求文档、接口契约和测试验收文档。
3. 调度 5 个子代理围绕账号权限模块进行正式方案讨论。
4. 在独立任务分支实现账号权限模块，完成测试和本地浏览器验收后通知用户查验。
