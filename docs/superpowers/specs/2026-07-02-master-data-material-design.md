# 基础资料与物料管理设计规格

## 目标

在账号与权限基础模块之后，建立一期生产主链路所需的主数据底座。该阶段只沉淀可被 BOM、库存、工单、采购和销售后续复用的基础资料与物料档案，不进入业务流转。

## 范围

本阶段采用“轻量基础资料 + 物料管理”范围：

- 计量单位：维护单位编码、单位名称、数量精度、状态、排序和备注。
- 仓库：维护仓库编码、仓库名称、仓库类型、负责人、地址、状态和备注。
- 供应商：维护供应商编码、供应商名称、联系人、联系电话、状态和备注。
- 客户：维护客户编码、客户名称、联系人、联系电话、状态和备注。
- 物料分类：维护分类编码、分类名称、上级分类、状态、排序和备注。
- 物料档案：维护物料编码、物料名称、规格型号、物料类型、来源属性、所属分类、基本单位、状态和备注。

本阶段明确不做：

- BOM 结构、BOM 版本、替代料、损耗率。
- 库存数量、库存流水、锁定库存、批次、序列号、条码。
- 采购订单、销售订单、价格、应收应付、财务核算。
- 复杂审批、多组织、多租户、附件图片、复杂导入导出。
- 单位换算规则。计量单位一期只记录基本单位和精度。

## 用户场景

- 管理员维护单位、仓库、供应商、客户、物料分类，为后续业务模块提供可引用主数据。
- 工艺或计划人员创建原材料、半成品、成品、辅料等物料档案。
- 生产相关人员按编码、名称、分类、类型、状态快速检索物料。
- 管理员停用错误、废弃或暂不使用的基础资料，避免后续模块继续选用。
- 只读用户只能查看资料，不能新增、编辑、启用或停用。

## 数据模型

所有主数据表都应包含：

- `id`：后端生成主键。
- `code`：业务编码，同类主数据内唯一。
- `name`：业务名称。
- `status`：`ENABLED` 或 `DISABLED`，由后端枚举定义。
- `remark`：备注，可为空。
- `created_at`、`updated_at`：后端生成。

各实体补充字段：

- 计量单位：`precision_scale`、`sort_order`。
- 仓库：`warehouse_type`、`manager_name`、`address`。
- 供应商：`contact_name`、`contact_phone`。
- 客户：`contact_name`、`contact_phone`。
- 物料分类：`parent_id`、`sort_order`。
- 物料：`category_id`、`unit_id`、`material_type`、`source_type`、`specification`。

状态规则：

- 新建数据默认启用，除非创建请求显式传入停用。
- 已停用主数据仍可在历史数据中展示，后续业务新增或编辑选择时应限制或明确提示。
- 分类停用不自动停用子分类和物料；存在启用子分类或启用物料时禁止停用该分类，并返回明确原因。

编码规则：

- 编码由用户录入，后端校验唯一性和非空。
- 编码被业务数据引用后，后续阶段不得通过简单改名破坏追溯。
- 本阶段不提供系统自动编号，避免过早绑定企业编码规则。

## 接口设计

接口保持账号权限模块已建立的统一响应结构、分页结构、错误码和鉴权方式。

建议接口分组：

- `/api/admin/master/units`
- `/api/admin/master/warehouses`
- `/api/admin/master/suppliers`
- `/api/admin/master/customers`
- `/api/admin/master/material-categories`
- `/api/admin/master/materials`

每组提供一致能力：

- `GET` 列表：支持分页、关键词、状态筛选。
- `GET /{id}` 详情。
- `POST` 创建。
- `PUT /{id}` 更新。
- `PUT /{id}/enable` 启用。
- `PUT /{id}/disable` 停用。

物料列表额外支持分类、物料类型、来源属性筛选。物料分类列表一期返回平铺列表加 `parentId`，前端负责组装树形展示，避免后端为展示形态增加额外接口复杂度。

## 权限设计

菜单建议：

- 基础资料
  - 计量单位
  - 仓库管理
  - 供应商管理
  - 客户管理
- 物料管理
  - 物料分类
  - 物料档案

权限编码建议：

- `master:unit:view`、`master:unit:create`、`master:unit:update`
- `master:warehouse:view`、`master:warehouse:create`、`master:warehouse:update`
- `master:supplier:view`、`master:supplier:create`、`master:supplier:update`
- `master:customer:view`、`master:customer:create`、`master:customer:update`
- `master:material-category:view`、`master:material-category:create`、`master:material-category:update`
- `master:material:view`、`master:material:create`、`master:material:update`

启用和停用归入 `update` 权限。前端按钮显示只做体验控制，后端接口鉴权是最终安全边界。`/api/admin/**` 未映射接口应默认拒绝。

## 前端设计

页面结构沿用账号权限模块后台样式：

- 页面标题区：标题、说明、主操作按钮。
- 查询区：关键词、状态和必要业务筛选。
- 表格区：加载、空状态、错误状态、分页。
- 表单弹窗或抽屉：保存中、保存失败、必填校验。
- 操作区：编辑、启用、停用。

页面建议：

- 计量单位、仓库、供应商、客户：列表 + 创建/编辑弹窗。
- 物料分类：左侧分类树 + 右侧列表或详情，支持创建、编辑、启停。
- 物料档案：列表 + 详情抽屉 + 创建/编辑弹窗或抽屉。

表格列优先级：

- 基础资料默认列：编码、名称、状态、更新时间、操作。
- 仓库补充：仓库类型、负责人。
- 供应商和客户补充：联系人、联系电话。
- 物料档案默认列：物料编码、物料名称、规格型号、分类、基本单位、物料类型、状态、更新时间、操作。

移动端要求：

- 桌面生产办公优先。
- 窄屏下主内容不能被导航挤压，查询区允许折行。
- 表格允许横向滚动，不能出现列遮挡、文字穿透或按钮重叠。
- 弹窗或抽屉在窄屏下接近全宽，底部操作按钮可见。

## 错误处理与审计

必须覆盖的错误场景：

- 编码重复。
- 必填为空。
- 非法状态。
- 数据不存在。
- 物料引用不存在或已停用的单位、分类。
- 无权限访问。
- 保存失败后按钮恢复可操作，表单不误关闭。

写操作必须记录审计日志，至少覆盖创建、更新、启用、停用，记录操作人、目标类型、目标标识、动作、请求来源和操作时间。

## 测试与验收

自动化准出：

- 前端 `npm test`、`npm run typecheck`、`npm run build` 通过。
- 后端 Docker 化 Maven/Testcontainers 测试通过。
- 数据库迁移和初始化可从空库重复执行。

浏览器验收路径：

1. 使用管理员登录。
2. 进入基础资料菜单，新增计量单位、仓库、供应商、客户。
3. 进入物料管理菜单，新增物料分类。
4. 新增原材料、半成品、成品三类物料。
5. 查询、查看详情、编辑、停用物料。
6. 验证重复编码、必填缺失、停用引用等错误提示。
7. 使用只读角色登录，确认菜单、按钮和后端越权请求被限制。
8. 查看审计日志中基础资料和物料变更记录。

视觉验收截图：

- 基础资料菜单展开状态。
- 计量单位、仓库、供应商、客户列表和表单弹窗。
- 物料分类树和右侧内容联动。
- 物料档案列表、详情、表单。
- 必填校验、保存失败、启停确认、无权限状态。
- 窄屏下导航、查询区、表格和弹窗。

阻断缺陷：

- 本地无法启动或浏览器无法访问。
- 管理员无法完成核心新增、编辑、启停路径。
- 编码唯一约束失效。
- 无权限用户可绕过后端接口鉴权。
- 物料可引用不存在的单位或分类。
- 关键页面在桌面或窄屏下不可操作。

## 实施拆分建议

该规格可拆为以下实施任务：

1. 后端数据模型、迁移、初始化权限点和接口契约。
2. 后端基础资料与物料 API、鉴权、审计和测试。
3. 前端 API 封装、路由、菜单白名单和通用页面组件。
4. 前端基础资料页面和测试。
5. 前端物料分类、物料档案页面和测试。
6. 本地部署、浏览器功能验收、接口越权验收和视觉分析。

拆分时必须保持本规格范围，不得把 BOM、库存、采购销售或财务业务提前并入本阶段。
