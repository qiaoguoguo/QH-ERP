# BOM 管理设计规格

## 目标

在基础资料与物料管理之后，建立可被后续生产工单引用的标准 BOM 主数据能力。该阶段聚焦成品、半成品的用料结构、版本、明细、启用状态和数据校验，不进入库存扣减、生产执行或成本计算。

## 范围

本阶段采用“标准 BOM 主数据 + 版本管理 + 明细维护”范围：

- BOM 主表：维护 BOM 编码、父项物料、版本、名称、基准数量、基准单位、状态、生效日期、失效日期和备注。
- BOM 明细：维护行号、子项物料、用量、单位、损耗率和备注。
- 版本管理：同一父项物料可有多个版本，但同一时间只允许一个启用版本。
- 状态管理：草稿、启用、停用。
- 复制版本：从现有 BOM 复制主表和明细，生成新版本草稿。
- 数据校验：父项、子项、数量、损耗率、重复明细、父子相同、启用版本唯一和循环引用。
- 权限控制：查看、创建、更新、复制、启用、停用。
- 浏览器视觉分析：覆盖列表、详情、表单、明细、权限差异、异常、空状态和窄屏视口。

本阶段明确不做：

- 库存数量、库存流水、库存占用、缺料计算和批次序列号。
- 生产工单、领料、退料、报工、完工入库和排程。
- 成本卷算、成本核算、财务凭证、应收应付和总账。
- 采购订单、销售订单、采购销售价格和合同。
- 工艺路线、工序、工时、设备、模具和产能。
- 替代料、选配 BOM、配置 BOM、工程变更审批和附件图纸。
- 复杂多级展开 UI 或功能；本阶段只做可检测循环引用校验，不提供多层用料树。

## 用户场景

- 工艺或计划人员创建成品 BOM，维护原材料、半成品和辅料明细。
- 工艺或计划人员复制旧版本，调整用量后启用新版本。
- 生产相关人员查看当前启用版本，确认后续工单可引用的用料结构。
- 管理员停用不再使用的 BOM 版本，保留历史追溯。
- 只读用户只能查看 BOM，不能新增、编辑、复制、启用或停用。

## 数据模型

### BOM 主表

建议表名：`mfg_bom`。

字段：

- `id`：后端生成主键。
- `bom_code`：BOM 编码，全局唯一。
- `parent_material_id`：父项物料标识。
- `version_code`：版本号，同一父项物料内唯一。
- `name`：BOM 名称。
- `base_quantity`：基准数量，必须大于 0。
- `base_unit_id`：基准单位，默认等于父项物料基本单位。
- `status`：`DRAFT`、`ENABLED`、`DISABLED`。
- `effective_from`：生效日期，可为空。
- `effective_to`：失效日期，可为空。
- `remark`：备注，可为空。
- `created_by`、`created_at`、`updated_by`、`updated_at`、`enabled_by`、`enabled_at`、`version`：审计和乐观锁字段。

约束：

- `bom_code` 全局唯一。
- `(parent_material_id, version_code)` 唯一。
- PostgreSQL 部分唯一索引保证同一 `parent_material_id` 只允许一个 `status = 'ENABLED'`。
- `base_quantity > 0`。

### BOM 明细表

建议表名：`mfg_bom_item`。

字段：

- `id`：后端生成主键。
- `bom_id`：BOM 主表标识。
- `line_no`：行号，同一 BOM 内唯一。
- `child_material_id`：子项物料标识。
- `unit_id`：用量单位标识，默认等于子项物料基本单位。
- `quantity`：用量，必须大于 0。
- `loss_rate`：损耗率，默认 0，必须满足 `0 <= loss_rate < 1`。
- `remark`：备注，可为空。
- `created_at`、`updated_at`：时间字段。

约束：

- `(bom_id, line_no)` 唯一。
- `(bom_id, child_material_id)` 唯一。
- `quantity > 0`。
- `loss_rate >= 0 and loss_rate < 1`。

## 状态规则

- 新建 BOM 默认状态为 `DRAFT`。
- 草稿可编辑主表和明细。
- 启用版本不可直接编辑主表关键字段和明细；需要调整时复制为新版本草稿。
- 停用版本不可编辑明细，不作为后续生产工单默认可选。
- 启用 BOM 必须有至少一条明细。
- 启用 BOM 时必须再次校验父项、子项物料状态和循环引用。
- 启用新版本时，同一父项物料不得同时存在其他启用版本；一期采用显式失败返回 `BOM_ENABLED_VERSION_EXISTS`，由用户先停用旧版本后再启用新版本。

## 物料引用规则

- 父项物料必须为已启用物料。
- 父项物料类型必须为 `FINISHED_GOOD` 或 `SEMI_FINISHED`。
- 父项来源属性必须为 `SELF_MADE` 或 `OUTSOURCED`。
- 子项物料必须为已启用物料。
- 子项可为 `RAW_MATERIAL`、`SEMI_FINISHED`、`FINISHED_GOOD` 或 `AUXILIARY`，但不能等于父项物料。
- 同一 BOM 内同一子项物料不能重复。
- 明细单位必须为已启用计量单位；一期默认用子项物料基本单位，不提供单位换算。

## 循环引用规则

保存和启用时必须拒绝可检测循环引用。

校验口径：

- 当前 BOM 的父项物料不能直接出现在当前明细中。
- 若当前明细中的子项物料，沿现有 `DRAFT` 或 `ENABLED` BOM 的父子关系可追溯回当前父项物料，则判定为循环引用。
- 草稿也参与循环引用检测，避免保存明显不可用的 BOM 数据。
- 本阶段只给出错误提示，不提供多级展开树、路径可视化或成本展开。

错误码使用 `BOM_CYCLE_DETECTED`。

## 接口设计

接口保持账号权限模块和主数据模块已建立的统一响应结构、分页结构、错误码和鉴权方式。

接口路径：

- `/api/admin/boms`

能力：

- `GET /api/admin/boms`：列表，支持分页、关键词、父项物料和状态筛选。
- `GET /api/admin/boms/{id}`：详情，返回主表和明细。
- `POST /api/admin/boms`：创建草稿。
- `PUT /api/admin/boms/{id}`：更新草稿。
- `POST /api/admin/boms/{id}/copy`：复制为新版本草稿。
- `PUT /api/admin/boms/{id}/enable`：启用。
- `PUT /api/admin/boms/{id}/disable`：停用。

## 权限设计

菜单建议：

- 物料管理
  - BOM 管理

权限编码：

- `material:bom`：BOM 管理菜单，路由 `/materials/boms`，归属物料管理。
- `material:bom:view`：查看 BOM。
- `material:bom:create`：创建 BOM。
- `material:bom:update`：更新草稿 BOM。
- `material:bom:copy`：复制 BOM 版本。
- `material:bom:enable`：启用 BOM。
- `material:bom:disable`：停用 BOM。

前端按钮显示只做体验控制，后端接口鉴权是最终安全边界。

## 前端设计

页面结构沿用现有后台样式：

- 页面标题区：标题、说明、主操作按钮。
- 查询区：关键词、父项物料、状态。
- 表格区：加载、空状态、错误状态、分页和行操作。
- 详情抽屉或详情页：主表信息、版本状态、审计信息和明细列表。
- 创建/编辑页或大抽屉：基础信息表单和明细编辑表格。
- 操作确认：复制、启用、停用需要确认，启用失败要保留当前页面状态。

列表列优先级：

- BOM 编码。
- 父项物料编码。
- 父项物料名称。
- 版本。
- 明细数量。
- 状态。
- 更新时间。
- 操作。

明细编辑表格：

- 行号。
- 子项物料。
- 子项类型。
- 用量。
- 单位。
- 损耗率。
- 备注。
- 操作。

交互要求：

- 动态明细行使用稳定行标识，避免新增、删除、校验时错行。
- 表单错误需要有顶部汇总和行内提示。
- 草稿显示编辑、复制、启用、停用；启用显示查看、复制、停用；停用显示查看、复制。
- 无权限用户不显示写按钮，直接路由和接口仍由后端限制。
- 桌面端保持 ERP 后台表格信息密度；窄屏允许表格横向滚动，关键按钮不得重叠。

## 错误处理与审计

必须覆盖的错误场景：

- BOM 编码重复。
- 同父项版本重复。
- 同父项已有启用版本。
- BOM 不存在。
- 状态非法。
- 父项物料不存在、停用或类型不允许。
- 子项物料不存在或停用。
- 基准单位或明细单位不存在、停用或不允许引用。
- 明细为空。
- 明细用量不合法。
- 损耗率不合法。
- 明细子项重复。
- 父项等于子项。
- 可检测循环引用。
- 启用版本被直接编辑。
- 无权限访问。
- 保存失败后按钮恢复可操作，表单不误关闭。

写操作必须记录审计日志，至少覆盖创建、更新、复制、启用、停用，记录操作人、目标类型、目标标识、动作、请求来源和操作时间。

## 测试与验收

自动化准出：

- 前端 `npm test`、`npm run typecheck`、`npm run build` 通过。
- 后端 Docker 化 Maven/Testcontainers 测试通过。
- 数据库迁移和初始化可从空库重复执行。

浏览器验收路径：

1. 使用管理员登录。
2. 进入物料管理下的 BOM 管理。
3. 在空列表或初始列表状态下确认页面可用。
4. 新建成品或半成品 BOM 草稿，添加原材料、半成品和辅料明细。
5. 验证必填、重复版本、重复子项、用量错误、父子相同和循环引用错误。
6. 启用 BOM，确认列表状态、详情只读和审计记录。
7. 复制为新版本草稿，调整用量后验证启用版本唯一规则。
8. 使用只读角色登录，确认菜单、按钮和后端越权请求被限制。
9. 保存桌面和窄屏视觉截图。

视觉验收截图：

- BOM 列表默认状态。
- BOM 列表筛选结果。
- BOM 空状态。
- BOM 创建表单。
- BOM 明细编辑表格。
- 表单校验错误。
- 循环引用或重复版本错误。
- BOM 详情只读状态。
- 复制版本表单。
- 启用和停用确认。
- 只读权限状态。
- 窄屏列表、表单和明细表格。

阻断缺陷：

- 本地无法启动或浏览器无法访问。
- 管理员无法完成 BOM 创建、明细维护、启用、复制或停用。
- 同父项多个启用版本。
- 循环引用可保存或启用。
- 启用 BOM 无明细。
- 父项或子项可引用停用物料。
- 无权限用户可绕过后端接口鉴权。
- 关键页面在桌面或窄屏下不可操作。

## 实施拆分建议

该规格拆为以下实施任务：

1. 后端 BOM 迁移、错误码、权限种子和基础测试。
2. 后端 BOM 接口、状态规则、版本规则、循环引用校验、审计和集成测试。
3. 前端 BOM API 封装、路由、菜单白名单和权限测试。
4. 前端 BOM 列表、详情、表单、明细编辑和组件测试。
5. 本地部署、浏览器功能验收、接口越权验收和视觉分析。

拆分时必须保持本规格范围，不得把库存、工单、成本、采购销售、工艺路线、替代料、选配 BOM、工程变更审批或复杂多级展开 UI 提前并入本阶段。
