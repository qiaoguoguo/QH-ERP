# 成本归集基础设计规格

## 目标

成本归集基础阶段建立成本业务记录与来源追溯能力。系统必须把生产领料、报工、完工入库等生产执行结果沉淀为可查询、可鉴权、可审计的成本记录，并明确这些记录只是一期业务口径，不是正式财务成本核算结果。

## 阶段边界

本阶段覆盖：

- 成本记录列表、详情和手工记录。
- 已过账生产领料自动生成材料数量口径记录。
- 已过账生产报工自动生成人工数量口径记录。
- 已过账完工入库作为工单产出数量追溯。
- 工单成本汇总和生产工单详情成本归集分区。
- 成本记录到生产工单、来源单据、来源明细、物料和产品的追溯。
- 权限控制、审计记录、异常反馈和浏览器视觉分析。

本阶段不覆盖：

- 正式财务成本核算、总账、凭证、会计期间和结账。
- 标准成本、移动加权、先进先出、成本差异分析。
- 在产品成本、完工成本分摊、制造费用自动分配算法。
- 采购销售联动、应收应付联动。
- 多币种、多组织、多公司、多账套、多租户。
- 审批流、BI 报表和经营分析大屏。

## 已确认业务口径

- 成本归集一期只做业务记录和追溯。
- 自动材料来源记录只记录领料数量、单位、物料和来源，不自动计算材料金额。
- 自动人工来源记录只记录报工数量口径，不自动计算人工金额。
- 完工入库作为产出数量和归集对象追溯，不作为成本类型计入金额。
- 手工记录可补充人工金额、制造费用、其他成本或必要材料金额。
- 手工记录必须关联有效生产工单，产品从工单带出；本阶段不支持仅按产品录入成本记录。
- 自动生成记录不得修改来源工单、产品、来源单据、来源数量和来源物料。
- 成本记录不得反写库存余额、生产工单状态或生产单据状态。
- 页面文案必须避免暗示正式财务核算结果。

## 核心对象

### 成本记录

成本记录承载成本类型、来源单据、关联工单、产品、金额或数量口径、记录人和记录时间。

成本类型：

- 材料。
- 人工。
- 制造费用。
- 其他。

来源类型：

- 生产自动来源。
- 手工录入。

### 工单成本汇总

工单成本汇总用于在生产工单上下文中查看该工单已有成本记录和产出数量。汇总金额只汇总已有金额口径的业务记录；无金额的自动材料和人工来源记录只作为数量和来源口径展示。

### 来源追溯

来源追溯必须支持：

- 材料记录追溯到生产领料单、领料明细、工单 BOM 用料快照和库存流水。
- 人工记录追溯到生产报工单和工单。
- 完工产出追溯到完工入库单和库存流水。
- 手工记录追溯到工单、产品、记录人和说明。

## 后端设计

### 数据表

建议新增：

- `mfg_cost_record`：成本记录主表，包含自动来源记录和手工记录。

关键字段：

- `record_no`：成本记录编号，全局唯一。
- `work_order_id`：生产工单。
- `product_material_id`：产品物料快照，便于筛选。
- `cost_type`：成本类型。
- `source_type`：来源类型。
- `source_document_type`：来源单据类型。
- `source_document_id`：来源单据主表标识。
- `source_line_id`：来源明细标识。
- `work_order_material_id`：工单用料快照行。
- `material_id`、`unit_id`、`quantity`：物料和数量口径。
- `unit_price`、`amount`：手工单价和金额口径，允许为空。
- `basis_type`：口径类型。
- `business_date`、`remark`、`recorded_by`、`recorded_at`。
- `created_by`、`created_at`、`updated_by`、`updated_at`、`version`。

关键约束：

- 生产领料自动来源记录对 `source_document_type + source_line_id + cost_type` 唯一。
- 生产报工自动来源记录对 `source_document_type + source_document_id + cost_type` 唯一。
- 完工入库如保存产出追溯记录，对 `source_document_type + source_document_id + basis_type` 唯一。
- 手工记录必须有 `work_order_id`。
- `quantity`、`unit_price`、`amount` 如不为空必须大于等于 0，数量必须大于 0。
- 手工记录必须满足 `quantity` 或 `amount` 至少一个有效。
- `cost_type`、`source_type`、`source_document_type`、`basis_type` 必须受约束。

### 服务边界

建议新增独立成本模块：

- `CostAdminController`：成本管理接口入口。
- `CostAdminService`：列表、详情、手工记录、工单成本汇总。
- `CostRecordWriter` 或等价组件：供生产过账事务写入自动来源记录。

生产服务只在领料、报工、完工入库过账成功的同一事务中调用成本写入组件，不直接承担成本查询和维护职责。

### 事务规则

领料过账自动记录：

1. 生产领料过账完成库存扣减和流水写入。
2. 在同一事务内为每条领料明细创建材料成本来源记录。
3. 记录来源领料单、领料明细、工单、产品、物料、单位和数量。
4. 写入审计日志。
5. 成本写入失败时，整个领料过账回滚。

报工过账自动记录：

1. 生产报工过账完成工单累计数量更新。
2. 在同一事务内创建人工数量口径记录。
3. 记录来源报工单、工单、产品和报工数量。
4. 写入审计日志。
5. 成本写入失败时，整个报工过账回滚。

完工入库过账追溯：

1. 完工入库过账完成库存入账和流水写入。
2. 工单成本汇总读取完工入库作为产出数量追溯。
3. 如保存产出追溯记录，必须使用 `OUTPUT_QUANTITY_TRACE` 并明确不计入金额。

手工记录：

1. 校验工单存在且未取消。
2. 校验成本类型、口径类型、金额和数量。
3. 生成记录编号并写入成本记录。
4. 写入审计日志。

### 接口范围

- 成本记录：列表、详情、创建手工记录、更新手工记录。
- 工单成本汇总：按工单查看成本记录、金额口径和完工产出追溯。

## 前端设计

### 路由

- `/cost/records`：成本记录列表。
- `/cost/records/create`：新建手工成本记录。
- `/cost/records/:id`：成本记录详情。
- `/cost/records/:id/edit`：编辑手工成本记录。

### 页面

成本记录列表：

- 筛选项包括关键词、工单号、产品、成本类型、来源类型、来源单号和业务日期范围。
- 表格展示记录编号、成本类型、来源类型、来源单据、工单、产品、数量、金额、业务日期、记录人、记录时间、状态和操作。
- 操作列保持稳定宽度，主要操作为详情。

成本记录详情：

- 顶部展示成本类型、金额或数量口径、来源单据和状态摘要。
- 基础信息区展示工单、产品、来源类型、来源单号、业务日期、记录人、记录时间和备注。
- 来源区展示领料明细、报工数量、完工入库数量或手工说明。
- 追溯入口返回生产工单详情和来源单据所在工单分区。

手工记录表单：

- 字段包括工单、成本类型、口径类型、业务日期、数量、单位、单价、金额、来源说明和备注。
- 编辑模式只允许手工记录进入，不允许修改自动记录来源字段。
- 表单校验贴近字段，保存失败保留输入。
- 不提供复杂分摊、凭证或结账相关控件。

生产工单详情成本分区：

- 放置在库存流水摘要之后。
- 展示成本记录号、成本类型、来源单据、数量、金额、记录时间和状态。
- 有创建权限时提供新增手工成本记录入口。

### 视觉设计

- 延续现有后台 ERP 的高密度表格和克制状态标签。
- 金额和数量右对齐，使用稳定列宽和等宽数字。
- 长工单号、物料编码和来源单号使用省略或提示策略。
- 窄屏允许表格横向滚动，但关键按钮和错误提示必须可触达。
- 不使用财务看板、趋势图、营销式卡片或装饰性图表。

## 权限设计

建议权限点：

- `cost`。
- `cost:record:view`。
- `cost:record:create`。
- `cost:record:update`。

角色口径：

- 管理员：成本记录全量权限。
- 成本管理角色：查看、创建和更新手工成本记录。
- 生产管理人员：查看工单上下文成本追溯。
- 只读用户：查看成本记录和追溯信息。
- 无权限用户：无成本菜单和接口权限。

## 错误处理

必须提供明确业务错误：

- 成本记录不存在。
- 成本关联工单不存在。
- 工单状态不允许成本记录。
- 来源单据不存在。
- 来源单据未过账或状态无效。
- 来源记录重复归集。
- 成本类型不合法。
- 成本口径不合法。
- 金额或数量不合法。
- 自动生成记录来源字段不可修改。

前端展示原则：

- 接口错误用业务文案展示，不只显示错误码。
- 表单校验错误贴近字段。
- 保存或生成失败后保留当前输入，恢复按钮状态。
- 权限不允许的操作不可见或禁用，后端仍必须拒绝越权请求。

## 测试策略

后端测试：

- 迁移、枚举约束和唯一约束。
- 领料过账自动生成材料来源记录。
- 报工过账自动生成人工数量记录。
- 完工入库在工单成本汇总中可追溯。
- 手工记录创建、更新、金额和数量校验。
- 重复来源、草稿来源、无效工单、自动记录不可修改。
- 权限、CSRF、审计和错误码。
- 成本记录不影响库存余额和生产状态。

前端测试：

- 路由守卫和菜单权限。
- 成本记录列表筛选、分页、空状态、错误状态。
- 成本记录详情展示和追溯入口。
- 手工记录表单校验和保存失败。
- 手工记录编辑权限、编辑模式和自动记录不可编辑。
- 生产工单详情成本分区和只读权限。
- 窄屏关键状态。

浏览器验收：

- 真实本地服务完成管理员主路径。
- 覆盖成本管理角色、只读用户和无权限用户。
- 归档 `docs/testing/cost-collection-visual-audit/` 截图和分析结论。

## 主代理决策结论

- 下一阶段名称定为“成本归集基础记录”。
- 采用一个阶段完成成本业务记录、自动来源、手工记录和工单成本追溯。
- 成本归集一期只做业务记录和追溯，不做正式财务核算。
- 自动材料和人工记录只做数量来源口径，不自动计算金额。
- 完工入库作为产出数量追溯，不作为成本金额来源。
- 手工记录用于人工金额、制造费用、其他成本和必要金额补录。
- 不新增第 6 类子代理；审查仍由既定五类角色承担。

## 自检记录

- 本规格没有使用未确认的总账、凭证、标准成本、移动加权、成本差异、制造费用自动分配或复杂财务报表范围。
- 设计范围与 `docs/product/phase-one-scope.md` 中第六个里程碑一致。
- 接口、测试和任务文档均采用同一阶段名称和同一视觉分析目录。
