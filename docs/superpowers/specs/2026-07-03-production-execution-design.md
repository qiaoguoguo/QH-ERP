# 生产执行基础设计规格

## 目标

生产执行基础阶段建立生产工单、领料、报工和完工入库能力。系统必须让生产任务、BOM 快照、库存变动和生产结果可追溯、可鉴权、可审计，并为后续成本归集提供稳定来源。

## 阶段边界

本阶段覆盖：

- 生产工单创建、编辑草稿、发布、完成和取消。
- 工单发布时生成 BOM 用料快照。
- 从工单发起生产领料，过账后扣减库存。
- 从工单发起生产报工，过账后记录合格、不良和累计报工数量。
- 从工单发起完工入库，过账后增加库存。
- 工单详情追溯 BOM 快照、领料、报工、完工入库和库存流水。
- 权限控制、审计记录、异常反馈和浏览器视觉分析。

本阶段不覆盖：

- 高级排产、工艺路线、工序流转、设备负荷。
- 批次、序列号、库位、条码。
- 倒冲、退料、补料、返工、拆解、委外。
- 采购销售联动、审批流、附件图纸、工程变更。
- 正式成本核算、成本差异、财务凭证。
- 多单位换算、多组织、多公司、多租户。

## 已确认业务口径

- 生产执行围绕工单上下文展开，不建立独立生产单据中心。
- 工单编号由后端自动生成。
- 工单发布时引用启用 BOM，并生成用料快照。
- 工单发布后产品、BOM、计划数量、领料仓库、入库仓库不再编辑。
- 一期不允许超领、超报、超入。
- 一期不启用库存占用锁定，领料过账时实时扣减库存。
- 报工不直接影响库存。
- 完工入库数量不得超过累计合格报工数量。
- 工单完成要求累计入库数量等于计划数量。
- 所有写操作必须写审计。
- 生产相关库存流水必须能追溯到来源生产明细、生产单据、工单和 BOM 快照。

## 核心对象

### 生产工单

生产工单承载生产任务、计划数量、产品物料、关联 BOM、计划日期和状态流转。

状态：

- 草稿：可编辑、可发布、可取消。
- 已发布：已生成 BOM 快照，可领料、报工、完工入库。
- 生产中：已有领料、报工或完工入库记录。
- 已完成：生产闭环完成，以查看和追溯为主。
- 已取消：已终止，只允许在未发生已过账业务时取消。

### 工单用料快照

工单用料快照由启用 BOM 明细在发布时生成，固定记录生产任务所依据的用料结构。

关键字段：

- 来源 BOM 和 BOM 明细。
- 子项物料。
- 单位。
- 应领数量。
- 已领数量。
- 损耗率。

### 生产领料

生产领料从工单用料快照发起，过账后扣减库存并生成库存流水。

状态：

- 草稿：可编辑、可过账。
- 已过账：已扣库存，不可编辑。

### 生产报工

生产报工记录生产结果，包括合格数量、不良数量、报工日期和报工人。报工过账后更新工单累计数量，但不直接影响库存。

### 完工入库

完工入库从工单发起，过账后增加产品库存并生成库存流水。入库数量不得超过累计合格报工数量扣除已入库数量。

## 后端设计

### 数据表

建议新增：

- `mfg_work_order`：生产工单主表。
- `mfg_work_order_material`：工单用料快照。
- `mfg_material_issue`：生产领料主表。
- `mfg_material_issue_line`：生产领料明细。
- `mfg_work_report`：生产报工单。
- `mfg_completion_receipt`：完工入库单。

建议修改：

- `inv_stock_movement` 的变动类型约束增加 `PRODUCTION_ISSUE` 和 `PRODUCTION_RECEIPT`。

关键约束：

- 工单编号全局唯一。
- 生产领料单号、报工单号、完工入库单号全局唯一。
- 工单用料快照对 `work_order_id + line_no` 唯一。
- 领料明细对 `issue_id + work_order_material_id` 唯一。
- 库存变动继续使用 `source_type + source_line_id` 唯一约束防止重复入账。

### 事务规则

发布工单必须在单个事务内完成：

1. 校验工单存在且为草稿。
2. 校验产品物料启用且类型允许。
3. 校验 BOM 启用、父项匹配且明细非空。
4. 校验领料仓库、入库仓库和单位启用。
5. 生成工单用料快照。
6. 更新工单状态为已发布。
7. 写入审计日志。

领料过账必须在单个事务内完成：

1. 校验领料单存在且为草稿。
2. 锁定领料单、工单和用料快照行。
3. 校验累计领料不超过应领数量。
4. 锁定库存余额。
5. 校验库存充足。
6. 写入生产领料库存流水。
7. 更新库存余额和用料快照已领数量。
8. 更新领料单状态和工单状态。
9. 写入审计日志。

报工过账必须在单个事务内完成：

1. 校验报工单存在且为草稿。
2. 锁定报工单和工单。
3. 校验合格数量和不良数量合计大于 0。
4. 校验累计报工不超过计划数量。
5. 更新工单累计报工、合格和不良数量。
6. 更新报工单状态和工单状态。
7. 写入审计日志。

完工入库过账必须在单个事务内完成：

1. 校验入库单存在且为草稿。
2. 锁定入库单和工单。
3. 校验累计入库不超过累计合格报工数量。
4. 锁定库存余额。
5. 写入完工入库库存流水。
6. 更新库存余额和工单累计入库数量。
7. 更新入库单状态和工单状态。
8. 写入审计日志。

### 接口范围

- 工单：列表、详情、创建、更新草稿、发布、完成、取消。
- 领料：工单内列表、详情、创建、更新草稿、过账。
- 报工：工单内列表、详情、创建、更新草稿、过账。
- 完工入库：工单内列表、详情、创建、更新草稿、过账。

## 前端设计

### 路由

- `/production/work-orders`：生产工单列表。
- `/production/work-orders/create`：新建生产工单。
- `/production/work-orders/:id`：生产工单详情。
- `/production/work-orders/:id/edit`：编辑草稿工单。
- `/production/work-orders/:id/material-issues`：工单领料。
- `/production/work-orders/:id/reports`：工单报工。
- `/production/work-orders/:id/completion-receipts`：工单完工入库。

### 页面

工单列表：

- 筛选项包括关键词、状态、产品物料和计划日期范围。
- 表格展示工单号、产品、BOM 版本、计划数量、已报工、已入库、计划日期、状态和操作。
- 操作列按权限和状态展示详情、编辑、发布、领料、报工、完工入库、完成和取消。

工单表单：

- 独立页面承载，不使用普通小弹窗。
- 字段分组为基础信息、产品与 BOM、计划信息、仓库和备注。
- 选择产品和 BOM 后展示 BOM 明细预览。

工单详情：

- 顶部展示状态、产品、计划数量、报工数量、入库数量和关键操作。
- 分区展示 BOM 用料快照、领料记录、报工记录、完工入库记录和库存流水摘要。
- 详情页是领料、报工和完工入库主入口。

领料、报工、完工入库页面：

- 以工单上下文打开。
- 保留工单摘要，避免用户脱离业务上下文。
- 表单错误贴近字段展示，并保留顶部错误摘要。

### 视觉设计

- 延续现有后台 ERP 的高密度表格和克制状态标签。
- 数量列右对齐。
- 操作列设置稳定宽度。
- 长工单号、物料名称和 BOM 版本使用省略规则，并确保详情页可查看完整值。
- 窄屏允许表格横向滚动，但关键按钮和错误提示必须可触达。

## 权限设计

建议权限点：

- `production`。
- `production:work-order:view/create/update/release/complete/cancel`。
- `production:issue:view/create/update/post`。
- `production:report:view/create/update/post`。
- `production:receipt:view/create/update/post`。

角色口径：

- 管理员：生产执行全量权限。
- 生产管理人员：工单创建、更新、发布、完成、取消和查看执行记录。
- 仓库人员：查看已发布工单，创建、更新、过账领料和完工入库。
- 生产人员：查看已发布工单，创建、更新、过账报工。
- 只读用户：查看工单和追溯信息。
- 无权限用户：无生产菜单和接口权限。

## 错误处理

必须提供明确业务错误：

- 工单不存在。
- 工单状态不允许当前操作。
- 产品物料无效。
- BOM 无效或未启用。
- BOM 明细为空。
- 仓库、物料或单位无效。
- 数量不合法。
- 领料超过应领数量。
- 库存不足。
- 报工超过计划数量。
- 入库超过累计合格报工数量。
- 已过账单据不可编辑。
- 重复过账。

前端展示原则：

- 接口错误用业务文案展示，不只显示错误码。
- 表单校验错误贴近字段。
- 过账失败后保留当前输入，恢复按钮状态。
- 权限不允许的操作不可见或禁用，后端仍必须拒绝越权请求。

- BOM 快照生成和追溯。

- 报工过账、超报。

- 真实本地服务完成管理员主路径。
- 覆盖仓库人员、生产人员、只读用户和无权限用户。
- 归档 `docs/testing/production-execution-visual-audit/` 截图和分析结论。

## 主代理决策结论

- 下一阶段名称定为“生产执行基础能力”。
- 采用一个阶段打通工单、领料、报工、完工入库闭环，不拆成多个更小模块。
- 领料、报工、完工入库先作为工单上下文能力，不建立独立单据中心。
- 报工采用草稿和过账模式，便于统一审计和防重复提交。
- 一期禁止超领、超报、超入。
- 一期不启用库存锁定和缺料分析。
- 窄屏侧栏占用空间不单独作为阻断；若导致核心操作不可见、表单不可提交或权限状态无法识别，则作为阻断视觉缺陷。

- 接口、测试和任务文档均采用同一阶段名称和同一视觉分析目录。
