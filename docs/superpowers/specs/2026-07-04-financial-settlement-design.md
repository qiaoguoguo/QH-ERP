# 财务往来基础模块设计规格

## 目标

财务往来基础模块建立制造业采购入库和销售出库之后的业务往来台账，让系统从“采购、生产、销售的业务单据闭环”推进到“应收、应付和收付款登记”的基础管理闭环。

本阶段主链路为：

```text
销售出库 -> 应收台账 -> 收款登记 -> 应收余额 -> 来源追溯
采购入库 -> 应付台账 -> 付款登记 -> 应付余额 -> 来源追溯
```

本阶段只建立业务往来台账、收付款登记、余额核销和来源追溯能力，不承诺总账、凭证、会计科目、会计期间、结账、税务申报、完整发票生命周期、银行对账、正式收入确认、正式成本结转或 BI 报表。

## 阶段边界

本阶段覆盖：

- 财务往来菜单，包含应收台账、收款记录、应付台账和付款记录四个入口。
- 基于已过账销售出库生成应收草稿。
- 应收列表、详情、编辑草稿、确认、取消、关闭。
- 基于已确认或部分收款应收登记收款草稿。
- 收款列表、详情、编辑草稿、过账和取消草稿。
- 基于已过账采购入库生成应付草稿。
- 应付列表、详情、编辑草稿、确认、取消、关闭。
- 基于已确认或部分付款应付登记付款草稿。
- 付款列表、详情、编辑草稿、过账和取消草稿。
- 应收、收款、应付、付款与来源销售出库、销售订单、采购入库、采购订单之间的追溯。
- 管理员、财务角色、销售角色、采购角色、只读用户和无权限用户的权限差异验收。
- 空状态、加载状态、异常状态、无权限状态和浏览器视觉分析。

本阶段不覆盖：

- 总账、凭证、科目、会计期间、结账、正式财务报表。
- 税务申报、税率管理、销项税、进项税、发票红冲、发票认证或发票作废。
- 银行对账、票据生命周期、收款认领、付款审批、报销、固定资产或工资。
- 多币种、汇兑损益、多组织、多公司、多账套、多租户。
- 销售退货、采购退货、退款、退票、已过账收付款反冲或调整单。
- 经营 BI、指标看板、财务驾驶舱或大屏。

## 信息架构

财务往来作为独立一级菜单，不并入采购、销售、库存或成本模块。

建议菜单结构：

- 财务往来
  - 应收台账
  - 收款记录
  - 应付台账
  - 付款记录

信息架构原则：

- 应收台账表达销售出库之后客户应付给企业的业务金额，不等同于收入确认。
- 应付台账表达采购入库之后企业应付给供应商的业务金额，不等同于成本结转。
- 收款和付款表达业务登记与余额核销，不等同于银行对账或凭证入账。
- 来源追溯必须让用户知道金额来自哪张采购入库或销售出库。
- 页面不得出现总账、凭证、税务、正式发票或报表看板入口。

## 路由与页面形态

建议路由：

- `/finance`：按当前账号拥有的首个财务查看权限动态重定向，优先级为应收、收款、应付、付款；仅有应付查看权限的采购角色应进入 `/finance/payables`；无任一财务查看权限时进入无权限页。
- `/finance/receivables`：应收台账列表。
- `/finance/receivables/create`：基于销售出库生成应收。
- `/finance/receivables/:id`：应收详情。
- `/finance/receivables/:id/edit`：编辑应收草稿。
- `/finance/receivables/:id/receipts/create`：基于应收登记收款。
- `/finance/receipts`：收款记录列表。
- `/finance/receipts/:id`：收款详情。
- `/finance/receipts/:id/edit`：编辑收款草稿。
- `/finance/payables`：应付台账列表。
- `/finance/payables/create`：基于采购入库生成应付。
- `/finance/payables/:id`：应付详情。
- `/finance/payables/:id/edit`：编辑应付草稿。
- `/finance/payables/:id/payments/create`：基于应付登记付款。
- `/finance/payments`：付款记录列表。
- `/finance/payments/:id`：付款详情。
- `/finance/payments/:id/edit`：编辑付款草稿。

页面形态沿用采购、销售和成本模块已经通过视觉审计的后台 ERP 模式：整页列表、整页表单、整页详情，不使用营销式首屏、卡片堆叠看板或复杂弹窗承载主流程。

## 业务对象与状态

### 应收台账

应收台账承载客户、来源销售出库、来源销售订单、业务日期、到期日期、应收金额、已收金额、未收金额和状态。

中文状态：

- 草稿
- 待收款
- 部分收款
- 已收清
- 已关闭
- 已取消

### 收款记录

收款记录承载应收、客户、收款日期、收款金额、收款方式、状态和备注。

中文状态：

- 草稿
- 已过账
- 已取消

已过账收款必须只读，不允许编辑或取消。

### 应付台账

应付台账承载供应商、来源采购入库、来源采购订单、业务日期、到期日期、应付金额、已付金额、未付金额和状态。

中文状态：

- 草稿
- 待付款
- 部分付款
- 已付清
- 已关闭
- 已取消

### 付款记录

付款记录承载应付、供应商、付款日期、付款金额、付款方式、状态和备注。

中文状态：

- 草稿
- 已过账
- 已取消

已过账付款必须只读，不允许编辑或取消。

## 状态标签视觉口径

状态标签沿用现有 Element Plus 标签和采购、销售模块视觉口径，颜色克制，不使用大面积状态底色。

建议标签类型：

- 草稿：默认灰色。
- 待收款、待付款：蓝色或信息色。
- 部分收款、部分付款：黄色或警示色。
- 已收清、已付清、已过账：绿色或成功色。
- 已关闭：灰色或信息色。
- 已取消：红色或危险色。

不得在页面可见文案中直接暴露 `DRAFT`、`CONFIRMED`、`PARTIALLY_RECEIVED`、`RECEIVED`、`PARTIALLY_PAID`、`PAID`、`POSTED`、`CLOSED`、`CANCELLED` 等枚举值。

## 应收列表

用途：按客户、状态、日期、来源和关键词扫描销售来源应收，判断收款进度。

筛选项：

- 关键词：应收单号、客户、来源销售出库、来源销售订单。
- 客户。
- 状态。
- 业务日期范围。
- 到期日期范围。
- 来源单号。

表格列：

- 应收单号。
- 客户。
- 来源销售出库。
- 来源销售订单。
- 业务日期。
- 到期日期。
- 状态。
- 应收金额。
- 已收金额。
- 未收金额。
- 更新时间。
- 操作。

行操作：

- 详情。
- 编辑，仅草稿且有更新权限。
- 确认，仅草稿且有确认权限。
- 取消，仅草稿或未收款待收状态且有取消权限。
- 关闭，仅待收款或部分收款且有关闭权限。
- 登记收款，仅待收款或部分收款且有收款创建权限。

## 应收生成与编辑表单

用途：基于已过账销售出库生成应收草稿，或编辑应收草稿的非来源字段。

来源摘要：

- 来源销售出库单号。
- 来源销售订单号。
- 客户。
- 业务日期。
- 来源金额。
- 来源明细数量。

可编辑字段：

- 到期日期，必填。
- 备注，可选。

交互要求：

- 来源选择只允许已过账且未生成应收的销售出库。
- 来源选择通过财务候选来源接口加载，避免只在提交后才发现来源已生成往来。
- 来源和金额只读展示，不允许前端改写。
- 重复来源、未过账来源、来源不存在必须显示业务化错误。
- 保存失败后保留输入并恢复按钮。

## 应收详情

用途：查看应收主信息、金额进度、来源追溯、收款记录和操作记录。

顶部操作：

- 返回列表。
- 编辑，仅草稿且有更新权限。
- 确认，仅草稿且有确认权限。
- 取消，仅草稿或未收款待收状态且有取消权限。
- 关闭，仅待收款或部分收款且有关闭权限。
- 登记收款，仅待收款或部分收款且有收款创建权限。

摘要区：

- 应收金额。
- 已收金额。
- 未收金额。
- 收款进度。
- 状态。

分区：

- 基础信息。
- 来源追溯。
- 收款记录。
- 操作记录。

交互要求：

- 已收清、已关闭、已取消状态展示只读提示，不依赖按钮消失让用户猜。
- 没有收款记录时展示“暂无收款记录”。
- 来源追溯必须能进入来源销售出库和销售订单详情。

## 收款列表

用途：扫描收款草稿和已过账收款，查看客户收款情况。

筛选项：

- 关键词：收款单号、应收单号、客户、来源单号。
- 客户。
- 状态。
- 收款日期范围。
- 应收标识或应收单号。

表格列：

- 收款单号。
- 应收单号。
- 客户。
- 收款日期。
- 收款金额。
- 收款方式。
- 状态。
- 创建人。
- 过账人。
- 过账时间。
- 操作。

行操作：

- 详情。
- 编辑，仅草稿且有更新权限。
- 过账，仅草稿且有过账权限。
- 取消，仅草稿且有取消权限。

## 收款表单与详情

收款表单顶部展示应收摘要：

- 应收单号。
- 客户。
- 应收金额。
- 已收金额。
- 未收金额。
- 来源销售出库。
- 来源销售订单。

表单字段：

- 收款日期，必填。
- 收款金额，必填，必须大于 0 且不得超过未收金额。
- 收款方式，必填。
- 备注，可选。

交互要求：

- 收款金额输入和展示均使用两位小数。
- 超额收款在前端给出提示，后端仍必须最终拒绝。
- 过账操作必须二次确认，确认文案包含收款单号和金额。
- 已过账详情展示“已过账收款只读”。

## 应付与付款页面

应付和付款页面与应收、收款对称，但业务对象替换为供应商、采购入库和采购订单。

应付生成表单的来源选择通过财务候选来源接口加载，只允许已过账且未生成应付的采购入库。

应付列表重点列：

- 应付单号。
- 供应商。
- 来源采购入库。
- 来源采购订单。
- 业务日期。
- 到期日期。
- 状态。
- 应付金额。
- 已付金额。
- 未付金额。
- 操作。

付款表单顶部展示应付摘要：

- 应付单号。
- 供应商。
- 应付金额。
- 已付金额。
- 未付金额。
- 来源采购入库。
- 来源采购订单。

付款字段：

- 付款日期。
- 付款金额。
- 付款方式。
- 备注。

交互要求：

- 付款金额不得超过未付金额。
- 已过账付款只读。
- 来源追溯必须能进入来源采购入库和采购订单详情。

## 金额与进度展示

- 金额列全部右对齐。
- 金额统一展示两位小数。
- 摘要区优先展示数字，例如“已收 3,000.00 / 应收 10,000.00”。
- 列表中可使用窄进度条辅助识别，但不得把进度条做成页面视觉中心。
- 状态和金额必须同时可见，不能只靠颜色判断结清状态。

## 权限展示

建议权限点：

- `finance`：财务往来菜单。
- `finance:receivable:view/create/update/confirm/cancel/close`。
- `finance:receipt:view/create/update/post/cancel`。
- `finance:payable:view/create/update/confirm/cancel/close`。
- `finance:payment:view/create/update/post/cancel`。

角色口径：

- 管理员：财务往来全量权限。
- 财务角色：应收、收款、应付、付款查看和维护权限。
- 销售角色：方向级查看销售来源应收和收款进度，不做个人数据范围过滤；不显示应付和付款入口，不显示财务确认、登记和过账按钮。
- 采购角色：方向级查看采购来源应付和付款进度，不做个人数据范围过滤；不显示应收和收款入口，不显示财务确认、登记和过账按钮。
- 只读用户：查看台账、详情和追溯，不显示写操作。
- 无权限用户：无财务往来菜单和接口权限，直接访问财务路由进入无权限页。

页面表现：

- 菜单按权限显示，缺少财务权限不展示财务往来入口。
- 按钮按权限和状态共同收敛，不能只按权限展示。
- 只读用户不显示生成、编辑、确认、取消、关闭、登记、过账按钮。
- 后端接口鉴权是最终安全边界。

## 空、加载、异常和无权限状态

空状态：

- 暂无应收台账。
- 暂无收款记录。
- 暂无应付台账。
- 暂无付款记录。
- 暂无来源追溯。
- 暂无操作记录。

加载状态：

- 应收台账加载中。
- 收款记录加载中。
- 应付台账加载中。
- 付款记录加载中。
- 财务往来详情加载中。

异常状态：

- 应收不存在。
- 应付不存在。
- 收款不存在。
- 付款不存在。
- 来源单据不存在。
- 来源单据未过账，不能生成往来台账。
- 来源已生成往来台账，不能重复生成。
- 收款金额不能超过未收金额。
- 付款金额不能超过未付金额。
- 已收清台账不可继续登记收款。
- 已付清台账不可继续登记付款。
- 已过账收款不可编辑或取消。
- 已过账付款不可编辑或取消。

无权限状态：

- 使用统一无权限页和文案“当前账号没有访问该功能的权限”。
- 无权限状态不得显示半截业务页面或空白页。

## 视觉设计口径

整体风格：

- 延续现有 ERP 后台风格，左侧导航、顶部账号区、内容区标题、筛选区、表格和详情分区保持一致。
- 页面应服务财务往来录入、核对和追溯，不做经营看板、财务驾驶舱或装饰化大留白。
- 列表、表单、详情优先保证扫描效率和业务语义准确性。

表格：

- 应收、收款、应付、付款均为高密度表格。
- 金额字段右对齐。
- 状态标签使用中文。
- 操作列固定右侧，宽度稳定。
- 长客户名、长供应商名、长单号允许省略，但详情页必须能查看完整值。
- 宽表允许横向滚动；滚动条、固定列阴影和操作列不得遮挡关键金额和状态。

表单：

- 来源摘要必须位于表单顶部。
- 主表字段按两到三列布局，窄屏下纵向排列。
- 金额输入必须明确单位和精度。
- 错误摘要在表单上方可见，字段级错误贴近控件。

详情：

- 顶部摘要区展示金额进度、日期和状态。
- 基础信息、来源追溯、收付款记录、操作记录分区清楚。
- 已完成或已过账只读状态使用提示条或状态提示。

窄屏：

- 窄屏侧栏未折叠是已知全站风险，本阶段可作为阶段内优化项，不作为阶段启动阻断。
- 若窄屏侧栏、表格或按钮遮挡核心金额、状态、来源或保存过账操作，则必须阻断修复。
- 窄屏下筛选区、详情摘要和表单主字段应可纵向阅读；宽表允许横向滚动。

## 规格自检

- 本规格没有纳入总账、凭证、科目、税务、完整发票、银行对账、正式收入确认、正式成本结转或 BI 报表。
- 页面模式复用采购、销售和成本阶段已经通过视觉审计的列表、筛选、表单、详情、状态标签、权限按钮和追溯模式。
- 应收、收款、应付、付款状态均要求中文标签，不允许枚举值直接外露。
- 收付款核销只描述业务台账与余额更新，不表达正式银行到账、付款审批或凭证入账。
- 窄屏侧栏作为阶段内可优化项，不作为启动阻断；若遮挡核心操作则升级为阻断缺陷。
