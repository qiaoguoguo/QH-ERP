# 销售管理基础模块设计规格

## 目标

销售管理基础模块建立制造业成品或可销售物料离开企业的业务入口，让系统从“采购入库增加库存”推进到“销售出库扣减库存”的基础经营闭环。

本阶段主链路为：

```text
客户 -> 销售订单 -> 销售出库 -> 库存扣减 -> 库存流水 -> 来源追溯
```

本阶段只建立销售订单、销售出库、库存扣减和来源追溯能力，不承诺应收、发票、收款、税务、正式财务凭证、销售退货、物流发运、审批流、CRM 客户画像或报表看板。

## 阶段边界

本阶段覆盖：

- 销售管理菜单，包含销售订单和销售出库两个入口。
- 销售订单列表、详情、新建草稿、编辑草稿、确认、取消和关闭。
- 销售订单明细维护，包括物料、单位、销售数量、销售单价、预计交付日期和备注。
- 基于已确认或部分出库销售订单创建销售出库草稿。
- 销售出库草稿编辑、列表、详情查看和过账。
- 销售出库过账后扣减库存余额，生成销售出库库存流水。
- 销售订单、销售出库和库存流水三方追溯。
- 管理员、销售员、仓库角色、只读用户和无权限用户的权限差异验收。
- 空状态、加载状态、异常状态、无权限状态和浏览器视觉分析。

本阶段不覆盖：

- 报表看板化界面、经营大屏、营销页或装饰化概览页。
- CRM 客户画像、客户跟进、商机、报价、合同或价格策略。
- 应收账款、发票、收款、税务、总账联动或正式财务凭证。
- 销售退货、换货、物流发运、承运商、运单、签收或发货批次。
- 销售审批流、信用额度审批、价格审批或合同审批。
- 多币种、多组织、多公司、多账套、多租户。
- 批次、序列号、库位、条码、销售包装单位和多单位换算。

销售单价只作为销售业务记录字段和后续扩展基础，不生成应收、不计算税额、不形成正式收入或财务核算结果。

## 主数据展示口径

### 可销售物料

本阶段不新增单独“可销售”字段，销售订单表单按现有物料类型判断可选范围。

- 可作为销售订单明细物料：启用的成品 `FINISHED_GOOD`、启用的半成品 `SEMI_FINISHED`。
- 不可作为销售订单明细物料：原材料 `RAW_MATERIAL`、辅料 `AUXILIARY`，即使处于启用状态，本阶段也不展示在销售订单物料选择器中。
- 若接口、历史数据或异常输入导致不可销售物料进入校验链路，UI 错误文案使用“本阶段仅支持成品和半成品作为销售订单明细物料”，避免出现“可销售字段”之类不存在的模型暗示。

### 客户启停

- 销售订单创建、更新和确认时，客户必须处于启用状态。
- 销售订单确认后，若客户后续被停用，不阻止基于该订单创建销售出库、编辑销售出库草稿或过账销售出库。
- 销售出库表单和详情继续展示来源客户名称、来源订单号和订单状态，不把来源客户停用展示为出库阻断提示。
- 销售出库是否可继续流转主要由来源订单状态、出库数量、库存可用量、仓库启用状态和出库单状态决定。

## 信息架构

销售管理作为独立一级菜单，不并入库存管理、采购管理或客户主数据。

建议菜单结构：

- 销售管理
  - 销售订单
  - 销售出库

信息架构原则：

- 销售订单表达客户需求和销售承诺，不直接改变库存。
- 销售出库表达仓库实际发出货物，是库存扣减动作。
- 库存流水只展示库存变化事实，并能跳回销售出库来源。
- 页面文案必须持续区分销售订单、销售出库和库存单据，避免用户误以为进入应收、发票、收款、物流或 CRM 流程。

## 业务对象与状态

### 销售订单

销售订单承载客户、订单日期、预计交付日期、销售物料、销售数量、销售单价和累计出库进度。

状态标签：

- `DRAFT`：草稿，可编辑、可确认、可取消，不允许出库。
- `CONFIRMED`：已确认，可创建销售出库，可关闭，不允许编辑关键字段。
- `PARTIALLY_SHIPPED`：部分出库，可继续出库，可关闭未出库余量。
- `SHIPPED`：全部出库，不再允许继续出库，可关闭归档。
- `CLOSED`：已关闭，不允许出库。
- `CANCELLED`：已取消，不允许出库。

中文展示：

- 草稿
- 已确认
- 部分出库
- 全部出库
- 已关闭
- 已取消

### 销售订单明细

销售订单明细记录客户订购物料和数量口径。

关键字段：

- 行号。
- 物料。
- 单位，使用物料基本单位。
- 销售数量。
- 已出库数量。
- 未出库数量。
- 销售单价。
- 预计交付日期。
- 备注。

### 销售出库

销售出库必须基于销售订单创建，过账后扣减库存余额并生成库存流水。

状态标签：

- `DRAFT`：草稿，可编辑、可过账。
- `POSTED`：已过账，已写库存余额和库存流水，不可编辑，不可重复过账。

中文展示：

- 草稿
- 已过账

销售出库明细必须引用销售订单明细，物料、单位和来源行必须一致。

## 状态标签视觉口径

状态标签沿用现有 Element Plus 标签和采购管理视觉口径，颜色克制，不使用大面积状态底色。

建议标签类型：

- 草稿：默认灰色。
- 已确认：绿色或成功色。
- 部分出库：蓝色或信息色，用于提示仍有后续动作。
- 全部出库：绿色或成功色。
- 已关闭：灰色或信息色。
- 已取消：红色或危险色。
- 已过账：绿色或成功色。

不得在页面可见文案中直接暴露 `DRAFT`、`CONFIRMED`、`PARTIALLY_SHIPPED`、`SHIPPED`、`CLOSED`、`CANCELLED`、`POSTED` 等枚举值。

## 路由与页面形态

建议路由：

- `/sales`：重定向到 `/sales/orders`。
- `/sales/orders`：销售订单列表。
- `/sales/orders/create`：新建销售订单。
- `/sales/orders/:id`：销售订单详情。
- `/sales/orders/:id/edit`：编辑销售订单草稿。
- `/sales/orders/:id/shipments/create`：基于销售订单创建销售出库。
- `/sales/shipments`：销售出库列表。
- `/sales/shipments/:id`：销售出库详情。
- `/sales/shipments/:id/edit`：编辑销售出库草稿。

页面形态沿用采购订单、采购入库、库存流水和 `MasterDataTableView` 的后台 ERP 模式：整页列表、整页表单、整页详情，不使用营销式首屏、卡片堆叠看板或复杂弹窗承载主流程。

用途：按客户、状态、日期和关键词扫描销售订单，执行订单级状态操作。

筛选项：

- 关键词：订单号、客户、物料。
- 客户。
- 状态。
- 订单日期范围。
- 预计交付日期范围。

表格列：

- 订单号。
- 客户。
- 订单日期。
- 预计交付日期。
- 状态。
- 总数量。
- 已出库。
- 未出库。
- 行数。
- 创建人。
- 更新时间。
- 操作。

行操作：

- 详情。
- 编辑，仅草稿且有更新权限。
- 确认，仅草稿且有确认权限。
- 取消，仅草稿或已确认且有取消权限。
- 关闭，仅已确认或部分出库且有关闭权限。
- 创建出库，仅已确认或部分出库且有销售出库创建权限。

交互要求：

- 查询和重置按钮位于筛选区末尾。
- 表格空态展示“暂无销售订单”。
- 加载态展示“销售订单加载中”。
- 错误态展示可理解中文错误，并保留筛选条件。
- 操作列固定在右侧；按钮较多时允许换行或保持小号文字按钮，不得遮挡状态列。

## 销售订单表单

用途：创建或编辑销售订单草稿。

主表字段：

- 客户，必填，仅可选择启用客户；创建、更新和确认时客户停用应展示明确错误。
- 订单日期，必填。
- 默认预计交付日期，可选。
- 备注，可选。

明细编辑器字段：

- 物料，必填，仅可选择启用的成品 `FINISHED_GOOD` 和半成品 `SEMI_FINISHED`。
- 单位，只读展示物料基本单位。
- 销售数量，必填，必须大于 0。
- 销售单价，必填，必须大于等于 0。
- 预计交付日期，可默认取主表预计交付日期。
- 备注，可选。

交互要求：

- 明细使用表格编辑器，支持新增明细和删除明细。
- 物料下拉不得展示原材料 `RAW_MATERIAL` 和辅料 `AUXILIARY`；若用户通过异常路径提交这些物料，错误提示为“本阶段仅支持成品和半成品作为销售订单明细物料”。
- 数量和单价按字符串提交，避免前端浮点误差。
- 同一销售订单内同一物料是否允许重复，应由业务规则明确；若不允许，错误文案为“同一销售订单内物料不能重复”。
- 表单顶部保留错误摘要，字段附近展示校验提示。
- 保存按钮在提交中禁用，失败后恢复并保留输入。
- 非草稿订单进入编辑页时展示“仅草稿销售订单可编辑”，保存按钮不可用。

- 关闭，仅已确认或部分出库且有关闭权限。
- 创建出库，仅已确认或部分出库且有出库创建权限。

摘要区：

- 总数量。
- 已出库。
- 未出库。
- 订单日期。
- 状态。

基础信息：

- 订单号。
- 客户。
- 预计交付日期。
- 明细行数。
- 创建人。
- 创建时间。
- 更新时间。
- 备注。

销售明细：

- 行号、物料、规格、单位、销售数量、已出库、未出库、销售单价、预计交付日期、备注。

出库记录：

- 出库单号。
- 出库仓库。
- 业务日期。
- 状态。
- 出库数量。
- 过账人。
- 过账时间。
- 操作：详情。

交互要求：

- 确认、取消、关闭使用二次确认。
- 已取消、已关闭、全部出库状态下不显示创建出库。
- 没有出库记录时展示“暂无出库记录”。
- 出库记录状态必须展示中文标签“草稿”或“已过账”。

## 销售出库表单

用途：基于销售订单创建或编辑销售出库草稿。

来源订单摘要：

- 来源订单号。
- 客户，展示来源订单客户名称；来源订单确认后客户被停用时不展示阻断提示。
- 订单未出库数量。
- 订单状态。

主表字段：

- 出库仓库，必填，仅可选择启用仓库。
- 业务日期，必填。
- 备注，可选。

出库明细编辑器字段：

- 来源订单行，必填。
- 物料，只读或随来源订单行带出。
- 单位，只读展示物料基本单位。
- 订单数量。
- 已出库。
- 未出库。
- 本次出库数量，必填，必须大于 0 且不得超过未出库数量。
- 仓库，表头或主表展示出库仓库；如后续支持行级仓库，本阶段不启用。
- 备注，可选。

交互要求：

- 默认带出来源订单仍有未出库余量的订单行。
- 编辑草稿时，来源行候选必须包含当前出库单已有来源行和同一订单仍有未出库余量的来源行。
- 同一销售出库内来源订单行不得重复。
- 来源订单确认后客户被停用，不影响销售出库草稿创建、编辑和过账；页面只展示客户名称和订单状态，不出现与客户停用相关的出库阻断文案。
- 库存不足应在保存或过账时给出明确错误，不得让用户误以为已过账。
- 已过账销售出库进入编辑页时展示“已过账销售出库不可编辑”，保存按钮不可用。

## 追溯展示

销售订单详情：

- 展示出库记录摘要。
- 出库记录可跳转到销售出库详情。
- 出库数量、过账状态、过账人和过账时间必须清楚。

销售出库详情：

- 展示来源订单摘要。
- 展示库存流水追溯。
- 库存变动方向必须为“出库”，数量变化应易扫描。

库存流水列表：

- 增加销售出库来源类型，底层固定使用 `SALES_SHIPMENT`，页面展示为“销售出库”。
- 销售出库流水行应支持查看来源单据，跳转到销售出库详情。
- 不得把销售出库流水跳到库存单据详情或采购入库详情。

## 权限展示

建议权限点：

- `sales`：销售管理菜单。
- `sales:order:view`：查看销售订单。
- `sales:order:create`：创建销售订单。
- `sales:order:update`：更新销售订单草稿。
- `sales:order:confirm`：确认销售订单。
- `sales:order:cancel`：取消销售订单。
- `sales:order:close`：关闭销售订单。
- `sales:shipment:view`：查看销售出库。
- `sales:shipment:create`：创建销售出库。
- `sales:shipment:update`：更新销售出库草稿。
- `sales:shipment:post`：过账销售出库。

角色口径：

- 管理员：销售订单和销售出库全量权限。
- 销售员：销售订单查看、创建、更新、确认、取消、关闭；销售出库查看。
- 仓库角色：销售订单查看；销售出库查看、创建、更新、过账；库存流水查看。
- 只读用户：销售订单、销售出库和追溯查看。
- 无权限用户：无销售菜单和销售接口权限，直接访问销售路由进入无权限页。

页面表现：

- 菜单按权限显示，缺少销售权限不展示销售管理入口。
- 按钮按权限和状态共同收敛，不能只按权限展示。
- 只读用户不显示编辑、确认、取消、关闭、创建出库、过账。
- 仓库角色不显示销售订单确认、取消、关闭等销售员动作。
- 销售员不显示创建出库和过账动作。
- 后端接口鉴权是最终安全边界，前端权限表现只用于减少误操作。

## 空、加载、异常和无权限状态

空状态：

- 销售订单列表：暂无销售订单。
- 销售出库列表：暂无销售出库。
- 订单详情出库记录：暂无出库记录。
- 出库详情库存流水：暂无库存流水。

加载状态：

- 销售订单加载中。
- 销售订单详情加载中。
- 销售订单表单加载中。
- 销售出库加载中。
- 销售出库详情加载中。
- 销售出库表单加载中。

异常状态：

- 销售订单不存在。
- 销售出库不存在。
- 销售订单详情加载失败。
- 销售出库详情加载失败。
- 仅草稿销售订单可编辑。
- 仅已确认或部分出库销售订单可创建销售出库。
- 已过账销售出库不可编辑。
- 本阶段仅支持成品和半成品作为销售订单明细物料。
- 本次出库数量不能超过未出库数量。
- 库存不足，不能过账销售出库。
- 来源订单行与出库明细不一致。
- 客户已停用，不能创建、更新或确认销售订单。
- 仓库已停用，不能过账销售出库。

异常展示边界：

- 销售出库表单和详情不得因来源订单客户后续停用展示阻断错误。
- 若销售出库创建或过账失败，应优先展示订单状态、库存不足、仓库停用、出库数量超限等与出库直接相关的错误。

无权限状态：

- 使用统一无权限页和文案“当前账号没有访问该功能的权限”。
- 无权限状态不得显示半截业务页面或空白页。

## 视觉设计口径

整体风格：

- 延续现有 ERP 后台风格，左侧导航、顶部账号区、内容区标题、筛选区、表格和详情分区保持一致。
- 页面应服务高频业务处理，不做营销页、客户画像页、经营看板或装饰化大留白。
- 列表、表单、详情优先保证扫描效率和业务语义准确性。

表格：

- 销售订单、销售出库和库存流水均为高密度表格。
- 数量和单价字段右对齐。
- 状态标签使用中文。
- 操作列固定右侧，宽度稳定。
- 长客户名、长物料名、长单号允许省略，但详情页必须能查看完整值。
- 宽表允许横向滚动；滚动条、固定列阴影和操作列不得遮挡关键字段。

表单：

- 主表字段按两到三列布局，窄屏下纵向排列。
- 明细编辑器使用稳定列宽，关键输入包括物料、单位、销售数量、销售单价、预计交付、备注。
- 销售出库明细必须突出订单数量、已出库、未出库、本次出库和仓库信息。
- 错误摘要在表单上方可见，字段级错误贴近控件。

详情：

- 顶部摘要卡展示数量进度、日期和状态。
- 基础信息、明细、出库记录、库存流水追溯分区清楚。
- 已过账只读状态使用提示条或状态提示，不依赖按钮缺失让用户猜测。

窄屏：

- 窄屏侧栏未折叠是已知全站风险，本阶段可作为阶段内优化项，不作为阶段启动阻断。
- 若窄屏侧栏、表格或按钮遮挡核心操作，导致无法查看状态、无法保存、无法过账或无法返回，则必须阻断修复。
- 窄屏下筛选区、详情摘要和表单主字段应可纵向阅读；明细表允许横向滚动。

## 视觉风险与处理口径

宽表风险：

- 销售订单、销售出库、库存流水列数多，可能出现横向滚动。
- 处理口径：保留固定操作列和稳定列宽；关键数量、状态和操作必须在桌面视口可识别。

长文本风险：

- 客户名、物料名、规格、订单号、出库单号可能较长。
- 处理口径：列表省略，详情完整展示；不得因省略导致来源单据、状态或数量不可判断。

按钮拥挤风险：

- 销售订单行操作在草稿和已确认状态较多。
- 处理口径：按状态和权限收敛按钮；危险动作使用克制文字按钮和二次确认。

语义混淆风险：

- 销售出库容易被误读为普通库存出库、物流发运或应收确认。
- 处理口径：页面标题、按钮、状态、流水类型统一使用“销售出库”；不得出现发票、收款、应收、发货物流等文案。

权限误导风险：

- 销售员和仓库角色职责不同。
- 处理口径：销售员侧重订单动作，仓库侧重出库动作，只读无动作，无权限进入无权限页。

## 规格自检

- 本规格没有纳入报表看板化界面、营销页、CRM 客户画像、应收、发票、收款、税务、凭证、销售退货、物流发运或审批流。
- 页面模式复用采购管理基础阶段已经通过视觉审计的列表、筛选、表单、详情、状态标签、权限按钮和追溯模式。
- 销售订单状态和销售出库状态均要求中文标签，不允许枚举值直接外露。
- 销售出库库存联动只描述 UI 和交互口径；业务实现必须保证过账后库存扣减、流水生成和来源追溯一致。
- 窄屏侧栏作为阶段内可优化项，不作为启动阻断；若遮挡核心操作则升级为阻断缺陷。
