# 任务记录：项目启动基线

## 任务目标

建立项目启动前的基础文档体系，为后续技术选型、工程初始化和模块开发提供统一依据。

## 输入

- 用户确认项目方向：制造业单公司 Web ERP，生产管理优先。
- 用户确认主代理负责整体项目推进。
- `AGENTS.md` 中的协作、计划、分支和验收规则。
- 5 个子代理围绕启动前准备事项输出的建议。

## 预期成果

- 恢复并完善 `README.md`。
- 建立 `docs/` 文档索引。
- 建立产品、技术、UI、测试、部署和任务模板基线。

## 验收标准

- 文档能够说明项目定位、阶段目标和协作规则。
- 文档能够指导下一步技术选型和首个模块任务拆分。
- 不写入未确认的完整 ERP 细节方案。
- 不编写业务代码。

## 当前结论

启动基线作为第一阶段准备成果，不代表项目最终方案定稿。后续每个模块仍需单独任务文档、需求说明、验收标准和测试结论。
