# 任务记录：技术选型与工程骨架

## 任务目标

形成技术选型与工程骨架决策，为后续初始化前端、后端、数据库、测试和本地验收环境提供明确依据。

## 输入

- [项目启动基线](../project-startup-baseline.md)
- [技术基线](../architecture/technical-baseline.md)
- [接口契约基线](../api/api-contract-baseline.md)
- [前端工程基线](../frontend/frontend-baseline.md)
- [测试计划](../testing/test-plan.md)
- 5 个子代理围绕技术选型与工程骨架输出的建议。
- 本机运行环境检查结果。

## 本次包含

- 明确前后端分离分层单体架构。
- 明确前端、后端、数据库、测试和本地部署的推荐技术路线。
- 明确工程骨架最小内容。
- 明确当前环境缺口和处理原则。

## 本次不包含

- 不初始化业务代码。
- 不创建数据库表。
- 不实现账号权限模块。
- 不推送远端分支。

## 角色分工

- 产品经理：确认技术路线必须服务一期可验收业务能力。
- UI 设计师：确认工程骨架应支撑后台高信息密度界面和设计验收。
- 前端开发：提出前端技术路线、工程骨架内容和风险。
- 后端开发：提出后端技术路线、数据一致性、权限、审计和迁移要求。
- 测试：提出质量门槛、验收命令和账号权限模块验证关注点。
- 主代理：汇总决策、落地文档、执行验证，并准备下一阶段工程初始化。

## 验收标准

- 技术选型能支撑制造业单公司 ERP 的长期扩展。
- 不因本机 Java 8 环境降低后端技术标准。
- 后续工程初始化有明确目录、技术、测试和部署依据。
- 文档通过基础格式检查。

## 风险与处理

- Java 环境不足：工程初始化前升级 JDK 21 或更高版本。
- 组件库能力不满足 ERP 表格场景：工程骨架阶段必须做表格能力验证页面。
- 后端权限粒度不清：账号权限模块任务文档必须先定义角色、权限、菜单和接口鉴权边界。
- 测试环境不可复现：使用 Docker 和项目内脚本固化启动与验证步骤。
