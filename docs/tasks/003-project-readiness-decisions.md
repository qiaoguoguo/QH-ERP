# 任务记录：项目开工准备决策

## 任务目标

汇总 5 个子代理对项目正式开工前缺口的讨论结论，形成首个模块、一期验收方式、接口契约、主数据、UI、前端、后端、测试和变更控制的可执行基线。

## 输入

- 用户授权主代理作为项目负责人独立推动项目落地。
- [仓库规范](../../AGENTS.md)
- [项目启动基线](../project-startup-baseline.md)
- [一期范围与里程碑](../product/phase-one-scope.md)
- [技术选型决策记录](../architecture/technology-decision.md)
- 5 个子代理按 `/goal` 输出的产品、UI、前端、后端、测试建议。

## 本次包含

- 确认首个业务模块为账号与权限基础。
- 确认一期采用模块逐步验收，并在生产主链路完成后追加端到端闭环验收。
- 确认成本归集一期先做到业务记录，不承诺形成正式财务核算结果。
- 补齐接口契约、主数据、UI、前端、测试和部署验收基线。
- 补充需求变更、缺陷阻断和阶段成果通知规则。

## 本次不包含

- 不实现账号权限模块代码。
- 不创建生产、库存、BOM 或财务业务表。
- 不合入或推送主分支。
- 不通知用户进行业务成果验收。

## 角色分工

- 产品经理：确认首个模块、一期验收方式、成本归集深度和账号权限需求边界。
- UI 设计师：确认后台 ERP 页面模式、权限表现和浏览器设计验收标准。
- 前端开发：确认前端模块规范、权限表现、接口联调和验证命令。
- 后端开发：确认接口契约、数据迁移、初始化数据、审计和权限模型。
- 测试：确认测试准入准出、账号权限验收范围、缺陷阻断规则和回归边界。
- 主代理：汇总决策、更新文档、维护后续任务边界。

## 决策结论

### 首个模块

首个业务模块确认为账号与权限基础。原因是后续物料、BOM、库存、生产、采购、销售、财务和报表都依赖登录、用户、角色、菜单、按钮和接口权限边界。

### 一期验收方式

一期采用“模块逐步验收 + 生产主链路端到端闭环验收”。每个模块完成后必须本地部署、浏览器可访问、用户可操作；生产主链路完成后追加从物料、BOM、库存、工单、领料、报工、完工入库到成本记录的闭环验收。

### 成本归集深度

成本归集一期做到业务记录，记录材料、人工、制造费用或其他成本来源及其关联工单、产品和单据。正式财务核算结果不作为一期承诺，避免在物料、库存和生产数据尚未稳定前提前承诺核算准确性。

## 后续任务顺序

1. 完成工程骨架初始化并通过基础验证。
2. 创建账号与权限基础模块任务文档、需求文档和测试验收范围。
3. 调度 5 个子代理围绕账号与权限模块进行正式设计讨论。
4. 在独立任务分支实现账号权限模块。
5. 完成测试、部署和浏览器验收后，再通知用户进行阶段成果查验。

## 验收标准

- 项目开工前关键决策已写入文档。
- 后续账号权限模块有明确输入和边界。
- 接口、数据、UI、前端、测试、部署文档具备可执行口径。
- 文档未扩展到完整 ERP 详细方案。

## 执行记录

- 计划：先收集子代理结论，再补齐文档，最后继续工程骨架验证。
- 变更：形成首个模块和一期验收方式决策。
- 验证：文档完成后执行基础文件检查和 Git 状态检查。
- 结论：已补齐项目开工准备决策、账号权限范围、主数据规则、接口契约、前端、UI、测试、部署和变更控制基线。
