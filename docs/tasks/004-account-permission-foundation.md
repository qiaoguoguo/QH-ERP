# 任务记录：账号与权限基础模块

## 任务目标

实现 QH ERP 的首个业务基础模块，使企业内部用户能够登录系统，并按角色权限访问菜单、页面、按钮和后端接口，为后续物料、BOM、库存和生产模块提供统一权限底座。

## 背景与输入

- 关联业务背景：本项目是制造业单公司内部 Web ERP，所有后续模块都依赖账号、角色、菜单、按钮和接口权限。
- 关联文档：
  - [账号与权限基础模块范围](../product/account-permission-scope.md)
  - [账号与权限需求说明](../product/account-permission-requirements.md)
  - [账号与权限接口契约](../api/account-permission-api.md)
  - [账号与权限测试计划](../testing/account-permission-test-plan.md)
  - [接口契约基线](../api/api-contract-baseline.md)
  - [本地开发与验证说明](../ops/local-development.md)
- 已确认约束：
  - 只支持企业内部用户。
  - 不做外部账号、多租户、复杂审批、单点登录和第三方登录。
  - 后端接口鉴权是最终安全边界，前端权限表现只作为用户体验控制。
  - 阶段成果必须本地部署、浏览器可访问、用户可操作。

## 范围

### 本次包含

- 登录、退出、当前用户信息。
- 初始超级管理员初始化。
- 用户列表、创建、编辑、启用、停用、重置密码。
- 角色列表、创建、编辑、启用、停用。
- 用户分配角色。
- 角色分配菜单权限和按钮权限。
- 后端接口权限校验。
- 停用账号访问拦截。
- 账号、角色、权限变更审计记录。
- 前端后台布局、登录页、用户管理、角色管理、权限配置、无权限反馈。
- 本地部署、接口、前端、后端和浏览器验收。

### 本次不包含

- 外部客户、供应商或会员账号。
- 多公司、多组织、多租户隔离。
- 复杂审批流。
- 单点登录或第三方登录。
- 完整字段级权限配置界面。
- 按仓库、部门或业务线配置复杂数据范围。
- 完整操作审计查询报表，仅实现账号权限变更的基础审计写入和必要查询。

- 测试：执行功能、接口、权限、异常、回归和部署验收。
- 主代理：统筹计划、调度子代理、集成实现、运行验证、提交阶段结果。

## 验收标准

- 功能验收：
  - 初始超级管理员可登录。
  - 管理员可创建用户、角色并分配角色权限。
  - 不同角色登录后菜单和按钮表现不同。
  - 用户停用后不能登录，已登录访问也应被后端拒绝。
- 数据验收：
  - 用户名、角色编码、权限编码具备唯一约束。
  - 初始化数据可从空库迁移并幂等创建。
  - 账号、角色、权限变更有基础审计记录。
- 权限验收：
  - 未登录访问受保护页面和接口会被拦截。
  - 无权限用户直接请求接口返回无权限。
  - 权限变更以后端实时校验为准；前端菜单和按钮通过刷新当前用户或重新登录更新。
- UI 验收：
  - 登录、后台布局、用户列表、角色列表、权限配置、无权限页具备加载、空状态、错误和成功反馈。
  - 页面保持 ERP 后台信息密度，不做营销化布局。
- 测试验收：
  - 前端 `npm test`、`npm run typecheck`、`npm run build` 通过。
  - 后端 Docker 化 JDK 21 + Maven + Testcontainers 测试通过。
  - 权限和接口关键场景具备自动化测试或明确浏览器验收记录。
- 部署验收：
  - `docker compose up -d postgres` 可启动本地数据库。
  - 后端和前端可本地启动。
  - 用户可通过浏览器完成核心验收路径。

## 风险与处理

- 权限只做前端隐藏：必须由后端接口鉴权覆盖，测试直接调用接口验证。
- 权限变更生效口径不清：本模块明确后端实时生效，前端通过刷新当前用户或重新登录更新展示。
- 初始化管理员不可用：作为阻断缺陷处理。
- 密码和会话安全不足：采用 HttpOnly Cookie + 服务端 Session，新增、修改、删除接口保留 CSRF 或等效校验。
- 菜单树和权限树半选规则争议：需求和接口文档中明确父子层级和半选只属于前端展示状态，后端保存明确权限点集合。

## 变更控制

- 变更原因：账号权限模块是首个业务基础模块，需要在实现前明确可验收范围。
- 影响范围：前端、后端、数据库、接口、测试、部署验收。
- 涉及文件：`docs/product`、`docs/api`、`docs/testing`、`apps/web`、`apps/api`。
- 验收影响：任何范围变更都会影响浏览器验收路径和自动化测试。
- 决策依据：5 个子代理 `/goal` 输出结论和项目负责人汇总判断。

## 缺陷与阻断

- 阻断缺陷：初始化管理员不可用、登录不可用、未登录可访问受保护资源、越权接口成功、停用账号仍可访问、本地无法部署、浏览器核心路径不可执行。
- 严重缺陷：权限展示与后端结果不一致、审计缺失、错误提示不可定位、角色权限保存后无法恢复展示。
- 已知问题：完整字段级权限和复杂数据范围不在本期实现范围。
- 是否允许进入验收：无阻断缺陷，严重缺陷有明确处理结论且不影响本次范围时才允许。
