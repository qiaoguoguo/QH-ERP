# 任务记录：基础资料与物料管理

## 任务目标

建立制造业 ERP 一期生产主链路所需的主数据底座，覆盖计量单位、仓库、供应商、客户、物料分类和物料档案，为后续 BOM、库存和生产工单提供可引用资料。

## 验收标准

- 管理员可完成六类主数据的新增、查询、编辑、启用、停用。
- 物料可引用已启用单位和已启用分类，不能引用不存在或停用的单位/分类。
- 单位、仓库、供应商、客户、分类、物料编码各自唯一。
- 存在启用子分类或启用物料时，分类不能停用。
- 存在启用物料引用时，计量单位不能停用。
- 只读角色只能查看，不能新增、编辑、启用、停用。
- 无权限用户直接调用后端写接口返回 `AUTH_FORBIDDEN`。
- 写操作生成审计日志。
- 前端 `npm test`、`npm run typecheck`、`npm run build` 通过。
- 后端 Docker 化 Maven/Testcontainers 测试通过。
- 本地部署后浏览器完成核心验收路径，并保存视觉分析记录。

## 风险与处理

- 权限只在前端隐藏：必须通过后端接口鉴权覆盖，测试直接调用写接口验证 `AUTH_FORBIDDEN`。
- 主数据编码重复或后续引用混乱：后端唯一约束和服务层校验必须同时覆盖，错误码使用 `MASTER_DATA_CODE_EXISTS`。
- 物料引用停用资料：创建和编辑物料时必须校验单位与分类存在且启用，失败返回 `MASTER_DATA_REFERENCE_INVALID`。
- 分类停用影响后续引用：存在启用子分类或启用物料时禁止停用，失败返回 `MASTER_DATA_CATEGORY_IN_USE`。
- 计量单位停用影响启用物料：存在启用物料引用时禁止停用，失败返回 `MASTER_DATA_UNIT_IN_USE`。
- 页面信息密度不足或视觉遮挡：浏览器验收必须覆盖桌面和窄屏截图，发现影响核心操作的问题先修复再验收。

## 变更控制

- 变更原因：基础资料与物料管理是账号权限后的第二个业务模块，需要在实施前明确范围、接口和测试边界。
- 影响范围：后端数据模型、接口、权限、审计、前端菜单页面、自动化测试、本地部署和浏览器验收。
- 涉及文件：`docs/tasks`、`docs/api`、`docs/testing`、`docs/product`、后续实现涉及 `apps/api` 和 `apps/web`。
- 验收影响：任何范围、字段、权限或验收路径变更都会影响接口契约、测试计划和浏览器验收记录。
- 决策依据：已确认设计规格、实施计划和产品决策记录。

## 缺陷与阻断

- 阻断缺陷：系统无法启动、核心页面无法浏览器访问、关键接口不可用、权限绕过、关键业务数据错误、数据库迁移失败、阶段验收路径不可执行。
- 严重缺陷：编码唯一性失效、物料引用停用或不存在资料、分类停用规则失效、计量单位被启用物料引用时仍可停用、写操作审计缺失、权限表现与后端鉴权不一致、视觉问题影响核心操作。
- 已知问题：BOM、库存数量与流水、采购销售单据、财务核算、复杂审批、多组织、多租户、附件图片、复杂导入导出和单位换算规则不在本阶段范围。
- 是否允许进入验收：无阻断缺陷，严重缺陷完成影响评估并不影响本阶段核心路径时才允许。
