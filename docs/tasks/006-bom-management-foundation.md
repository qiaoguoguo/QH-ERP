# 任务记录：BOM 管理

## 任务目标

建立制造业 ERP 一期生产主链路所需的标准 BOM 主数据能力，使成品和半成品能够维护可追溯、可鉴权、可审计的用料结构和版本，为后续库存、生产工单、领料、报工、完工入库和成本归集提供稳定引用依据。

## 背景与输入

- 关联业务背景：本项目是制造业单公司内部 Web ERP，当前阶段在基础资料与物料管理之后推进，BOM 是生产主链路中连接物料档案、生产工单、领料和成本归集的关键主数据。

  - [BOM 管理接口契约](../api/bom-management-api.md)
  - [BOM 管理测试计划](../testing/bom-management-test-plan.md)
  - [BOM 管理设计规格](../superpowers/specs/2026-07-03-bom-management-design.md)
  - [BOM 管理实施计划](../superpowers/plans/2026-07-03-bom-management-implementation-plan.md)
  - [一期范围与里程碑](../product/phase-one-scope.md)
  - [核心业务流程](../product/business-flow.md)
- 已确认约束：
  - 本阶段只建立 BOM 主数据、版本、明细和启停规则，不进入库存、生产工单、成本或采购销售业务流转。
  - BOM 可保存为草稿；启用版本可被后续工单默认引用；停用版本保留历史可追溯。
  - 启用 BOM 时同一父项物料只能存在一个启用版本。
  - 保存和启用时必须拒绝可检测的循环引用；本阶段不提供复杂多级展开页面或功能。
  - 权限必须以后端接口鉴权为最终边界，前端按钮控制只作为用户体验。
  - 阶段成果必须本地部署、浏览器可访问，并保存核心路径视觉分析记录。
  - 视觉问题如影响核心操作、关键数据识别、权限状态判断或工作台扫描效率，必须修复后再进入阶段验收。

## 范围

### 本次包含

- BOM 列表：按关键词、父项物料和状态查询，支持分页、状态标签和行操作。
- BOM 详情：展示父项物料、版本、状态、基准数量、有效日期、审计信息和明细行。
- BOM 新增：选择父项物料、填写 BOM 编码、版本、名称、基准数量、有效日期、备注并维护明细。
- BOM 编辑：草稿可编辑主表和明细；启用和停用版本只读，不直接修改明细。
- BOM 明细：维护行号、子项物料、用量、单位、损耗率和备注，校验必填、数量、重复子项和父子关系。
- 版本管理：复制现有 BOM 为新版本，启用新版本时确保同父项仅一个启用版本。
- 状态管理：草稿、启用、停用；支持启用和停用操作。
- 数据规则：父项、子项必须引用已启用物料；父项与子项不能相同；可检测循环引用必须拒绝。
- 后端权限点、菜单入口、接口鉴权和审计记录。
- 前端菜单、路由守卫、按钮权限、错误反馈和浏览器视觉验收。

### 本次不包含

- 库存数量、库存流水、锁定库存、批次、序列号、条码。
- 生产工单、领料、报工、完工入库、生产排程和工序流转。
- 成本卷算、成本核算、财务凭证、应收应付和总账。
- 采购订单、销售订单、销售报价、采购询价和价格体系。
- 工艺路线、工序、工时、设备、模具和产能规则。
- 替代料、选配 BOM、配置 BOM、工程变更审批和附件图纸。
- 复杂多级展开 UI 或功能；本阶段只做循环引用校验，不展示多层用料树。

## 角色分工

- 产品经理：确认 BOM 目标、字段范围、状态规则、版本规则、循环校验口径和验收路径。
- UI 设计师：确认列表、详情、表单、明细表格、版本操作、只读状态、异常状态和视觉验收标准。
- 前端开发：实现菜单、路由、页面、明细编辑、按钮权限、错误反馈、响应式适配和前端测试。
- 后端开发：实现数据模型、接口、事务、唯一约束、循环校验、状态规则、接口鉴权、审计记录和后端测试。
- 测试：验证功能、接口、权限、异常、审计、本地部署和浏览器视觉验收。
- 主代理：统筹任务边界、分支实施、自检验证、缺陷处理、提交记录和阶段交付判断。

## 验收标准

- 管理员可创建 BOM 草稿，并维护至少两条明细。
- BOM 编码全局唯一；同一父项物料的版本唯一。
- 草稿可编辑；启用和停用版本不可直接编辑明细。
- 启用 BOM 必须有至少一条明细。
- 同一父项物料只能存在一个启用 BOM 版本。
- 启用新版本时必须明确处理旧启用版本，系统不得同时保留两个启用版本。
- 父项物料必须为已启用的成品或半成品，且来源属性为自制或委外。
- 子项物料必须为已启用物料，可覆盖原材料、半成品、辅料等类型。
- 基准单位和明细单位必须为已启用计量单位。
- 明细用量必须大于 0；损耗率为空时按 0 处理，填写时必须在 `0 <= 损耗率 < 1` 范围内。
- 同一 BOM 内同一子项物料不能重复。
- 父项物料不能作为自己的子项。
- 保存和启用时必须拒绝可检测的循环引用。
- 只读角色只能查看，不能新增、编辑、复制、启用、停用。
- 无权限用户直接调用后端写接口返回 `AUTH_FORBIDDEN`。
- 写操作生成审计日志。
- 前端 `npm test`、`npm run typecheck`、`npm run build` 通过。
- 后端 Docker 化 Maven/Testcontainers 测试通过。
- 本地部署后浏览器完成核心验收路径，并保存视觉分析记录。

## 风险与处理

- 循环引用导致后续工单用料展开异常：保存和启用时基于现有 BOM 关系加当前明细做可检测循环校验，失败返回 `BOM_CYCLE_DETECTED`。
- 同父项多个启用版本导致工单引用歧义：数据库部分唯一索引和服务层事务校验必须同时覆盖，失败返回 `BOM_ENABLED_VERSION_EXISTS`。
- 启用 BOM 引用停用物料：创建、编辑和启用时都校验父项、子项状态，失败返回明确错误码。
- 明细表格字段多导致页面拥挤：桌面优先保证表格扫描效率，窄屏允许横向滚动，关键按钮和错误提示不得遮挡。
- 前端只隐藏按钮但接口可越权：后端接口鉴权覆盖所有写接口，测试直接调用写接口验证 `AUTH_FORBIDDEN`。
- 版本编辑破坏追溯：启用版本不可直接修改明细，必须复制新版本调整。

## 变更控制

- 变更原因：BOM 管理是一阶段生产主链路的第三个业务模块，需要在实施前明确范围、接口、数据规则和视觉验收要求。
- 影响范围：后端数据模型、接口、权限、审计、前端菜单页面、自动化测试、本地部署和浏览器验收。
- 涉及文件：`docs/tasks`、`docs/api`、`docs/testing`、`docs/product`、后续实现涉及 `apps/api` 和 `apps/web`。
- 验收影响：任何范围、字段、状态、权限、循环校验或验收路径变更都会影响接口契约、测试计划和浏览器视觉分析记录。
- 决策依据：一期范围、核心业务流程、产品决策记录和五类子代理讨论结论。

## 缺陷与阻断

- 阻断缺陷：系统无法启动、核心页面无法浏览器访问、关键接口不可用、权限绕过、关键业务数据错误、数据库迁移失败、阶段验收路径不可执行。
- 严重缺陷：同父项多个启用版本、循环引用可保存、启用 BOM 无明细、父项或子项引用停用物料、启用版本可直接改明细、复制版本丢失明细、写操作审计缺失、权限表现与后端鉴权不一致、视觉问题影响核心操作。
- 已知问题：库存、生产工单、领料、报工、完工入库、成本、采购销售、工艺路线、替代料、选配 BOM、工程变更审批、复杂多级展开 UI 或功能不在本阶段范围。
- 是否允许进入验收：无阻断缺陷，严重缺陷完成影响评估并不影响本阶段核心路径时才允许。
