# 任务记录：仓库与库存基础能力

## 任务名称

仓库与库存基础能力。

## 任务目标

建立制造业 ERP 一期生产主链路所需的库存余额、库存变动流水、期初库存和库存调整能力，使后续生产工单、领料、完工入库和成本归集能够依赖可信、可追溯、可鉴权的库存数据。

## 背景与输入

- 关联业务背景：本项目是制造业单公司内部 Web ERP，当前已完成账号权限、基础资料与物料管理、BOM 管理。库存是 BOM 之后、生产工单和领料之前的关键基础能力。

- 已确认约束：
  - 本阶段只按“仓库 + 物料”维度管理库存，不引入批次、库位、序列号、条码或多单位换算。
  - 一期不允许负库存。
  - 同一仓库物料只允许存在一次已过账期初，后续修正通过库存调整处理。
  - 已过账单据不可编辑。
  - 一期可用数量等于现存数量，锁定数量固定为 0 或仅响应预留。
  - 所有影响库存余额的动作必须生成库存变动流水和审计记录。
  - 阶段成果必须本地部署、浏览器可访问，并保存核心路径视觉分析记录。

## 范围

### 本次包含

- 库存余额查询：按物料、仓库、物料类型、关键词和正库存筛选，展示现存数量、可用数量、单位和最近更新时间。
- 库存变动流水查询：展示期初、调增、调减导致的库存变化，包含变动前数量、变动数量、变动后数量和来源单据。
- 库存单据：支持期初库存和库存调整两类单据的草稿保存、编辑和过账。
- 期初库存：过账后增加库存余额并生成变动流水；同一仓库物料只能过账一次期初。
- 库存调整：支持调增和调减；调减后库存不得小于 0。
- 权限控制：库存菜单、余额查看、流水查看、单据查看、单据创建、单据更新、单据过账。
- 审计记录：创建、更新、过账等写操作必须写入审计日志。
- 前端页面：库存余额、库存变动流水、库存单据列表、期初/调整表单、单据详情和权限差异展示。
- 自动化测试、本地部署、浏览器功能验收和视觉分析。

### 本次不包含

- 生产工单、领料、报工、完工入库。
- 成本核算、成本归集、财务凭证、应收应付和总账。
- 采购订单、销售订单、采购入库、销售出库。
- 批次、序列号、库位、条码、先进先出、保质期。
- 调拨、盘点任务、库存占用锁定、缺料分析。
- 多单位换算、多组织、多公司、多租户。
- 仓储 WMS 或图形化库位管理。

## 验收标准

- 功能验收：
  - 管理员可创建期初库存草稿并过账。
  - 管理员可创建库存调整草稿并过账。
  - 库存余额和库存变动流水与过账结果一致。
  - 已过账单据不可编辑。
- 数据验收：
  - 所有库存数量使用 `NUMERIC(18,6)` 口径。
  - 同一仓库物料只允许一次已过账期初。
  - 调减后库存不得小于 0。
  - 所有过账都生成来源可追溯的库存变动流水。
- 权限验收：
  - 管理员具备库存全量权限。
  - 仓库人员按权限查看、创建或过账库存单据。
  - 只读用户仅能查看余额、流水和详情。
  - 无权限用户不能访问库存页面或接口。
- UI 验收：
  - 页面符合后台 ERP 信息密度和现有设计风格。
  - 数量列右对齐，正负变动、状态和来源信息可识别。
  - 表单错误、库存不足、重复期初和只读状态清晰可见。
  - 窄屏下筛选区、表格和表单不重叠、不穿出容器。
- 测试验收：
  - 后端集成测试覆盖迁移、权限、期初、调整、过账、异常、并发和审计。
  - 前端测试覆盖路由、权限、列表、表单、动态明细、错误反馈和窄屏关键状态。
  - 全量后端测试、前端测试和前端构建通过。
- 部署验收：
  - 本地 PostgreSQL、后端和前端可启动。
  - 浏览器可以访问库存管理核心页面。
  - 健康检查和核心验收路径通过。

## 风险与处理

- 余额和流水不一致：所有库存变化必须通过过账事务完成，事务内同时更新余额、写流水和写审计。
- 并发过账丢失更新：过账时按仓库物料锁定库存余额行。
- 重复提交导致重复入账：单据状态和来源明细唯一约束共同防重。
- 负库存破坏后续领料：一期明确禁止负库存，调减超过现存时拒绝。
- 停用主数据被入账：过账前校验仓库、物料、单位均启用。
- 范围滑入生产或采购销售：本阶段只由期初和调整产生库存变动，后续来源类型只做预留。

## 缺陷与阻断

- 阻断缺陷：服务无法启动、核心页面无法访问、库存可为负、重复期初可过账、余额和流水不一致、无权限用户可写库存、过账无审计、视觉分析缺失。
- 严重缺陷：数量精度错误、只读权限前端展示不一致但后端能拦截、筛选结果不稳定、表单失败后状态丢失。
- 已知问题：批次、库位、序列号、库存占用、调拨、盘点、生产单据来源和采购销售来源不在本阶段。
- 是否允许进入验收：必须完成任务文档、接口契约、设计规格、实施计划、自动化测试、本地部署、浏览器验收和视觉分析后再判断。
