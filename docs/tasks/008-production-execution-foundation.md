# 任务记录：生产执行基础能力

## 任务名称

生产执行基础能力。

## 任务目标

建立制造业 ERP 一期生产主链路中的生产工单、领料、报工和完工入库基础能力，使已完成的物料、BOM 和库存能力能够被真实生产业务调用，并为后续成本归集提供可追溯的数据来源。

## 背景与输入

- 关联业务背景：本项目是制造业单公司内部 Web ERP，当前已完成账号权限、基础资料与物料管理、BOM 管理、仓库与库存基础能力。下一阶段应承接一期主链路中的生产工单、领料、报工和完工入库。

- 已确认约束：
  - 本阶段以工单上下文打通生产执行闭环，不建立高级排产、工艺路线或独立生产单据中心。
  - 工单发布时生成 BOM 用料快照，后续 BOM 修改不得影响已发布工单。
  - 一期不启用库存占用锁定，领料过账时实时扣减库存，库存不得为负。
  - 一期不允许超领、超报、超入。
  - 报工只记录生产结果，不直接修改库存。
  - 领料和完工入库必须通过过账写入库存余额和库存变动流水。
  - 阶段成果必须本地部署、浏览器可访问，并保存核心路径视觉分析记录。

## 范围

### 本次包含

- 生产工单：
  - 工单列表、详情、新增、编辑草稿、发布、完成、取消。
  - 工单字段包括工单编号、产品物料、关联 BOM、计划数量、计划日期、领料仓库、入库仓库、状态、备注和审计信息。
  - 发布工单时校验产品物料、BOM、仓库和单位均有效，并生成用料快照。
- 工单用料快照：
  - 从启用 BOM 明细生成子项物料、单位、应领数量和 BOM 行来源。
  - 展示应领数量、已领数量和未领数量。
- 领料：
  - 从工单详情发起领料，生成领料草稿。
  - 明细默认来自工单用料快照，支持本次领料数量录入。
  - 领料过账后扣减库存，生成库存变动流水和审计记录。
- 报工：
  - 从工单详情发起报工，记录合格数量、不良数量、报工日期、报工人和备注。
  - 报工过账后更新工单累计报工数量，不直接影响库存。
- 完工入库：
  - 从工单详情发起完工入库，生成入库草稿。
  - 入库过账后增加成品或半成品库存，生成库存变动流水和审计记录。
- 状态与追溯：
  - 工单状态覆盖草稿、已发布、生产中、已完成、已取消。
  - 工单详情展示 BOM 快照、领料记录、报工记录、完工入库记录和相关库存流水。
  - 库存流水可反查来源生产单据、工单和 BOM 快照。
- 权限控制：
  - 生产管理菜单、生产工单、领料、报工、完工入库的查看、创建、更新、过账和状态动作权限。
  - 后端接口鉴权是最终安全边界，前端按钮控制只作为体验控制。
- 自动化测试、本地部署、浏览器功能验收和视觉分析。

### 本次不包含

- 高级排产、产能计划、设备负荷。
- 工艺路线、工序级派工、工序流转卡。
- 批次、序列号、库位、条码。
- 替代料、倒冲领料、退料、补料、返工、拆解、委外加工。
- 采购、销售联动触发生产。
- 库存占用锁定、缺料分析、齐套分析。
- 多单位换算、多组织、多公司、多租户。
- 审批流、附件图纸、工程变更。
- 正式成本核算、标准成本、移动加权、成本差异分析。

## 验收标准

- 功能验收：
  - 管理员可创建生产工单草稿并发布。
  - 发布工单必须引用启用 BOM，并生成 BOM 用料快照。
  - 已发布工单可创建领料单、报工单和完工入库单。
  - 领料、报工、完工入库支持多次执行，但累计数量不得超过规则上限。
  - 工单达到完成条件后可完成；无已过账业务的工单可取消。
- 数据验收：
  - 所有生产数量使用 `NUMERIC(18,6)` 口径。
  - 领料过账后原材料或半成品库存减少，完工入库过账后产成品或半成品库存增加。
  - 报工只更新生产结果，不直接修改库存。
  - 库存余额、库存流水和生产来源单据一致。
  - BOM 快照可追溯，后续 BOM 修改不影响已发布工单。
- 权限验收：
  - 管理员具备生产执行全量权限。
  - 生产管理人员可创建、编辑草稿、发布、完成和取消工单，可查看执行记录。
  - 仓库人员可查看已发布工单，可创建、编辑和过账领料单、完工入库单。
  - 生产人员可查看已发布工单，可创建和过账报工单。
  - 只读用户只能查看工单和追溯信息。
  - 无权限用户不能访问生产页面或接口。
- UI 验收：
  - 页面符合后台 ERP 信息密度和现有设计风格。
  - 工单列表、详情页和明细表格的状态、数量和操作可扫描。
  - 操作列预留稳定宽度，关键按钮不可被截断。
  - 表单错误、库存不足、BOM 不可用、只读状态和无权限状态清晰可见。
  - 窄屏下筛选区、表格、表单和底部操作区不重叠；如需横向滚动，关键操作可触达。
- 测试验收：
  - 后端集成测试覆盖迁移、权限、工单发布、领料过账、报工过账、完工入库过账、异常、并发和审计。
  - 前端测试覆盖路由、权限、列表、表单、详情、动态明细、错误反馈和窄屏关键状态。
  - 全量后端测试、前端测试和前端构建通过。
- 部署验收：
  - 本地 PostgreSQL、后端和前端可启动。
  - 浏览器可以访问生产管理核心页面。
  - 健康检查和核心验收路径通过。
- 浏览器视觉分析：
  - 视觉分析记录保存到 `docs/testing/production-execution-visual-audit/`。
  - 覆盖工单列表、查询结果、空状态、新建表单、详情、领料、报工、完工入库、错误状态、只读权限和窄屏视口。

## 风险与处理

- BOM 修改影响已发布工单：发布时生成 BOM 快照，工单详情读取快照，不回读可变 BOM 明细。
- 库存被扣成负数：领料过账时锁定库存余额并校验过账后数量不小于 0。
- 重复提交导致重复扣减或入库：来源明细唯一约束、单据状态和事务校验共同防重。
- 超领、超报、超入破坏生产数据可信度：一期全部禁止，返回明确业务错误。
- 工单状态不清导致按钮错显：后端状态机定义最终可操作性，前端只做展示和体验保护。
- 详情页信息过密：采用状态摘要加分区表格，不使用嵌套卡片堆叠。
- 窄屏侧栏占用空间：若不影响核心访问和操作，记录为后续优化；若导致关键操作不可见或表单不可提交，视为阻断缺陷。

## 缺陷与阻断

- 阻断缺陷：服务无法启动、核心页面无法访问、工单无法发布、BOM 快照缺失、领料后库存未扣减、完工入库后库存未增加、库存可为负、余额和流水不一致、无权限用户可写生产接口、写操作无审计、浏览器视觉分析缺失。
- 严重缺陷：数量精度错误、超领超报超入未拦截、已过账单据可编辑、状态按钮与后端规则不一致、详情追溯不完整、只读权限前端展示不一致但后端能拦截。
- 已知问题：排产、工艺路线、批次、序列号、库位、条码、倒冲、退料、返工、委外、成本核算、采购销售联动不在本阶段。
