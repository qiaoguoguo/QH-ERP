# 任务记录：成本归集基础记录

## 任务名称

成本归集基础记录。

## 任务目标

建立制造业 ERP 一期生产主链路中的成本归集基础记录能力，把生产领料、报工、完工入库等生产执行结果沉淀为可查询、可追溯、可鉴权、可审计的成本业务记录，为后续正式成本核算、财务查询和报表能力提供可信数据基础。

本阶段只做业务记录与来源追溯，不承诺正式财务核算结果。

## 背景与输入

- 关联业务背景：当前项目已完成账号权限、基础资料与物料管理、BOM 管理、仓库与库存基础能力、生产执行基础能力。下一阶段应承接一期主链路中的成本归集基础记录。

- 已确认约束：
  - 成本归集一期只做业务记录和追溯，不做正式成本核算。
  - 成本记录至少说明成本来源、关联生产工单、产品、成本类型、金额或数量口径、记录人、记录时间和来源单据。
  - 生产领料可形成材料数量口径记录；生产报工可形成人工数量口径记录；完工入库作为产出追溯和归集对象，不自动形成正式金额成本。
  - 当前库存与生产执行没有稳定单价来源，不允许把自动生成记录解释为正式材料成本金额。
  - 阶段成果必须本地部署、浏览器可访问，并保存核心路径视觉分析记录。

## 范围

### 本次包含

- 成本记录：
  - 成本记录列表、详情、新增手工记录。
  - 按关键词、工单、产品、成本类型、来源类型、来源单号、日期范围查询。
  - 成本记录字段包括记录编号、工单、产品、成本类型、来源类型、来源单据、数量、单位、单价、金额、业务日期、记录人、记录时间、备注和审计信息。
- 自动来源记录：
  - 已过账生产领料明细自动形成材料成本来源记录，记录物料、单位、数量、工单、产品和领料单来源。
  - 已过账生产报工自动形成人工数量口径记录，记录报工数量、工单、产品和报工单来源。
  - 已过账完工入库在工单成本汇总中作为产出数量追溯，不自动计入成本金额。
  - 自动来源记录必须防重复生成。
- 手工成本记录：
  - 支持人工、制造费用、其他成本或必要的材料金额补录。
  - 手工记录必须关联有效生产工单，产品从工单带出；本阶段不支持仅按产品录入成本记录。
  - 手工记录必须有金额或数量口径，必须填写业务日期和说明。
  - 手工记录不得反写库存、生产单据状态或财务凭证。
- 工单成本追溯：
  - 生产工单详情展示成本归集记录摘要。
  - 成本记录详情可追溯回生产工单和来源单据。
  - 工单成本汇总展示材料、人工、制造费用、其他记录和完工产出数量口径。
- 权限控制：
  - 成本管理菜单、成本记录查看、创建、更新或补录权限。
  - 后端接口鉴权是最终安全边界，前端按钮控制只作为体验控制。
- 自动化测试、本地部署、浏览器功能验收和视觉分析。

### 本次不包含

- 正式财务成本核算结果。
- 总账、凭证、会计期间、结账。
- 标准成本、移动加权平均、先进先出、成本差异分析。
- 在产品成本、完工成本分摊、制造费用自动分配算法。
- 采购、销售、应收、应付和财务深度联动。
- 多币种、多组织、多公司、多账套、多租户。
- 审批流、BI 报表、经营分析大屏。
- 对已完成生产执行阶段进行排产、工艺路线、批次、序列号、库位、条码、倒冲、退料、补料、返工或委外扩展。

## 验收标准

- 功能验收：
  - 已过账生产领料明细自动形成材料来源成本记录。
  - 已过账生产报工自动形成人工数量口径成本记录。
  - 完工入库可在工单成本汇总中作为产出数量追溯。
  - 有权限用户可手工录入制造费用、其他成本或金额补录记录。
  - 成本记录可按工单、产品、成本类型、来源类型、来源单号和日期范围查询。
  - 成本记录详情可查看来源单据、工单、产品、金额或数量口径、记录人和记录时间。
  - 生产工单详情可查看该工单成本记录摘要并进入成本详情。
- 数据验收：
  - 成本数量和金额使用 `NUMERIC(18,6)` 口径。
  - 自动来源记录与来源单据、来源明细、工单和产品保持一致。
  - 同一来源明细不得重复生成同类型自动成本记录。
  - 成本记录不得反写库存余额、生产工单状态或生产单据状态。
  - 自动生成记录不得被改写来源单据、来源数量、工单或产品。
  - 手工记录必须有金额或数量口径，负数金额和负数数量不得保存。
- 权限验收：
  - 管理员具备成本记录全量查看、创建、更新或补录权限。
  - 成本管理角色可查看和维护成本记录，但不自动获得生产、库存无关写权限；浏览器验收必须覆盖该角色。
  - 只读用户只能查看成本记录和追溯信息。
  - 无权限用户不能访问成本页面或接口。
  - 前端隐藏按钮不能替代后端接口鉴权。
- UI 验收：
  - 页面符合后台 ERP 信息密度和现有设计风格。
  - 成本记录列表、详情和工单成本分区中的成本类型、来源、金额、数量和状态可扫描。
  - 表单错误、重复生成、来源缺失、只读状态和无权限状态清晰可见。
  - 窄屏下筛选区、表格、详情和表单不重叠；如需横向滚动，关键操作可触达。
  - 页面文案不得暗示正式财务核算结果。
- 测试验收：
  - 后端集成测试覆盖迁移、权限、自动生成、手工记录、查询详情、异常、审计和回滚。
  - 前端测试覆盖路由、权限、列表、详情、表单、工单成本分区、错误反馈和窄屏关键状态。
  - 全量后端测试、前端测试和前端构建通过。
- 部署验收：
  - 本地 PostgreSQL、后端和前端可启动。
  - 浏览器可以访问成本管理核心页面。
  - 健康检查和核心验收路径通过。
- 浏览器视觉分析：
  - 视觉分析记录保存到 `docs/testing/cost-collection-visual-audit/`。
  - 覆盖成本列表、查询结果、空状态、详情、手工记录表单、表单错误、重复来源错误、工单成本分区、成本管理角色、只读权限、无权限状态和窄屏视口。

## 风险与处理

- 材料金额来源不足：自动材料来源记录只记录数量口径，金额必须手工补录或留空，界面和文档明确不代表正式材料成本。
- 报工没有工时字段：一期人工来源记录使用报工数量口径，人工金额通过手工记录补充。
- 完工入库被误解为成本金额：完工入库仅作为产出数量追溯和归集对象，不自动计入成本金额。
- 重复生成成本记录：自动来源记录建立唯一约束和事务校验，重复请求返回明确错误。
- 生产服务过大：新增独立成本服务，生产过账点只调用成本来源写入组件，避免继续堆叠生产服务职责。
- 手工记录失控：手工记录必须关联有效生产工单，必须填写成本类型、口径和说明，并写审计。
- 视觉误导：页面不做财务看板、利润分析或正式核算报表，只展示业务记录和追溯。

## 缺陷与阻断

- 阻断缺陷：服务无法启动、核心页面无法访问、生产过账后成本来源缺失、成本记录无法追溯工单或来源单据、重复生成导致重复成本、无权限用户可写成本接口、写操作无审计、浏览器视觉分析缺失。
- 严重缺陷：金额或数量精度错误、自动记录可修改来源字段、成本记录反写库存或生产状态、页面暗示正式成本核算、只读权限前端展示不一致但后端能拦截。
- 已知问题：标准成本、移动加权、成本差异、凭证、总账、制造费用分配算法不在本阶段。
- 是否允许进入验收：必须完成任务文档、接口契约、设计规格、实施计划、自动化测试、本地部署、浏览器验收和视觉分析后再判断。
