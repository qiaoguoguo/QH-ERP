# 任务记录：一期生产主链路端到端闭环验收与稳定化

## 任务名称

一期生产主链路端到端闭环验收与稳定化。

## 任务目标

在账号权限、基础资料与物料、BOM、库存基础、生产执行和成本归集基础记录均已进入主分支后，执行一期生产主链路端到端闭环验收，确认链路真实可用、权限一致、数据可追溯、页面可操作，并集中识别、修复和复验主链路缺陷。

本阶段不是新增业务模块，不扩大到采购、销售、正式财务、报表或审批能力。

## 范围

### 本次包含

- 管理员端到端主路径：
  - 登录并确认菜单入口可用。
  - 准备启用单位、仓库、物料分类、原材料、辅料、成品。
  - 创建或选用启用 BOM，并确认版本和明细可用。
  - 创建并过账期初库存，确认库存余额与库存流水。
  - 创建生产工单，引用启用 BOM，发布后确认 BOM 用料快照。
  - 创建并过账生产领料，确认原料库存扣减、库存流水和材料成本来源记录。
  - 创建并过账生产报工，确认工单报工数量和人工成本来源记录。
  - 创建并过账完工入库，确认成品库存增加、库存流水和产出追溯。
  - 创建并编辑手工成本记录，确认成本记录关联工单和产品。
  - 从成本记录、工单详情、库存流水完成跨模块追溯。
- 权限一致性：
  - 管理员、基础资料维护、库存角色、生产角色、成本角色、只读用户、无权限用户的菜单、路由、按钮和后端接口一致性。
  - 前端隐藏按钮不能替代后端接口鉴权。
- 数据一致性：
  - 库存余额、库存流水、工单状态、报工数量、完工入库数量、成本记录和审计记录的关键断言。
  - 失败或重复操作不得产生部分写入或重复影响。
- 异常路径：
  - 未登录、无权限、BOM 不可用、库存不足、超领、超报、超入、重复过账、非法成本金额或数量、无效工单。
- 页面与视觉：
  - 核心列表、表单、详情、异常、权限状态和窄屏视口截图。
  - 页面文案不得暗示正式财务核算结果。
- 缺陷管理：
  - 缺陷分级、修复派发、复审、复验和最终结论记录。

### 本次不包含

- 采购、销售、应收、应付、正式财务核算、总账、凭证、会计期间、结账。
- 标准成本、移动加权、先进先出、成本差异分析、在产品成本和完工成本分摊。
- 报表看板、BI 分析、审批流、多组织、多公司、多账套、多租户。
- 工艺路线、工序流转、高级排产、批次、序列号、库位、条码、倒冲、退料、补料、返工或委外。
- 重做已有模块的信息架构、设计系统或组件库。

## 角色分工

- 产品经理：确认端到端业务主路径、异常路径、阻断缺陷标准和最终验收结论口径。
- UI 设计师：确认浏览器页面可操作性、视觉截图清单、桌面和窄屏验收标准、视觉阻断标准。
- 前端开发：确认菜单、路由守卫、按钮权限、追溯跳转和浏览器脚本覆盖点；修复前端缺陷。
- 后端开发：确认事务边界、数据一致性、接口鉴权、审计和来源追溯；修复后端缺陷。
- 测试：执行测试矩阵、接口核查、浏览器验收、视觉分析和缺陷分级建议。
- 主代理：统筹五角色讨论、建立阶段文档、派发执行任务、组织规格审查和质量审查、复核证据并判断是否可进入主分支验收。

## 验收标准

- 功能验收：
  - 管理员能够完成基础资料、BOM、期初库存、生产工单、领料、报工、完工入库、成本归集和追溯闭环。
  - 库存流水能够追溯到期初、生产领料和完工入库来源。
  - 成本记录能够追溯到生产工单、产品、来源单据、记录人和记录时间。
  - 页面可完成必要查询、创建、过账、详情查看和追溯操作。
- 数据验收：
  - 期初过账后原料库存增加。
  - 领料过账后原料库存按领料数量减少。
  - 完工入库过账后成品库存按入库数量增加。
  - 报工和完工入库数量符合一期规则，超领、超报、超入被拒绝。
  - 领料和报工自动生成对应成本业务记录，不重复生成。
  - 成本记录不反写库存余额、工单状态或生产单据状态。
- 权限验收：
  - 管理员可完成全链路。
  - 库存角色不能执行生产和成本无关写操作。
  - 生产角色不能执行库存调整和成本手工维护。
  - 成本角色可查看和维护成本记录，但不能执行库存和生产写操作。
  - 只读用户不能新增、编辑、发布、过账、取消或完成。
  - 无权限用户不能访问业务页面或接口。
- UI 验收：
  - 菜单、表格、表单、详情、异常提示、权限状态和追溯入口清晰可见。
  - 桌面和窄屏视口下关键操作可触达，密集表格可通过可信滚动方式查看。
  - 页面不出现阻断级重叠、错页、空白、加载中截图或误导性成本核算文案。
- 测试验收：
  - 后端全量测试通过。
  - 前端全量测试、类型检查和构建通过。
  - 端到端浏览器主路径、权限路径、异常路径和视觉分析完成。
  - `git diff --check` 通过。
- 部署验收：
  - 本地 PostgreSQL、后端和前端可启动。
  - 后端健康检查返回 `UP`。
  - 前端核心路由可浏览器访问。
- 浏览器视觉分析：
  - 视觉分析记录保存到 `docs/testing/phase-one-e2e-visual-audit/`。
  - 截图覆盖主路径、追溯、权限差异、异常状态和窄屏视口。

## 风险与处理

- 跨模块链路长：用同一批目标数据贯穿截图、接口和数据断言，避免旧数据或不同数据集拼接证据。
- 权限矩阵复杂：先覆盖管理员、成本角色、只读和无权限，再扩展库存、生产和基础资料角色。
- 重复过账和并发风险高：重复提交必须验证不会重复扣库存、重复累计数量或重复生成成本记录。
- 菜单兜底易复发：重点检查后端菜单 children 缺失时库存、生产、成本入口仍可显示且路由受权限保护。
- 视觉截图容易失真：密集表格、长详情和固定侧边栏使用可信视口截图，不用失真全页截图替代结论。
- 成本边界易误读：所有成本相关页面、记录和结论只表达业务记录与来源追溯，不承诺正式财务核算。

## 缺陷与阻断

- 阻断缺陷：服务无法启动、核心页面无法访问、管理员无法完成主链路、关键业务数据错误、权限绕过、库存/生产/成本不可追溯、过账失败产生部分写入、视觉分析缺失或阻断视觉问题。
- 严重缺陷：追溯入口断链、异常提示不清导致业务结果不可判断、菜单/按钮权限与后端权限不一致但未形成写入绕过、重复操作存在数据风险但未污染数据。
- 一般缺陷：不影响主路径、数据可信度、权限判断和关键操作的文案、排序、轻微布局或体验问题。
- 是否允许进入验收：阻断缺陷必须全部修复并复验；严重缺陷必须有修复或明确风险处理结论；一般缺陷可记录为后续优化但不得影响最终验收判断。
