# 任务记录：采购管理基础模块

## 任务名称

采购管理基础模块。

## 任务目标

建立制造业 ERP 一期之后的采购管理基础能力，让有权限用户能够维护采购订单，并基于采购订单完成采购入库。采购入库过账后必须增加库存余额、生成库存流水，并能从采购订单、采购入库和库存流水之间追溯来源。

本阶段只做采购业务基础和库存入库追溯，不做应付、发票、付款、税务、审批、MRP、采购退货、质检入库或正式财务凭证。

## 范围

### 本次包含

- 采购订单：
  - 采购订单列表、详情、新建草稿、编辑草稿。
  - 确认、取消、关闭状态操作。
  - 采购订单明细维护。
  - 订单行已入库数量和未入库数量。
- 采购订单明细：
  - 供应商。
  - 物料。
  - 数量。
  - 单位。
  - 采购单价。
  - 预计到货日期。
  - 备注。
- 采购入库：
  - 基于已确认或部分入库采购订单创建采购入库单。
  - 采购入库草稿维护。
  - 采购入库明细引用采购订单明细。
  - 采购入库过账。
  - 过账后增加库存余额。
  - 过账后生成 `PURCHASE_RECEIPT` 库存流水。
- 追溯：
  - 采购订单详情查看入库记录。
  - 采购入库详情查看来源采购订单。
  - 库存流水追溯到采购入库来源。
- 权限：
  - 采购管理菜单。
  - 采购订单查看、新建、编辑、确认、取消、关闭。
  - 采购入库查看、新建、编辑、过账。
  - 管理员、采购员、仓库角色、只读用户和无权限用户权限差异。
  - 后端接口鉴权作为最终安全边界。
- 审计：
  - 采购订单创建、更新、确认、取消、关闭。
  - 采购入库创建、更新、过账。
- 验收：
  - 自动化测试。
  - 本地部署。
  - 浏览器功能验收。
  - 权限路径验收。
  - 异常路径验收。
  - 视觉分析。

### 本次不包含

- 应付账款、发票、付款、税务、暂估入库、总账联动、正式财务凭证。
- 采购审批流、询价、比价、供应商评分、采购价格策略和价格历史分析。
- MRP 自动采购建议、采购需求计划、报表看板或经营分析。
- 采购退货、到货检验、质检入库、待检库存或让步接收。
- 多币种、多组织、多公司、多账套、多租户。
- 批次、序列号、库位、条码、采购包装单位和多单位换算。
- 对既有生产、库存或成本模块进行范围外重构。

## 验收标准

- 功能验收：
  - 管理员能够创建采购订单草稿、编辑草稿、确认订单、取消订单和关闭订单。
  - 管理员能够基于已确认或部分入库订单创建采购入库草稿。
  - 管理员能够过账采购入库，过账后入库单不可编辑且不可重复过账。
  - 采购订单详情显示入库记录、已入库数量和未入库数量。
  - 采购入库详情显示来源订单和过账前后库存。
  - 库存流水显示 `PURCHASE_RECEIPT` 来源，并能追溯到采购入库。
- 数据验收：
  - 采购订单草稿、确认、取消和关闭不改变库存余额。
  - 采购入库草稿不改变库存余额，不生成库存流水。
  - 采购入库过账后，库存余额增加量等于本次入库数量。
  - 每条采购入库明细生成一条可追溯库存流水。
  - 同一入库明细不得重复生成库存流水。
  - 订单行已入库数量不得超过采购数量。
  - 异常失败不得产生部分库存余额、部分流水或错误累计入库数量。
- 权限验收：
  - 管理员具备采购订单和采购入库全量权限。
  - 采购员可维护采购订单并查看采购入库，不自动获得采购入库过账权限。
  - 仓库角色可查看订单必要信息，并维护、过账采购入库，不可确认、取消或关闭采购订单。
  - 只读用户只能查看采购订单、采购入库和追溯信息。
  - 无权限用户不能访问采购页面或接口。
  - 前端隐藏按钮不能替代后端接口鉴权。
- UI 验收：
  - 页面符合后台 ERP 信息密度和现有设计风格。
  - 采购订单列表、详情、表单和采购入库列表、详情、表单状态清晰。
  - 采购入库与库存单据入库文案不混淆。
  - 桌面和窄屏视口下关键操作可触达，密集表格可通过可信横向滚动查看。
  - 表单错误、异常提示、只读状态和无权限状态清晰可见。
- 测试验收：
  - 后端全量测试通过。
  - 前端全量测试、类型检查和构建通过。
  - 浏览器主路径、权限路径、异常路径和视觉分析完成。
  - `git diff --check` 通过。
- 部署验收：
  - 本地 PostgreSQL、后端和前端可启动。
  - 后端健康检查返回 `UP`。
  - 前端采购核心路由可浏览器访问。

## 风险与处理

- 库存过账逻辑分叉：先抽取库存域统一过账端口，再接入采购入库，避免采购、生产和库存三处规则不一致。
- 并发入库导致超收：过账时锁定采购订单行，事务内计算剩余未入库数量并回写累计。
- 数据库约束遗漏：采购迁移必须显式更新 `inv_stock_movement` 变动类型约束，自动化测试覆盖 `PURCHASE_RECEIPT`。
- 采购单价被误解为财务入账：接口文档、页面文案和测试结论均说明采购单价只是业务字段。
- 采购入库与库存单据混淆：采购页面和库存流水来源明确显示“采购入库”。
- 视觉证据不足：阶段验收必须保存真实浏览器截图和视觉分析结论。

## 缺陷与阻断

- 阻断缺陷：服务无法启动、核心页面无法访问、采购主路径无法走通、采购入库不增加库存、库存流水缺失、追溯断链、权限绕过、数据库迁移失败、视觉分析缺失。
- 严重缺陷：订单累计入库数量错误、异常失败产生部分写入、重复过账存在数据风险、采购入库与库存单据语义混淆、只读权限前端展示不一致但后端能拦截。
- 一般缺陷：不影响主路径、数据可信度、权限判断和关键操作的文案、排序、轻微布局或体验问题。
- 是否允许进入验收：阻断缺陷必须全部修复并复验；严重缺陷必须有修复或明确风险处理结论。
