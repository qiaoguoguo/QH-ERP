# 任务记录：财务往来基础模块

## 任务名称

财务往来基础模块。

## 任务目标

建立制造业 ERP 在采购入库和销售出库之后的基础往来台账能力，让有权限用户能够基于已过账销售出库生成应收台账并登记收款，基于已过账采购入库生成应付台账并登记付款。系统必须能维护应收、收款、应付、付款的状态、金额余额和来源追溯，并保证收付款后累计金额、未收未付余额和状态一致。

本阶段只做财务往来业务台账、收付款登记、核销余额和来源追溯，不做总账、凭证、会计科目、会计期间、结账、税务申报、完整发票生命周期、银行对账、报销、固定资产、工资、多币种汇兑损益、多组织、多公司、多账套、多租户、正式收入确认、正式成本结转或 BI 报表。

## 背景与输入

- 关联业务背景：采购管理基础模块已完成“采购订单 -> 采购入库 -> 库存增加 -> 来源追溯”，销售管理基础模块已完成“销售订单 -> 销售出库 -> 库存扣减 -> 来源追溯”。下一阶段需要把已过账采购入库和销售出库沉淀为可追溯的应付、应收和收付款业务台账。

## 范围

### 本次包含

- 应收台账：
  - 基于已过账销售出库生成应收草稿。
  - 应收列表、详情、编辑草稿、确认、取消、关闭。
  - 展示客户、来源销售订单、来源销售出库、业务日期、到期日期、应收金额、已收金额、未收金额和状态。
  - 同一销售出库明细不得重复生成应收来源。
- 收款记录：
  - 基于应收台账登记收款草稿。
  - 收款列表、详情、编辑草稿、过账、草稿取消。
  - 支持单笔应收多次部分收款，累计不得超过未收金额。
  - 过账后更新应收已收金额、未收金额和状态。
- 应付台账：
  - 基于已过账采购入库生成应付草稿。
  - 应付列表、详情、编辑草稿、确认、取消、关闭。
  - 展示供应商、来源采购订单、来源采购入库、业务日期、到期日期、应付金额、已付金额、未付金额和状态。
  - 同一采购入库明细不得重复生成应付来源。
- 付款记录：
  - 基于应付台账登记付款草稿。
  - 付款列表、详情、编辑草稿、过账、草稿取消。
  - 支持单笔应付多次部分付款，累计不得超过未付金额。
  - 过账后更新应付已付金额、未付金额和状态。
- 追溯：
  - 应收详情追溯来源销售出库和销售订单。
  - 收款详情追溯对应应收。
  - 应付详情追溯来源采购入库和采购订单。
  - 付款详情追溯对应应付。
- 权限：
  - 财务往来菜单。
  - 应收查看、生成、编辑、确认、取消、关闭。
  - 收款查看、创建、编辑、过账、取消草稿。
  - 应付查看、生成、编辑、确认、取消、关闭。
  - 付款查看、创建、编辑、过账、取消草稿。
  - 管理员、财务角色、销售角色、采购角色、只读用户和无权限用户权限差异。
  - 后端接口鉴权作为最终安全边界。
- 审计：
  - 应收生成、更新、确认、取消、关闭。
  - 收款创建、更新、过账、取消草稿。
  - 应付生成、更新、确认、取消、关闭。
  - 付款创建、更新、过账、取消草稿。
- 验收：
  - 自动化测试。
  - 本地部署。
  - 浏览器功能验收。
  - 权限路径验收。
  - 异常路径验收。
  - 视觉分析。

### 本次不包含

- 总账、凭证、会计科目、会计期间、期末结账、财务报表或正式核算结果。
- 税务申报、税率管理、销项税、进项税、发票红冲、发票认证、发票作废或完整发票生命周期。
- 银行账户管理、银行对账、票据管理、收款认领、付款审批、报销、固定资产、工资。
- 多币种、汇兑损益、多组织、多公司、多账套、多租户。
- 正式收入确认、正式成本结转、利润核算或经营 BI 看板。
- 销售退货、采购退货、退款、退票、已过账收付款反冲或调整单。
- 对既有采购、销售、库存、生产或成本模块进行范围外重构。

## 产品口径

- 本阶段财务往来不是正式财务核算模块，而是面向制造业单公司的业务往来台账。
- 应收来源固定为已过账销售出库，金额按销售出库明细数量和来源销售订单单价形成业务金额快照。
- 应付来源固定为已过账采购入库，金额按采购入库明细数量和来源采购订单单价形成业务金额快照。
- 应收或应付生成后，来源、客户或供应商、核心金额不可由前端随意改写。
- 草稿应收或应付只允许编辑到期日期和备注等非来源字段。
- 已确认且已发生收付款的应收或应付不可取消，不可改核心金额和来源。
- 已收清或已付清台账不可继续登记收款或付款。
- 已过账收款或付款不可编辑、不可取消；错误修正不纳入本阶段。
- 关闭用于停止未结清台账继续收付，已关闭台账不可继续登记收付款。
- 所有金额字段以人民币业务金额口径展示，精度为两位小数，不做多币种或含税拆分。
- 页面文案使用“应收台账、收款记录、应付台账、付款记录、登记、过账、来源追溯”，不得使用“凭证生成、会计分录、收入确认、成本结转、税务处理”等范围外口径。
- 本阶段销售角色和采购角色的查看能力是方向级授权，不做个人、部门或业务员数据范围过滤；销售角色若被授予查看权限，可查看销售方向应收和收款记录，不可查看应付和付款；采购角色若被授予查看权限，可查看采购方向应付和付款记录，不可查看应收和收款。

## 状态口径

- 应收状态：`DRAFT` 草稿、`CONFIRMED` 待收款、`PARTIALLY_RECEIVED` 部分收款、`RECEIVED` 已收清、`CLOSED` 已关闭、`CANCELLED` 已取消。
- 收款状态：`DRAFT` 草稿、`POSTED` 已过账、`CANCELLED` 已取消。
- 应付状态：`DRAFT` 草稿、`CONFIRMED` 待付款、`PARTIALLY_PAID` 部分付款、`PAID` 已付清、`CLOSED` 已关闭、`CANCELLED` 已取消。
- 付款状态：`DRAFT` 草稿、`POSTED` 已过账、`CANCELLED` 已取消。

## 验收标准

- 功能验收：
  - 管理员或财务角色能够基于已过账销售出库生成应收草稿，并确认应收。
  - 管理员或财务角色能够基于已确认或部分收款应收登记收款草稿，并过账部分收款和全额收款。
  - 管理员或财务角色能够基于已过账采购入库生成应付草稿，并确认应付。
  - 管理员或财务角色能够基于已确认或部分付款应付登记付款草稿，并过账部分付款和全额付款。
  - 应收详情显示来源销售订单、销售出库和收款记录。
  - 应付详情显示来源采购订单、采购入库和付款记录。
  - 收款详情显示对应应收和核销金额。
  - 付款详情显示对应应付和核销金额。
- 数据验收：
  - 同一销售出库明细不得重复生成应收来源。
  - 同一采购入库明细不得重复生成应付来源。
  - 应收金额等于已收金额加未收金额。
  - 应付金额等于已付金额加未付金额。
  - 收款或付款过账失败不得留下部分核销明细、错误累计金额或错误状态。
  - 收款金额不得超过未收金额，付款金额不得超过未付金额。
  - 状态必须由金额关系和业务动作共同约束，不得出现金额与状态矛盾。
- 权限验收：
  - 管理员具备财务往来全量权限。
  - 财务角色可查看和维护应收、收款、应付、付款。
  - 销售角色可查看销售方向应收和收款进度，不得查看应付和付款，不得确认应收、登记或过账收款。
  - 采购角色可查看采购方向应付和付款进度，不得查看应收和收款，不得确认应付、登记或过账付款。
  - 只读用户只能查看财务往来台账、详情和追溯信息。
  - 无权限用户不能访问财务往来页面或接口。
  - 前端隐藏按钮不能替代后端接口鉴权。
- UI 验收：
  - 页面符合后台 ERP 信息密度和现有设计风格。
  - 财务往来菜单、应收列表、收款列表、应付列表、付款列表、详情和登记表单状态清晰。
  - 金额字段右对齐，状态标签中文展示，来源单据可追溯。
  - 页面不得把本阶段能力表达为正式财务核算、凭证、税务或报表。
  - 桌面和窄屏视口下关键操作可触达，密集表格可通过可信横向滚动查看。
  - 表单错误、异常提示、只读状态和无权限状态清晰可见。
- 测试验收：
  - 后端全量测试通过。
  - 前端全量测试、类型检查和构建通过。
  - 浏览器主路径、权限路径、异常路径和视觉分析完成。
  - `git diff --check` 通过。
- 部署验收：
  - 本地 PostgreSQL、后端和前端可启动。
  - 后端健康检查返回 `UP`。
  - 前端财务往来核心路由可浏览器访问。

## 风险与处理

- 范围滑向正式财务核算：所有文档、接口、页面和测试均限制为业务台账与收付登记，禁止凭证、科目、税务和结账文案进入交付能力。
- 金额一致性风险：金额累计只由后端事务内根据核销明细更新，前端不得传最终累计值。
- 来源重复风险：来源表和数据库唯一约束必须防止同一来源明细重复生成应收或应付。
- 超额收付款风险：收付款过账必须锁定目标台账并校验余额，超额时整体拒绝。
- 并发核销风险：对同一应收或应付并发过账时，必须使用行锁或条件更新兜底，避免超收或超付。
- 采购销售历史变更风险：财务台账保留生成时金额快照，不反向依赖来源单据后续变更。
- 视觉证据不足：阶段验收必须保存真实浏览器截图和视觉分析结论。

- 影响范围：新增财务往来文档、后续财务往来后端和前端实现、权限种子、测试与浏览器验收路径。
- 涉及文件：

  - `docs/product/business-flow.md`
  - `docs/tasks/013-financial-settlement-foundation.md`
  - 后续财务往来接口契约、测试计划、设计规格和实施计划。
- 验收影响：后续阶段验收新增应收、收款、应付、付款、来源追溯、权限和视觉分析要求。
- 决策依据：销售管理基础完成版交接文档、固定五角色阶段讨论结论和用户授权主代理接手下一阶段财务往来基础模块开发落地。

## 缺陷与阻断

- 阻断缺陷：服务无法启动、核心页面无法访问、应收或应付任一主路径无法完成、收付款金额累计错误、状态和金额矛盾、来源重复生成、超额收付款被接受、权限绕过、数据库迁移失败、视觉分析缺失。
- 严重缺陷：异常失败产生部分写入、来源追溯错误、金额精度污染、已结清继续登记、已过账收付款可编辑、只读权限前端展示不一致但后端能拦截。
- 一般缺陷：不影响主路径、数据可信度、权限判断和关键操作的文案、排序、轻微布局或体验问题。
- 是否允许进入验收：阻断缺陷必须全部修复并复验；严重缺陷必须有修复或明确风险处理结论。
