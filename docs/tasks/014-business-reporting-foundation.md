# 任务记录：经营报表基础模块

## 任务名称

经营报表基础模块。

## 任务目标

建立制造业单公司 ERP 的固定口径经营报表基础能力，让有权限用户能够在浏览器中查看采购、销售、库存、生产、成本和财务往来的经营汇总、明细追溯和异常清单。报表结果必须来自已完成业务模块的真实单据和状态，支持按期间、客户、供应商、物料、仓库、工单和状态筛选，并能追溯到来源业务单据。

本阶段只做固定口径经营报表、口径说明、权限控制、空态异常态和来源追溯，不做可配置 BI、拖拽报表、总账、凭证、会计科目、会计期间、结账、正式财务报表、税务申报、多组织、多公司、多账套或多租户。

- 已确认约束：

  - 报表只消费已有业务数据，不新增采购、销售、库存、生产、成本或财务往来业务流程。
  - 报表金额为经营业务金额，不等同正式财务入账金额。
  - 草稿、取消和作废业务单据默认不进入经营汇总，除非作为异常清单明确展示。
  - 来源追溯必须受原业务模块权限约束，不得通过报表泄露无权查看的业务详情。

## 业务口径

- 销售经营金额以已过账销售出库为主，金额按出库数量和来源销售订单单价形成经营金额。
- 采购经营金额以已过账采购入库为主，金额按入库数量和来源采购订单单价形成经营金额。
- 库存收发存以库存流水和当前库存余额为准，期初和期末按业务日期范围推算。
- 生产执行以生产工单、领料、报工和完工入库状态为准。
- 成本归集只统计有效成本业务记录，并明确 `formalAccounting=false`。
- 应收应付余额以已确认及后续状态的往来台账为准，排除草稿和已取消记录。
- 已关闭且仍有未结余额的往来台账单独列示，不并入正常未结余额。
- 日期口径优先使用业务日期；订单按订单日期，入库和出库按过账业务日期，生产按工单和生产单据业务日期，往来按发生日期和到期日期。
- 报表行必须能追溯来源，不允许只给出无法解释的孤立汇总数。

## 验收标准

- 功能验收：
  - 经营概览展示销售、采购、库存、生产、成本、应收应付和收付款关键指标。
  - 固定报表支持日期范围筛选和模块相关维度筛选。
  - 固定报表展示汇总指标、明细表格、口径说明和来源追溯入口。
  - 经营异常清单展示库存不足、生产逾期、应收逾期、采购到货逾期等可行动事项。
- 数据验收：
  - 汇总值与来源明细合计一致。
  - 草稿和已取消单据不污染经营汇总。
  - 金额、数量和百分比按定点口径处理，不产生浮点误差。
  - 空数据返回零值和空数组，不返回错误。
  - 报表不得生成或修改业务单据。
- 权限验收：
  - 管理员具备全部报表查看权限。
  - 有部分报表权限的角色只能访问对应报表。
  - 无报表权限用户不能访问菜单、路由或接口。
  - 来源跳转继续受来源模块权限控制。
- UI 验收：
  - 页面符合后台 ERP 信息密度。
  - 不做经营大屏或营销式展示。
  - 桌面宽表可读，关键金额和状态可扫描。
  - 窄屏下筛选、指标和表格仍可操作。
  - 空态、异常态、无权限态清晰区分。
- 测试验收：
  - 后端全量测试通过。
  - 前端全量测试、类型检查和构建通过。
  - 浏览器主路径、权限路径、异常路径和视觉分析完成。
  - `git diff --check` 通过。
- 部署验收：
  - 本地 PostgreSQL、后端和前端可启动。
  - 后端健康检查返回 `UP`。
  - 前端经营报表核心路由可浏览器访问。

## 风险与处理

- 报表口径滑向正式财务：页面、接口和文档统一使用经营金额、业务口径和来源追溯，不使用总账、凭证、财务报表等交付语义。
- 汇总与来源不一致：每个报表必须提供追溯接口，后端测试断言明细合计等于汇总。
- 权限绕过：报表接口和来源跳转分别鉴权，前端隐藏不能替代后端鉴权。
- 宽表不可读：UI 设计和视觉验收覆盖桌面与窄屏，必要时采用固定关键列、横向滚动和来源详情展开。
- 性能风险：后端采用固定 SQL 聚合和分页追溯，不通过实体全量加载后在 Java 循环聚合。
- 历史期初推算风险：库存收发存第一版明确按库存流水业务日期和流水顺序推算，不承诺期间快照。

## 缺陷与阻断

- 阻断缺陷：服务无法启动、报表核心页面无法访问、关键报表接口不可用、汇总值与来源明细对不上、权限绕过、来源追溯不可用、视觉分析缺失、报表页面表达为总账或正式财务报表。
- 严重缺陷：状态过滤错误、空态和异常态误导用户、金额精度污染、来源跳转错误、宽表遮挡关键金额或状态。
- 已知问题：第一版不做期间快照、不做缓存、不做导出、不做图表配置，不支持退货退款或业务反冲后的净额口径。
- 是否允许进入验收：阻断缺陷必须全部修复并复验；严重缺陷必须有修复或明确风险处理结论。
