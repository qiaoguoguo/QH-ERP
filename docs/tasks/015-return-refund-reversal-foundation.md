# 任务记录：退货退款与业务反冲基础模块

## 任务名称

退货退款与业务反冲基础模块。

## 任务目标

建立制造业单公司 ERP 的退货退款与业务反冲基础能力，让已过账采购、销售、生产、库存、往来和成本业务能够通过可追溯的反向单据表达修正事实。已过账原单不得被直接改写或删除，所有退货、退款、退料、补料、冲减和反冲都必须形成独立业务记录，并同步影响库存流水、往来余额、成本口径和经营报表净额。

本阶段只做业务层反向闭环、来源追溯、权限控制、异常拦截、经营报表净额同步和浏览器验收，不做审批流、正式红字发票、税务处理、总账凭证、会计期间、结账、多组织、多币种、高级质量体系、复杂售后维修或正式财务核算。

## 背景与输入

- 014 经营报表基础模块已完成并合入主分支，系统已具备采购、销售、库存、生产、成本、往来和固定经营报表的正向链路。
- 014 明确不包含销售退货、采购退货、退款、反冲、生产退料和补料，这些能力进入 015。

## 范围

### 本次包含

- 销售退货：
  - 基于已过账销售出库行创建退货草稿。
  - 退货过账后生成销售退货入库库存流水。
  - 冲减关联应收业务余额。
  - 支持部分退货、全量退货、防超退、防重复过账和来源追溯。
- 采购退货：
  - 基于已过账采购入库行创建退货草稿。
  - 退货过账后生成采购退货出库库存流水。
  - 冲减关联应付业务余额。
  - 支持库存不足拦截、防超退、防重复过账和来源追溯。
- 生产退料与补料：
  - 基于已过账生产领料行创建退料入库。
  - 基于工单物料创建补料出库。
  - 同步生成生产反向或补充库存流水。
  - 影响工单材料消耗和成本归集业务口径。
- 往来冲减与退款记录：
  - 对应收、应付、收款和付款形成业务冲减记录。
  - 控制冲减金额不得超过可冲余额。
  - 区分余额冲减和实际退款业务记录，不承诺正式银行或总账处理。
- 已过账业务反冲：
  - 已过账业务不能编辑、删除或直接取消。
  - 通过反向业务记录表达冲减影响。
  - 建立原单、原行、反向单据、库存流水、往来冲减和成本影响之间的追溯链。
  - 第一版不建设泛化“反冲中心”或独立 `BUSINESS_REVERSAL` 单据；业务反冲由销售退货、采购退货、生产退料、生产补料和往来冲减这些具体反向单据表达。
- 经营报表同步：
  - 销售、采购、库存、生产、成本和往来固定报表增加原发生、反向发生和净额口径。
  - 销售报表使用 `salesOriginalAmount`、`salesReturnAmount`、`salesNetAmount`、`salesOriginalQuantity`、`salesReturnQuantity`、`salesNetQuantity`。
  - 采购报表使用 `purchaseOriginalAmount`、`purchaseReturnAmount`、`purchaseNetAmount`、`purchaseOriginalQuantity`、`purchaseReturnQuantity`、`purchaseNetQuantity`。
  - 库存报表使用 `inboundOriginalQuantity`、`inboundReverseQuantity`、`inboundNetQuantity`、`outboundOriginalQuantity`、`outboundReverseQuantity`、`outboundNetQuantity`、`inventoryNetChangeQuantity`。
  - 生产报表使用 `issuedOriginalQuantity`、`materialReturnQuantity`、`materialSupplementQuantity`、`issuedNetQuantity`、`completedQuantity`。
  - 成本报表使用 `materialOriginalCost`、`materialReturnCost`、`materialSupplementCost`、`materialNetCost`、`totalNetCost`。
  - 往来报表使用 `receivableOriginalAmount`、`receivableAdjustmentAmount`、`receivableNetAmount`、`payableOriginalAmount`、`payableAdjustmentAmount`、`payableNetAmount`、`settlementRemainingAmount`。
  - 报表来源追溯可定位到退货、退料、补料、冲减和反冲记录。
- 权限和审计：
  - 反向业务独立权限点。
  - 后端接口鉴权作为最终安全边界。
  - 来源权限不足时继续脱敏，不泄露单号、主键、数量、金额、日期、状态或路由参数。
  - 写操作记录审计日志。
- 验收：
  - 自动化测试。
  - 本地部署。
  - 浏览器主路径、权限路径和异常路径验收。
  - 视觉分析。

### 本次不包含

- 审批流、工作流引擎、消息中心。
- 正式红字发票、税务申报、银行对账、退款支付通道。
- 总账、凭证、会计科目、会计期间、结账、正式财务报表。
- 多组织、多公司、多账套、多币种、多租户。
- 高级质量检验、售后维修、换货、返工流程、质量索赔。
- 期间关账和正式成本结转。
- 可配置 BI、经营大屏和报表设计器。

## 业务口径

- 已过账原单是业务事实，退货、退款、反冲不得直接覆盖原单核心字段。
- 草稿反向单不影响库存、往来、成本和报表。
- 已过账反向单按自身业务日期进入经营数据，不回写原单期间。
- 取消的草稿不影响经营数据；已过账反向单第一版不允许取消，只能再建立新的受控反向记录。
- 销售退货数量累计不得超过原销售出库行数量减已退数量。
- 采购退货数量累计不得超过原采购入库行数量减已退数量，且过账时必须校验可用库存。
- 生产退料数量累计不得超过原领料行已领数量减已退数量。
- 补料出库必须校验工单状态、物料来源和库存可用数量。
- 往来冲减金额不得超过目标应收、应付、收款或付款的可冲余额。
- 报表同时展示原发生、反向发生和净额，不能只给出无法解释的净额。
- 来源追溯必须双向可查：原单能看到反向影响，反向单能追溯原单和影响记录。

## 验收标准

- 功能验收：
  - 销售退货从已过账销售出库发起，过账后生成退货入库、冲减应收并可追溯。
  - 采购退货从已过账采购入库发起，过账后生成退货出库、冲减应付并可追溯。
  - 生产退料/补料围绕生产工单和领料来源形成库存反向或补充流水。
  - 往来冲减不能超过可冲金额，已收、已付、未收、未付变化可解释。
  - 经营报表净额口径体现正向与反向业务。
- 数据验收：
  - 已过账原单核心事实不被直接修改或删除。
  - 反向单据、库存流水、往来冲减、成本影响和报表来源一致。
  - 超退、超冲、库存不足、来源状态非法和重复反冲均被拦截。
  - 金额和数量使用定点口径，不产生浮点误差。
  - 并发重复过账不得重复影响库存、往来或成本。
- 权限验收：
  - 管理员具备全部反向业务权限。
  - 模块角色只能访问授权范围内的退货或冲减能力。
  - 只读角色只能查看，不能创建、过账或取消草稿。
  - 无权限用户不能访问菜单、路由或接口。
  - 来源受限时脱敏响应不泄露敏感字段。
- UI 验收：
  - 页面保持后台 ERP 信息密度。
  - 原单、反向单、数量、金额、状态和影响范围可扫描。
  - 桌面宽表关键列清晰；窄屏表单、状态和追溯不重叠。
  - 空态、异常态、无权限态和来源受限态清晰区分。
- 测试验收：
  - 后端全量测试通过。
  - 前端全量测试、类型检查和构建通过。
  - 浏览器主路径、权限路径、异常路径和视觉分析完成。
  - `git diff --check` 通过。
- 部署验收：
  - 本地 PostgreSQL、后端和前端可启动。
  - 后端健康检查返回 `UP`。
  - 前端退货退款与业务反冲核心路由可浏览器访问。

## 风险与处理

- 反向业务误改历史：后端禁止直接修改已过账原单核心字段，统一新增反向记录。
- 超退超冲：过账事务中锁定来源行或目标台账，按累计已退和已冲重新校验。
- 库存负数：采购退货和补料出库过账前校验库存可用数量。
- 往来与库存不同步：退货过账在单事务内写库存影响和往来冲减；任一步失败整体回滚。
- 成本净额不可信：生产退料和补料只影响业务成本口径，不承诺正式成本结转。
- 报表口径混淆：报表同时展示原发生、反向发生和净额，并提供来源追溯。
- 权限泄露：延续 014 来源脱敏规则，前端隐藏不能替代后端鉴权。

## 缺陷与阻断

- 阻断缺陷：服务无法启动、核心页面无法访问、关键接口不可用、数据库迁移失败、已过账原单被直接改写、允许超退超冲、库存或往来余额错误、报表净额与来源不一致、权限绕过、视觉分析缺失。
- 严重缺陷：状态误导、追溯断链、来源脱敏不完整、反向业务口径文案混淆、窄屏关键操作不可见。
