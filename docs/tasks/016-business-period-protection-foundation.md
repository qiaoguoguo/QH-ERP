# 任务记录：业务期间与历史数据保护基础

## 任务目标

建立按业务日期维护、锁定和解锁业务期间的基础能力。锁定期间后，库存、采购、销售、生产、成本、往来和反向业务中会影响经营口径的写入必须被后端拒绝；历史数据仍可查询、追溯和用于报表查看。

本阶段只做业务期间锁定与历史写入保护，不做正式财务关账。

## 范围

### 本次包含

- 业务期间列表、新增、编辑、按自然月生成、锁定和解锁。
- 不重叠日期范围校验；锁定和解锁原因必填并留存审计记录。
- 服务层统一校验业务日期，覆盖库存、采购、销售、生产、成本、往来和反向业务写入。
- 期间查看、维护、锁定和解锁的独立权限；前端菜单、路由和按钮体验控制，以及后端最终鉴权。
- 锁定状态展示、写入失败提示、浏览器验收和视觉分析。

### 本次不包含

- 正式财务关账、会计期间、总账、凭证、税务、银行对账、期间快照、库存月结、成本结转和损益结转。
- 审批流、消息通知、电子签核、多组织、多公司、多账套、多币种和历史数据批量修复。
- 质量检验、库存质量状态、MRP、排产和产能计划。

## 业务口径

- 期间默认按自然月维护，编码为 `YYYY-MM`；允许管理员维护不重叠的日期范围。
- `OPEN` 期间允许原有业务规则下的新增、编辑、确认、发布、过账和反向业务。
- `LOCKED` 期间拒绝影响经营口径的写入，但不限制查询、来源追溯和报表查看。
- 无匹配期间按开放处理；页面和接口可提示期间未配置风险。
- 开放期间可以创建引用锁定期间历史原单的反向业务；反向业务按自身开放期间的业务日期计入经营影响，绝不回写历史原单或锁定期间。
- 锁定和解锁均为高风险动作，必须有独立权限、必填原因和审计记录。解锁不自动重算或改写既有业务数据。

## 角色分工

- 产品经理：维护业务范围、业务口径、变更控制和验收矩阵。
- UI 设计师：定义期间列表、确认弹窗、权限态、错误态和窄屏视觉验收。
- 前端开发：实现 API 封装、页面、路由、权限体验和前端测试。
- 后端开发：实现期间模型、权限、接口、审计和统一写入保护。
- 测试：执行定向自动化、浏览器路径、视觉分析和回归风险检查。
- 主代理：统筹工作包、集中审查、整改和阶段交付判断。

## 验收标准

- 锁定业务期间后，业务日期落入该期间的库存、采购、销售、生产、成本、往来和反向业务写入被拒绝。
- 锁定期间仍可查询、追溯和报表查看。
- 开放期间允许创建反向业务引用锁定期间历史原单。
- 无权限用户不能锁定、解锁或绕过接口校验。
- 视觉分析覆盖期间列表、锁定确认、写入失败、只读权限、无权限和 390px 窄屏。
- 期间日期范围不可重叠；锁定和解锁均有原因和审计记录。

## 风险与处理

- 历史写入入口遗漏：以 `BusinessPeriodGuard` 服务层统一校验为边界，并按模块覆盖测试。
- 跨期反向口径混淆：反向影响只进入自身开放期间，保留对历史来源的追溯。
- 解锁被滥用：使用独立权限、原因、审计和确认弹窗。
- 将业务期间误解为正式财务交付：所有页面和文档使用业务期间与历史数据保护表述。

## 变更控制

- 变更原因：015 已保护已过账原单不被直接改写，仍需限制历史业务日期的新写入，保护已验收的经营口径。
- 影响范围：期间管理、跨模块写服务、权限、接口、页面、自动化测试、浏览器验收和视觉分析。
- 涉及文件：本任务文档、接口契约、测试计划、产品决策、业务流程、验收标准和文档索引。
- 验收影响：新增锁定拒绝、跨期反向、查询可用、权限不可绕过和视觉状态的验收要求。
