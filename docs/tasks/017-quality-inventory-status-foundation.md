# 任务记录：质量检验与库存质量状态基础

## 任务目标

建立库存质量状态基础能力，将库存余额从仓库+物料升级为仓库+物料+质量状态，并让合格库存成为销售出库、生产领料、生产补料和后续计划净算的唯一可用库存口径。

本阶段只建立质量状态对库存可用性的基础约束，不建设完整质量管理体系。

## 背景与输入

- 关联业务背景：016 已完成业务期间与历史数据保护，017 必须继续保护已锁定期间内的库存事实和经营口径。
- 关联文档：`docs/superpowers/specs/2026-07-10-quality-inventory-status-design.md`。
- 已确认约束：当前库存余额以仓库和物料为主，`availableQuantity` 一期等于现存量；017 后需调整为合格可用库存。
- 业务范围前提：采购入库、完工入库、销售退货和生产退料增加库存；销售出库、生产领料、采购退货和生产补料消耗库存。

## 范围

### 本次包含

- 建立 `PENDING_INSPECTION` 待检、`QUALIFIED` 合格、`REJECTED` 不合格、`FROZEN` 冻结四类库存质量状态。
- 库存余额按仓库、物料和质量状态分层，库存流水记录质量状态。
- 历史库存默认迁移为合格库存，保证既有库存、采购、销售、生产和报表口径平滑延续。
- `availableQuantity` 只代表合格可用库存，不再等同总现存量。
- 采购入库、完工入库、销售退货和生产退料默认形成待检库存。
- 轻量质量确认支持待检转合格、不合格或冻结，并记录原因、操作人、业务日期和审计。
- 合格库存支持冻结；冻结库存支持解冻回合格。
- 销售出库、生产领料和生产补料只能消耗合格库存。
- 采购退货按显式质量状态退货，允许待检、不合格或合格库存退回供应商；冻结库存需先解冻。
- 质量状态写入、冻结、解冻和相关库存过账受权限、原因、审计和 016 业务期间保护约束。
- 质量确认、库存余额质量状态分布、候选库存禁用原因、权限态、异常态、空态和视觉验收。

### 本次不包含

- 完整 QMS、检验标准库、抽样方案、缺陷等级、供应商质量评分、让步接收和不合格评审闭环。
- SPC、质量趋势分析、质量成本归集或质量经营报表。
- 批次、序列号、库位、条码、拣配、调拨、盘点和仓储执行深化。
- MRP、计划净算实现、主生产计划、排产、产能计划和缺料分析。
- 审批流、消息通知、工作流设计器和复杂流程配置。
- 正式财务、总账、凭证、税务、银行对账、成本重算和财务结账。
- 复杂返工返修、售后维修、召回、质量索赔和多组织、多公司、多账套、多币种能力。

## 业务口径

- 合格库存是唯一普通可用库存，待检、不合格和冻结库存不得参与销售出库、生产领料、生产补料或后续计划净算。
- 总现存量等于同一仓库和物料下各质量状态库存数量合计；合格可用量只是其中的 `QUALIFIED` 数量。
- 入库先进入待检库存，完成质量确认后才能转为合格可用库存；017 不提供免检配置。
- 质量确认允许把待检数量一次性拆分到合格、不合格和冻结，结果数量合计必须等于本次处理数量。
- 状态转换前后同一仓库和物料的总库存不变，不允许任一质量状态库存出现负数。
- 采购退货是供应商退回场景，不按普通可用量消耗规则处理；退货时必须显式选择质量状态，冻结库存需先解冻。
- 业务日期落入已锁定期间时，质量确认、冻结、解冻、入库质量状态落账和按质量状态扣减库存都必须被后端拒绝。
- 锁定期间历史质量记录、库存余额、库存流水和来源追溯仍可查询。

## 核心业务流程

```text
采购入库/完工入库/销售退货/生产退料 -> 待检库存 -> 质量确认 -> 合格/不合格/冻结库存 -> 合格库存参与销售出库和生产领料
```

```text
合格库存 -> 冻结 -> 不参与销售出库、生产领料、生产补料或采购退货 -> 解冻回合格
```

```text
用户提交质量确认或冻结 -> BusinessPeriodGuard 校验业务日期 -> 开放期间继续 -> 已锁定期间返回 BUSINESS_PERIOD_LOCKED
```

## 验收标准

- 历史库存默认迁移为合格库存。
- 库存余额按仓库、物料和质量状态分层，汇总数量一致。
- `availableQuantity` 只代表合格可用库存。
- 采购入库、完工入库、销售退货和生产退料默认形成待检库存。
- 质量确认可以把待检库存转为合格、不合格或冻结，且转换前后总库存不变。
- 销售出库、生产领料和生产补料不能消耗非合格库存。
- 采购退货按显式质量状态退货，冻结库存需先解冻。
- 质量状态写入受权限、原因、审计和业务期间保护约束。
- 后端接口校验不可被前端绕过；无权限用户不能处理质量确认、冻结或解冻。
- 页面必须同时展示总现存、合格可用量和质量状态分布，避免把总库存误读为可用库存。
- 异常提示必须区分普通库存不足、非合格库存不可用、指定质量状态库存不足和期间锁定。
- 浏览器验收覆盖采购入库、完工入库、销售退货、生产退料、销售出库、生产领料、生产补料、采购退货、冻结解冻、权限差异和期间保护。
- 视觉分析覆盖质量确认列表、处理抽屉、库存余额质量分布、候选库存禁用原因、权限态、异常态、空态和 390px 窄屏。

## 风险与处理

- 余额维度调整影响既有库存、采购、销售、生产、反向业务和报表查询：先以库存服务统一口径承载质量状态，再逐个业务入口接入，并验证总库存汇总一致。
- `availableQuantity` 口径变化导致页面误读：接口和页面同时展示总现存、合格可用量和状态分布，前端不得自行推导最终可用量。
- 入库默认待检改变库存即时可用预期：历史库存默认合格，新增入库通过质量确认转合格，页面明确展示待检状态。
- 采购退货与普通出库规则混淆：采购退货必须显式选择质量状态，冻结库存先解冻，不走普通合格可用量默认规则。
- 质量确认出现部分写入或重复提交：状态转换、流水、余额、单据状态和审计必须在同一事务内完成，并校验来源状态和可处理数量。
- 017 被扩大成完整质量系统：文档、接口、页面和测试均限制为库存质量状态基础，完整 QMS、SPC、批次、库位、MRP 和正式财务保持排除。

## 变更控制

- 变更原因：当前库存只有仓库和物料余额，无法阻止待检、不合格或冻结库存被销售、生产和后续计划口径误认为可用库存。
- 影响范围：库存余额、库存流水、采购入库、完工入库、销售退货、生产退料、销售出库、生产领料、生产补料、采购退货、权限、审计、期间保护、接口、页面和测试。
- 涉及文件：本任务文档、接口契约、测试计划、产品流程、产品决策、验收标准、文档索引和后续实现文件。
- 验收影响：新增质量状态分层、合格可用量、入库待检、质量确认、冻结解冻、非合格不可消耗、采购退货显式状态、权限审计和期间保护验收要求。
- 决策依据：`docs/superpowers/specs/2026-07-10-quality-inventory-status-design.md`、017 阶段启动讨论结论和 016 业务期间保护边界。

## 缺陷与阻断

- 阻断缺陷：服务无法启动、核心质量确认页面无法访问、库存余额汇总不一致、`availableQuantity` 包含非合格库存、普通消耗可扣减非合格库存、期间保护可绕过、权限可绕过、关键接口不可用或视觉分析缺失。
- 严重缺陷：质量确认部分写入、重复提交造成重复库存、冻结库存可被普通消耗、采购退货状态来源错误、错误提示掩盖质量状态不可用原因。
- 已知问题：完整 QMS、SPC、批次、库位、MRP、正式财务、复杂返工返修和质量报表不在本阶段。
- 是否允许进入验收：阻断缺陷和严重缺陷修复并通过定向复验后，才允许进入阶段交付前全量验证窗口。
