# 任务记录：库存可用量、占用、冻结与预留基础

## 任务目标

建立库存可用量口径底座和轻量业务占用、预留控制，区分账面库存、合格现存、占用库存、预留库存、冻结库存、现货净可用、物料级采购在途参考、仓库级现货可承诺量和计划净需求缺口，为后续销售可承诺量和计划净需求建立可信口径。

本阶段承接 `017 质量检验与库存质量状态基础`，不建设完整 MRP、复杂 ATP 引擎、排产、批次、库位、正式财务或库存月结。

## 背景与输入

- 017 已将库存余额升级为仓库、物料、质量状态维度，并让 `availableQuantity` 只代表合格可用库存。
- 当前 `inv_stock_balance.locked_quantity` 存在但缺少业务维护逻辑，销售、生产候选库存基本仍按合格现存计算。
- 销售订单、生产工单和采购订单已具备确认、取消、关闭、部分执行和过账状态，是 018 占用、预留和在途的来源。
- 用户确认 018 为当前阶段目标，并授权主代理独立推进，但实现、设计、测试和审核仍按固定五角色协作完成。

## 范围

### 本次包含

- 建立库存占用、预留来源台账，记录来源类型、来源单据、来源行、仓库、物料、单位、质量状态、数量、状态、业务日期、原因、操作人和审计。
- 保留 `inv_stock_balance.locked_quantity` 作为合格库存占用和预留汇总缓存，由统一库存可用量服务维护。
- 库存余额接口展示账面库存、合格现存、占用库存、预留库存、冻结库存、现货净可用、物料级采购在途参考、仓库级现货可承诺量和净需求缺口。
- `availableQuantity` 在 018 后调整为现货净可用，即合格现存减合格占用和合格预留。
- 销售订单行新增“预留仓库”，确认前必填；销售订单确认时按订单行预留仓库生成 `SALES_ORDER + RESERVATION`；销售出库仓库必须与来源订单行预留仓库一致；取消、关闭和数量减少释放剩余预留。
- 生产工单发布按 BOM 快照和领料仓库生成 `PRODUCTION_WORK_ORDER + RESERVATION`；生产领料草稿如建立执行锁定则为 `OCCUPATION`；领料过账消费占用或预留并扣减合格库存；不得把工单发布直接记为占用。
- 采购订单确认和部分入库形成物料级采购在途参考；本阶段不在采购订单行补仓库字段；采购在途不进入仓库级账面库存或仓库级现货可承诺量；采购入库过账后才进入具体仓库待检库存，质量确认后才进入合格现存。
- 冻结只能作用于未被占用、未被预留的合格现货净可用库存；解冻后回到合格现存，不自动改写历史承诺。
- 销售出库、生产领料、生产补料、采购退货和库存调减不得绕过占用、预留、质量状态和期间保护。
- 库存余额、销售候选、生产候选、采购在途、占用追溯、权限态、异常态、空态和视觉验收。

### 本次不包含

- 完整 MRP、复杂分时段 ATP、自动补货建议、计划订单生成、排产和产能计划。
- 批次、序列号、库位、波次拣货、调拨深化、盘点和仓储执行深化。
- 多层 BOM 展开净需求、替代料净算、跨仓自动调拨承诺。
- 采购订单行入库仓库预设字段、在途库存预留、销售延期承诺和基于在途的确认门禁。
- 审批流、消息通知、流程设计器。
- 正式财务、总账、凭证、税务、银行对账、库存月结、成本结转和财务结账。
- BI 平台或复杂经营分析。

## 业务口径

- 账面库存：同一仓库、物料下所有质量状态现存合计，包含待检、合格、不合格、冻结，不包含采购在途。
- 合格现存：质量状态为 `QUALIFIED` 的现存库存，尚未扣减占用和预留。
- 冻结库存：质量状态为 `FROZEN` 的现存库存；待检和不合格也是不可用，但不归入冻结库存。
- 占用库存：已被执行单据硬锁定、即将消耗的库存，例如销售出库草稿、生产领料草稿、生产补料草稿、采购退货草稿。
- 预留库存：已被确认需求软预留、尚未进入执行单据的库存，例如已确认销售订单行按预留仓库形成的未出库量、已发布生产工单按领料仓库形成的未领料需求。
- 现货净可用：合格现存减合格占用、合格预留。
- 物料级采购在途参考：已确认或部分入库采购订单的未入库数量，按物料和预计到货日期展示；本阶段采购订单行没有仓库字段，不按仓库计算在途，不进入仓库级账面库存，也不进入仓库级现货可承诺量。
- 仓库级现货可承诺量：等于该仓库、该物料的现货净可用；销售确认门禁只看订单行预留仓库的现货净可用是否足够。
- `netRequirementShortageQuantity`：计划净需求前置缺口，公式为 `max(需求数量 - 可覆盖供给数量, 0)`；本阶段不输出供需差额，不生成 MRP 建议。
- 销售可承诺门禁：018 不允许预留在途，物料级采购在途只做参考；现货不足但在途足够时销售订单保持草稿并提示现货不足、在途仅供参考。
- 来源专属候选接口必须把当前来源自己的预留或占用算回可选量，避免销售订单或生产工单被自己的预留锁死。

## 核心业务流程

```text
销售订单行选择预留仓库 -> 销售订单确认 -> 按预留仓库生成销售预留 -> 同仓库销售出库草稿可转占用 -> 出库过账消费占用或预留 -> 合格库存扣减
```

```text
生产工单发布 -> 按 BOM 快照和领料仓库生成用料预留 -> 领料草稿可转占用 -> 领料过账消费占用或预留 -> 合格库存扣减
```

```text
采购订单确认 -> 形成物料级采购在途参考 -> 采购入库选择仓库并过账 -> 在途减少并进入具体仓库待检库存 -> 质量确认 -> 合格现存增加
```

```text
合格现货净可用 -> 冻结 -> 冻结库存不可承诺、不可占用、不可普通消耗 -> 解冻回合格现存
```

## 验收标准

- 同一仓库、同一物料存在待检、合格、不合格、冻结、占用、预留和在途时，各口径数量公式正确且不重复计算。
- `availableQuantity` 不包含待检、不合格、冻结、已占用或已预留库存。
- 销售订单行必须填写预留仓库后才能确认；确认生成 `SALES_ORDER + RESERVATION`；取消、关闭、数量减少和同仓库出库过账能释放或消费预留；销售出库仓库与预留仓库不一致必须被拒绝。
- 生产工单发布生成 `PRODUCTION_WORK_ORDER + RESERVATION`；取消、完成、用料减少和领料过账能释放或消费预留；工单发布不得直接写为 `OCCUPATION`。
- 采购订单确认形成物料级采购在途参考，部分入库正确减少在途，入库后先进入具体仓库待检，不直接进入现货净可用。
- 冻结已占用或已预留库存被拒绝；冻结库存不能被承诺、占用或普通消耗。
- 销售出库、生产领料、生产补料、采购退货和库存调减不能绕过可用量、质量状态和占用预留校验。
- 同一来源行不能重复预留、重复占用或过账后残留占用。
- 占用、预留、释放、消费、在途和冻结相关写入受权限、审计和业务期间保护约束。
- 页面必须清晰展示账面库存、合格现存、占用、预留、冻结、现货净可用、物料级采购在途参考和仓库级现货可承诺量，且不能把在途当作现货或销售确认门禁。
- 浏览器验收覆盖库存余额、销售出库候选、生产领料候选、采购在途、来源追溯、权限态、异常态和 390px 窄屏。

## 风险与处理

- 数量口径混淆：接口字段必须由后端统一计算，前端不得自行推导最终可用量或可承诺量。
- `locked_quantity` 悬挂：所有预留、占用、释放和消费必须在同一事务内更新台账和余额缓存。
- 并发超占：创建预留或占用时必须锁定对应合格余额行，并校验现货净可用。
- 来源重复：台账必须对活跃来源类型、来源单据和来源行建立唯一约束。
- 在途误读：采购在途不进入账面库存，不参与立即出库、领料上限或销售确认门禁；库存余额页面必须标明其为物料级参考。
- 质量冲突：冻结、待检、不合格不得进入可承诺或普通消耗。
- 范围膨胀：计划净需求只输出前置口径和缺口，不生成 MRP 建议、不做排产。

## 缺陷与阻断

- 阻断缺陷：库存公式错误、`availableQuantity` 包含非合格或已占用预留库存、重复占用、出库或领料绕过预留导致超卖、采购在途被当成账面库存或仓库级现货可承诺量、冻结库存可被普通消耗、销售出库仓库绕过订单行预留仓库、生产工单发布被记为占用、来源释放后残留占用、权限或期间保护绕过、关键页面不可访问、视觉分析缺失。
- 严重缺陷：错误提示混淆普通库存不足和可承诺不足、占用追溯断链、列表和候选区口径不一致、并发场景偶发重复占用但未造成负数。
- 已知排除：完整 MRP、复杂 ATP、批次、库位、库存月结、正式财务和自动计划建议不在本阶段。
