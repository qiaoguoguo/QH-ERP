# 任务记录：批次、序列号与来源去向追溯基础

## 任务目标

建立制造业 ERP 的批次、序列号与来源去向追溯基础能力，让采购、质量、库存、销售、生产、退货反冲、冻结解冻和库存追溯能够持续记录并校验物料追踪身份。

本阶段承接 `017 质量检验与库存质量状态基础` 和 `018 库存可用量、占用、冻结与预留基础`。交付重点是补齐核心业务单据链路的追踪数据采集、库存维度校验、基础来源去向查询和浏览器可验收页面；不是建设完整 WMS、MES、扫码设备、法规级召回、批次成本或正式财务。

## 背景与输入

- 017 已将库存余额升级到质量状态维度，并建立采购入库、生产完工、销售退货、生产退料到质量确认的基础链路。
- 018 已建立合格现存、占用、预留、冻结、采购在途参考、现货净可用和仓库级现货可承诺量口径。
- 当前 `mst_material` 没有追踪方式字段；`inv_stock_balance`、`inv_stock_movement` 和 `inv_stock_reservation` 也没有批次或序列号维度。
- 当前采购入库、销售出库、生产领料、完工入库、退货反冲、冻结解冻已经接入库存过账，019 必须在这些入口统一引入追踪上下文。
- 用户要求主代理只做总体规划、范围控制、并行协调、审查汇总和交付判断；业务实现和审核由固定五角色完成。

## 范围

### 本次包含

- 物料基础资料增加追踪方式：不追踪、批次管理、序列号管理。
- 建立批次主档、序列号主档和通用库存追踪分配明细，支撑草稿保存、过账校验、详情回显和追溯查询。
- 库存余额、库存流水、库存占用预留支持批次或序列号维度。
- 采购入库、生产完工入库、销售退货入库、生产退料入库按物料追踪方式生成或继承批次/序列号。
- 销售出库、生产领料、采购退货、生产补料按物料追踪方式选择可用批次/序列号。
- 质量确认、冻结、解冻保留同一批次/序列身份，并正确影响 018 可用量口径。
- 销售退货必须继承原销售出库批次/序列；生产退料必须继承原生产领料批次/序列。
- 库存余额和库存流水支持按批次号、序列号筛选和追溯入口。
- 建立批次追溯和序列号追溯接口与抽屉，展示来源、当前状态、库存流水、质量状态变化、占用预留和去向单据。
- 增加批次/序列查看权限、追溯查看权限、后端错误码、审计和期间保护。
- 建立自动化测试、浏览器验收和视觉审计记录目录 `docs/testing/019-batch-serial-trace-visual-audit/`。

### 本次不包含

- 完整 WMS、库位、容器、托盘、波次、拣货路径、库存调拨深化和盘点深化。
- 条码、二维码、RFID、PDA、扫码枪、标签打印和外部设备集成。
- MES 实时采集、设备采集、工序级追溯和工艺路线追溯。
- 法规级召回流程、召回通知、召回报表和复杂图谱化追溯。
- 保质期预警、先进先出或效期优先自动推荐。
- 批次属性模板、批次合并、批次拆分专项管理。
- 批次成本、序列号成本、完整成本核算深化。
- 完整 MRP、排产、计划订单和在途预留。
- 正式财务、总账、凭证、库存月结和财务结账。
- 维修、售后和资产生命周期类序列号管理。
- 自动批号规则的复杂配置；本阶段只支持手工录入和必要的基础默认生成。

## 核心决策

- 旧物料和旧库存默认保持 `NONE` 不追踪，不自动伪造历史批次或序列号。
- 有正库存、库存流水或活动预留的物料，不允许直接从不追踪切换为批次或序列号追踪；如确需追踪旧库存，后续另做历史结存追踪初始化。
- 一个物料只支持一种主追踪方式，不支持同一物料同时强制批次和序列号。
- 批号和序列号唯一范围采用物料内唯一：`material_id + batch_no`、`material_id + serial_no`。
- 序列号按单件管理，数量固定为 1，不允许小数、部分领用、部分退货或部分冻结。
- 销售订单和生产工单预留仍保持物料、仓库、质量状态维度；执行出库、领料、补料、采购退货时精确到批次或序列号，避免过早锁死批次。
- 质量状态决定库存是否业务可用，冻结是额外限制；可用量必须同时扣除待检、不合格、冻结、占用和预留。
- 追溯粒度到核心单据、批次/序列号和生产工单关系；不做工序级、成本级或图谱化追溯。

## 验收标准

- 物料可设置不追踪、批次管理或序列号管理；不同追踪方式驱动后续单据必填、禁用和校验。
- 批次物料入库可一行拆多批，分配数量合计必须等于业务行数量。
- 序列号物料入库数量必须为整数，序列号数量必须等于业务行数量，重复序列号被后端拦截。
- 库存余额按追踪方式正确汇总：非追踪按仓库+物料+质量状态，批次按仓库+物料+质量状态+批次，序列按仓库+物料+质量状态+序列。
- 库存流水记录来源类型、来源单据、来源行、仓库、物料、质量状态、批次/序列、数量、前后结存、业务日期和操作人。
- 销售出库和生产领料只能选择合格、未冻结、未被占用或预留导致不可用的批次/序列；过账后精确扣减对应库存。
- 采购退货和生产补料作为出库类动作，必须选择当前可用批次/序列并受期间保护和权限控制。
- 销售退货和生产退料必须继承原出库或原领料批次/序列，不允许任意改选。
- 质量确认、冻结、解冻后批次/序列身份不丢失，质量状态和冻结结果正确影响可用量。
- 批次追溯能从批次看到采购/完工来源、质量状态变化、当前结存、生产领用、销售出库和退货反冲。
- 序列号追溯能看到单件当前状态、来源、质量状态变化、冻结解冻、去向和历史流水。
- 无追溯查看权限的用户不能越权访问追溯详情；来源单据详情按原模块权限脱敏。
- 锁定期间内影响库存事实、追踪身份或质量状态的写入被拒绝；历史查询和追溯仍可访问。
- 桌面和移动端页面无关键文字溢出、控件重叠、抽屉遮挡核心操作或权限状态不可识别问题。

## 风险与处理

- 数据模型膨胀：追踪维度统一放入库存余额、库存流水、库存预留和通用分配明细，避免每个业务模块自建不同结构。
- 历史库存失真：旧库存默认不追踪，禁止直接切换有存量物料的追踪方式，后续如需追踪旧库存另做专项治理。
- 批次/序列与 018 预留冲突：预留保持物料级，执行时精确占用，降低销售和生产提前锁批次的复杂度。
- 序列号并发抢占：序列号选择、占用和过账必须锁定序列记录和对应余额行，避免同一序列号重复在库或重复出库。
- 追溯越权：追溯接口返回链路摘要，来源详情跳转仍按采购、销售、生产、质量等模块权限控制。
- UI 信息过密：表格默认显示批次/序列摘要，明细和链路放入抽屉或展开行，移动端优先保障查询和查看。

## 变更控制

- 变更原因：019 从库存可用量基础继续补齐追踪维度，提升制造业 ERP 的库存可信度、质量追溯和生产来源去向查询能力。
- 影响范围：物料、库存、质量、采购、销售、生产、退货反冲、权限、审计、文档、测试和视觉验收。
- 涉及文件：以设计规格、接口契约、测试计划和实施计划为准。
- 验收影响：新增批次/序列主路径、异常路径、权限路径、期间锁定路径和视觉审计证据。
- 决策依据：固定五角色 019 阶段讨论结论和当前仓库 017、018 已完成能力边界。

## 缺陷与阻断

- 阻断缺陷：迁移失败、核心页面不可访问、追踪方式无法驱动业务校验、批次/序列身份丢失或串链、序列号重复在库、库存余额错误、负库存、冻结或待检库存可被普通出库、退货可篡改来源身份、期间锁定或权限绕过、追溯关键链路错误、视觉证据缺失。
- 严重缺陷：非主路径追踪展示缺失、错误提示不清但后端已拦截、移动端布局拥挤但可操作、追溯抽屉缺少非关键元数据、筛选组合存在边界问题。
- 已知排除：完整 WMS、扫码、标签、库位、工序级追溯、成本级追溯、完整召回、MRP 和正式财务不在本阶段。
- 是否允许进入验收：只有功能完整、集中审查、定向复审和交付前全量验证通过后才允许进入主分支阶段验收。
