## 2026-07-13 范围决策记录

### 决策一：审批首批对象

- 原因：测试角色建议只接 ECO，前端与后端建议同时接合同和 ECO。只接一个对象不能充分证明公共底座跨业务域复用；接入合同全部动作和 ECO 取消又会扩大到 025 合同深化或混淆业务取消。
- 最终决策：只接入 `SALES_PROJECT_CONTRACT_ACTIVATION` 与 `BOM_ECO_APPLICATION` 两个真实动作。合同关闭、终止、草稿取消和 ECO 取消继续走原业务状态机，不纳入 022 审批。
- 验收影响：原合同 `activate` 与 ECO `apply` 公共 HTTP 入口必须封住绕过；只有审批最终通过的内部适配器能执行原业务动作。

### 决策二：文件与对象存储

- 原因：仓库当前没有通用附件、S3、PDF、XLSX 或后台文件任务能力；数据库不适合保存大文件正文。
- 最终决策：引入 MinIO 作为本地和验收环境的 S3 兼容私有对象存储；文件元数据、对象关系、权限、状态和审计保存在 PostgreSQL。浏览器不直连对象存储，上传、下载和删除全部经后端。
- 验收影响：MinIO 停止、伪造类型、超限文件、猜测附件 ID、权限撤销后下载和对象存储失败一致性均为必测路径。

### 决策三：导入导出深度

- 原因：用户要求公共批量能力，但明确不要求 022 把系统大部分手工表单一次性改造成导入录入。
- 最终决策：手工表单继续保留；首批仅实现新增物料导入/物料导出，以及单个 BOM 草稿创建或更新导入/BOM 草稿导出。后续业务对象通过适配器逐阶段接入，034 再补完整模板、批量工具和历史数据导入。
- 验收影响：导入必须先全文件校验、再人工确认、最后通过既有业务服务原子写入；任一错误不得部分提交。BOM 导入不得发布、应用 ECO 或改写已发布工单快照。

### 决策四：平台复杂度

- 最终决策：固定审批定义由迁移和代码种子维护，022 不提供审批定义管理页面；采用 PostgreSQL 任务队列与前端轮询，不引入 Kafka、RabbitMQ、Redis、WebSocket、BPM 引擎、动态表单或模板设计器。
- 依赖原则：S3 使用 AWS SDK v2 兼容协议，不绑定 MinIO 专用 SDK；XLSX 使用 Apache POI；PDF 使用许可证清晰、支持 Java 21 的固定版式库。具体版本由后端在实现时按 Spring Boot 4.0.7 依赖兼容性固定，不使用浮动版本。PDF 必须内置并登记许可的中文字体，不能依赖宿主机字体。

## 阶段目标

为制造业单公司 ERP 建立受权限、状态机、并发、审计和历史兼容约束的固定审批与业务单据公共底座，使后续合同变更、价格、盘差、期间和财务场景能够复用审批、待办、消息、附件、固定打印和批量任务能力，而不重复建设基础设施或演变成通用工作流平台。

022 完成后必须同时满足：

- 合同生效和 BOM ECO 应用只有在固定审批通过后才能执行，业务状态与审批状态始终分离。
- 我的待办、我发起的、已处理和消息只向有权限的相关用户展示，不泄露对象摘要、数量或文件信息。
- 合同与 ECO 附件通过私有对象存储保存，所有下载重新校验平台权限和业务对象权限。
- 合同生效审批单与 BOM ECO 应用审批单可按固定模板预览并生成 PDF，模板代码和版本可追溯。
- 物料和 BOM 草稿具备受控 XLSX 导入导出样板；导入走全文件校验、确认和业务服务原子提交，失败有行级明细。
- 数据库任务具备幂等、租约、重试、过期和重启恢复语义，不需要外部消息中间件。
- 审批、附件、打印、导入导出、任务状态、权限拒绝和关键状态变化均可审计，且不记录敏感正文、完整文件内容或密钥。
- 000 至 021 历史对象、业务状态、BOM 时效规则、工单快照、权限和审计继续可用，不补造审批历史、不重算业务事实。

## 固定审批产品契约

### 首批场景

| 场景代码 | 业务对象 | 提交前业务状态 | 最终通过动作 | 通过后业务状态 | 不纳入动作 |
|---|---|---|---|---|---|
| `SALES_PROJECT_CONTRACT_ACTIVATION` | 销售项目合同 | `DRAFT` | 调用原合同 `activate` 业务语义 | `EFFECTIVE` | `close`、`terminate`、草稿 `cancel` |
| `BOM_ECO_APPLICATION` | BOM 工程变更 | `DRAFT` | 调用原 ECO `apply` 事务语义 | `APPLIED` | ECO `cancel` |

历史已生效合同和已应用 ECO 不补审批实例；页面显示“历史对象，无 022 审批记录”。历史草稿从 022 上线后按新规则提交。

### 状态

审批实例状态固定为：

- `SUBMITTED`：已提交，至少有一个待处理任务。
- `APPROVED`：所有固定步骤通过，且业务动作在同一事务成功。
- `REJECTED`：审批人驳回，业务对象仍为草稿。
- `WITHDRAWN`：提交人在最终通过前撤回，业务对象仍为草稿。
- `CANCELLED`：仅平台管理员或系统异常治理取消审批，不等同于任何业务作废或取消。

提交前没有审批实例，不设置审批 `DRAFT`。审批任务状态为 `PENDING`、`APPROVED`、`REJECTED`、`CANCELLED`；撤回或审批取消时尚未完成的任务统一转 `CANCELLED`。状态只允许单向转换，不允许终态复活。

### 定义、步骤与候选人

- `approval_definition` 和 `approval_definition_step` 由 V19 固定种子维护，包含场景代码、定义版本、业务对象类型、业务动作、固定步骤序号、步骤名称和候选权限码。
- 022 两个样板均使用一个固定审批步骤；数据模型允许后续阶段通过迁移增加有序固定步骤，但不允许运行时条件编排。
- 任务是权限候选池任务。当前用户只有同时具备步骤候选权限和业务对象查看权限时，才能在“我的待办”看到并处理；首个成功处理者通过行锁和版本占有任务。
- 提交人不得审批自己提交的实例。候选权限变更后，每次查询和处理都按当前权限重新判断。
- 定义版本、步骤、候选权限码、业务对象摘要和对象版本必须在实例中形成快照，后续种子升级不改写历史审批。

### 动作规则

| 动作 | 允许人 | 前置条件 | 业务结果 |
|---|---|---|---|
| 提交 | 具备原业务动作权限的用户 | 对象为 `DRAFT`、版本匹配、规则完整、无同场景开放实例 | 只创建实例、任务和消息，不改变业务状态 |
| 通过 | 非提交人的候选审批人 | 实例和任务为待处理、版本匹配、对象仍满足原动作条件 | 在一个事务中完成业务动作、任务和实例终态 |
| 驳回 | 非提交人的候选审批人 | 实例和任务为待处理，原因必填 | 实例终态，业务对象仍为 `DRAFT` |
| 撤回 | 提交人 | 尚无审批任务完成，原因必填、版本匹配 | 实例终态，业务对象仍为 `DRAFT` |
| 审批取消 | 平台治理权限 | 平台异常或对象失效，原因必填 | 实例终态，不代替业务取消 |

- 同一对象同一场景最多一个非终态实例；终态后重新提交会创建新实例，不覆盖旧记录。
- 所有动作携带实例或任务 `version`；SQL 更新绑定当前版本，0 行返回稳定并发错误。
- 最终通过必须锁定审批实例、任务和业务对象，再调用业务适配器；业务动作、审批终态、消息和审计一致成功或整体回滚。
- 合同 `PUT /api/admin/sales-project-contracts/{id}/activate` 和 ECO `PUT /api/admin/bom-engineering-changes/{id}/apply` 不再允许普通 HTTP 调用直接完成动作。审批启用后的外部调用返回 `APPROVAL_REQUIRED`；内部适配器调用无公共绕过参数、请求头或可伪造标志。
- 对象在审批中被合法改变而版本不一致时，处理返回 `APPROVAL_BUSINESS_OBJECT_CHANGED`，不静默覆盖。提交人撤回并按新版本重新提交。

## 待办与消息契约

- “我的待办、已处理、我发起的”由审批实例和任务查询形成，不另造可脱离审批的数据事实。
- 消息只覆盖审批提交、通过、驳回、撤回、取消，以及导入、导出、打印任务成功或失败。
- 消息状态固定为 `UNREAD`、`READ`；支持单条已读和当前用户全部已读。
- 审批提交时向当前候选用户生成站内消息；审批终态和文档任务终态向提交人或任务发起人生成消息。
- 列表返回最小业务摘要。当前用户失去业务对象查看权限后，待办、消息数量、标题、对象编号和跳转均不得继续泄露；消息可保留不可跳转的通用结果文案，但不返回对象敏感字段。
- 前端使用有界 `setTimeout` 轮询；页面卸载、切换账号、任务终态或路由离开必须停止。022 不使用 WebSocket。

## 对象存储与附件契约

### 存储结构

- `compose.yaml` 新增固定版本 MinIO 服务、健康检查、独立命名卷和私有网络访问；不得改变现有 PostgreSQL 服务名、端口或数据卷。
- 后端通过 S3 兼容配置连接私有 bucket；配置至少包含 endpoint、region、bucket、access key、secret key、path-style 和连接超时，凭据只来自环境变量或本地开发默认值，不写入响应、日志或审计。
- 对象键由后端随机生成，按 `attachments/`、`imports/`、`exports/`、`prints/` 前缀隔离；不得包含用户提供的原文件名或可猜业务 ID。
- `file_object` 保存对象键、原文件名、实际 MIME、字节数、服务端 SHA-256、ETag、用途、状态、创建人、时间和版本；`business_attachment` 保存白名单对象类型、对象 ID、文件 ID、说明、状态和版本。

### 首批对象与限制

- 业务附件首批只支持 `SALES_PROJECT_CONTRACT` 与 `BOM_ENGINEERING_CHANGE`。导入源文件、失败明细、导出结果和打印结果属于任务文件，不伪装成业务附件。
- 单附件上限 `20 MiB`，单业务对象最多 `20` 个有效附件。
- 允许类型：PDF、PNG、JPEG、TXT、CSV、DOCX、XLSX；拒绝可执行文件、脚本、压缩包、宏文档和双重扩展名。
- 后端必须同时校验扩展名、魔数或实际 MIME、大小和文件结构，不能信任浏览器 `Content-Type`。022 不建设杀毒平台，安全边界以严格白名单、私有存储和后端下载为准。
- SHA-256 由服务端流式计算；同一业务对象上传同哈希文件返回既有有效附件，不重复落对象。

### 权限和一致性

- 列表、上传、下载和删除都先通过对象类型适配器校验当前用户的业务查看或维护权限，再校验附件动作权限。
- 下载由后端流式返回并设置安全的 `Content-Disposition`；不得返回 bucket、object key、公开 URL或预签名 URL。权限撤销后，历史下载 ID 立即失效。
- 上传失败不得生成有效附件；若对象写入成功而元数据事务失败，必须补偿删除并由清理任务兜底。用户列表只返回 `AVAILABLE` 文件。
- 删除携带版本。对象存储删除失败时返回 `FILE_STORAGE_UNAVAILABLE` 且不把元数据伪标为已删除；成功后软删除关系和元数据，保留审计但不可再次下载。
- 审批提交时记录附件 ID 与哈希快照。审批中的附件不可修改；需要增删时先撤回，修改后重新提交。

## 固定打印契约

- 固定模板只有 `CONTRACT_ACTIVATION_APPROVAL_V1` 与 `BOM_ECO_APPLICATION_APPROVAL_V1`，分别打印合同生效审批单和 BOM ECO 应用审批单。
- 模板由代码和迁移固定登记，不提供模板设计器。预览和生成都重新校验审批实例与业务对象查看权限。
- 预览返回固定结构的只读数据，由前端按同一字段顺序展示；PDF 由后端任务生成并写入对象存储。
- PDF 元数据保存模板代码、模板版本、审批实例版本、业务对象版本、生成主体和生成时间。模板升级不改写旧任务元数据。
- 中文字体作为受许可的应用资源随构建分发，Windows 与容器环境输出必须一致；不得依赖本机字体。
- 打印结果保留 `7` 天，到期转 `EXPIRED` 并拒绝下载；元数据与审计继续保留。022 不把生成 PDF 自动转为永久业务附件。

## 导入导出与数据库任务契约

### 文档任务

- `document_task` 统一承载 `MATERIAL_IMPORT`、`MATERIAL_EXPORT`、`BOM_DRAFT_IMPORT`、`BOM_DRAFT_EXPORT`、`APPROVAL_PRINT`。
- 状态固定为 `QUEUED`、`RUNNING`、`READY_TO_COMMIT`、`VALIDATION_FAILED`、`SUCCEEDED`、`FAILED`、`CANCELLED`、`EXPIRED`；阶段字段区分 `VALIDATE`、`COMMIT`、`EXPORT`、`PRINT`。
- worker 使用 PostgreSQL `for update skip locked` 领取任务，持有 `leaseUntil`、`heartbeatAt`、`attemptCount` 和版本。租约超时可由其他 worker 接管。
- 校验、导出和打印最多自动重试 3 次。导入提交只有在业务事务明确回滚时才能重试；提交标记和幂等记录必须能判断未知结果，禁止重复创建业务数据。
- 创建任务和导入确认都要求 `Idempotency-Key`。同一用户、任务类型和 key 重试返回既有任务；同 key 不同文件哈希或不同请求返回冲突。
- 排队或待确认任务可取消；运行中的业务提交不可强制取消。任务源文件、结果文件和失败文件默认保留 `7` 天，过期后拒绝下载。
- 任务列表默认只返回当前用户任务；具备查看全部权限的管理员也必须叠加对象域权限。所有下载在执行时重新鉴权。

### XLSX 通用约束

- 只接受 `.xlsx`，单文件上限 `10 MiB`；物料最多 `10,000` 行，单个 BOM 最多 `5,000` 条明细。
- 拒绝宏、外部链接、公式单元格、隐藏业务列、超限工作表和压缩炸弹；读取必须使用受限流式或事件模式，不能无界加载。
- 文件校验分为结构、字段、引用、业务和权限五层。所有错误保存行号、列名、稳定错误码和中文建议；不得在错误明细中泄露凭据或 021 敏感结算税务字段。
- 校验无误后状态进入 `READY_TO_COMMIT`。用户确认时重新校验权限、任务版本和关键引用；业务条件变化时返回新的失败明细，不使用过期校验强行提交。
- 每个文件默认全有或全无，不提供“跳过错误行”或部分成功模式。

### 物料导入导出

- 物料导入首批只新增物料，不批量更新、启停或修改历史已引用字段。
- 字段覆盖现有创建口径：编码、名称、规格、物料类型、来源类型、追踪方式、分类、基本单位、状态、成本分类、库存计价分类和成本启用字段。
- 编码为空时在提交事务中调用 021 编码规则生成；显式编码按现有唯一性校验。校验阶段不得提前占号。
- 提交必须逐行复用物料业务服务的同一校验与审计语义，不允许批量 SQL 直接插入 `mst_material`。任一行失败，整个文件事务回滚。
- 物料导出按用户当前筛选快照生成 XLSX，包含当前用户有权查看的 021 成本属性，不包含客户、供应商敏感结算税务资料。

### BOM 草稿导入导出

- 一个导入文件只处理一个 BOM 草稿，模式为 `CREATE` 或 `UPDATE_DRAFT`。更新必须携带目标 BOM ID 和版本，并以整套明细替换方式原子提交。
- 头字段：父项物料、BOM 编码、版本、名称、基准数量、基准单位、有效期和备注；行字段：子项物料、业务单位、业务数量、损耗率、仓库和备注。
- 必须复用现有 BOM 业务服务校验父子项、单位换算、重复行、循环引用、数量、损耗率、版本唯一性、有效期和基本单位快照。
- 只能创建或更新 `DRAFT`；不得发布、启用、应用 ECO、修改已发布 BOM、生成替代料动作或改写工单用料快照。
- BOM 草稿导出只允许导出一个现有草稿的头与明细；已发布 BOM 不通过“草稿导出”入口伪装成可回写模板。

## 审计契约

- 继续使用 `sys_audit_log` 作为统一审计入口；V19 可增加结构化 `detail_json`，但不得另建不可统一查询的日志孤岛。
- 必须审计：审批提交/通过/驳回/撤回/取消，附件上传/下载/删除，打印预览/生成/下载，导入上传/校验/确认/失败，导出创建/生成/下载，任务领取/重试/取消/过期，新增平台接口权限拒绝，以及业务状态变化。
- 审计至少包含主体、动作、目标类型、目标 ID、结果、稳定错误码、请求路径、时间、关联审批或任务 ID；失败审计不得记录完整请求体、文件正文、原始错误行敏感值、对象存储凭据或签名。
- 审批最终通过同时保留审批动作审计和原业务动作审计，两者以关联 ID 串联，不用一个日志替代另一个。

## 数据模型与迁移原则

- 022 首个迁移为 `V19__fixed_approval_document_platform_schema.sql`，只追加平台表、索引、约束、审计结构和必要种子；不删除或破坏 V1 至 V18 对象。
- 最小平台表：审批定义、审批定义步骤、审批实例、审批任务、审批动作历史、站内消息、文件对象、业务附件、文档任务、导入暂存批次/行/错误、固定打印模板。
- 所有外部业务对象引用使用受控 `object_type + object_id`，并由代码白名单适配器校验；不依赖无外键、可任意写入的字符串对象类型。
- 同一对象同一场景开放审批、任务幂等键、附件对象关系、候选任务查询、任务租约、消息收件人、文档结果过期和文件 SHA-256 均建立必要索引或唯一约束。
- 不给 020 合同、021 ECO、BOM、物料或工单快照增加破坏性非空字段；历史记录不回写审批状态或文件关系。
- 权限种子幂等，`SYSTEM_ADMIN` 自动获得新增权限，重复初始化不产生重复权限、菜单或角色关系。
- 空库必须从 V1 顺序迁移到 V19；存量验证必须从真实 V18 结构和代表性 000-021 数据升级到 V19，并执行物料、BOM/ECO、工单快照、销售项目合同、库存、采购、销售、往来、报表和权限关键查询。

## 阻断缺陷

存在以下任一问题时不得合入或推送 `main`，不得通知用户验收：

- 合同生效或 ECO 应用可绕过审批，审批人可自批，重复/陈旧任务可再次改变业务状态，或审批通过与业务动作不是事务一致。
- 审批状态写入业务状态、历史对象被补造审批、既有合同/ECO/BOM 时效或已发布工单快照被改写。
- 待办、消息、审批摘要、任务数量、附件或结果文件跨用户或跨业务权限泄露。
- 附件可猜 ID 下载/删除、返回对象存储地址、伪造类型或超限上传成功，或存储故障造成有效元数据与对象不一致。
- PDF 缺少模板版本、中文不可读、权限撤销后仍可预览/下载，或过期结果仍可访问。
- 导入直接写业务表、绕过业务服务、产生部分提交、幂等失败造成重复数据，或 BOM 导入发布/应用/污染工单快照。
- 任务无租约或恢复语义导致重复执行，导出/失败明细下载绕过权限，或过期文件继续可用。
- 新增 `/api/admin` 路径无权限映射，权限种子不幂等，关键权限拒绝缺少审计。
- V19 迁移失败，V18 存量升级破坏 000-021 数据或关键路径。
- 核心页面无法访问或操作；桌面端布局、状态、错误、权限或主操作不可识别，导致验收路径不能执行。
- PostgreSQL、MinIO、后端或前端任一交付服务无法正常启动和访问。
