## 业务对象与状态

### 缺料分析运行

持久状态固定为：

| 状态 | 含义 | 允许操作 |
|---|---|---|
| `RUNNING` | 已锁定规范化范围，正在形成快照 | 查看运行信息，不允许建议动作 |
| `COMPLETED` | 快照完整且当前未发现来源变化 | 查看、导出、重算、按权限处理建议 |
| `FAILED` | 本轮计算失败，未形成可消费结果 | 查看错误摘要、重新运行 |
| `STALE` | 当前来源指纹与快照不同 | 查看历史、重算；禁止确认或转换建议 |
| `EXPIRED` | 距 `asOfTime` 超过 30 分钟且尚未重算 | 查看历史、重算；禁止确认或转换建议 |

- `STALE` 优先于 `EXPIRED`；状态可在查询时按当前来源指纹和 `expiresAt` 派生并持久化，不需要引入定时调度器。
- 失败运行只保存运行头、规范化范围、失败码、失败摘要和审计，不保存半成品需求、分配或建议。
- 重算总是创建新运行并保留 `previousRunId`，不得覆盖原快照。

### 建议

持久状态固定为：

| 状态 | 含义 | 允许操作 |
|---|---|---|
| `OPEN` | 新生成、尚未人工处理 | 确认、驳回 |
| `CONFIRMED` | 已人工确认 | 采购建议可转请购草稿；可驳回 |
| `CONVERTED` | 已生成请购草稿 | 查看来源和目标，不得重复转换 |
| `DISMISSED` | 已人工驳回并记录原因 | 查看历史 |

- 父运行处于 `STALE` 或 `EXPIRED` 时，建议展示对应失效状态并禁用所有写动作，但不改写建议历史状态。
- 建议类型固定为 `PURCHASE_REQUISITION`、`PRODUCTION_ORDER`、`USE_PUBLIC_STOCK`、`USE_EXISTING_SUPPLY`。
- `OUTSOURCED` 物料沿用 `PRODUCTION_ORDER` 建议类型并返回 `materialSourceType=OUTSOURCED`、`conversionAllowed=false`，界面显示“外协生产建议，027 后可执行”，不得在 026 创建外协事实。

## 输入契约

### 有效销售需求

- 只读取 `GET /api/admin/sales/effective-demands?countedOnly=true` 对应的可信后端事实，不允许前端拼接或重新判定计入口径。
- 需求粒度必须保留订单行和交付计划行；内部缺失的 `unitId`、仓库或来源版本由后端关联销售行/计划补齐。
- 项目订单形成项目需求，普通订单形成公共需求。
- `countedOnly=false` 仅用于诊断抽屉展示稳定排除原因。
- 销售退货不自动重开有效需求；026 不从退货推导新需求。

### BOM、单位与物料来源

- 对 `SELF_MADE`、`OUTSOURCED` 成品/半成品未覆盖量，按需求日期命中 021 已发布且有效的 BOM 版本。
- BOM 必须递归展开到可采购或没有下级有效 BOM 的物料；每层保存 BOM 版本、行、父需求和路径。
- 循环 BOM、无有效 BOM、物料停用和单位换算缺失必须形成稳定异常，不得默认为 0 或继续猜算。
- 数量统一使用基本单位和 `NUMERIC(18,6)`；接口返回十进制字符串，前端不得用浮点数形成可信净算。
- BOM 损耗率按当前 021 契约应用；每层计算后按基本单位精度舍入，不能只在最终结果一次舍入。
- `PURCHASED` 物料形成采购建议；`SELF_MADE` 形成生产建议；`OUTSOURCED` 形成不可转换的外协生产建议。

### 库存、采购和工单供给

- 库存只使用 023 `availableQuantity` 的合格现货净可用口径；待检、不合格、冻结、已预留、已占用不得计入。
- 项目库存只能覆盖同项目需求；其他项目库存不得覆盖当前项目或公共需求。
- 公共库存可以参与项目/公共需求净算，但只形成 `USE_PUBLIC_STOCK` 分配和建议，不自动转换所有权或预留。
- 采购只使用 024 `countedOnly=true` 有效供给；关闭、取消、全部入库或剩余量为 0 的来源不得计入。
- 项目采购只能覆盖同项目需求；公共采购可参与项目/公共需求，其他项目采购不得计入。
- 既有工单只使用 `RELEASED`、`IN_PROGRESS` 且未完工入库数量大于 0 的供给；当前无项目归属，统一视为公共供给或潜在提示。
- `DRAFT`、`COMPLETED`、`CANCELLED` 工单不得计入；026 不补造项目归属。

## 净算与供给分配契约

### 规范化范围

- 运行请求支持项目、销售订单、客户、合同、物料、需求日期止和是否包含公共需求；开始日期默认包含全部逾期开放需求。
- 默认需求窗口截至当前业务日期后 180 天；显式日期不得超过 365 天。
- 规范化范围形成 `scopeHash`，并原样保存在运行快照中。

### 确定性顺序

- 需求排序：需求日期升序、项目标识空值后置、项目编号、销售订单号、交付计划行 ID、需求物料 ID、BOM 路径。
- 供给排序：可用日期升序、来源优先级、来源单号、来源行 ID。
- 相同公共供给可服务多个需求时必须按上述需求顺序逐行扣减剩余量；同一供给分配总量不得超过快照时剩余量。

### 供给优先级

项目需求固定按以下顺序分配：

1. 同项目合格可用库存。
2. 同项目且能在需求日期前到达的有效采购供给。
3. 公共合格可用库存。
4. 公共且能在需求日期前到达的有效采购供给。
5. 能在需求日期前完工的既有公共工单供给。

公共需求固定按以下顺序分配：

1. 公共合格可用库存。
2. 公共且能在需求日期前到达的有效采购供给。
3. 能在需求日期前完工的既有公共工单供给。

- 晚于需求日期的采购或工单供给不覆盖准时需求，只记录为延迟供给并参与预计可用日期解释。
- 其他项目资源只输出 `CROSS_PROJECT_NOT_ALLOWED` 提示，不能进入覆盖数量。

### 公式与日期

- 直接需求量 = 有效销售需求 `openQuantity`。
- 成品未覆盖量 = `max(直接需求量 - 准时成品供给覆盖量, 0)`。
- BOM 毛需求 = `父项未覆盖量 × 子项基本数量 ÷ BOM 基准数量 × (1 + lossRate)`，每层换算并舍入到基本单位精度。
- 子项净缺料 = `max(BOM 毛需求 - 准时子项供给覆盖量, 0)`。
- 库存可用日期为快照业务日期；采购使用预计到货日；工单使用计划完工日。
- 准时覆盖后的短缺若存在晚到供给，预计可用日期取补足当前缺口所需最后一笔晚到供给日期；仍不足则为空。
- 金额、价格和正式成本不进入 026 核心净算或建议数量。

## 快照、过期、并发与幂等

- 运行保存 `asOfTime`、业务日期、需求窗口、规范化范围、`scopeHash`、`sourceFingerprint`、来源最大更新时间、运行人、完成时间和 `expiresAt`。
- `sourceFingerprint` 必须覆盖本范围实际读取的有效需求、BOM/单位、库存/预留、有效采购和有效工单的稳定主键、版本/更新时间及参与数量。
- 查询运行时按相同范围重算轻量来源指纹；不同则为 `STALE`。未变化但超过 `expiresAt` 则为 `EXPIRED`。
- 同一 `idempotencyKey` 和相同规范化请求返回同一运行；同键不同请求返回 409。
- 以 `scopeHash` 获取并发锁。同范围并发请求在锁内若发现 30 分钟内相同来源指纹的已完成运行，返回该运行，不重复生成快照；来源变化或已过期才创建新运行。
- 运行头创建、计算结果写入和失败状态必须明确事务边界；计算失败不得保留半成品需求、分配和建议。
- 所有建议写动作要求 `version`、`idempotencyKey`；409 不得由前端自动重放。

## 采购建议转请购草稿

- 只有父运行仍为 `COMPLETED`、建议为 `CONFIRMED`、类型为 `PURCHASE_REQUISITION`、剩余建议数量大于 0 时允许转换。
- 操作者同时需要 `planning:material-requirement:convert-requisition` 与 `procurement:requisition:create`。
- 转换要求建议版本和幂等键；同键重试返回同一请购，不得重复创建。
- 项目需求生成 `PROJECT` 请购并保留项目；公共需求生成 `PUBLIC` 请购且项目为空。
- 请购状态固定为 `DRAFT`；物料、基本单位、建议数量、需求日期、来源运行/建议写入来源链。供应商、价格和审批信息保持为空。
- V28 为请购增加可空的 026 来源字段或等价唯一来源关系，并对同一建议只允许一个非取消来源目标。
- 转换不提交审批、不选价、不生成采购订单；后续仍完全遵循 024 状态机、权限和固定审批。

## 验收矩阵

### 主路径

- 有效项目需求按有效 BOM 展开，项目库存、项目采购、公共库存、公共采购、既有工单按冻结顺序覆盖，分配不重复。
- 公共需求只使用公共资源，不使用任何项目专属资源。
- 多需求共享同一公共供给时，按需求日期和稳定键分配，重复运行同一来源得到相同结果。
- `PURCHASED`、`SELF_MADE`、`OUTSOURCED` 短缺生成正确建议及转换边界。
- 采购建议确认后幂等生成 024 `DRAFT` 请购，保留项目/公共模式和双向来源。

### 异常路径

- 未来/历史/无效 BOM、循环 BOM、单位换算缺失、物料停用均返回稳定异常且不猜算。

- 来源变化、30 分钟过期、并发重算、版本冲突、幂等冲突和计算失败均可解释且无半成品。
- 空需求范围形成可查看的已完成空快照，不以 500 或伪造建议结束。

### 权限与追溯

- 无查看权限不能访问列表、详情或导出；无计算权限不能运行/重算；无建议管理权限不能确认/驳回。

- 来源详情权限不足时脱敏或拒绝，不通过汇总、导出或抽屉泄露。
