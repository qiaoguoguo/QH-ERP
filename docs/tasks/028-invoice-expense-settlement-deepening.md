## 方案比较与采用结论

1. 把现有应收/应付直接改名或升级为发票：实现表面简单，但会混淆余额台账与开票事实、破坏历史来源与状态语义，不采用。
2. 前向追加发票、费用、预收预付、核销和凭证草稿子账，通过稳定关系桥接既有应收/应付余额台账：历史兼容、职责清晰、可供后续阶段消费，采用。
3. 直接建设正式财务子系统：会提前引入正式科目、凭证、总账、月结、税务和银行对账，明显越界，不采用。

主代理合并五角色意见后作出以下阶段裁决：

- 允许 V30 移除收付款单目标分配限制，升级为一笔资金多目标核销；历史单目标记录原样保留并继续可读。
- 标准采购三单匹配首版采用零容差。数量、未税单价、税率、未税金额、税额、含税金额、供应商或项目归属任一不一致时进入差异状态，只能修改草稿或业务源头，不允许确认采购发票、生成应付链接或凭证草稿。
- 销售/采购发票确认时优先绑定已有有效应收/应付；没有对应台账时，按原收发货来源调用既有规则自动生成确认态应收/应付，再建立发票链接。不得新增第二份来源余额。

## 范围

### 纳入范围

#### 销售发票

- 基于已过账销售出库行的不可变税价、客户、项目、合同和订单快照创建发票草稿。
- 同一发票只能属于同一客户、同一人民币币种和同一项目/公共归属；不允许一张发票混合多个项目或项目与公共来源。
- 支持一个发票引用同归属下多个销售出库行；支持来源行部分开票，但累计开票数量和金额不得超过来源快照净可开票余额。
- 草稿允许编辑开票日期、外部发票号码、发票类型、来源分配、备注和必要税务快照；来源客户、项目、合同、税价与累计边界由后端校验。
- 确认时绑定或生成基础应收，并建立发票、应收和来源行关系；确认后不可普通编辑或取消。
- 展示来源、价税、应收、收款/预收核销、审计和凭证草稿追溯。

#### 采购发票与三单匹配

- 标准采购发票基于同一供应商、同一项目/公共归属的采购订单行与已过账采购入库行创建。
- 保存供应商发票号码、发票日期、类型、采购订单、采购入库、数量、未税单价、税率、未税金额、税额和含税金额快照。
- 每次保存或显式匹配都由后端重算三单结果；完全一致为 `MATCHED`，未执行为 `UNMATCHED`，任一差异为 `EXCEPTION`。
- `EXCEPTION` 只记录结构化差异，不允许人工例外确认；确认采购发票必须处于 `MATCHED`。
- 确认时绑定或生成基础应付，并建立发票、应付、采购订单行和入库行关系；不得更改采购、入库、库存数量或价值。
- 外协供应商结算允许引用同一供应商、同一项目/公共归属的 027 外协订单与已过账外协收货，保存暂估与实际结算差异，但不套用标准采购订单三单匹配，不回写库存暂估或正式项目成本。

#### 费用单

- 费用单固定区分 `PROJECT` 项目费用和 `PUBLIC` 公共费用；项目费用必须有有效项目，公共费用项目为空。
- 首版费用单的往来方固定为启用供应商；不引入员工、报销人、临时往来方或新的人员主数据。
- 费用行包含受控费用分类、供应商、可选采购/外协来源、项目/公共归属、业务日期、未税金额、税率、税额、含税金额和备注。
- 项目费用可以引用销售项目、采购来源、外协订单或已过账外协收货；公共费用不能伪挂项目来源。
- 有可信来源时保存来源快照；无来源的普通供应商费用使用 021 结算税务资料作为默认输入，最终金额和税额仍由后端校验。
- 确认费用单生成或链接应付余额；028 只形成“待 029 消费的费用归属事实”，不写库存价值、成本层、工单成本或项目利润。

#### 预收预付与收付款深化

- 现有 `fin_receipt`、`fin_payment` 继续代表真实资金发生；V30 支持一笔收款/付款分配多个应收/应付，并允许保留未核销余额。
- 新建收款/付款可以带零到多个分配目标。零分配或分配后仍有余额时，过账后形成客户预收或供应商预付可用余额。
- 一般收款、付款和预收预付都必须固定往来方、人民币、项目/公共归属和业务日期；项目资金只能核销同项目目标，公共资金只能核销公共目标。
- 核销只允许同客户/供应商、同币种、同项目/公共归属；支持部分、多次、多目标分配，禁止超额、重复、跨主体、跨项目和并发穿透。
- 核销过账只减少资金可用余额并增加目标已核销金额，不再次增加收款/付款发生额。
- V11 `adjusted_amount`、往来冲减、退款和退货事实继续参与可核销余额；已关闭、已取消、已结清或无余额目标不可核销。
- 历史单目标收付款无需补造预收预付，不改编号、金额、状态或分配关系。

#### 对账核销

- 提供客户与供应商方向的对账核销工作台，按往来方、项目/公共归属、日期和状态展示应收/应付、发票、费用、收付款、预收预付和 V11 冲减。
- 后端返回每一目标的原金额、已收/已付、已冲减、已核销、未结余额和来源摘要；前端不重算可信余额。
- 工作台提交核销草稿或过账请求时携带版本与幂等键，服务端锁定资金与全部目标后一次提交；失败不得产生部分分配。

#### 凭证草稿

- 支持从已确认销售/采购发票、已确认费用单、已过账收款/付款和已过账核销事件幂等生成一份活动凭证草稿。
- 草稿包含来源事件、草稿号、业务日期、摘要、往来方、项目、借贷方向、业务分类、未税金额、税额、含税金额和生成版本。
- 草稿分录借贷金额必须相等；金额和分类由后端按固定业务映射生成，前端不得自由改金额或构造平衡结果。
- 状态固定为 `DRAFT`、`READY`、`CANCELLED`；`READY` 只表示待 031 正式制证，不表示已审核、已过账或已记账。
- 不引入正式科目表、正式科目编码、会计期间、正式凭证号、总账、试算平衡或记账接口。

#### 权限、审计、追溯与兼容

- 新资源具备独立查看、创建、更新、确认/过账、取消草稿、匹配、核销和凭证草稿生成权限。
- 发票快照中的完整税号、开户行、银行账号等字段必须同时受财务敏感字段权限控制；普通发票查看权限不自动获得完整敏感字段。
- 所有服务层在缺少 `CurrentUser` 或权限时失败关闭；路径权限不能替代服务层对象、动作、项目、敏感字段和成本权限校验。
- 所有确认、过账、匹配、核销、取消草稿和凭证草稿生成记录系统审计；摘要不得泄露完整税号、银行账号、内部对象存储地址或无权成本金额。
- 新写入继续受 016 业务期间保护；历史查询、追溯和已锁定期间只读访问保持可用。028 不建设 030 月结。
- 现有应收、收款、应付、付款、往来冲减页面和 API 保持兼容，并增加发票/费用/核销摘要与受限态，不改变历史响应核心语义。

## 状态机与动作

### 发票和费用

| 对象 | 状态 | 允许动作 | 禁止动作 |
|---|---|---|---|
| 销售/采购发票 | `DRAFT` | 查询、更新、来源分配、匹配（采购）、确认、取消草稿 | 核销、正式制证、记账 |
| 销售/采购发票 | `CONFIRMED` | 查询、追溯、核销摘要、生成凭证草稿 | 普通编辑、取消、重复确认、改来源 |
| 销售/采购发票 | `CANCELLED` | 查询、追溯 | 任何写动作 |
| 费用单 | `DRAFT` | 查询、更新、确认、取消草稿 | 核销、生成凭证草稿 |
| 费用单 | `CONFIRMED` | 查询、追溯、核销摘要、生成凭证草稿 | 普通编辑、取消、改归属 |
| 费用单 | `CANCELLED` | 查询、追溯 | 任何写动作 |

发票结算状态、费用结算状态均由应收/应付余额派生为“未结清、部分结清、已结清”，不是可直接写入的业务状态。

### 资金与核销

| 对象 | 状态 | 允许动作 | 禁止动作 |
|---|---|---|---|
| 收款/付款 | `DRAFT` | 更新资金信息、分配目标、过账、取消草稿 | 产生余额影响 |
| 收款/付款 | `POSTED` | 查询、追溯、对未核销余额发起核销、生成凭证草稿 | 普通编辑、取消、重复增加现金 |
| 收款/付款 | `CANCELLED` | 查询 | 任何写动作 |
| 核销 | `DRAFT` | 更新分配、过账、取消草稿 | 改变真实资金金额 |
| 核销 | `POSTED` | 查询、追溯、生成凭证草稿 | 普通编辑、取消、重复过账 |
| 核销 | `CANCELLED` | 查询 | 任何写动作 |

预收/预付展示状态由真实资金单状态和未核销余额派生为“草稿、可用、部分核销、已核销、已取消”。

### 凭证草稿

| 状态 | 含义 | 允许动作 |
|---|---|---|
| `DRAFT` | 已生成但尚未确认移交 | 查询、校验、标记待制证、取消 |
| `READY` | 待 031 正式制证 | 查询、来源追溯、取消 |
| `CANCELLED` | 草稿已取消 | 只读查询 |

## 金额、税额、来源与余额规则

- API 中数量、单价、税率和金额统一使用十进制字符串；前端不得使用 `Number` 或浮点运算形成可信业务结果。
- 数量与单价使用不低于 `numeric(18,6)`，税率使用 `numeric(9,6)`，金额使用 `numeric(18,2)`；税率存储为 0 至 1。
- 行级未税金额、税额、含税金额由后端按来源快照和数量计算，统一四舍五入到两位；头金额必须等于行金额汇总。
- 来源剩余可开票数量和金额等于合法来源净额减去所有非取消发票行累计；退货和 V11 冲减必须纳入净额，结果不得为负。
- 采购三单匹配逐行比较订单、已过账入库与发票的供应商、物料、单位、项目/公共归属、数量、未税单价、税率和三类金额；不以头汇总相等代替逐行一致。
- 发票确认、费用确认、收付款过账、核销过账和凭证草稿生成必须带版本和幂等键；重复同键返回同一结果，冲突键整体拒绝。
- 一笔资金的金额等于已核销金额与未核销余额之和；应收/应付继续满足 V11 原金额等于收付、冲减与未结余额之和。
- 任何并发写入必须锁定资金头、全部目标余额和来源累计；超额、负余额或版本冲突整体回滚。

## 接口边界

以下为稳定资源边界；具体请求字段必须遵守本文状态、金额、权限和版本契约，不能通过路径差异规避服务层校验。

- `/api/admin/finance/sales-invoices`：候选来源、列表、详情、创建、更新草稿、确认、取消草稿、来源追溯。
- `/api/admin/finance/purchase-invoices`：候选来源、列表、详情、创建、更新草稿、匹配、确认、取消草稿、来源追溯。
- `/api/admin/finance/expenses`：分类/来源候选、列表、详情、创建、更新草稿、确认、取消草稿。
- `/api/admin/finance/advance-receipts`、`/prepayments`：列表、详情、创建资金草稿、过账、取消草稿、可用余额与核销记录。
- `/api/admin/finance/settlement-workbench`：客户/供应商待核销汇总、候选资金与目标、核销草稿、过账、取消草稿、追溯。
- `/api/admin/finance/voucher-drafts`：列表、详情、从来源生成、标记待制证、取消和来源追溯。
- 既有 `/receivables`、`/receipts`、`/payables`、`/payments`、往来冲减接口保持兼容；新增通用收款/付款创建与多目标分配入口时，原嵌套单目标入口继续可用并调用同一服务。
- 所有详情响应必须返回 `version`、`allowedActions`、业务状态、派生结算状态、十进制字符串、来源摘要、权限受限原因和审计摘要；无权字段不得用 0、空串或内部 ID 伪装。
- 所有失败使用稳定业务错误码，至少区分不存在、状态不允许、来源不一致、匹配差异、累计超额、余额不足、跨主体/项目、并发冲突、幂等冲突、期间锁定、权限不足和敏感字段受限。

### 权限组

- `finance:sales-invoice:view/create/update/confirm/cancel`
- `finance:purchase-invoice:view/create/update/match/confirm/cancel`
- `finance:expense:view/create/update/confirm/cancel`

- `finance:voucher-draft:view/generate/ready/cancel`

角色方向：

- 系统管理员拥有全部 028 权限。
- 财务角色拥有业务维护、确认、过账、核销和凭证草稿权限；敏感字段权限独立授予。
- 销售角色只按授权查看销售发票、应收与收款进度，不得确认发票、过账资金或核销。
- 采购角色只按授权查看采购发票、匹配状态、应付与付款进度，不得确认发票、过账资金或核销。
- 项目角色只查看有权项目的归属摘要；无财务金额权限时不得返回金额明细。
- 只读用户只有明确授予的查看权限；无权限用户菜单、路由和 API 均不可访问。

### 审计要求

- 记录对象类型、对象 ID/编号、动作、前后状态、往来方、项目/公共归属、来源摘要、金额摘要、幂等键、版本、操作者、时间、请求路径和成功/失败。
- 三单匹配审计记录匹配版本与差异类别，不写无关大字段。
- 核销审计记录资金单、全部目标和分配金额；失败事务不得写成功审计或部分分配。
- 敏感税务与银行资料只记录“新增/修改/清空/受限访问”类型，不明文保存值。

## V30 数据迁移与兼容要求

- 新增销售发票、销售发票行与来源分配、采购发票、采购发票行与匹配快照、费用单与费用行、资金未核销余额/核销记录、凭证草稿与分录、发票/费用到应收应付的链接表，以及必要索引、检查约束、权限和菜单种子。

- 迁移前后正式文件对象与 MinIO 对象数量必须一致；任何测试不得命中正式 `qherp/qherp-private`。

### 通用壳层、密度和状态契约

- 页面壳层：优先复用现有 `MasterDataTableView`、`.module-page`、`.page-heading`、`.query-card`、`.table-card`、`.table-scroll` 和 `.table-pagination` 模式；复杂工作台可在同一壳层内使用不嵌套卡片的分区，不得在卡片内再堆卡片。
- 页头：每页必须有清晰标题和一句业务口径说明；页头右侧只放页面级主操作，例如新增、返回列表、保存草稿、确认、过账、生成凭证草稿。行级操作、查询条件和批量说明不得放入页头。
- 首屏扫描顺序：状态和归属先于金额，金额先于来源明细。首屏从左到右、从上到下必须能识别对象编号、往来方、项目/公共归属、业务状态、派生结算状态、未税金额、税额、含税金额、未结余额和允许动作。
- 查询区：所有列表和工作台筛选必须使用标签置顶网格；日期范围使用两个日期选择器，清空值落为 `''`；查询、重置和方向切换不得改变无关筛选，查询、重置、每页条数变化后分页回到第一页。
- 指标卡：只用于需要首屏金额扫描的列表或工作台，每组桌面首屏不超过 4 个主指标；必须包含指标名称、金额、口径提示和受限态。无权限金额显示“无权查看”，不得显示 `0.00`、空字符串或内部 ID。
- 表格：主列表列数按 1280×720 可扫描控制在 10 至 14 个业务主列，额外字段进入详情或抽屉；金额列右对齐并使用等宽数字；操作列固定在右侧或横向滚动后可访问，操作按钮不得挤压换行遮挡。
- 分页：所有主列表默认 10 条并提供 10/20/50/100；候选池、来源池和核销池不套用主列表分页限制，必须支持独立分页、搜索或加载足够候选。
- 弹窗/抽屉：来源追溯、差异明细、核销明细、审计明细优先使用抽屉或受视口约束的弹窗；内容区滚动、底部关闭/确认动作固定可达，抽屉内宽表也必须有横向滚动容器。
- 禁用原因：按钮因状态、权限、期间锁定、匹配差异、余额不足、来源受限、敏感受限或成本受限不可执行时，必须在按钮旁、按钮提示或状态提示中给出中文原因；不得只隐藏关键阻断原因。
- 状态表达：加载、空、错误、无权限、只读、禁用、敏感受限、成本受限和来源受限必须真实可辨；内部枚举、来源类型和错误码必须映射为中文业务文案。

- 列表筛选：关键词、供应商、项目/公共归属、采购订单、采购入库、外协订单、发票状态、匹配状态、结算状态、发票日期起止、外部发票号码、来源类型。
- 列表首屏列：发票号、外部发票号码、供应商、来源类型、项目/公共归属、采购订单/外协订单、入库/收货摘要、发票日期、状态、匹配状态、含税金额、未结余额、差异数、操作。标准采购和外协行必须通过来源类型标签区分。
- 新建/编辑页：标准采购发票的首屏扫描顺序为供应商、项目/公共归属、采购订单、已过账入库、匹配状态、价税合计；外协结算的首屏扫描顺序为供应商、项目/公共归属、外协订单、已过账外协收货、暂估金额、实际结算金额、差异金额。来源候选池必须独立于主列表分页，按供应商、归属、订单/入库、收货日期、可开票余额搜索。
- 标准采购来源表：展示采购订单行、入库行、物料、单位、订单数量、入库数量、发票数量、订单未税单价、入库快照税率、发票未税单价、未税金额、税额、含税金额；差异字段必须同时有颜色和文字原因。
- 详情页：页头动作为返回、编辑草稿、执行匹配、确认、取消草稿、生成凭证草稿；`EXCEPTION` 状态的确认按钮必须禁用并显示“存在三单差异，零容差规则禁止确认”。分区顺序为基础信息、匹配摘要、三单明细、应付链接、付款/预付核销、凭证草稿、审计。
- 三单匹配工作台：页面主体分为订单事实、入库事实、发票事实和差异结果四个扫描区；同一行按物料/来源行对齐，差异抽屉展示差异类型、订单值、入库值、发票值、阻断动作和可返回源单的链接。不得以头汇总相等替代逐行匹配结论。
- 外协状态：外协结算不显示“采购订单三单匹配通过”，只显示“外协结算来源已引用”和暂估/实际差异；成本受限时隐藏无权成本金额但保留状态和动作禁用原因。

### 费用单逐页契约

- 列表页：标题为“费用单”，说明必须表达“记录项目/公共供应商费用归属，供后续成本和凭证阶段消费，不形成正式项目成本”。页头主操作为“新增费用单”。
- 列表筛选：关键词、供应商、费用分类、项目/公共归属、来源类型、费用状态、结算状态、业务日期起止、成本受限状态。
- 列表首屏列：费用单号、供应商、费用分类、项目/公共归属、来源类型、来源编号、业务日期、状态、结算状态、未税金额、税额、含税金额、未结余额、操作。
- 新建/编辑页：表单分组为往来方与归属、费用分类、来源候选、费用行、税务快照、备注。项目费用必须先选择项目再显示项目来源候选；公共费用不得展示可挂项目来源的候选结果。
- 来源候选：采购来源、外协订单和外协收货候选使用独立分页或足够加载，支持来源类型、供应商、项目/公共归属、业务日期和编号搜索；候选为空显示“当前条件下无可用来源”，不能显示为无权限或加载中。
- 详情页：状态条先显示费用状态、供应商、项目/公共、费用分类、业务日期、价税合计、应付/核销摘要；分区为费用行、来源快照、应付链接、付款/预付核销、非正式成本提示、凭证草稿、审计。确认后只读，必须显示“028 不写库存价值、工单成本或项目利润”。

### 预收预付和收付款深化逐页契约

- 预收款列表：标题为“预收款”，说明为“展示已过账收款形成的客户未核销余额”。筛选包括关键词、客户、项目/公共归属、资金状态、核销状态、收款日期起止、可用余额是否大于 0；列为预收单号/收款单号、客户、项目/公共、收款日期、收款金额、已核销金额、可用余额、状态、最近核销时间、操作。
- 预付款列表：标题为“预付款”，供应商方向文案、筛选和列与预收款保持一致，所有“客户/收款”文案替换为“供应商/付款”。
- 登记/编辑页：预收和预付登记页复用收款/付款草稿语义，首屏显示往来方、项目/公共归属、业务日期、资金金额、分配摘要和未核销余额；资金方式必须为受控选项或后端返回枚举中文，不能继续用裸文本占位。
- 详情页：分区为资金事实、可用余额、已核销目标、未核销余额、凭证草稿、审计。已过账记录不可编辑，未核销余额大于 0 时显示“发起核销”入口；无核销权限时显示只读余额。
- 收款/付款详情受影响契约：既有详情必须增加多目标核销明细、预收/预付余额、发票/费用链接和凭证草稿摘要；历史单目标记录继续按原对象可读，不能因新核销表而隐藏历史来源。旧嵌套登记入口保留，但新通用登记和多目标分配入口必须在文案上区分。

### 对账核销工作台契约

- 页面壳层：标题为“对账核销工作台”，说明必须表达“按往来方和项目/公共归属核销资金余额与应收应付，不新增现金发生额”。页头可放“新建核销草稿”或“过账核销”，但必须依赖已选分配和 `allowedActions`。
- 首屏结构：顶部为方向切换、往来方、项目/公共归属和日期筛选；中部左侧为可用资金池，右侧为可核销目标池；底部或右侧固定本次核销摘要和提交区。1280×720 下提交区必须可见，不得被表格纵向滚动推出视口。
- 资金池列：资金单号、资金类型、业务日期、原金额、已核销金额、可用余额、项目/公共、状态、受限原因、操作。目标池列：目标类型、目标单号、来源摘要、业务日期、原金额、已收/已付、已冲减、已核销、未结余额、状态、受限原因、操作。
- 多目标交互：资金池和目标池均不得使用主列表当前 10 条分页作为候选全集；必须支持独立查询、分页或足够加载。用户选择资金或目标后，已选态、剩余可分配金额、目标未结余额、本次核销金额、超额错误和提交禁用原因必须即时同步。
- 提交确认：过账前使用统一确认弹窗，展示往来方、项目/公共归属、资金笔数、目标笔数、本次核销总额和幂等键口径；并发冲突、余额不足、跨主体/跨归属失败必须保留页面输入并显示中文错误。
- 空错权限态：无资金、无目标、方向无权限、往来方无权限、项目无权限、金额受限、来源受限分别有明确空态或受限态，不得混用。

### 凭证草稿逐页契约

- 列表页：标题为“凭证草稿”，说明必须包含“仅为 031 正式制证提供业务分类建议，不是正式凭证”。页头主操作只能是“从来源生成草稿”或批量刷新建议，不得出现审核、记账、过账、生成正式凭证。
- 列表筛选：关键词、来源类型、来源编号、往来方、项目/公共归属、草稿状态、业务日期起止、生成日期起止、借贷平衡状态。
- 列表首屏列：草稿号、来源类型、来源编号、业务日期、往来方、项目/公共、状态、借方合计、贷方合计、平衡状态、生成版本、更新时间、操作；借贷金额右对齐，平衡异常使用文字和颜色双重提示。
- 详情页：页头动作为返回、刷新、标记待制证、取消草稿；状态条必须显示“非正式凭证草稿”标签。分区为来源事件、分录建议、金额平衡、生成版本、来源追溯、审计；分录表展示借贷方向、业务分类、摘要、未税金额、税额、含税金额、往来方和项目，不展示正式科目编码或正式凭证号。
- 来源受限：无来源查看权限时保留来源类型、来源编号脱敏摘要和“来源受限”标签；不得用空白分录或金额 0 伪装权限不足。

### 受影响既有页面契约

- 应收/应付列表：保留历史台账口径，在说明中补充“含历史业务台账，028 发票/费用通过链接衔接”；新增发票/费用链接、核销摘要和凭证草稿状态列时，仍需控制主列不超过 14 个，旧来源列和金额列不得被挤出首屏扫描顺序。
- 应收/应付详情：顶部摘要新增关联发票/费用、派生结算状态、多目标核销摘要和凭证草稿摘要；来源追溯扩展为通用来源类型，但历史 `SALES_SHIPMENT`、`PURCHASE_RECEIPT` 文案和跳转不变。
- 收款/付款列表与详情：增加未核销余额、已核销金额、多目标数、预收/预付状态、凭证草稿摘要；已过账记录只读，普通编辑入口只对草稿可见。行操作过多时使用“详情”主入口承载次级操作，不能让操作列宽度无上限增长。
- 收款/付款表单：现有单目标登记保留；新增通用资金登记必须展示可选分配目标和未分配余额，资金方式、业务日期和项目/公共归属为明确控件。单目标入口进入时，应在摘要区显示目标应收/应付并允许保存为部分分配。
- 往来冲减列表/表单/详情：保留 V11 冲减语义；新增核销可用余额提示和来源受限态时，不得把冲减错误展示为预收预付，不得改变既有退款/退货追溯。
- 财务导航与根路由：财务菜单新增销售发票、采购发票、费用单、预收款、预付款、对账核销、凭证草稿，并按权限隐藏；财务根路由优先跳到当前用户有权的 028 主页面，若仅有历史权限则回落到既有应收、收款、应付、付款或往来冲减。
- 来源详情入口：销售项目、销售出库、采购订单、采购入库、外协订单和外协收货详情只增加财务摘要或抽屉入口；摘要显示发票状态、结算/核销状态和来源受限原因，不得在来源模块页面提供财务确认、过账或核销写动作。

### 复用组件与必须重构边界

- 可复用：`MasterDataTableView` 页面壳层、`.query-form` 标签置顶网格样式、`.table-scroll` 宽表滚动、`.table-pagination` 分页、`confirmAction` 统一确认弹窗、`returnTo` 导航工具、`formatFinanceAmount` 展示格式、状态标签组件模式和财务来源追溯表格结构。
- 必须扩展：`financePageHelpers` 需要新增 028 权限、状态中文映射、来源类型中文映射和首个可访问财务路由；`FinanceSourceTracePanel` 需要扩展为销售发票、采购发票、费用、外协、核销和凭证草稿可复用的通用追溯组件或拆分组件。
- 必须治理：新增或受影响列表不得继续新增第二套横向 label 搜索栏；涉及 028 改动的既有财务查询区应保持标签置顶网格、日期清空为空字符串、分页 10/20/50/100、固定或可达操作列。
- 必须替换：收款/付款“方式”不得继续以裸文本占位表达关键枚举；应使用受控选项或后端返回的中文枚举。涉及多目标核销的表单不得沿用“一个资金单只能选一个目标”的摘要和校验。
- 禁止模式：禁止把候选来源、来源追溯或核销目标限制为当前主列表 10 条；禁止在列表下方展开长追溯表；禁止显示裸内部码、裸错误码、裸来源类型；禁止用空白、`0.00` 或隐藏列表示无权限金额、敏感字段或成本字段。

### 真实桌面验收门禁

- UI 集中审查和测试视觉检验必须使用 1280×720 真实桌面页面逐页核对本节页面范围；不截图，不测试移动端，不用源码检查、单测或功能冒烟替代页面规范结论。
- 每个新增页面至少检查加载、空、错误、无权限、只读或禁用、敏感受限或成本受限、正常数据、关键动作确认弹窗、抽屉打开和关闭、宽表横向滚动、分页、返回链。
- 对账核销工作台必须真实检查资金池和目标池多候选、多目标选择、超额禁用、提交区可达和候选池不受主列表分页限制。
- 三单匹配工作台必须真实检查完全匹配和至少一种差异状态，确认按钮禁用原因、差异抽屉、三列对齐和页面级溢出。
- 最终记录必须逐页说明检查路由、数据状态、适用检查项、通过项、不满足项、严重度、建议归属和数据/账号缺口；不得只写“视觉通过”。

## 完整业务工作包

工作包数量为六个，因为发票、匹配、费用、资金核销和凭证草稿具有独立状态与数据所有权，机械合并会扩大单服务和页面复杂度。接口与验收矩阵冻结后，前端、后端、UI 和测试在真实依赖允许时并行；每个工作包包含实现、受影响测试和定向验证，不拆红测/实现/转绿流程任务。

### 工作包一：V30 兼容底座与财务前端基础

成果：V30 表/约束/索引/权限/错误码、历史 V10/V11 多目标迁移、业务期间/审计/幂等/敏感权限公共边界；前端财务 API 分域、路由菜单、共用状态/金额/来源/候选池/核销组件，以及受影响既有页面查询规范治理。

定向验证：V30 空库和重点存量迁移、历史财务查询、权限映射、十进制类型、路由菜单和公共组件测试。

### 工作包二：销售发票与应收衔接

成果：销售发票候选、草稿、部分开票、确认、应收绑定/生成、来源追溯、收款/预收核销摘要、列表/表单/详情和定向测试。

定向验证：累计边界、防重、税额舍入、项目/公共一致、历史应收兼容、权限/敏感受限和页面状态。

### 工作包三：采购发票、外协结算与三单匹配

成果：标准采购发票、零容差三单匹配、结构化差异、应付绑定/生成；外协订单/收货供应商结算来源、暂估差异只读追溯；列表/表单/详情/匹配工作台和定向测试。

定向验证：完全匹配、各类差异、重复/超额、供应商/项目不一致、退货净额、外协不回写库存价值、权限和页面差异识别。

### 工作包四：项目/公共费用单

成果：费用分类、项目/公共归属、供应商与采购/外协来源、价税、确认应付链接、非正式成本提示、列表/表单/详情和定向测试。

定向验证：归属二选一、终态项目、来源一致、无来源默认税务、应付防重、不写项目成本/库存价值、成本/敏感权限和页面状态。

### 工作包五：预收预付与多目标对账核销

成果：收付款多目标分配、未核销余额、预收款/预付款、客户/供应商对账核销工作台、版本/幂等/并发锁、历史单目标兼容和定向测试。

定向验证：零/部分/全额/多目标核销、超额、重复、并发、跨主体/项目、V11 冲减、终态目标、事务原子性和资金页面。

### 工作包六：凭证草稿与跨链追溯

成果：固定业务分类、平衡分录、幂等生成、待正式制证/取消、来源追溯、列表/详情；销售项目、采购、外协和报表相关财务摘要入口。

定向验证：来源事件覆盖、借贷相等、不得正式记账、重复生成、权限/受限来源、返回链和页面“非正式凭证”边界。

## 测试隔离数据最小集合

- 客户、供应商各至少 2 个，覆盖完整/缺失税务资料、敏感权限、税率 13%、6%、0%。
- 项目 A、项目 B 和公共归属，覆盖活动与终态项目；禁止把终态项目用于新费用或资金归属。
- 销售：合同/订单、部分与全部已过账出库、历史 V10 应收/收款、退货与冲减、未生成应收来源。
- 采购：项目专采/公共采购、多个订单、部分与多批已过账入库、历史 V10 应付/付款、采购退货与冲减。
- 外协：已发布订单、已过账发料、一个或多个已过账收货、暂估单价与实际结算差异。
- 费用：项目、公共、采购来源、外协来源和无来源普通供应商费用。
- 资金：普通收款/付款、多笔预收/预付、部分/全额/多目标/超额/并发核销。
- 匹配：完全匹配、数量、价格、税率、税额舍入、供应商、项目归属和重复来源差异。
- 权限：管理员、财务维护、财务只读、销售查看、采购查看、项目查看、敏感受限、成本受限、来源受限和无权限账号。

## 阻断与严重规则

### 阻断

- V30 迁移失败、修改历史迁移或历史 V10/V11 台账被改写/不可查询。
- 重复或超额开票、错误三单匹配仍可确认、应收应付重复生成。
- 收付款、预收预付或核销出现负余额、重复现金、跨主体/项目、并发穿透或部分写入。
- 税额、价税合计、资金余额或 V11 冲减口径错误。
- 外协/费用回写库存价值或被表达为正式项目成本。
- 凭证草稿产生正式凭证号、科目余额、总账或记账事实。
- 权限绕过、完整敏感税务/银行字段或未授权成本金额泄露。
- 核心接口/页面不可用，或新增/受影响页面未完成真实页面规范矩阵。

### 严重

- 来源、项目/公共/外协归属或应收应付链接错误。
- 状态、`allowedActions` 和页面动作不一致；确认/过账/核销不可幂等。
- 前端对业务金额做浮点可信计算，或重要来源/税价/余额只显示内部码。
- 审计缺关键来源、金额、目标或幂等信息。
- 页面问题影响关键数据识别、动作可达、权限判断或工作台扫描效率。

### 一般

- 不影响数据可信度、权限、来源、主流程和页面关键判断的文案、排序或轻微布局问题。
