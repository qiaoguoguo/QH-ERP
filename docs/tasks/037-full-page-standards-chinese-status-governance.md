## 中文状态与技术编码契约

### 总原则

1. 内部值保持原样，中文化只作用于用户可见展示层。
2. 映射键必须至少包含业务域、字段语义和原始编码；禁止建立不分业务语境的全局 `CLOSED -> 已关闭` 单表。
3. 优先复用或收敛各模块现有 `*PageHelpers`、`*StatusTag` 和显示字段；共享层只承载真正跨域且语义相同的映射与未知值策略。
4. 任何 `statusName || status`、`labels[value] ?? value`、直接 `prop="status"` 等用户可见原值兜底都必须被清点并分类，不得靠肉眼抽样放行。
5. 未识别状态的主界面文案显示“未知状态”；需要诊断时可在 tooltip、详情辅助信息或日志中显示“原编码：CODE”，不得把裸露英文编码作为唯一用户说明。
6. 错误码可以保留，但必须同时给出可理解的中文错误说明。
7. 后端已有 `statusName`、`typeName`、`reasonMessage` 等显示字段只有在确认稳定返回中文且语义正确时才可使用；仍须提供确定性中文兜底。

### 业务语境示例

| 原编码 | 语境 | 推荐中文 | 禁止误用 |
|---|---|---|---|
| `CLOSED` | 业务月结/请购/订单与期间 | 已月结、已结案或已关闭，按字段冻结 | 不得自动用于所有订单、项目或期间 |
| `POSTED` | 总账凭证/其他业务单据 | 总账为已记账，其他业务单据为已过账 | 不得跨域统一或翻译成泛化“已完成” |
| `READY` | 业务月结/财务关账/凭证草稿/任务 | 可月结、可结账、待正式制证或待确认提交，按字段冻结 | 不得跨域统一成“就绪” |
| `OPEN` | 差异/会计期间/项目履约 | 待处理、开放或履约中，按字段冻结 | 不得只按字面翻译 |
| `LOCKED`/`FROZEN` | 结账、快照、配置 | 已锁定/已冻结 | 不得混同禁用状态 |
| `CURRENT`/`STALE` | BOM、物料需求、项目成本、关账检查、报表时效 | 当前有效、来源已变化、已失效或已过期，按字段冻结 | 不得改变来源计算口径 |
| `RELEASED` | 生产工单 | 已下达 | 不得显示裸英文 |
| `IN_PROGRESS` | 生产/任务执行 | 进行中 | 不得显示裸英文 |
| `COMPLETED` | 已实际完成的对象 | 已完成 | 未完成的结束态应按业务字段区分 |

最终中文必须由产品经理核对业务语义、UI 设计师核对可读性和状态色、前端开发落地、测试在真实页面核验。

### 页面规范例外

- `/gl/trial-balance` 是单次试算结果页，表格只列示本次试算差异；分页会割裂一次试算的完整差异集合。保留无分页，使用表格内部滚动并在 1280×720 验证完整可达。

- `/platform/delivery-assets` 是后端一次返回的只读版本化交付目录，由多个有明确计数的小型目录表组成，不是可查询主列表。保留无分页，各目录独立使用表格内部滚动。

### 操作列统一展示契约

1. 当前代码中的 109 个 `label="操作"` 表格列全部纳入，不区分主列表、详情内表格、表单行编辑器、抽屉、面板或共享组件；避免同一列型在不同表面复制出第二套规则。
2. 列属性统一为 `fixed="right" width="184"`，不得使用 `min-width` 替代固定宽度。本阶段曾取消右固定的操作列全部恢复，既有 14 个右固定列也统一宽度。
3. 单元格操作容器保持单行、不换行，按钮间距 8px。正常最多直显两个操作：优先“详情/查看”，其次为当前最高频主流程动作。
4. 第三个及以后操作收入“更多”下拉。危险或终止类操作（删除、取消、关闭、作废、停用等）默认收入“更多”；若两枚直显按钮在 184px 内无法完整显示，改为一个直显按钮加“更多”。
5. “更多”只在当前行至少存在一个可见溢出操作时显示；无可用操作时沿用现有空态表达。每个动作原有权限、`v-if`、禁用条件、禁用原因、事件、加载态、确认流程和危险/成功/警告语义必须原样保留。
6. 1280×720 复验必须覆盖无权限、部分权限、超过两个可见操作、长文案、危险动作、横向滚动最左与最右位置；完成条件为操作列始终在视口右侧，关键字段不被永久遮挡，按钮和下拉项不重叠且完整可点击。

## 阻断与问题处理

### 阻断问题

- 无法生成三类完整性清单，或清单无法解释路由/页面数量差异。
- 核心页面或路由无法访问，导致全量核对不能继续。
- 任一非白名单的用户可见英文状态、阶段、类型、动作或原因仍裸露。
- 页面级横向溢出、关键金额/状态/操作列不可访问、弹窗/抽屉 footer 不可访问。
- 页面治理改变 API 编码、状态机、权限、业务金额、库存价值、会计口径或报表口径。
- 全量后端/前端测试、类型检查、生产构建或主分支验收服务失败。

### 严重问题

- 英文原值只在少数异常、权限、空态或回退路径暴露。
- 固定列、列宽、tooltip 或状态标签影响业务判断但仍存在替代路径。
- 查询、重置、分页、返回上下文、候选联动或不可编辑原因不符合规范。
- 中文翻译语义错误、颜色误导或同一业务状态出现多个不一致译法。

### 一般问题

- 不影响业务识别和操作可达性的轻微间距、文案或局部视觉意见。
- 一般问题先登记，集中审查后统一决定是否纳入本阶段；不得用一般问题反复开启审核循环。
