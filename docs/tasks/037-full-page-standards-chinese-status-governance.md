## 中文状态与技术编码契约

### 总原则

1. 内部值保持原样，中文化只作用于用户可见展示层。
2. 映射键必须至少包含业务域、字段语义和原始编码；禁止建立不分业务语境的全局 `CLOSED -> 已关闭` 单表。
3. 优先复用或收敛各模块现有 `*PageHelpers`、`*StatusTag` 和显示字段；共享层只承载真正跨域且语义相同的映射与未知值策略。
4. 任何 `statusName || status`、`labels[value] ?? value`、直接 `prop="status"` 等用户可见原值兜底都必须被清点并分类，不得靠肉眼抽样放行。
5. 未识别状态的主界面文案显示“未知状态”；需要诊断时可在 tooltip、详情辅助信息或日志中显示“原编码：CODE”，不得把裸露英文编码作为唯一用户说明。
6. 错误码可以保留，但必须同时给出可理解的中文错误说明。
7. 后端已有 `statusName`、`typeName`、`reasonMessage` 等显示字段只有在确认稳定返回中文且语义正确时才可使用；仍须提供确定性中文兜底。

### 业务语境示例

| 原编码 | 语境 | 推荐中文 | 禁止误用 |
|---|---|---|---|
| `CLOSED` | 业务期间/财务结账 | 已关闭 | 不得自动用于订单、项目或工单 |
| `POSTED` | 凭证/总账 | 已过账 | 不得翻译成泛化“已完成” |
| `READY` | 任务/候选/交付资产 | 已就绪、待执行或可交付，按字段冻结 | 不得跨域统一成一个词 |
| `OPEN` | 差异/会计期间/业务对象 | 待处理、开放中或未关闭，按字段冻结 | 不得只按字面翻译 |
| `LOCKED`/`FROZEN` | 结账、快照、配置 | 已锁定/已冻结 | 不得混同禁用状态 |
| `CURRENT`/`STALE` | 数据新鲜度 | 当前有效/已过期 | 不得改变来源计算口径 |
| `RELEASED` | 生产工单 | 已下达 | 不得显示裸英文 |
| `IN_PROGRESS` | 生产/任务执行 | 进行中 | 不得显示裸英文 |
| `COMPLETED` | 已实际完成的对象 | 已完成 | 未完成的结束态应按业务字段区分 |

最终中文必须由产品经理核对业务语义、UI 设计师核对可读性和状态色、前端开发落地、测试在真实页面核验。

## 阻断与问题处理

### 阻断问题

- 无法生成三类完整性清单，或清单无法解释路由/页面数量差异。
- 核心页面或路由无法访问，导致全量核对不能继续。
- 任一非白名单的用户可见英文状态、阶段、类型、动作或原因仍裸露。
- 页面级横向溢出、关键金额/状态/操作列不可访问、弹窗/抽屉 footer 不可访问。
- 页面治理改变 API 编码、状态机、权限、业务金额、库存价值、会计口径或报表口径。
- 全量后端/前端测试、类型检查、生产构建或主分支验收服务失败。

### 严重问题

- 英文原值只在少数异常、权限、空态或回退路径暴露。
- 固定列、列宽、tooltip 或状态标签影响业务判断但仍存在替代路径。
- 查询、重置、分页、返回上下文、候选联动或不可编辑原因不符合规范。
- 中文翻译语义错误、颜色误导或同一业务状态出现多个不一致译法。

### 一般问题

- 不影响业务识别和操作可达性的轻微间距、文案或局部视觉意见。
- 一般问题先登记，集中审查后统一决定是否纳入本阶段；不得用一般问题反复开启审核循环。
