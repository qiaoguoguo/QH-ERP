## 链接与按钮语义边界

### 必须使用按钮或链接按钮

- 页面标题动作区中的新增、返回、历史导入、导入、导出、下载、上传、打印、生成、提交、处理等动作。
- 所有 `label="操作"` 列中的详情、查看、查看业务、过账、编辑、取消、关闭等行级动作。
- 详情动作区中以“查看……”表述、并明确承担操作入口的关联资源跳转；可以保留原位置和路由，但必须使用轻量按钮承载。
- 固定打印等复合工具动作；操作列只显示一个紧凑触发入口，其内部子动作在受视口约束且 teleport 到 body 的弹出层、对话框或抽屉中承载。

### 可以保留文本链接

- 菜单、页签、固定报表入口。
- 单号、项目号、来源编号、追溯对象、业务对象名称等数据本体下钻。
- 生产工单来源追溯、报表数据钻取、审批对象字段中的数据本体导航。
- 权限不足时不可跳转的普通文本和受限原因。

### 冲突决策

- 审批中心“查看业务单据”不是单号本体，而是动作式文案；本阶段纳入轻量按钮化。集中审查发现放在对象列会被长业务对象文本挤压，因此移入统一的右固定操作列，白名单路由、权限和返回上下文保持不变。
- 历史导入详情“查看文档任务”是详情关联动作；本阶段纳入轻量按钮化，但保留原路由与关联关系，不改为业务写动作。
- 上述两项解决 UI 审计与产品初步分类差异，不扩大为对所有正文链接的盲目替换。

## 页面规范冻结决策

### 列表操作列

- 所有受影响操作列继续 `fixed="right" width="184"`。
- 单元格内为单行、垂直居中、稳定间距、禁止换行；不得放标题、说明、表单、警告条或完整面板。
- 最多两个行级动作直显；第三个及以后进入“更多”。
- 详情/查看类优先，其次当前状态主流程动作；下载、固定打印等辅助动作可进入“更多”；取消、关闭、删除、终止等危险或终结动作靠后并保留危险语义。
- “更多”菜单必须 teleport 到 body，不能被表格、抽屉或固定列裁剪。

### 详情标题动作

- 生产工单详情标题区最多两个动作直显：“返回列表”和当前状态最高频的主流程推进动作。
- 其余原动作进入“更多”，排序与显示条件固定；危险/终结动作置于菜单末尾并保留原确认、禁用和语义。
- 不能为了满足数量限制删除、合并或隐藏原动作。

### 固定打印

- 页面级固定打印可保留为清晰的详情工具分区，必须与相邻信息分区保持稳定间距。
- 行级固定打印只显示“固定打印”紧凑按钮；点击后在 teleport 到 body 的受视口约束承载中展示“预览”和“生成 PDF”。
- 固定打印按一个复合动作参与直显数量计算；内部预览、生成 PDF 仍按原顺序、权限、允许状态、禁用原因和错误提示执行。

### 页面工具动作

- “新增修复申请”和五类“历史导入”进入标题按钮组；正向新增保持主要层级，“历史导入”使用次要按钮，不抢占主创建动作。
- 路由入口可使用 `RouterLink custom` 包装 `el-button` 或按钮触发 `router.push`；不得丢失 `returnTo`、查询参数或浏览器导航语义。
- 详情关联动作使用 `el-button size="small" text` 或等价统一轻量按钮，不与普通正文混淆。

## 接口、权限与数据不变契约

- 后端实现工作包为 0，现有接口和字段足以支持全部整改。
- 生产执行过账、固定打印预览/PDF 任务、数据修复、历史导入、审批和消息接口完全不变。
- `allowedActions`、`availableActions`、权限判断、危险确认、`version`、`idempotencyKey`、`returnTo` 和各适配器 `adapterCode` 原样保留。
- 共享组件只能承载展示和事件透传，不得在组件内重新解释权限、状态或业务规则。

## 阻断规则

- 用户点名的生产工单详情、数据修复或供应商历史导入未闭合。
- 任一 B 项缺少修复后截图、固定 UI 结论或固定测试真实页面复核。
- 页面无法访问、关键接口失败、控制台新增错误、页面级横向溢出、操作列移出视口、按钮重叠或不可点击。
- 操作列取消 `fixed="right"`、宽度不是 `184`、超过两个动作仍直显，或完整固定打印面板仍嵌在表格单元格内。
- 动作、权限、按钮条件、禁用原因、确认、事件、接口、状态机、适配器参数、返回上下文或持久数据被改变。
- 视觉验收误触发发布、领料、报工、完工入库、完成、取消、过账、标记已读、导入确认等业务写动作。
