# 任务文档模板

每个业务阶段优先只建立一份权威任务说明，作为产品、UI、前端、后端、测试、迁移、审查和验收的共同边界。不得为同一能力机械重复创建规格、红测、实现、转绿、审查和部署文档；小型治理任务可直接在既有权威文档或交接中记录。

执行流程、后台固定角色任务派发信息和交叉审查矩阵见 `docs/ops/stage-collaboration-delivery-process.md`。

## 任务名称

填写任务名称。

## 任务目标

说明本任务要达成的明确成果，不写泛化目标。

## 背景与输入

- 关联业务背景：
- 权威工作区、分支与基准提交：
- 上游阶段与关联文档：
- 已确认约束：

## 范围

- 填写本次包含的工作内容。

### 本次不包含
