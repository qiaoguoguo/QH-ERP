# 任务文档模板

每个开发、设计、测试或部署任务都必须先建立任务文档。任务文档是执行边界和验收依据。

## 任务名称

填写任务名称。

## 任务目标

说明本任务要达成的明确成果，不写泛化目标。

## 背景与输入

- 关联业务背景：
- 关联文档：
- 已确认约束：

## 范围

### 本次包含

- 填写本次包含的工作内容。

### 本次不包含

- 填写本次明确不包含的工作内容。

## 角色分工

- 产品经理：
- UI 设计师：
- 前端开发：
- 后端开发：
- 测试：
- 主代理：

## 预期成果

- 文档成果：
- 代码成果：
- 验收成果：

## 验收标准

- 功能验收：
- 数据验收：
- 权限验收：
- UI 验收：
- 测试验收：
- 部署验收：
- 浏览器视觉分析：

## 风险与处理

- 填写已识别风险和处理方式。

## 变更控制

- 变更原因：
- 影响范围：
- 涉及文件：
- 验收影响：
- 决策依据：

## 缺陷与阻断

- 阻断缺陷：
- 严重缺陷：
- 已知问题：
- 是否允许进入验收：

## 执行记录

- 计划：
- 变更：
- 验证：
- 视觉分析：
- 结论：
