# 验收标准

## 总则

阶段成果必须能在浏览器中实际访问和操作。仅代码完成、接口完成或文字说明不能作为验收完成依据。

## 功能验收

- 满足任务文档中列出的功能范围。
- 正常流程和异常流程都有明确结果。
- 状态流转符合业务规则。

## 数据验收

- 数据新增、修改、删除或状态变更可追溯。
- 库存、数量、金额和成本数据口径一致。
- 关键业务数据不能依赖前端假数据。
- 成本归集一期只验收业务记录与来源追溯，不把业务金额或数量口径验收为正式财务核算结果。
- 成本记录必须能追溯到工单、产品、来源单据、记录人和记录时间。
- 采购入库过账必须增加库存余额、生成采购来源库存流水，并能从采购订单、采购入库和库存流水之间追溯。
- 采购单价只作为采购业务记录字段，不验收为应付、付款、税务、凭证或正式成本核算结果。
- 销售出库过账必须扣减库存余额、生成销售来源库存流水，并能从销售订单、销售出库和库存流水之间追溯。
- 销售来源库存流水必须回跳对应销售出库，不能误跳库存单据、采购入库或其他来源单据；缺少销售出库查看权限时应隐藏对应来源跳转。
- 销售单价只作为销售业务记录字段，不验收为应收、发票、收款、税务、凭证、收入确认或正式财务核算结果。
- 销售数量、出库数量和销售单价必须按 decimal 边界验收，避免浮点精度污染；零值、负数、超精度和非法格式按任务规格返回受控结果。
- 财务往来基础只验收业务台账、收付款登记、余额核销和来源追溯，不验收为总账、凭证、税务、完整发票、正式收入确认、正式成本结转或 BI 报表。
- 应收台账必须能追溯到已过账销售出库和销售订单；同一销售出库明细不得重复生成应收来源。
- 应付台账必须能追溯到已过账采购入库和采购订单；同一采购入库明细不得重复生成应付来源。
- 应收金额必须等于已收金额加未收金额；应付金额必须等于已付金额加未付金额。
- 收款金额不得超过未收金额，付款金额不得超过未付金额；失败操作不得产生部分核销、错误余额或错误状态。
- 已过账收款和付款不可编辑、不可取消；已收清应收和已付清应付不可继续登记收付款。
- 经营报表基础只验收固定口径经营汇总、来源追溯、权限控制、空态异常态和视觉分析，不验收为可配置 BI、总账、凭证、正式财务报表、结账或税务结果。
- 经营报表汇总值必须能由来源明细解释；销售、采购、库存、生产、成本和往来报表必须明确业务口径。
- 经营报表接口不得写入或修改业务单据，不得绕过来源模块权限泄露来源详情。
- 退货退款与业务反冲基础只验收业务反向单据、库存反向流水、往来冲减、成本业务影响、经营报表净额和来源追溯，不验收为审批流、红字发票、税务、总账凭证、结账或银行退款支付。
- 已过账采购、销售、生产、库存、往来和成本原单不得被退货、退款或反冲直接改写或删除。
- 销售退货、采购退货、生产退料、生产补料和往来冲减必须能追溯原单、原行、反向单据、库存流水、往来记录、成本影响和经营报表来源。
- 退货数量不得超过可退数量，冲减金额不得超过可冲金额，反向出库不得造成负库存。
- 经营报表必须能区分原发生、反向发生和净额，不能只展示无法解释的净额。
- 业务期间与历史数据保护基础只验收按业务日期的锁定、解锁、审计和历史写入保护，不验收正式财务关账、会计期间、总账、凭证、税务、期间快照或成本结转。
- 锁定期间必须拒绝库存、采购、销售、生产、成本、往来和反向业务中影响经营口径的写入；查询、来源追溯和经营报表查看不得受阻。
- 开放期间可创建引用锁定期间历史原单的反向业务，反向影响必须按自身业务日期进入开放期间，不得回写锁定期间。
- 质量检验与库存质量状态基础只验收库存质量状态、轻量质量确认、库存可用量口径、非合格不可用、采购退货质量状态、权限、期间保护和来源追溯，不验收完整 QMS、SPC、批次、序列号、库位、MRP、审批流、正式财务或复杂返工返修。
- 历史库存必须默认迁移为合格库存；迁移后同一仓库、物料下各质量状态数量合计必须等于迁移前现存数量。
- 库存余额必须按仓库、物料和质量状态分层；库存流水必须记录质量状态；余额、流水、质量确认记录和来源单据之间的数量与状态必须一致。
- 017 阶段 `availableQuantity` 必须只代表合格可用库存，不得包含待检、不合格或冻结库存；018 起在库存余额和业务候选中进一步表示合格现货净可用，即扣除已占用和已预留库存，采购在途单列参考。
- 采购入库、生产完工入库、销售退货和生产退料默认形成待检库存，并生成可追溯的质量确认记录。
- 质量确认可以把待检库存转为合格、不合格或冻结，转换前后总库存不变，数量不平、超过待检余额、重复提交或状态非法必须被拒绝。
- 销售出库、生产领料和生产补料不能消耗非合格库存；总库存充足但合格库存不足时必须返回可识别的质量状态不可用或指定质量状态库存不足错误。
- 采购退货必须按显式质量状态退货，允许扣减待检、不合格或合格库存；冻结库存必须先解冻后才可退货。
- 质量确认、冻结、解冻、入库质量状态落账和按质量状态扣减必须受业务期间保护；锁定期间历史质量记录、库存余额、库存流水和来源追溯仍可查询。
- 质量状态相关失败事务不得产生部分写入，不能出现单据状态、库存余额、库存流水、质量记录或审计之间不一致。
- 质量状态库存必须能追溯来源单据、来源行、质量确认记录、状态转换流水和后续消耗记录。
- 库存可用量、占用、冻结与预留基础只验收库存可用量口径、轻量业务预留占用、物料级采购在途参考、仓库级现货可承诺量和计划净需求缺口前置口径，不验收完整 MRP、复杂 ATP、排产、批次、库位、库存月结或正式财务。
- 库存余额必须区分账面库存、合格现存、占用库存、预留库存、冻结库存、现货净可用、物料级采购在途参考、仓库级现货可承诺量和净需求缺口。
- `availableQuantity` 在 018 后必须代表现货净可用，不得包含待检、不合格、冻结、已占用或已预留库存。
- 销售订单行确认前必须填写预留仓库；确认必须按预留仓库生成 `SALES_ORDER + RESERVATION`；取消、关闭、数量减少和同仓库出库过账必须释放或消费对应预留；销售出库仓库必须与来源订单行预留仓库一致。
- 生产工单发布必须按 BOM 快照和领料仓库生成 `PRODUCTION_WORK_ORDER + RESERVATION`；不得把工单发布直接记为占用；取消、完成、用料减少和领料过账必须释放或消费对应预留。
- 采购订单确认或部分入库必须形成物料级采购在途参考；采购在途不按仓库计算，不进入仓库级账面库存或仓库级现货可承诺量；采购入库过账必须减少在途并形成具体仓库待检库存，质量确认后才进入合格现存。
- 冻结已占用或已预留的合格库存必须被拒绝；冻结库存不能进入可承诺量、占用、预留或普通消耗。
- 账面库存充足但现货净可用不足时，销售出库、生产领料、生产补料、采购退货或库存调减必须返回可识别的可用量不足或占用预留不可用错误。
- 销售确认门禁是选定预留仓库的现货净可用是否足够；现货不足但物料级采购在途足够时订单保持草稿并提示现货不足、在途仅供参考；本阶段不允许预留在途。
- 占用、预留、释放、消费和在途变化必须有来源追溯、审计记录和业务期间保护。
- `netRequirementShortageQuantity` 只输出非负缺口 `max(需求数量 - 可覆盖供给数量, 0)`，不生成 MRP 建议、计划订单或排产结果。

## 权限验收

- 菜单、按钮、字段和数据范围符合角色权限规则。
- 无权限操作应给出明确反馈。
- 业务期间的查看、创建、维护、锁定和解锁使用独立权限；无权限用户不得通过页面、路由或接口锁定、解锁或绕过写入保护。
- 质量确认查看、质量确认处理、冻结和解冻必须使用独立权限；未登录、无权限和只读用户不得通过菜单、路由、按钮或接口绕过质量状态控制。
- 前端隐藏或禁用质量操作不能替代后端鉴权；越权调用质量确认、冻结、解冻或非合格消耗接口必须被后端拒绝。
- 库存可用量查看和占用预留查看必须使用明确权限；只读用户可查看库存口径和追溯，不得执行影响占用、预留、释放、消费或在途口径的写动作。
- 销售、生产、采购模块触发的预留、占用、释放和在途变化必须由对应模块写权限和后端接口鉴权共同约束。

## UI 验收

- 页面结构、表单、列表、弹窗和提示符合设计基线。
- 异常、加载、空状态和成功反馈完整。
- 浏览器验收阶段必须保存关键页面截图并输出视觉分析结论。
- 每个阶段必须把截图证据和视觉分析结论保存到对应的 `docs/testing/<模块标识>-visual-audit/` 目录，并在验收通知中明确记录位置。
- 视觉分析必须覆盖布局稳定性、信息密度、关键操作可见性、文案溢出、控件重叠、响应式适配和业务状态识别。
- 视觉分析必须以可信浏览器截图为依据；截图失真时必须改用可信视口截图，并记录失真原因。
- 如视觉问题影响核心操作、关键数据识别、权限状态判断或明显降低工作台扫描效率，必须修复并重新截图验证后才允许进入阶段验收。
- 业务期间阶段视觉分析覆盖期间列表、锁定确认、写入失败、只读权限、无权限和 390px 窄屏。
- 质量状态阶段视觉分析覆盖质量确认列表、质量确认处理抽屉、库存余额质量状态分布、销售出库候选、生产领料候选、采购退货候选、期间保护拦截、权限差异、空态、异常态和 390px 窄屏。
- 质量状态页面必须同时展示总现存、合格可用、待检、不合格和冻结数量；质量状态不得只依靠颜色表达，必须有中文文案。
- 质量确认处理抽屉、候选库存区域和库存余额宽表在桌面和 390px 窄屏下不得遮挡关键字段、错误提示、底部操作或质量状态禁用原因。
- 018 阶段视觉分析覆盖库存余额可用量总览、占用预留追溯、物料级采购在途参考、销售订单行预留仓库、销售出库候选、生产领料候选、冻结拦截、期间保护、权限差异、空态、异常态和 390px 窄屏。
- 库存余额页面必须清晰展示账面库存、合格现存、占用库存、预留库存、冻结库存、现货净可用、物料级采购在途参考、仓库级现货可承诺量和净需求缺口；采购在途不得被表现为现货可用或销售确认门禁。
- 销售和生产候选库存区域必须展示本次最多出库或本次最多领料、禁用原因和后端返回的可用量口径，不能只显示笼统“库存不足”。

## 阻断条件

- 无法本地启动或部署。
- 核心页面无法浏览器访问。
- 关键业务数据错误。
- 权限绕过。
- 浏览器视觉分析缺失，或视觉问题影响核心操作、关键数据识别、权限状态判断。
- 视觉分析没有可追溯截图证据，或截图存在明显失真却被用于验收结论。
- 库存、工单、成本相关流程不可追溯。
- 采购订单、采购入库和采购来源库存流水不可追溯。
- 销售订单、销售出库和销售来源库存流水不可追溯。
- 销售出库库存扣减错误、产生负库存，或重复过账污染库存和流水。
- 页面或接口把销售单价作为应收、发票、收款、税务、收入确认、凭证或正式财务结果交付。
- 应收、收款、应付、付款任一财务往来主路径无法完成。
- 应收或应付来源追溯断链、重复来源未阻断、金额余额错误或状态和金额矛盾。
- 超额收付款被接受，或收付款失败后产生脏金额、半提交核销或错误状态。
- 页面或接口把财务往来作为总账、凭证、税务、完整发票、正式收入确认、正式成本结转或 BI 报表交付。
- 经营报表核心页面无法访问、关键报表接口不可用，或汇总值与来源明细对不上。
- 经营报表来源追溯不可用、追溯到错误单据，或无权限用户可查看报表或来源详情。
- 页面或接口把经营报表作为可配置 BI、总账、凭证、结账、正式财务报表或税务结果交付。
- 退货退款与业务反冲核心页面无法访问、关键接口不可用，或数据库 V11 迁移失败。
- 反向业务直接改写或删除已过账原单。
- 系统接受超退、超冲、重复反冲，或反向出库造成负库存。
- 库存余额、往来余额、成本净额或经营报表净额与反向业务来源不一致。
- 无权限用户可访问退货退款反冲接口，或来源受限响应泄露单号、主键、数量、金额、业务日期、状态或路由参数。
- 数据库迁移失败或初始化数据不可用。
- 账号权限模块中未登录、无权限或停用账号仍可访问受保护资源。
- 阻断缺陷未修复或未回归。
- 锁定期间仍允许影响经营口径写入、跨期反向业务回写历史期间、期间锁定或解锁权限绕过、锁定期间查询受阻，均为阻断缺陷。
- 质量状态数据库迁移失败、历史库存未默认迁移为合格库存、迁移后库存总量不一致，均为阻断缺陷。
- 库存余额、库存流水、质量确认记录、来源单据之间数量或质量状态不一致，均为阻断缺陷。
- `availableQuantity` 包含待检、不合格或冻结库存，或页面把总库存误展示为可销售、可领用、可补料数量，均为阻断缺陷。
- 采购入库、生产完工入库、销售退货或生产退料未默认形成待检库存，或未生成可追溯质量确认记录，均为阻断缺陷。
- 质量确认转换前后总库存变化，数量不平、超过待检余额、重复提交或状态非法仍被接受，均为阻断缺陷。
- 销售出库、生产领料或生产补料成功消耗待检、不合格或冻结库存，均为阻断缺陷。
- 采购退货未按显式质量状态扣减，或直接扣减冻结库存，均为阻断缺陷。
- 无权限、只读或未登录用户能处理质量确认、冻结、解冻，或通过接口绕过质量状态校验，均为阻断缺陷。
- 锁定期间仍允许质量确认、冻结、解冻、入库质量状态落账或按质量状态扣减，均为阻断缺陷；锁定期间历史质量记录、库存余额、库存流水或来源追溯无法查询，也按阻断缺陷处理。
- 质量状态失败事务产生单据状态、库存余额、库存流水、质量记录或审计的部分写入，均为阻断缺陷。
- 质量状态来源追溯无法关联来源单据、来源行、质量确认记录、状态转换流水或后续消耗，均为阻断缺陷。
- 018 库存可用量公式错误、数量重复计算、`availableQuantity` 包含非合格或已占用预留库存，均为阻断缺陷。
- 销售订单、生产工单、销售出库、生产领料、采购入库、采购退货或生产补料绕过占用预留校验导致超卖、超领、负库存或残留占用，均为阻断缺陷。
- 销售订单行缺少预留仓库仍可确认、销售出库仓库与来源订单行预留仓库不一致仍可过账、或系统自动跨仓释放改仓分配，均为阻断缺陷。
- 生产工单发布直接生成占用而不是预留，均为阻断缺陷。
- 采购在途被计入账面库存、立即可用库存或仓库级现货可承诺量，均为阻断缺陷。
- 现货不足但以物料级采购在途为依据确认销售订单或预留在途，均为阻断缺陷。
- 已占用或已预留库存仍可被冻结、承诺、出库、领料或补料，均为阻断缺陷。
- 占用预留来源重复、释放后仍残留、过账后未消费或失败事务产生部分写入，均为阻断缺陷。
- 锁定期间仍允许影响占用、预留、释放、消费或采购在途的写入，或历史占用预留和在途追溯无法查询，均为阻断缺陷。
- 库存余额、销售候选、生产候选或采购在途页面缺失 018 视觉分析证据，或视觉问题影响关键数量口径识别，均为阻断缺陷。
