# 验收标准

## 总则

阶段成果必须能在浏览器中实际访问和操作。仅代码完成、接口完成或文字说明不能作为验收完成依据。

## 功能验收

- 满足任务文档中列出的功能范围。
- 正常流程和异常流程都有明确结果。
- 状态流转符合业务规则。

## 数据验收

- 数据新增、修改、删除或状态变更可追溯。
- 库存、数量、金额和成本数据口径一致。
- 关键业务数据不能依赖前端假数据。

## 权限验收

- 菜单、按钮、字段和数据范围符合角色权限规则。
- 无权限操作应给出明确反馈。

## UI 验收

- 页面结构、表单、列表、弹窗和提示符合设计基线。
- 异常、加载、空状态和成功反馈完整。

## 阻断条件

- 无法本地启动或部署。
- 核心页面无法浏览器访问。
- 关键业务数据错误。
- 权限绕过。
- 库存、工单、成本相关流程不可追溯。
