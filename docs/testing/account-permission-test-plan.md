# 账号与权限测试计划

## 目标

验证账号与权限基础模块满足功能、接口、权限、异常、回归和部署验收要求，确保后续业务模块可以安全复用权限底座。

## 准入标准

- 任务文档、需求说明、接口契约、权限编码和测试账号已明确。
- 本地 PostgreSQL 可通过 Docker Compose 启动。
- 前端和后端基础验证命令可执行。
- 初始化管理员规则已明确。

## 准出标准

- 前端 `npm test`、`npm run typecheck`、`npm run build` 通过。
- 后端 Docker 化 JDK 21 + Maven + Testcontainers 测试通过。
- 本地部署成功，浏览器可完成核心验收路径。
- 登录、退出、当前用户、用户管理、角色管理、权限配置、后端鉴权、停用账号、权限变更全部通过。
- 无阻断或严重缺陷遗留。

## 测试账号和角色

| 类型 | 用途 |
|---|---|
| 初始超级管理员 | 初始化后首次登录和全权限验收 |
| 系统管理员 | 日常账号权限管理验收 |
| 只读用户 | 验证菜单可见但无操作按钮 |
| 无关键权限用户 | 验证无权限页面和接口拒绝 |
| 停用用户 | 验证停用账号不可登录和不可访问 |

## 功能测试范围

- 登录成功。
- 登录失败。
- 退出登录。
- 当前用户信息展示。
- 用户新增、编辑、启用、停用、重置密码。
- 角色新增、编辑、启用、停用。
- 用户分配角色。
- 角色配置权限。
- 菜单和按钮按权限展示。
- 权限变更后刷新当前用户或重新登录展示变化。

## 接口测试范围

- 未登录访问受保护接口。
- 无权限访问受保护接口。
- 停用账号访问受保护接口。
- 用户列表分页、查询和空结果。
- 角色列表分页、查询和空结果。
- 用户名唯一约束。
- 角色编码唯一约束。
- 权限树结构。
- 角色权限保存。
- 审计日志写入。

## 权限测试范围

- 前端无权限菜单不可见。
- 前端无权限按钮不可见或禁用。
- 直接访问无权限路由会被拦截。
- 直接调用无权限接口会被后端拒绝。
- 角色停用后用户不再获得该角色权限。
- 用户停用后不能登录，已登录请求也被拒绝。
- 超级管理员仍走统一鉴权链路。

## 浏览器验收路径

1. 启动 PostgreSQL、后端和前端。
2. 使用初始超级管理员登录。
3. 创建只读角色并只授予用户查看权限。
4. 创建管理角色并授予用户管理、角色管理和权限配置权限。
5. 创建两个用户并分别分配只读角色和管理角色。
6. 用只读用户登录，验证菜单、按钮和越权接口反馈。
7. 用管理用户登录，验证用户和角色操作。
8. 停用某用户后验证登录失败和受保护接口拒绝。
9. 修改角色权限后刷新当前用户或重新登录，验证权限表现变化。
10. 查看账号权限相关审计记录。

## 回归范围

- 登录、退出和当前用户。
- 用户状态和角色分配。
- 角色状态和权限分配。
- 菜单渲染和路由守卫。
- 按钮权限。
- 后端接口鉴权。
- 初始化数据。
- 数据库迁移。

## 阻断缺陷

- 初始化超级管理员不可用。
- 未登录可访问受保护资源。
- 无权限或越权接口请求成功。
- 停用账号仍可登录或访问。
- 权限只前端隐藏，后端未拦截。
- 本地无法部署或浏览器无法完成核心路径。
- 用户、角色、权限关键数据错误且不可追溯。
