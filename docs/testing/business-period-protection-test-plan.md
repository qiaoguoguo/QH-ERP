# 业务期间与历史数据保护测试计划

## 测试目标

确认业务期间的范围、状态、权限和审计正确；确认锁定期间不能再被影响经营口径的写入污染，同时查询、追溯、报表和开放期间的跨期反向业务保持可用。

## 自动化验收矩阵

| 范围 | 覆盖重点 |
|---|---|
| 期间管理 | 创建、编辑、按月生成、日期重叠、状态转换、原因和审计 |
| 权限 | 查看、创建、维护、锁定、解锁；未登录、无权限和只读角色不可绕过接口 |
| 写入保护 | 库存期初、调整、过账；采购确认和入库；销售确认和出库；生产发布、领料、报工、完工入库；成本记录；往来确认和收付款；反向业务与冲减 |
| 跨期反向 | 开放期间可引用锁定期间历史原单；反向影响按自身业务日期进入开放期间 |
| 查询追溯 | 锁定期间列表、详情、来源追溯和经营报表仍可访问 |
| 前端 | API 路径、请求体、错误信封、列表筛选分页、状态标签、表单、确认弹窗和权限控制 |

## 浏览器与视觉验收矩阵

| 场景 | 账号或权限 | 结果 | 视觉证据 |
|---|---|---|---|
| 创建自然月期间并锁定 | 管理员 | 原因必填、状态更新、列表字段可扫描 | 桌面视口的期间列表、锁定确认弹窗 |
| 锁定期间写入失败 | 各模块可写角色 | 库存、采购、销售、生产、成本、往来和反向业务代表动作收到统一错误 | 写入失败提示 |
| 锁定期间查询 | 可查看角色 | 列表、详情、报表和来源追溯可访问 | 锁定状态展示 |
| 开放期间反向业务 | 反向业务可写角色 | 可引用锁定期间历史原单，影响进入开放期间 | 来源与日期展示 |
| 只读权限 | 期间查看角色 | 可查看，不显示或禁用锁定、解锁和编辑操作 | 只读权限态 |
| 无权限访问 | 无期间权限角色 | 菜单、路由和接口均被拒绝 | 无权限态 |
| 桌面视口与列表操作 | 管理员与只读角色 | 桌面视口下列表字段可扫描；横向滚动可用且不遮挡首末列；锁定确认弹窗完整可操作；操作列中的编辑、锁定、解锁入口可访问 | 不小于 1280px 宽度的视口截图：默认列表、横向滚动至操作列、锁定确认弹窗 |
| 窄屏适配 | 管理员与只读角色 | 390px 宽度下列表、弹窗、错误提示和操作不重叠；列表横向滚动后操作列仍可访问 | 390px 视口：默认列表、横向滚动至操作列、锁定确认弹窗 |

视觉分析记录目录为 `docs/testing/business-period-protection-visual-audit/`。记录必须列明截图文件、视口尺寸、页面路径、账号权限、发现问题、处理结果和最终结论，并检查布局、信息密度、关键操作可见性、文案溢出、控件重叠、响应式适配和业务状态识别。桌面与 390px 窄屏均须分别保存列表横向滚动至操作列的可信视口截图；锁定确认弹窗须在桌面和 390px 窄屏至少各保存一张完整可信视口截图。

## 阶段交付检查

- 定向自动化、类型检查、迁移验证、浏览器验收和视觉分析在阶段交付窗口完成。
- 遇到 `BUSINESS_PERIOD_LOCKED`、权限绕过、日期范围重叠、跨期影响回写或锁定期间查询受阻时，按阻断缺陷处理。
- 视觉问题影响关键操作、数据识别或权限判断时，修复并重新截图后才能进入阶段验收。
