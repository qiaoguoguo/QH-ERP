# 经营报表基础测试计划

## 目标

验证经营报表基础模块满足固定口径汇总、筛选、权限、来源追溯、空态异常态、数据对账、部署和浏览器视觉验收要求。确保报表结果来自真实业务单据，且不会被误交付为总账、凭证、正式财务报表或可配置 BI。

## 准入标准

- 经营报表任务文档、设计规格、接口契约、权限编码和验收路径已确定。
- 采购、销售、库存、生产、成本和财务往来模块在当前分支可用。
- 本地 PostgreSQL、后端和前端启动方式可用。
- 已确认 014 不做退货退款、反冲、关账、总账、凭证或可配置 BI。

## 准出标准

- 后端 Maven/Testcontainers 全量测试通过。
- 前端 `npm test`、`npm run typecheck`、`npm run build` 通过。
- 管理员浏览器主路径通过。
- 部分报表权限用户、来源受限用户、无权限用户和未登录用户权限验收通过。
- 销售、采购、库存、生产、成本和往来收付报表均完成汇总与来源追溯明细对账。
- 经营异常清单完成异常来源追溯或来源受限脱敏验收。
- 空态、异常态、无权限态浏览器验收通过。

## 最小验收闭环

准备一组贯通数据，覆盖采购入库、库存变动、生产领料、报工、完工入库、成本归集、销售出库、应收收款和应付付款。管理员进入经营报表中心，查看经营概览和固定报表；按当前月筛选，核对销售、采购、库存、生产、成本和往来汇总；分别打开销售、采购、库存、生产、成本、往来收付 6 类报表的来源追溯，并核对追溯明细合计能够解释页面汇总；打开经营异常清单并验证异常来源可跳转或按来源权限脱敏；切换无数据筛选条件确认空态；切换来源受限、部分报表权限、无报表权限和未登录用户确认菜单、路由、接口和来源跳转拦截。

## 测试角色和权限矩阵

| 角色 | 报表权限 | 来源权限 | 验证重点 |
|---|---|---|---|
| 管理员 | `report` 和全部 `report:*:view` | 全部来源模块查看权限 | 可查看全部报表、追溯来源、跳转详情和异常清单 |
| 报表只读角色 | `report`、`report:overview:view`、`report:sales:view`、`report:inventory:view` | 销售出库、销售订单、库存流水查看权限 | 只显示已授权报表入口，未授权采购、生产、成本、往来和异常路由与接口返回无权限 |
| 来源受限角色 | `report` 和全部 `report:*:view` | 不授予销售出库、采购入库、生产、成本、财务详情查看权限；保留报表查看权限 | 可看汇总和追溯行数，但追溯来源单号、主键、数量、金额、日期和路由参数必须脱敏 |
| 无报表权限用户 | 无 `report` 和无 `report:*:view` | 可有普通业务查看权限 | 报表菜单不可见，报表路由和接口返回无权限 |
| 未登录用户 | 无 | 无 | 报表页面跳转登录，报表接口返回未登录 |

## 最小数据集

| 范围 | 数据要求 | 排除与边界 |
|---|---|---|
| 基础资料 | 2 个客户、2 个供应商、1 个主仓、原料、半成品、成品、单位、启用 BOM | 停用资料不得进入筛选默认结果 |
| 销售 | 当前月 1 张已过账销售出库，关联已确认或部分出库销售订单；1 张草稿或取消销售订单；1 条预计发货日已逾期且未发完订单行 | 销售汇总只统计已过账销售出库；异常只统计仍有未发数量的订单 |
| 采购 | 当前月 1 张已过账采购入库，关联已确认或部分收货采购订单；1 张草稿或取消采购订单；1 条预计到货日已逾期且未收完订单行 | 采购汇总只统计已过账采购入库 |
| 库存 | 期初入库、采购入库、生产领料出库、完工入库、销售出库、调整入库或调整出库各至少 1 条库存流水；另准备 1 个零库存或无库存流水但存在销售未发或生产未领需求的仓库物料 | 期初、入库、出库、调整和期末数量必须可由库存流水重算；库存不足异常不得因为没有库存流水而漏报 |
| 生产 | 1 张已下达或执行中工单，包含领料、报工、完工入库；1 张计划完成日已逾期且未完工工单 | 草稿和取消工单不得进入生产执行汇总 |
| 成本 | 1 条有效材料成本记录、1 条有效人工或制造费用记录、1 个已过账生产业务但缺成本记录的工单 | 成本报表返回 `formalAccounting=false`，不产生凭证或科目语义 |
| 往来 | 1 条已确认应收、1 张已过账收款、1 条已确认应付、1 张已过账付款；1 条逾期未收应收；1 条 7 天内到期未付应付 | 草稿和取消往来单据不得进入余额口径 |
| 日期 | 当前月数据、上月数据、月初和月末边界日期 | 默认日期范围为当前月；跨期筛选必须稳定 |

## 自动化测试矩阵

### 后端接口测试

- 报表权限初始化包含 `report` 和全部 `report:*:view` 权限。
- 报表路径鉴权覆盖 `/api/admin/reports/**`。
- 未登录访问报表接口返回 `AUTH_UNAUTHORIZED`。
- 无权限访问报表接口返回 `AUTH_FORBIDDEN`。
- 日期范围非法或超过最大跨度返回 `REPORT_DATE_RANGE_INVALID`。
- 未知参数和无关通用参数被忽略，已支持参数非法时返回确定错误。
- 分页参数非法返回 `REPORT_PARAMETER_INVALID` 或 `VALIDATION_ERROR`。
- 固定报表列表均返回 `summary + items`，且 `summary` 统计完整筛选结果集而非当前页。
- 经营概览空数据返回零值和 `formalAccounting=false`。
- 销售经营汇总只统计已过账销售出库，排除草稿、取消或未过账数据。
- 采购经营汇总只统计已过账采购入库，排除草稿、取消或未过账数据。
- 库存收发存按库存流水推算期初、入库、出库、调整和期末数量。
- 生产执行汇总按工单、报工和完工入库统计计划数量、报工数量、完工数量和完工率。
- 成本归集汇总只统计有效成本业务记录，并返回 `formalAccounting=false`。
- 往来收付汇总排除草稿和已取消台账，单独列示已关闭未结余额。
- 经营异常清单按固定判定口径返回逾期、库存不足、成本缺失和往来异常。
- 库存不足异常使用 `INVENTORY_BALANCE` 表示仓库和物料库存状态，追溯键必须稳定定位仓库和物料。
- 销售、采购、库存、生产、成本、往来和异常追溯接口均可分页返回来源明细。
- 库存、生产、成本和经营异常清单属于期间敏感追溯，打开追溯时必须携带与当前报表或异常列表筛选一致的 `dateFrom` 和 `dateTo`，非当前期间追溯明细必须能够解释筛选汇总或异常来源。
- 期间敏感追溯的非法日期或超过 12 个月期间必须返回既有受控日期范围错误。
- 销售、采购、库存、生产、成本和往来汇总值与追溯明细合计一致。
- 来源受限角色的追溯响应保留行数，但来源主键、单号、行号、数量、金额、业务日期、状态和路由参数均为脱敏值。
- 来源受限角色的异常列表也必须脱敏来源主键、来源编号、业务日期、对象名和追溯键，不能通过异常列表绕过来源模块权限。
- 金额和数量响应字段为字符串。
- 报表接口不写入业务单据、库存余额、往来台账、凭证或正式财报数据。

## 浏览器验收路径

1. 管理员登录。
2. 确认经营报表菜单和固定报表入口可见。
3. 进入经营概览，确认核心指标和口径说明可见。
4. 切换日期范围，确认指标和列表刷新。
5. 进入销售经营汇总，核对销售出库金额，打开追溯并跳转销售出库和销售订单。
6. 进入采购经营汇总，核对采购入库金额，打开追溯并跳转采购入库和采购订单。

8. 进入生产执行，核对工单计划、报工和完工入库数量，打开追溯并核对工单、领料、报工和完工入库。
9. 进入成本归集，确认页面明确业务成本记录且 `formalAccounting=false`，打开追溯并核对成本来源业务。
10. 进入往来收付，核对应收、应付、已收、已付和未结余额，打开追溯并核对应收、收款、应付和付款。
11. 进入经营异常清单，确认异常类型、判定说明和来源追溯。
12. 切换无数据筛选条件，确认空态。
13. 使用部分报表权限用户登录，确认只可访问授权报表。
14. 使用来源受限角色登录，确认报表可看但来源详情不可越权跳转，来源敏感字段脱敏。
15. 使用无报表权限用户访问报表页面和接口，确认拒绝。
16. 未登录访问报表页面和接口，确认认证拦截。
17. 保存桌面和窄屏视觉截图。

## 阻断标准

- 报表核心页面无法浏览器访问。
- 关键报表接口不可用。
- 任一固定报表缺失 `summary + items`。
- 任一固定报表缺失来源追溯接口。
- 汇总值与来源明细对不上。
- 来源受限角色可看到来源敏感字段或可越权跳转。
- 无权限用户可访问报表页面或接口。
- 来源追溯不可用或跳转错误。
- 空态、异常态或无权限态误导用户。
- 桌面或窄屏关键指标、筛选或表格严重遮挡不可读。
- 自动化质量门未通过。
- 视觉分析没有真实浏览器截图证据。
- 页面或接口把经营报表表达为总账、凭证、结账、正式财务报表或 BI 平台。
