# 缺陷流程

## 缺陷等级

- 阻断：系统无法启动、核心流程无法使用、关键数据错误、权限严重问题。
- 严重：主要功能异常，但存在有限绕行方式。
- 一般：局部功能、展示或交互问题。
- 建议：不影响当前验收的优化建议。

## 缺陷记录字段

- 标题。
- 所属模块。
- 严重等级。
- 复现步骤。
- 实际结果。
- 期望结果。
- 影响范围。
- 截图或日志。
- 处理人。
- 当前状态。

## 流转状态

新建 -> 已确认 -> 修复中 -> 待回归 -> 已关闭。

## 关闭标准

缺陷必须经过回归验证，确认实际结果符合期望后才能关闭。

## 合入与验收限制

- 阻断缺陷未关闭时，不得合入或推送主分支。
- 严重缺陷未处理时，必须记录影响范围和处理结论。
- 权限绕过、关键数据错误、数据库迁移失败、核心页面无法访问不得降级为一般缺陷。
- 阶段验收通知必须包含已知缺陷和风险，不得隐瞒未关闭问题。
