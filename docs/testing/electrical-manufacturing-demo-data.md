# 电气制造业验收模拟数据链

## 目标

本数据集用于验收环境演示当前系统已实现的完整业务流程。业务背景设定为单公司电气制造企业，核心产品为 `QH-MCC-400 智能低压控制柜`，先生产半成品控制抽屉模块，再总装成品低压柜，覆盖基础资料、BOM、库存、采购、生产、成本、销售、财务往来、退货退款反冲和经营报表。

本数据集强调三点：

- 数据链条清晰，可按业务发生顺序理解系统能力。
- 单据数量克制，避免历史测试数据干扰验收。
- 覆盖当前系统已实现业务功能，不把尚未开发的期间关账、MRP、质量检验、批次序列号、正式会计凭证等内容伪造成已完成能力。

## 运行方式

脚本位置：`scripts/seed-electrical-acceptance-data.mjs`。

默认运行命令：

```powershell
node scripts/seed-electrical-acceptance-data.mjs
```

默认连接：

| 项目 | 默认值 |
|---|---|
| 后端 API | `http://127.0.0.1:18082` |
| PostgreSQL 容器 | `qherp-postgres` |
| 管理员账号 | `admin` |
| 管理员密码 | `Qherp@2026!` |
| 演示账号密码 | `Qherp@2026!` |

可选环境变量：

| 变量 | 用途 |
|---|---|
| `QHERP_API_BASE_URL` | 指定后端 API 地址 |
| `QHERP_ADMIN_USERNAME` | 指定管理员账号 |
| `QHERP_ADMIN_PASSWORD` | 指定管理员密码 |
| `QHERP_DEMO_PASSWORD` | 指定演示账号初始密码 |
| `QHERP_SEED_SUMMARY_FILE` | 写出完整执行明细 JSON；默认只在控制台打印摘要 |

清理策略：

- 清理所有业务单据、库存余额、库存流水、基础资料、演示角色、演示用户和审计日志。
- 保留 `admin`、`SYSTEM_ADMIN`、系统权限种子和系统管理员权限绑定。
- 脚本不内置备份；如需保留现场数据，运行前先执行数据库备份。

## 推荐演示顺序

1. 登录 `admin`，查看账号、角色和权限。
2. 查看基础资料：单位、仓库、供应商、客户、物料分类和物料。
3. 查看 BOM：半成品 BOM 已启用，成品标准 BOM 已启用，旧版 BOM 已停用，试制 BOM 为草稿。
4. 查看库存期初和盘点调整：原材料仓有期初库存，二次线盘盈，铭牌盘亏。
5. 查看采购管理：星河订单备料采购，采购订单确认，到货入库并过账。
6. 查看应付：采购入库生成应付，应付确认后支付 25000。
7. 查看生产执行第一段：生产 20 件控制抽屉模块，按 BOM 领料、报工、完工入半成品仓并完工。
8. 查看生产执行第二段：生产 10 台智能低压控制柜，从半成品仓领控制抽屉模块，从原材料仓领柜体、铜排和铭牌，报工、完工入成品仓并完工。
9. 查看生产退补料：铜排裁切后退料 1 千克，铭牌刮花补料 2 件。
10. 查看成本归集：生产领料、报工自动生成成本记录，另有半成品人工 1800 和成品制造费用 2600 的手工成本记录。
11. 查看采购退货：接触器 5 件外观破损退回供应商，过账后同步形成应付退货冲减。
12. 查看销售管理：星河智能装备购买 8 台低压柜，销售订单确认、销售出库过账。
13. 查看应收：销售出库生成应收，应收确认后收款 30000。
14. 查看销售退货：客户退回 1 台运输变形低压柜，过账后成品库存回增，并同步形成应收退货冲减。
15. 查看往来冲减：客户退货后退回部分已收款 1200；供应商返还部分预付款 800。
16. 查看经营报表：经营总览、销售、采购、库存、生产、成本、往来、异常报表，时间范围使用 `2026-07-01` 至 `2026-07-31`。
17. 切换演示角色账号，验证各角色菜单、按钮和接口权限边界；`demo_no_permission` 用于无权限页和接口拒绝。

## 使用注意

- 这是验收环境数据，不是迁移脚本，不应在生产库执行。
- 脚本每次运行都会清空并重建验收库业务数据，单据号会随运行时间变化。
- 若新增业务模块，应扩展本数据链，而不是另起一组互不关联的数据。
- 业务演示优先使用 `admin` 走完整链，再用各角色账号验证权限边界。
- 若报表或页面展示与本文件核对值不一致，应先检查是否重新运行过脚本、服务是否指向同一数据库、时间范围是否为 `2026-07-01` 至 `2026-07-31`。
