# 仓库与库存基础测试计划

## 目标

验证仓库与库存基础阶段满足功能、接口、权限、异常、审计、部署和浏览器视觉验收要求，确保后续生产工单、领料、完工入库和成本归集可以依赖可信库存余额和库存变动追溯。

## 准入标准

- 任务文档、设计规格、接口契约、权限编码和验收路径已确定。
- 账号权限、基础资料与物料管理、BOM 管理在主分支可用。
- 本地 PostgreSQL、后端、前端启动方式可用。
- 测试仓库、物料、单位和 BOM 基础数据口径已明确。
- 已确认一期不允许负库存、期初同一仓库物料只允许过账一次、已过账单据不可编辑。

## 准出标准

- 前端 `npm test`、`npm run build` 通过。
- 后端 Maven/Testcontainers 测试通过。
- 管理员浏览器验收路径通过。
- 仓库人员和只读用户权限验收通过。
- 后端越权接口验证通过。
- 视觉分析截图与结论已保存到 `docs/testing/inventory-management-visual-audit/`。

## 测试角色

| 类型 | 用途 |
|---|---|
| 管理员 | 完成期初、调整、过账、余额查询、流水追溯和审计验收 |
| 仓库人员 | 验证库存查询、单据创建和过账权限边界 |
| 只读用户 | 验证余额、流水和详情可查看，写按钮不可见，写接口返回 `AUTH_FORBIDDEN` |
| 无权限用户 | 验证无菜单入口、路由受限和接口拒绝 |

## 后端接口测试范围

- 未登录访问库存余额、库存变动、库存单据列表和详情返回 `AUTH_UNAUTHORIZED`。
- 无对应权限访问库存查询和写接口返回 `AUTH_FORBIDDEN`。
- 写接口缺少 CSRF token 被拒绝。
- 库存余额按仓库、物料、物料类型、关键词、正库存筛选和分页。
- 库存变动按仓库、物料、变动类型、方向、业务日期范围筛选和分页。
- 创建期初草稿，明细为空、数量为 0、数量为负、重复明细返回明确错误。
- 创建调整草稿，缺少调整方向或数量非法返回明确错误。
- 停用仓库、停用物料、非物料基本单位参与单据返回明确错误。
- 更新草稿单据成功；更新已过账单据返回 `INVENTORY_DOCUMENT_POSTED_IMMUTABLE`。
- 过账期初后库存余额增加，库存变动记录包含变动前、变动数量和变动后。
- 同一仓库物料重复期初过账返回 `INVENTORY_OPENING_EXISTS`。
- 调增调整后余额增加并生成调增流水。
- 调减调整后余额减少并生成调减流水。
- 调减超过现存数量返回 `INVENTORY_STOCK_NOT_ENOUGH`，余额和流水不变化。
- 重复过账同一单据返回 `INVENTORY_DUPLICATE_POST` 或等价受控错误，不能重复入账。
- 并发过账同一仓库物料时余额不丢失更新，不产生负库存。
- 所有过账动作写入审计日志。

## 前端测试范围

- 路由权限：无权限访问库存页面跳转无权限页。
- 库存余额列表：查询、重置、分页参数正确。
- 库存变动列表：时间范围、方向、业务类型查询正确。
- 库存单据列表：按类型、状态、时间和关键词查询。
- 期初表单：仓库、物料、数量、业务日期、原因必填。
- 调整表单：调整方向、数量、原因、提交确认和失败反馈。
- 动态明细：新增行、删除行、重复仓库物料校验。
- 数量输入：小数、0、负数、超精度和大数校验。
- 保存失败：错误提示可见，loading 恢复，弹窗或页面不误关闭。
- 只读权限：无创建、更新、过账按钮。
- 窄屏视口：余额表、变动表、单据明细表可横向滚动，操作按钮不重叠。

## 浏览器验收路径

1. 管理员登录。
2. 进入库存管理下的库存余额页面，查看默认状态和空状态。
3. 新建原材料 X 在原料仓的期初库存草稿并过账。
4. 查询库存余额，确认原材料 X 的现存数量、可用数量和单位正确。
5. 查询库存变动流水，确认期初流水可追溯到来源单据。
6. 创建调增调整并过账，确认余额增加、流水新增。
7. 创建调减调整并过账，确认余额减少、流水新增。
8. 尝试调减超过现存数量，确认返回库存不足且余额不变。
9. 尝试同一仓库物料重复期初过账，确认返回重复期初错误。
10. 尝试使用停用仓库或停用物料创建单据，确认被拒绝。
11. 仓库人员登录，验证其可执行的查询、创建和过账范围。
12. 只读用户登录，验证余额、流水和详情可看，写按钮不可见。
13. 只读用户直接调用创建、更新或过账接口，验证返回 `AUTH_FORBIDDEN`。
14. 保存桌面和窄屏视觉截图。

## 阻断标准

- 本地服务无法启动或浏览器无法访问核心页面。
- 管理员无法完成期初、查询、调整、过账和追溯。
- 库存可被调整为负数。
- 重复期初可成功过账。
- 并发调整导致余额错误。
- 库存余额和变动流水不一致。
- 无权限用户可绕过后端接口鉴权。
- 写操作无审计或不可追溯。
- 浏览器视觉分析缺失，或视觉问题导致核心数据、关键按钮或权限状态无法识别。

## 阶段排除

本阶段不测试生产工单、领料、报工、完工入库、成本、采购销售、批次、序列号、库位、条码、调拨、盘点任务、库存占用锁定、缺料分析、先进先出、多单位换算、多组织或多租户能力。
