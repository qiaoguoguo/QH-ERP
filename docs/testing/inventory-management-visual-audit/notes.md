## 功能验收结果

- 管理员可查看库存余额，原材料 X 在原料仓当前余额为 `110`，与期初、调增、调减结果一致。
- 库存变动流水展示期初、调增、调减三条记录，变动前、变动数量、变动后和来源单据可追溯。
- 期初和调整表单可进入，调整表单显示调整方向列，期初表单不显示调整方向列。
- 表单未填写业务日期和原因时显示 `请填写业务日期和原因`，页面未误跳转。
- 调减超过现存数量时显示 `库存不足，调减后库存不能小于 0`，草稿仍保持未过账。
- 同一仓库物料重复期初过账时显示 `同一仓库物料已存在已过账期初`，草稿仍保持未过账。
- 仓库人员账号可创建并过账库存单据，单据 `INV-OPEN-20260703013113634-005` 已过账。
- 只读账号可查看库存单据列表和详情入口，不显示新增、编辑、过账按钮。
- 只读账号直接调用写接口返回 `403 / AUTH_FORBIDDEN`。
- 无权限账号访问库存余额接口返回 `403 / AUTH_FORBIDDEN`。
- 使用停用仓库创建库存单据返回 `400 / INVENTORY_WAREHOUSE_INVALID`。
- 使用停用物料创建库存单据返回 `400 / INVENTORY_MATERIAL_INVALID`。

## 视觉分析结论

- 布局稳定性：桌面端筛选区、表格区、分页和操作列位置稳定，错误提示出现后未造成表格跳动或按钮遮挡。
- 信息密度：库存余额、流水和单据列表保持后台 ERP 高密度信息呈现，适合查询、核对和重复操作。
- 关键操作可见性：管理员新增期初、新增调整、查询、重置、详情、编辑和过账入口可识别；只读账号写操作入口不可见。
- 数量列对齐：库存余额和流水数量列右对齐，数字显示为紧凑格式，余额 `110`、流水 `100 / 25 / 15` 可快速核对。
- 文案溢出：较长仓库编码、物料编码和单据编号在表格列内省略显示，未穿出容器或遮挡相邻列。
- 控件重叠：桌面端表单、列表、错误提示和按钮未出现重叠；窄屏端表格提供横向滚动，核心数据未被裁断。
- 响应式适配：窄屏下筛选项和表单字段纵向排列，库存表格可横向滚动；当前侧边栏仍占用窄屏首屏空间较多，属于后续体验优化项，不阻断本阶段验收。
- 权限状态识别：只读账号顶部身份、左侧菜单和列表操作列能共同表达只读权限，写按钮不可见，后端越权拒绝结果与前端表现一致。
- 可访问性风险：截图可确认文本可见性、错误提示位置和目标按钮大小；键盘焦点顺序、读屏语义和对比度精确值仍需后续专项测试验证。
