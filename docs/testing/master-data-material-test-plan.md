# 基础资料与物料管理测试计划

## 目标

验证基础资料与物料管理阶段满足功能、接口、权限、异常、审计、部署和浏览器视觉验收要求，确保后续生产主链路可以引用稳定主数据。

## 准入标准

- 任务文档、接口契约、权限编码和验收路径已确定。
- 账号权限模块在主分支可用。
- 本地 PostgreSQL、后端、前端启动方式可用。

## 准出标准

- 前端 `npm test`、`npm run typecheck`、`npm run build` 通过。
- 后端 Docker 化 Maven/Testcontainers 测试通过。
- 管理员浏览器验收路径通过。
- 只读与越权接口验证通过。
- 视觉分析截图与结论已保存。

## 测试角色

| 类型 | 用途 |
|---|---|
| 管理员 | 完成六类主数据新增、查询、编辑、启用、停用和审计验收 |
| 只读用户 | 验证菜单可查看但写按钮不可见，后端写接口返回 `AUTH_FORBIDDEN` |
| 无权限用户 | 验证无菜单入口、路由受限和接口拒绝 |

## 核心用例

- 六类主数据列表、查询、分页、详情、新增、编辑、启用、停用。
- 编码重复、必填为空、状态非法、ID 不存在。
- 物料引用不存在或停用单位/分类失败。
- 分类存在启用子分类或启用物料时停用失败。
- 计量单位存在启用物料引用时停用失败。
- 只读角色按钮不可见，后端越权返回 `AUTH_FORBIDDEN`。
- 写操作审计记录可查询。

## 功能测试范围

- 计量单位：按关键词和状态查询，新增、编辑、启用、停用，校验编码唯一和数量精度。
- 仓库：按关键词和状态查询，新增、编辑、启用、停用，校验编码唯一。
- 供应商：按关键词和状态查询，新增、编辑、启用、停用，校验编码唯一。
- 客户：按关键词和状态查询，新增、编辑、启用、停用，校验编码唯一。
- 物料分类：平铺接口返回 `parentId`，前端树形展示，新增、编辑、启用、停用。
- 物料档案：按关键词、状态、分类、物料类型、来源属性查询，详情、新增、编辑、启用、停用。

## 接口测试范围

- 未登录访问受保护接口返回认证错误。
- 无权限访问写接口返回 `AUTH_FORBIDDEN`。
- 列表接口返回 `PageResponse<T>`。
- 详情接口在 ID 不存在时返回 `MASTER_DATA_NOT_FOUND`。
- 创建和更新接口校验必填、编码唯一和状态枚举。
- 物料创建和更新接口校验单位、分类存在且启用。
- 物料分类创建和更新接口校验父级存在且启用，禁止自引用、挂到自身子级或形成环。
- 分类停用接口校验启用子分类和启用物料引用。
- 计量单位停用接口校验启用物料引用。
- 写接口校验 CSRF token。
- 创建、更新、启用、停用写入审计日志。

## 权限测试范围

- 管理员具备六类主数据查看、创建和更新权限。
- 只读用户具备查看权限时可进入列表和详情。
- 只读用户没有创建、编辑、启用、停用按钮。
- 只读用户直接调用创建、更新、启用、停用接口返回 `AUTH_FORBIDDEN`。
- 无权限用户不能通过直接路由访问受保护页面。
- 权限变更以后端实时鉴权为准，前端刷新当前用户或重新登录后更新展示。

## 浏览器验收路径

1. 管理员登录。
2. 新增单位、仓库、供应商、客户。
3. 新增物料分类。
4. 新增原材料、半成品、成品和辅料，最小覆盖 `PURCHASED`、`SELF_MADE`、`OUTSOURCED` 来源属性。
5. 查询、编辑、停用物料。
6. 验证重复编码和停用引用错误。
7. 只读用户登录验证权限表现。
8. 保存桌面和窄屏视觉截图。

## 视觉分析要求

- 浏览器验收阶段必须保存关键页面截图和分析结论，证据目录使用 `docs/testing/master-data-material-visual-audit/`。
- 截图至少覆盖桌面视口和一个窄屏视口。
- 关键页面截图必须覆盖基础资料菜单、计量单位列表、仓库表单、供应商列表、客户列表、物料分类树、物料档案列表、物料表单错误、只读权限状态和窄屏物料列表。
- 布局稳定性检查：页面标题、查询区、表格、分页、弹窗、抽屉和操作区不能在加载、空状态、错误状态或权限状态切换时出现异常跳动。
- 信息密度检查：桌面端应保持 ERP 后台可扫描的信息密度，表格行高、列宽和筛选区间距不能明显降低维护效率。
- 关键操作可见性检查：新增、编辑、启用、停用、保存、取消、查询和重置操作在有权限时清晰可见，在无权限时不可误导用户。
- 文案溢出检查：编码、名称、规格型号、联系人、联系电话和错误提示不能穿出容器或遮挡相邻内容。
- 控件重叠检查：表格固定列、横向滚动、弹窗底部按钮、下拉框和提示信息不得互相遮挡。
- 权限状态检查：只读视图写按钮不可见，越权或无权限反馈可识别。
- 异常状态检查：重复编码、停用引用、必填为空、列表为空和加载失败提示清楚，不应关闭未保存表单。
- 如发现影响核心操作、关键数据识别、权限状态判断或工作台扫描效率的视觉问题，必须修复并重新截图后才能进入阶段验收。

## 回归范围

- 登录、退出和当前用户权限。
- 菜单渲染、路由守卫和按钮权限。
- 六类主数据列表、表单、启停和错误反馈。
- 后端接口鉴权和审计日志。
- 数据库迁移和权限种子。
- 窄屏导航、查询区、表格横向滚动和弹窗可用性。

## 阻断缺陷

- 本地服务无法启动或浏览器无法访问核心页面。
- 管理员无法完成六类主数据核心维护路径。
- 编码唯一约束失效。
- 物料可引用不存在或停用的单位/分类。
- 分类存在启用子分类或启用物料时仍可停用。
- 计量单位存在启用物料引用时仍可停用。
- 无权限用户可绕过后端接口鉴权。
- 写操作没有审计记录。
- 关键页面在桌面或窄屏下主内容不可用、关键按钮不可见、表格列互相遮挡或文本严重错位。
