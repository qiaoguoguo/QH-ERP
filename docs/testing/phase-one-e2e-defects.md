# 一期生产主链路端到端缺陷清单

## 缺陷分级

| 等级 | 判定标准 | 处理要求 |
|---|---|---|
| 阻断 | 服务无法启动；核心页面不可访问；管理员主链路无法走通；库存、工单或成本核心数据错误；权限绕过；数据库迁移失败；视觉问题导致关键操作不可识别 | 必须修复并复验，通过前不得进入阶段验收 |
| 严重 | 主链路可走但关键异常无受控提示；追溯断链；菜单、按钮或路由权限与后端权限不一致；重复操作存在数据风险但未实际污染 | 必须修复或形成明确风险处理结论 |
| 一般 | 文案、排序、轻微布局或非核心体验问题，不影响主路径、数据可信度、权限判断和关键操作 | 可记录为后续优化，但需说明影响范围 |

## 当前结论

- 文档基线、自动化基线、管理员主路径、浏览器写操作补证、基础资料维护/库存角色/生产角色/成本角色/只读用户/无权限用户权限核查、未登录、BOM 不可用、库存不足、超领、超报、超完工入库、重复过账、非法成本金额或数量、无效工单手工成本、审计记录和成本不反写核查阶段尚未发现阻断、严重或一般缺陷。
- 产品规格复审提出的两项验收证据缺口已登记为复审发现项并补证关闭；它们不是业务功能缺陷，但属于阶段验收证据质量问题。
- 最终复审中产品经理、UI 设计师和测试同角色替代结论均为通过，未提出新的阻断、严重或一般缺陷。
- 当前缺陷列表为空；后续若在最终质量门、复审或用户验收中发现问题，将继续按本清单登记。
- 发现阻断缺陷时，必须先派发修复任务并完成规格审查、代码质量审查和复验。
