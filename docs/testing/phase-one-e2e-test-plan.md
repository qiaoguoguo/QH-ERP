# 一期生产主链路端到端测试计划

## 目标

验证一期生产主链路在当前分支真实可用，覆盖账号权限、基础资料与物料、BOM、库存、生产执行、成本归集、来源追溯、异常处理、页面操作和视觉证据。

本计划只验证一期已完成能力，不纳入采购、销售、正式财务、报表、审批、多组织或多租户。

## 准入标准

- `docs/tasks/010-phase-one-e2e-stabilization.md` 已建立并明确范围。
- `main` 或当前阶段分支包含一期六个业务里程碑的最新代码。
- 本地 PostgreSQL、后端、前端可启动。
- 管理员账号可登录。
- 已确认成本归集只作为业务记录与来源追溯，不作为正式财务核算结果。

## 准出标准

- 后端全量测试通过。
- 前端全量测试、类型检查和构建通过。
- 本地部署健康检查通过。
- 管理员浏览器主路径通过。
- 权限矩阵覆盖并通过。
- 高风险异常路径覆盖并通过。
- 数据一致性和追溯断言通过。
- 视觉分析截图与结论保存到 `docs/testing/phase-one-e2e-visual-audit/`。
- 缺陷清单已记录，阻断缺陷为 0。
- `git diff --check` 通过。

## 端到端测试矩阵

| 链路 | 主路径验证点 | 通过标准 |
|---|---|---|
| 账号权限 | 管理员登录、菜单加载、直接 URL 访问 | 菜单、路由、接口权限一致 |
| 基础资料 | 单位、仓库、物料分类、成品、原料、辅料创建并启用 | 后续 BOM、库存、生产可引用 |
| BOM | 成品 BOM 创建、明细维护、启用 | 工单创建时可选且明细正确带出 |
| 库存基础 | 期初库存单创建、过账、余额和流水更新 | 原料库存可用于生产领料 |
| 生产执行 | 工单创建、发布、领料、报工、完工入库 | 工单状态、数量、库存流水一致 |
| 成本归集 | 领料和报工自动生成成本记录，完工入库进入产出追溯 | 成本记录可查、可追溯、无重复 |
| 跨模块追溯 | 库存流水追来源单据，成本记录追生产工单，工单查看成本分区 | 来源链路不断裂 |
| 页面导航 | 左侧菜单、子菜单、直接 URL 访问 | 页面可达，权限不绕过 |

## 角色权限矩阵

| 角色 | 应允许 | 应拒绝 |
|---|---|---|
| 管理员 | 全链路配置、过账、查询、追溯 | 无 |
| 基础资料维护 | 单位、仓库、物料、BOM 维护 | 库存过账、生产过账、成本维护 |
| 库存角色 | 库存余额、流水、库存单据维护和过账 | 生产工单、成本记录维护 |
| 生产角色 | 工单、领料、报工、完工入库 | 基础资料关键配置、库存调整、成本手工维护 |
| 成本角色 | 成本记录查看、手工成本维护、工单成本追溯 | 库存写操作、生产写操作 |
| 只读用户 | 全链路列表、详情、追溯只读 | 所有新增、编辑、过账、取消 |
| 无权限用户 | 登录后仅无权限页或空菜单 | 所有业务页面和接口 |

权限验收必须同时覆盖菜单显示、按钮显示、直接 URL 和后端接口。

## 异常路径矩阵

| 模块 | 异常 | 预期 |
|---|---|---|
| 登录权限 | 未登录访问业务页 | 跳转登录 |
| 权限 | 无权限直接访问 URL 或 API | 无权限页或 `403` |
| 基础资料 | 停用仓库或物料被后续业务引用 | 明确业务错误 |
| BOM | 未启用 BOM 创建或发布工单 | 明确业务错误 |
| 库存 | 库存不足领料、重复期初、负数调整越界 | 不产生错误库存，提示清晰 |
| 生产工单 | 草稿不可领料、报工、入库，已取消不可执行 | 状态拦截 |
| 生产过账 | 重复过账领料、报工、入库 | 不重复影响库存、工单或成本 |
| 报工入库 | 完工入库数量超过合格数量 | 明确业务错误 |
| 成本 | 自动来源重复归集、草稿来源归集 | 不生成重复成本 |
| 手工成本 | 无效工单、非法金额、非法数量、超精度 | 表单或接口返回明确错误 |

## 数据一致性断言

- 期初过账后，目标原料库存余额等于期初数量。
- 领料过账后，目标原料库存余额减少领料数量。
- 完工入库过账后，目标成品库存余额增加入库数量。
- 每次库存变化都有对应库存流水和来源单据。
- 工单发布后 BOM 用料快照与 BOM 明细和计划数量匹配。
- 领料过账后，领料单状态为已过账，工单已领数量更新。
- 报工过账后，工单累计报工数量更新。
- 完工入库过账后，工单累计入库数量更新。
- 领料过账生成材料成本业务记录，来源指向领料单和领料行。
- 报工过账生成人工数量口径成本记录，来源指向报工单。
- 完工入库在工单成本汇总中作为产出追溯。
- 成本记录不反写库存余额、生产工单状态或生产单据状态。
- 关键写操作产生审计记录。

## 执行顺序

1. 自动化测试：后端全量测试、前端全量测试、前端类型检查、前端构建、`git diff --check`。
2. 接口核查：登录会话、菜单权限、基础资料、BOM、库存余额和流水、生产工单、成本记录、追溯接口。
3. 本地部署确认：PostgreSQL、后端健康检查、前端主页面、核心路由 HTTP 可达。
4. 浏览器管理员主路径：从测试数据准备到成本追溯完整走通。
5. 浏览器权限路径：按角色检查菜单、按钮、直接 URL 和关键写接口拒绝。
6. 浏览器异常路径：库存不足、重复过账、非法数量或金额、无权限访问。
7. 视觉分析：保存关键截图和分析结论。
8. 数据核对与缺陷归档：汇总库存余额、工单数量、成本记录、流水、审计记录，更新缺陷清单。

## 缺陷分级标准

| 等级 | 标准 |
|---|---|
| 阻断 | 服务无法启动；核心页面不可访问；主链路无法走通；库存、工单或成本核心数据错误；权限绕过；数据库迁移失败；视觉问题导致关键操作不可识别 |
| 严重 | 主链路可走但关键异常无受控提示；追溯断链；只读或菜单状态与后端权限不一致；重复操作有数据风险但未实际污染 |
| 一般 | 文案不清、列表排序不理想、轻微布局问题、非核心筛选体验问题、不影响主路径和数据可信度 |

## 阶段排除

本阶段不测试采购、销售、正式财务核算、总账、凭证、报表看板、审批、多组织、多租户、工艺路线、批次、库位、条码、标准成本、移动加权或成本差异分析。
