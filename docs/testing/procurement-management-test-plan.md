# 采购管理基础测试计划

## 目标

验证采购管理基础模块满足功能、接口、权限、异常、库存联动、来源追溯、审计、部署和浏览器视觉验收要求，确保采购入库能够可信增加库存，并能从采购订单、采购入库和库存流水之间完成追溯。

本计划不验收应付、发票、付款、税务、审批、MRP、采购退货、质检入库、多币种、多组织、正式财务凭证或报表能力。

## 准入标准

- 采购任务文档、设计规格、接口契约、权限编码和验收路径已确定。
- 账号权限、基础资料与物料、BOM、库存基础、生产执行和成本归集在当前分支可用。
- 本地 PostgreSQL、后端和前端启动方式可用。
- 已确认采购入库过账必须写库存余额和 `PURCHASE_RECEIPT` 库存流水。
- 已确认采购单价只作为业务字段，不代表应付、付款、税务或正式成本。
- 已确认采购阶段不做审批、MRP、采购退货或质检入库。

## 准出标准

- 后端 Maven/Testcontainers 全量测试通过。
- 前端 `npm test`、`npm run typecheck`、`npm run build` 通过。
- 管理员浏览器主路径通过。
- 采购员、仓库角色、只读用户和无权限用户权限验收通过。
- 未登录和越权接口验证通过。
- 异常路径验证通过，失败操作不产生部分写入。
- 视觉分析截图与结论已保存到 `docs/testing/procurement-management-visual-audit/`。
- `git diff --check` 通过。

## 后端接口测试范围

- 未登录访问采购接口返回 `AUTH_UNAUTHORIZED`。
- 无对应权限访问采购查询和写接口返回 `AUTH_FORBIDDEN`。
- 写接口缺少 CSRF token 被拒绝。
- 采购订单列表按关键词、供应商、状态、订单日期、预计到货日期筛选分页。
- 采购订单详情返回基础信息、明细、入库记录和库存流水摘要。
- 创建采购订单草稿成功，订单号后端生成。
- 更新采购订单草稿成功。
- 确认采购订单成功，状态从 `DRAFT` 到 `CONFIRMED`。
- 取消采购订单成功，且已发生入库的订单不可取消。
- 关闭采购订单成功，且关闭后不可继续入库。
- 采购入库列表按关键词、供应商、仓库、状态、业务日期和来源订单筛选分页。
- 基于已确认订单创建采购入库草稿成功。
- 更新采购入库草稿成功。
- 采购入库过账成功，库存余额增加，库存流水为 `PURCHASE_RECEIPT`。
- 过账前后库存差值必须等于本次入库数量；同一采购入库明细必须且只能生成一条库存流水。
- 库存流水必须断言 `movement_type = PURCHASE_RECEIPT`、`direction = IN`、`source_type = PURCHASE_RECEIPT`、`source_id = 采购入库单 id`、`source_line_id = 采购入库明细 id`。
- 采购订单行已入库数量必须等于所有已过账采购入库明细累计，未入库数量必须等于采购数量减已入库数量。
- 部分入库和全部入库过账后必须分别断言订单状态为 `PARTIALLY_RECEIVED` 和 `RECEIVED`。
- 订单行已入库数量和未入库数量与过账入库累计一致。
- 部分入库后订单状态为 `PARTIALLY_RECEIVED`。
- 全部入库后订单状态为 `RECEIVED`。
- 重复过账返回受控错误，库存余额和流水不变。
- 入库数量为 0、负数、超精度或超过未入库数量均返回受控错误。
- 停用供应商新建或确认采购订单被拒绝；已确认订单后续入库不因供应商停用失败。
- 停用物料新增采购订单明细被拒绝；停用仓库创建或过账采购入库被拒绝。
- 来源订单行与入库物料不一致被拒绝。
- 已过账采购入库不可编辑。
- 采购写操作均写入审计日志。
- 采购入库不会写成本记录、不会修改生产工单状态。

## 前端测试范围

- 采购 API 封装：请求路径、查询参数、写操作 CSRF、错误信封处理。
- 路由权限：无权限访问采购页面跳转无权限页。
- 采购菜单：有采购权限时显示采购管理入口，无权限时不显示。
- 采购订单列表：筛选、重置、分页、加载失败、空状态和权限按钮显示。
- 采购订单表单：必填校验、明细为空、数量校验、单价校验、保存失败提示、防重复提交。
- 采购订单详情：状态、明细、入库记录、追溯入口和操作按钮可见性。
- 采购入库列表：筛选、分页、空状态、详情入口和权限按钮显示。
- 采购入库表单：来源订单摘要、未入库数量、本次入库数量校验、超入库提示、防重复提交。
- 采购入库详情：来源订单、过账前后库存、库存流水来源和过账按钮状态。
- 只读权限：无创建、编辑、确认、取消、关闭或过账按钮。
- 窄屏视口：采购订单列表、采购订单详情、采购入库表单或详情可访问，关键操作不重叠。

## 浏览器验收路径

1. 管理员登录。
2. 确认采购菜单、采购订单和采购入库入口可见。
3. 准备启用供应商、启用原材料、启用仓库和基础单位。
4. 创建采购订单草稿，明细录入铜排、线缆、PLC、变频器或辅料。
5. 编辑采购订单草稿。
6. 确认采购订单。
7. 基于采购订单创建采购入库草稿，执行部分入库。
8. 过账部分入库，确认库存余额增加、库存流水出现 `PURCHASE_RECEIPT`。
9. 返回采购订单详情，确认已入库数量和未入库数量。
10. 创建第二张采购入库，完成剩余入库。
11. 过账后确认订单状态为 `RECEIVED`。
12. 从库存流水追溯到采购入库。
13. 从采购入库追溯到采购订单。
14. 使用采购入库后的物料创建生产工单并完成生产领料，证明采购库存进入既有生产消耗链路；该步骤只验证链路联通，不作为采购阶段新增生产能力验收依据。
15. 采购员登录，验证采购订单写权限和入库查看权限。
16. 仓库角色登录，验证采购入库创建、过账权限和订单决策操作拒绝。
17. 只读用户登录，验证只能查看订单、入库和追溯。
18. 无权限用户访问采购页面和接口，验证拒绝。
19. 未登录访问采购接口，验证认证拦截。
20. 保存桌面和窄屏视觉截图。

## 异常矩阵

| 场景 | 预期 |
|---|---|
| 未登录访问采购页面或接口 | 跳转登录或返回未认证 |
| 无权限访问采购页面或接口 | 显示无权限或返回拒绝 |
| 草稿订单无明细时确认 | 受控失败 |
| 停用供应商创建或确认订单 | 受控失败，不生成有效订单 |
| 停用物料加入订单明细 | 受控失败 |
| 非采购物料加入订单明细 | 受控失败 |
| 采购数量为 0、负数或超精度 | 受控失败 |
| 采购单价为负数或超精度 | 受控失败 |
| 未确认订单创建入库 | 受控失败 |
| 已取消、已关闭或已全部入库订单继续入库 | 受控失败 |
| 停用仓库创建或过账入库 | 受控失败 |
| 入库数量为 0、负数或超精度 | 受控失败 |
| 入库数量超过未入库数量 | 受控失败，累计数量不变 |
| 来源订单行与入库物料不一致 | 受控失败 |
| 已过账入库单编辑 | 受控失败 |
| 重复过账入库单 | 受控失败，库存和流水不变 |
| 异常过账中任一明细失败 | 整单回滚，无部分库存或流水 |

## 阻断标准

- 本地服务无法启动或浏览器无法访问核心页面。
- 管理员无法完成采购订单到采购入库过账主路径。
- 采购入库过账后库存余额未增加。
- 采购入库过账后库存流水缺失或来源不是 `PURCHASE_RECEIPT`。
- 采购订单、采购入库和库存流水追溯断链。
- 采购订单累计入库数量错误。
- 异常失败产生部分库存、部分流水或错误累计入库。
- 重复过账污染库存或流水。
- 无权限用户可绕过后端接口鉴权。
- 写操作无审计或关键链路不可追溯。
- 页面把采购单价表达为应付、付款、凭证或正式财务成本。
- 浏览器视觉分析缺失，或视觉问题导致核心数据、关键按钮或权限状态无法识别。
