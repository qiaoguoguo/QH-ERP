## UI/交互结论

- 页面布局：桌面端延续现有 ERP 后台风格，左侧导航、顶部账号区、内容区标题、筛选区、表格和详情卡片结构一致，没有营销页或看板化倾向。
- 信息密度：采购订单、采购入库和库存流水页面密度适中。列表以表格为主，筛选项集中在表格上方，行操作位于右侧，符合制造业后台扫描习惯。
- 关键操作可见性：桌面端新建、查询、重置、详情、确认/取消/关闭、创建入库、保存采购入库、查看来源订单等关键操作位置明确。权限账号截图显示按钮按角色收敛。
- 文案溢出：长单号、供应商、物料和仓库名称在表格中有省略；详情页可通过上下文和详情入口识别，不影响关键数据判断，记为非阻断。
- 控件重叠：桌面和窄屏截图均未发现按钮、表单项或表格内容互相遮挡。部分宽表依赖横向滚动，未阻断操作。
- 响应式适配：窄屏下侧栏未折叠，首屏空间占用较大，列表和表单需要较多滚动；但核心入口、页面标题和主要操作仍可访问，未达到阻断程度。
- 状态识别：采购订单状态“草稿 / 已确认 / 已取消 / 已关闭 / 全部入库”、采购入库状态“已过账”、库存流水方向“入库”均使用中文标签，颜色表达清楚，未出现 `POSTED` 等原始枚举直接外露。
- 采购入库语义：采购入库页面明确使用“来源订单”“入库仓库”“业务日期”“本次入库数量”“库存流水追溯”等文案，未与质检、退货、财务入账、发票、付款或审批混淆。
- 权限状态识别：采购员、仓库角色、只读角色和无权限角色的可见按钮差异清楚；无权限页文案明确。
- 范围边界：截图未发现应付、发票、付款、审批、MRP、质检、退货、财务凭证、报表等范围外 UI。
