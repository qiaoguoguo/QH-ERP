# 销售管理基础测试计划

## 目标

验证销售管理基础模块满足功能、接口、权限、异常、库存联动、来源追溯、审计、部署和浏览器视觉验收要求，确保销售出库能够可信扣减库存，并能从销售订单、销售出库和库存流水之间完成追溯。

本计划不验收应收、发票、收款、税务、凭证、收入确认、销售退货、物流、报表、多币种、多组织、多公司或正式财务核算能力。

## 准入标准

- 销售任务文档、设计规格、接口契约、权限编码和验收路径已确定。
- 账号权限、客户、物料、仓库、库存余额和库存流水基础能力在当前分支可用。
- 本地 PostgreSQL、后端和前端启动方式可用。
- 已确认销售出库过账必须扣减库存余额并写入销售来源库存流水。
- 已确认销售单价只作为业务字段，不代表应收、收款、税务、收入确认或正式财务凭证。
- 已确认销售阶段不做销售退货、物流配送、发票、收款、收入确认、报表或财务凭证。

## 准出标准

- 后端 Maven/Testcontainers 全量测试通过。
- 前端 `npm test`、`npm run typecheck`、`npm run build` 通过。
- 管理员浏览器主路径通过。
- 销售员、仓库角色、只读用户、无权限用户和未登录用户权限验收通过。
- 未登录和越权接口验证通过。
- 异常路径验证通过，失败操作不产生部分写入。
- 库存余额扣减、库存流水、订单累计出库数量和来源追溯验收通过。
- 视觉分析截图与结论已保存到 `docs/testing/sales-management-visual-audit/`。
- `git diff --check` 通过。

## 最小验收闭环

本阶段最小可准出的业务闭环为：客户、可销售物料、仓库和可用库存准备完成；管理员创建销售订单草稿并编辑；确认销售订单；基于已确认订单创建第一张销售出库并部分过账；库存余额按出库数量扣减，库存流水写入 `SALES_SHIPMENT` 来源；继续创建第二张销售出库完成剩余出库并过账；销售订单状态进入 `SHIPPED` / 全部出库；销售订单可查看两张出库记录，销售出库可查看库存流水，库存流水可回跳对应销售出库。

## 测试角色

| 类型 | 用途 |
|---|---|
| 管理员 | 完成销售订单、销售出库、库存扣减、库存流水、追溯和审计主路径 |
| 销售员 | 验证销售订单查看、创建、编辑、确认、取消、关闭和出库查看权限；不得过账销售出库 |
| 仓库角色 | 验证销售订单查看、销售出库查看、创建、编辑、过账和库存流水查看权限；不得确认、取消或关闭销售订单 |
| 只读用户 | 验证销售订单、销售出库和追溯可查看，写按钮不可见，写接口拒绝 |
| 无权限用户 | 验证无销售菜单入口、路由受限和接口拒绝 |
| 未登录用户 | 验证认证拦截 |

### 业务验收账号授权包口径

- 销售员、仓库角色和只读用户的浏览器验收账号，除下方销售订单、销售出库和库存流水权限外，应具备销售页面加载引用数据所需的基础资料只读权限。
- 必要基础资料只读权限至少包括：`master:customer:view`、`master:warehouse:view`、`master:material:view`、`master:unit:view`。
- 仓库角色和只读用户如需从库存流水完成销售来源追溯，应具备 `inventory:movement:view`。
- 该口径是制造业 ERP 业务角色的合理授权包，用于支持销售页面加载客户、仓库、物料和单位引用数据，不代表扩大销售阶段功能范围，也不改变销售接口自身的后端鉴权边界。

## 自动化验证范围

### 后端接口测试

- 未登录访问销售接口返回 `AUTH_UNAUTHORIZED`。
- 无对应权限访问销售查询和写接口返回 `AUTH_FORBIDDEN`。
- 写接口缺少 CSRF token 被拒绝。
- 销售订单列表按关键词、客户、状态、订单日期、预计交付日期筛选分页。
- 销售订单详情返回基础信息、明细、出库记录和库存流水摘要。
- 创建销售订单草稿成功，订单号后端生成。
- 更新销售订单草稿成功。
- 确认销售订单成功，状态从 `DRAFT` 到 `CONFIRMED`。
- 取消销售订单成功，且已发生出库的订单不可取消。
- 关闭销售订单成功，且关闭后不可继续出库。
- 销售出库列表按关键词、客户、仓库、状态、业务日期和来源订单筛选分页。
- 基于已确认订单创建销售出库草稿成功。
- 更新销售出库草稿成功。
- 销售出库过账成功，库存余额扣减，库存流水为销售出库来源。
- 过账前后库存差值必须等于本次出库数量；同一销售出库明细必须且只能生成一条库存流水。
- 库存流水必须断言 `movement_type = SALES_SHIPMENT`、`direction = OUT`、`source_type = SALES_SHIPMENT`、`source_id = 销售出库单 id`、`source_line_id = 销售出库明细 id`。
- 销售订单行已出库数量必须等于所有已过账销售出库明细累计，未出库数量必须等于销售数量减已出库数量。
- 部分出库和全部出库过账后必须分别断言订单状态为 `PARTIALLY_SHIPPED` 和 `SHIPPED`。
- 重复过账返回受控错误，库存余额和流水不变。
- 出库数量为 0、负数、超精度或超过未出库数量均返回受控错误。
- 库存不足返回受控错误，不能产生负库存、部分库存扣减或部分流水。
- 停用客户创建、更新或确认销售订单被拒绝。
- 订单确认后客户若被停用，不阻止基于该已确认订单创建或过账销售出库，但仍需校验订单状态、来源行、物料、单位、仓库和库存。
- 停用物料新增销售订单明细被拒绝；启用的 `RAW_MATERIAL` 与 `AUXILIARY` 创建或确认销售订单被拒绝；停用仓库创建或过账销售出库被拒绝。
- 来源订单行与出库物料不一致被拒绝。
- 已过账销售出库不可编辑。
- 销售写操作均写入审计日志。
- 销售出库不会写应收、发票、收款、税务、凭证、收入确认或报表数据。

### 前端测试

- 销售 API 封装：请求路径、查询参数、写操作 CSRF、错误信封处理。
- 路由权限：无权限访问销售页面跳转无权限页，未登录访问跳转登录。
- 销售菜单：有销售权限时显示销售管理入口，无权限时不显示。
- 销售订单列表：筛选、重置、分页、加载失败、空状态和权限按钮显示。
- 销售订单表单：客户必填、日期必填、明细为空、物料必填、数量校验、销售单价校验、重复物料或重复来源行校验、保存失败提示、防重复提交。
- 销售订单详情：状态、明细、出库记录、追溯入口和操作按钮可见性。
- 销售出库列表：筛选、分页、空状态、详情入口和权限按钮显示。
- 销售出库表单：来源订单摘要、未出库数量、本次出库数量、仓库、业务日期、超出库、库存不足、防重复提交。
- 销售出库详情：来源订单、出库明细、过账前后库存、库存流水来源和过账按钮状态。
- 只读权限：无创建、编辑、确认、取消、关闭或过账按钮。
- 仓库权限：可创建和过账销售出库，不显示销售订单确认、取消、关闭按钮。
- 销售员权限：可管理销售订单，不显示销售出库创建、编辑、过账按钮。
- 窄屏视口：销售订单列表、销售订单详情、销售出库表单或详情可访问，关键操作不重叠。

## 浏览器验收路径

1. 管理员登录。
2. 确认销售菜单、销售订单和销售出库入口可见。
3. 准备启用客户、启用可销售物料、启用仓库、基础单位和足够库存。
4. 创建销售订单草稿，明细录入可销售物料、销售数量和销售单价。
5. 编辑销售订单草稿。
6. 确认销售订单。
7. 基于销售订单创建销售出库草稿，执行部分出库。
8. 过账部分销售出库，确认库存余额扣减、库存流水出现 `SALES_SHIPMENT` 且方向为 `OUT`。
9. 返回销售订单详情，确认已出库数量和未出库数量。
10. 创建第二张销售出库，完成剩余出库。
11. 过账后确认订单状态为 `SHIPPED` / 全部出库。
12. 从库存流水追溯到销售出库，确认不会误跳库存单据或采购入库。
13. 从销售出库追溯到销售订单。
14. 从销售订单详情查看两张销售出库记录。
15. 销售员登录，验证销售订单写权限和销售出库过账拒绝。
16. 仓库角色登录，验证销售出库创建、编辑、过账权限和订单决策操作拒绝。
17. 只读用户登录，验证只能查看订单、出库和追溯。
18. 无权限用户访问销售页面和接口，验证拒绝。
19. 未登录访问销售页面和接口，验证认证拦截。
20. 保存桌面和窄屏视觉截图。

## 追溯验收口径

- 销售订单详情必须展示销售出库记录，至少包含出库单号、状态、仓库、业务日期、出库数量、过账人和详情入口。
- 销售出库详情必须展示来源销售订单摘要和库存流水追溯。
- 库存流水列表中销售来源流水必须标识为销售出库，方向为出库，数量、变动前、变动后、业务日期和操作人可见。
- 从库存流水点击销售来源单据必须进入对应 `/sales/shipments/{sourceId}` 销售出库详情路由。
- `INVENTORY_DOCUMENT` 来源流水仍必须进入库存单据详情，`PURCHASE_RECEIPT` 来源流水仍必须进入采购入库详情，生产来源或未知来源不能误跳销售出库。
- 无 `sales:shipment:view` 但有 `inventory:movement:view` 的账号可以查看库存流水列表，但不显示销售出库来源跳转。
- 同一销售出库明细只生成一条库存流水；重复过账不新增流水、不重复扣减库存。

## 销售单价验收口径

- 销售单价作为销售订单业务字段验收，不作为应收、发票、税务、收入确认、凭证或正式财务结果验收。
- 前端 payload 中销售数量、出库数量和销售单价应按字符串提交，避免浮点精度污染。
- 后端必须按 decimal 精度校验销售单价；负数、超精度和非法格式返回受控错误。
- 销售单价允许为 0 或必须大于 0 的规则，应在销售任务规格中固定；测试按最终规格执行，不允许前后端口径不一致。
- 编辑草稿时销售单价应保持原始精度语义；保存后详情、列表和后续追溯展示不得出现二进制浮点误差。

## 阻断标准

- 本地服务无法启动或浏览器无法访问核心页面。
- 管理员无法完成销售订单到销售出库过账主路径。
- 销售出库过账后库存余额未扣减，或扣减数量不等于本次出库数量。
- 销售出库导致负库存，或库存不足时产生部分扣减、部分流水。
- 销售出库过账后库存流水缺失、方向不是 `OUT`、来源不是销售出库，或来源行不准确。
- 销售订单、销售出库和销售来源库存流水追溯断链。
- 库存流水销售来源跳转误入库存单据、采购入库或错误销售出库。
- 销售订单累计出库数量或未出库数量错误。
- 异常失败产生部分库存、部分流水或错误累计出库。
- 重复过账污染库存或流水。
- 无权限用户可绕过后端接口鉴权，或销售员可过账出库，或仓库角色可确认、取消、关闭销售订单。
- 写操作无审计或关键链路不可追溯。
- 页面把销售单价表达为应收、发票、收款、税务、收入确认、凭证或正式财务结果。
- 浏览器视觉分析缺失，或视觉问题导致核心数据、关键按钮或权限状态无法识别。
