# 020 销售项目与合同管理基础测试计划

## 文档定位

本文是 020 阶段测试计划，服务于销售项目与合同管理基础的开发、集中审查、差异复审和交付前全量验证。本文不表示测试已经执行或通过。

020 的测试目标是确认销售项目作为一级成本、收入和利润对象的主维度能够稳定建立，并能通过 `projectId` 与 `contractId` 为后续阶段提供一致引用基础，同时不破坏 000-019 已交付链路。

## 范围边界

### 020 纳入

- 销售项目主档：项目编号、名称、客户、项目负责人、计划周期、状态、目标收入、目标成本、备注。
- 项目状态：`DRAFT`、`ACTIVE`、`CLOSED`、`CANCELLED`。
- 合同主档：主合同、补充合同、内部合同编号、外部纸质合同号、合同金额、签订日期、履约周期、状态、原因字段。
- 合同状态：`DRAFT`、`EFFECTIVE`、`CLOSED`、`TERMINATED`、`CANCELLED`。
- 数据关系：一个项目最多一个非取消主合同；一个项目可有多个补充合同；补充合同必须引用主合同。
- 状态规则：项目激活要求存在已生效主合同；项目关闭要求合同均为终态且关联销售订单均为 `SHIPPED`、`CLOSED` 或 `CANCELLED`；项目取消仅限未发生有效合同和销售订单的草稿。
- 项目更新：`ACTIVE` 项目允许更新负责人、计划日期、目标收入、目标成本和备注，必须写审计摘要；项目编号和客户锁定；`CLOSED`、`CANCELLED` 不可更新。
- 合同规则：合同草稿可取消；合同激活动作统一使用 `activate` 命名，状态进入 `EFFECTIVE`；已生效合同正常履约结束为 `CLOSED`，提前终止为 `TERMINATED`，两者均为终态。
- 补充合同规则：补充合同仅允许在项目 `ACTIVE` 且主合同 `EFFECTIVE` 时创建；激活补充合同时再次校验项目和主合同状态，否则返回稳定业务错误。
- 商业字段保护：生效前可编辑草稿；生效后关键商业字段不可直接覆盖，商业增减通过补充合同表达。
- 金额和日期规则：主合同金额必须大于 0；补充合同调整金额允许正负但不得为 0；目标收入和目标成本为非负管理目标；外部纸质合同号可空、可重复、可按关键词查询；合同日期允许超出项目计划周期。
- 销售订单关联：`projectId` 与 `contractId` 必须同时为空或同时有值；仅草稿创建和更新允许关联或解除；项目必须 `ACTIVE`、合同必须 `EFFECTIVE`、客户一致且合同属于项目；确认时再次校验。
- 历史兼容：历史销售订单允许 `projectId` 与 `contractId` 同时为空，不回填伪造归属。
- 权限与审计：项目、合同、关联销售订单摘要、候选接口、状态动作、草稿变更、原因字段均按规格受权限和审计约束；操作日志基于现有 `sys_audit_log`，不扩展审计表。
- 前端入口：销售菜单下“销售项目”，固定路由为 `/sales/projects`、`/sales/projects/create`、`/sales/projects/:id`、`/sales/projects/:id/edit`；`/sales` 根路径进入当前用户第一个可见销售子页面；合同在项目详情内通过抽屉维护；详情包含合同、关联订单、操作记录；390px 保持表格横向滚动。

### 后续阶段接口预留

- `projectId` 作为后续采购、库存、生产、费用、发票、收付款和成本统一引用基础。
- 020 只验证项目标识稳定、销售订单关联规则和历史兼容，不验证后续业务归集。
- 采购订单项目专采留到 024，生产工单项目归集留到 027，项目成本核算留到 029。

### 明确排除

- 项目专属库存、库存计价、项目专采、报价信用、缺料净算。
- 工单项目成本、发票、收入确认、凭证、总账、月结。
- 泛化合同版本、独立合同变更业务表、复杂审批、工艺路线、复杂质量。
- 合同金额拆税、开票依据、收入确认依据或正式财务结果。
- 对采购订单、生产工单增加无业务入口的项目字段。

## 测试分层

### 后端自动化

后端自动化应优先覆盖领域规则、事务一致性和权限鉴权。

- 项目创建：项目编号由后端生成且不可编辑，客户必须存在，负责人引用现有 `sys_user`，目标收入和目标成本不得为负。
- 项目状态：`DRAFT` 可激活、取消；激活必须存在已生效主合同；关闭必须满足合同均为终态且关联销售订单均为 `SHIPPED`、`CLOSED` 或 `CANCELLED`；取消仅允许未发生有效合同和销售订单的草稿。
- 项目更新：`ACTIVE` 项目只能更新负责人、计划日期、目标收入、目标成本和备注，并写入审计摘要；项目编号和客户不可改；`CLOSED`、`CANCELLED` 更新必须被拒绝。
- 合同创建：内部合同编号由后端生成且不可编辑；外部纸质合同号可空可重复；主合同金额必须大于 0；补充合同金额可正可负但不得为 0；补充合同必须引用所属项目的 `EFFECTIVE` 主合同，且项目必须为 `ACTIVE`。
- 主合同唯一：同一项目最多存在一个非取消主合同；已取消主合同不计入限制，但必须保留历史和审计；不得出现两个非取消主合同。
- 合同状态：草稿可编辑和取消；合同激活动作使用 `/activate`、`sales:contract:activate` 和 `SALES_PROJECT_CONTRACT_ACTIVATE`，状态进入 `EFFECTIVE`；合同生效后关键商业字段不可直接覆盖；`EFFECTIVE` 可正常关闭为 `CLOSED`；提前终止为 `TERMINATED`。
- 商业金额：合同金额只作为商业金额，不拆税，不生成收入、开票、应收、凭证或成本结果。
- 原因和版本：项目关闭、项目取消、合同关闭、合同终止、合同取消均要求 `reason`，长度 1-200；项目原因缺失返回项目原因错误码，合同原因缺失返回合同原因错误码；项目和合同更新请求与创建请求拆分，更新请求显式包含 `version`，所有状态动作也必须携带 `version`。
- 销售订单关联：`projectId` 与 `contractId` 同空或同有；只允许草稿创建和更新时关联或解除；项目必须 `ACTIVE`，合同必须 `EFFECTIVE`，客户一致，合同属于项目；确认时重复校验。
- 销售订单筛选：`projectLinked` 与 `projectId` 或 `contractId` 同时出现时返回 `VALIDATION_ERROR`，避免含糊组合。
- 历史兼容：既有销售订单两字段为空时可查询、编辑草稿、确认、出库和追溯，不被强制补项目归属。
- 审计：系统审计必须覆盖项目写入、合同写入、状态动作、草稿变更、销售订单关联或解除；至少可验证 `action`、目标、操作人、时间、`reason` 或变更字段摘要进入 `targetSummary`；不要求结构化 `before/after`。
- 销售订单关联审计：销售订单关联或解除写目标为 `SALES_PROJECT` 的审计记录，摘要包含订单号及旧/新关联，使项目 `operations` 可查询。
- 业务期间：020 项目和合同属于商业主数据或管理事实，不使用 `BusinessPeriodGuard`；测试不得期待按业务期间锁定拒绝项目或合同写入。
- 并发：并发创建主合同、并发激活项目、并发关联销售订单、并发关闭项目必须保持唯一约束、状态一致和事务回滚。
- 事务回滚：状态动作、合同激活、销售订单关联失败时不得产生部分写入、半条审计或错误状态。

### 接口集成

接口集成应覆盖后端契约、错误码、权限响应和跨模块影响。

- 项目接口：列表、详情、创建、更新、激活、关闭、取消、操作记录查询。
- 合同接口：创建主合同、创建补充合同、更新草稿、取消草稿、激活、关闭、终止、合同明细查询。
- 销售订单接口：草稿订单关联项目合同、解除关联、确认前重复校验、不同客户拒绝、非草稿拒绝、历史无项目订单继续可用。
- 关联摘要接口：项目详情中的关联销售订单摘要受 `sales:order:view` 权限约束，无权限时 `salesOrderCount` 和订单摘要返回 `null` 或空，`salesOrderSummaryRestricted=true`，不得显示为 0，不得泄露单号、金额、客户、状态、日期或路由参数等明细。
- 合同受限接口：项目查看与合同查看独立；无 `sales:contract:view` 时，项目列表和详情的合同字段返回 `null` 或空，`contractSummaryRestricted=true`，不得泄露合同编号、金额、状态或数量；有合同权限时项目列表返回 `supplementContractCount`，受限时为 `null`。
- 候选接口：销售订单项目合同选择使用专用最小候选接口，权限由 `sales:order:create` 或 `sales:order:update` 控制，不要求完整项目或合同查看权限；项目负责人使用专用最小用户候选接口，权限由 `sales:project:create` 或 `sales:project:update` 控制，不要求系统用户列表查看权限。
- 权限接口：未登录、无权限、只读权限、仅销售订单权限、仅项目权限、仅项目无合同权限、仅销售订单维护候选权限分别返回受控结果。
- 异常接口：编号不可编辑、合同金额非法、目标金额非法、补充合同缺主合同、补充合同在项目非 `ACTIVE` 或主合同非 `EFFECTIVE` 时创建或激活失败、合同不属于项目、项目状态不允许、合同状态不允许、客户不一致、原因长度不合法、版本缺失或冲突、筛选参数互斥。
- 回归接口：采购、生产、库存、质量、追踪和经营报表既有查询接口不得因为新增项目字段而失败。

### 前端自动化

前端自动化应覆盖页面状态、表单校验和权限表现。

- 菜单和路由入口：有权限用户在销售菜单下看到“销售项目”；无权限用户不可见或不可访问；`/sales/projects`、`/sales/projects/create`、`/sales/projects/:id`、`/sales/projects/:id/edit` 可按权限访问；`/sales` 根路径进入当前用户第一个可见销售子页面。
- 项目列表：分页、筛选、状态标签、空态、加载态、错误态、权限差异；有合同权限时展示 `supplementContractCount`，无合同时受限为空且显示受限状态。
- 项目表单：客户、负责人、计划周期、目标收入、目标成本、必填和格式校验；项目编号只读。
- 项目详情：基础信息、合同区域、关联订单区域、操作记录区域正确展示。
- 合同抽屉：主合同、补充合同、草稿编辑、激活、关闭、终止、取消、原因字段和错误提示；终态操作使用原因确认弹窗，含 1-200 字文本域、错误区域、确认和取消按钮。
- 商业字段保护：合同生效后关键商业字段在页面不可编辑，接口错误能被明确展示。
- 销售订单关联：草稿销售订单创建或编辑时项目合同联动选择；项目未激活、合同未生效、客户不一致时不可选或提交失败有明确提示。
- 候选选择：销售订单项目合同选择走最小候选体验，只有具备销售订单创建或更新权限时可用；项目负责人选择走最小用户候选体验，只有具备项目创建或更新权限时可用。
- 只读与无权限：只读用户可查看授权信息但不能写入；无销售订单查看权限时项目详情不展示关联订单明细且不得把数量显示为 0；无合同查看权限时项目详情不展示合同编号、金额、状态或数量。

### 浏览器验收

浏览器验收应使用真实服务和真实页面完成，不以接口通过替代。

- 主路径 1：创建项目草稿，创建主合同草稿，激活主合同使状态进入 `EFFECTIVE`，激活项目。
- 主路径 2：在已激活项目下创建正向补充合同和负向补充合同，验证补充合同引用 `EFFECTIVE` 主合同；激活补充合同时再次校验项目和主合同状态。
- 主路径 3：创建同客户销售订单草稿并关联项目合同，确认时再次校验通过。
- 异常路径：无主合同激活项目失败，主合同金额为 0 失败，补充合同金额为 0 失败，补充合同缺主合同失败，项目非 `ACTIVE` 或主合同非 `EFFECTIVE` 时创建或激活补充合同失败，客户不一致关联失败，非草稿订单变更关联失败，`projectLinked` 与项目或合同筛选同时使用失败。
- 状态路径：项目关闭、项目取消、合同关闭、合同终止、合同取消均必须填写 1-200 字原因；合同 `CLOSED` 表示正常履约结束，`TERMINATED` 表示提前终止；项目关闭仅在关联销售订单均为 `SHIPPED`、`CLOSED` 或 `CANCELLED` 时通过。
- 历史路径：历史无项目销售订单仍可列表、详情、确认、出库和来源追溯。
- 权限路径：无项目权限、只读项目权限、无合同查看权限、无销售订单查看权限、仅销售订单维护候选权限分别验证页面和接口表现。

## 权限与账号矩阵

测试账号按角色能力配置，不要求本文件固化真实账号密码。

| 账号角色 | 必需权限 | 验收重点 |
|---|---|---|
| 系统管理员 | 项目、合同、销售订单、审计全部权限 | 完整主路径、状态动作、审计核对 |
| 项目维护用户 | 项目和合同维护权限，销售订单查看权限 | 项目合同维护、关联订单摘要查看 |
| 项目只读用户 | 项目查看权限，合同查看权限 | 页面只读、按钮隐藏或禁用、接口写入拒绝 |
| 项目无合同查看用户 | 项目查看权限，无 `sales:contract:view` | 项目列表和详情合同字段为空或 `null`，`contractSummaryRestricted=true`，不得泄露合同编号、金额、状态或数量 |
| 销售订单维护用户 | `sales:order:create` 或 `sales:order:update`，无完整项目或合同查看权限 | 可使用项目合同最小候选接口，草稿订单关联和确认再校验 |
| 无销售订单查看用户 | 项目查看权限，无 `sales:order:view` | 项目详情不得泄露关联订单明细，`salesOrderCount` 与订单摘要为空或 `null`，`salesOrderSummaryRestricted=true`，不得显示为 0 |
| 项目负责人候选用户 | `sales:project:create` 或 `sales:project:update`，无系统用户列表查看权限 | 可使用负责人最小候选接口选择启用 `sys_user` |
| 无项目权限用户 | 无项目查看和维护权限 | 菜单、路由、接口均拒绝 |
| 停用用户 | 账号停用 | 登录或访问受保护资源失败 |

前端隐藏和禁用不能替代后端鉴权；所有写接口、状态动作和摘要接口都必须通过后端权限测试。

## 测试数据

基础数据建议至少包含：

- 客户 A：启用客户，用于项目、合同和销售订单一致性主路径。
- 客户 B：启用客户，用于客户不一致关联失败。
- 客户 C：停用客户，用于创建或关联失败边界。
- 用户 U1：有效项目负责人，来源于现有 `sys_user`。

- 销售订单 O1：客户 A，草稿，无项目合同。
- 销售订单 O2：客户 B，草稿，无项目合同。
- 销售订单 O3：客户 A，已确认，不允许变更项目合同。
- 历史销售订单 OH1：`projectId` 与 `contractId` 同时为空，用于兼容回归。

金额边界应覆盖：主合同金额 `0`、负数、合法正数、超精度小数、超大金额；补充合同金额 `0`、正数、负数、超精度小数；目标收入和目标成本 `0`、正数、负数。

原因边界应覆盖：空字符串、空白字符串、1 字、200 字、201 字；项目原因缺失使用项目原因错误码，合同原因缺失使用合同原因错误码。

版本边界应覆盖：更新请求缺少 `version`、状态动作缺少 `version`、过期 `version`、成功写入后版本递增。

日期边界应覆盖：签订日期为空或非法、履约开始晚于履约结束、项目计划开始晚于计划结束；合同日期允许超出项目计划周期，020 不做强制拒绝。

外部纸质合同号边界应覆盖：为空、重复、超长、关键词查询命中；不得因重复触发唯一约束错误。

## 000-019 受影响回归

020 的最小受影响回归应覆盖以下链路：

- 004 登录、菜单、角色权限、接口鉴权。
- 005 客户资料和用户引用仍可用于销售项目。
- 012 销售订单草稿、确认、关闭、取消、销售出库来源追溯。
- 013 应收来源仍可基于已过账销售出库生成，不要求项目归属。
- 014 经营报表销售来源和权限追溯不因项目字段为空失败。
- 015 销售退货和往来冲减仍可基于历史无项目销售出库执行。
- 016 业务期间锁定不误拦截 020 项目合同写入；既有受期间保护业务仍保持保护。
- 017 质量状态候选和质量确认不受项目合同字段影响。
- 018 库存可用量、预留、占用、采购在途口径不受项目合同字段影响。
- 019 批次、序列号、追踪分配、来源去向查询仍能处理历史无项目销售出库。

特别检查：020 只改造销售订单。采购订单和生产工单不得出现无业务入口的项目字段要求，也不得因缺少项目字段影响既有创建、查询和过账链路。

## 验证节奏

### 开发期定向验证

开发期只运行受影响范围验证，不把全量验证作为每个工作包门禁。

- 后端：项目、合同、销售订单关联、权限、审计、并发和迁移相关定向测试。
- 前端：销售项目页面、合同抽屉、销售订单关联区域和权限差异相关定向测试。
- 接口：项目合同接口、销售订单关联接口、关联摘要权限接口。
- 迁移：空数据库迁移和 019 存量库迁移的定向验证。
- 回归：销售订单到销售出库、应收、报表、退货、追踪的受影响路径。

### 集中审查

阶段功能完整后进入一轮集中审查。测试角色在集中审查中核对：

- 自动化覆盖是否覆盖本文列出的主路径、异常路径、权限、审计、并发、迁移和历史兼容。
- 浏览器验收脚本或手工记录是否覆盖关键业务路径。
- 视觉审计截图清单是否完整。
- 000-019 受影响回归是否有明确命令、结果和结论。
- 发现的阻断和严重缺陷是否已进入集中整改清单。

### 差异复审

集中整改后只复审修复差异和受影响路径：

- 阻断和严重缺陷的复测结果。
- 因修复改动影响的项目、合同、销售订单和回归链路。
- 新增或变更视觉截图。
- 不重新审查未变化的整个阶段。

### 唯一交付前全量验证

阶段功能完整、集中审查和差异复审通过后，进入唯一交付前全量验证窗口。

全量验证窗口应包含：

- 后端全量测试。
- 前端全量测试。
- 前端类型检查。
- 前端生产构建。
- 数据库迁移验证，包括空库和 019 存量库。
- 浏览器验收。
- 视觉审计和截图证据。
- 空白检查和最终验收记录。

全量验证窗口发现问题时，先用定向测试定位和修复，再验证受影响范围；最终仍需在同一窗口完成全量确认。

## 缺陷分级

### 阻断缺陷

- 系统无法启动、核心页面无法访问、020 关键接口不可用。
- 数据库迁移失败，或空库、019 存量库无法正常应用迁移。
- 权限绕过，包括无权限用户可查看或写入项目、合同、关联订单摘要。
- 项目编号或内部合同编号可被前端或接口篡改。
- 一个项目出现多个非取消主合同。
- 补充合同不引用主合同仍可创建或激活。
- 项目非 `ACTIVE` 或主合同非 `EFFECTIVE` 时仍可创建或激活补充合同。
- 主合同金额小于等于 0 被接受，补充合同金额为 0 被接受。
- 项目无已生效主合同仍可激活。
- 项目在合同非终态，或关联销售订单存在非 `SHIPPED`、`CLOSED`、`CANCELLED` 状态时仍可关闭。
- 项目存在有效合同或销售订单时仍可取消。
- 生效合同关键商业字段可被直接覆盖。
- 项目关闭、项目取消、合同关闭、合同终止、合同取消无 1-200 字原因仍可成功，或原因缺失错误码不稳定。
- 更新请求或状态动作缺少 `version` 仍可成功，或并发版本冲突被覆盖。
- 销售订单 `projectId` 与 `contractId` 只有一个有值仍可保存或确认。
- 销售订单关联非 `ACTIVE` 项目、非 `EFFECTIVE` 合同、不同客户合同或非所属项目合同仍可成功。
- 非草稿销售订单仍可变更项目合同关联。
- `projectLinked` 与 `projectId` 或 `contractId` 同时出现未返回 `VALIDATION_ERROR`。
- 无 `sales:contract:view` 仍可从项目列表或详情获得合同编号、金额、状态或数量。
- 无 `sales:order:view` 仍可从项目详情获得订单明细，或 `salesOrderCount` 被误返回为 0 造成权限受限与真实 0 条混淆。
- 销售订单项目合同最小候选接口绕过 `sales:order:create` 或 `sales:order:update`；项目负责人最小候选接口绕过 `sales:project:create` 或 `sales:project:update`。
- 历史无项目销售订单无法查询、确认、出库或追溯。
- 失败事务产生部分写入、错误状态、半条合同、半条关联或错误审计。
- 系统审计缺失，包括必需 `action`、目标、操作人、时间、`reason` 或变更字段摘要未写入 `targetSummary`；销售订单关联或解除未写目标为 `SALES_PROJECT` 的审计记录。
- 020 改动破坏 012-019 既有销售、库存、往来、报表、退货、质量、可用量或追踪核心链路。
- 视觉审计缺失，或视觉问题影响核心操作、关键数据识别、权限状态判断。

### 严重缺陷

- 金额精度、日期边界、目标收入或目标成本校验不一致但未直接造成核心数据污染。
- 并发操作返回结果不清晰或需要刷新后才能识别最终状态。
- 错误码和前端提示不一致，影响用户判断但未绕过规则。
- 项目详情关联订单摘要字段缺失，影响管理判断但未泄露越权数据。
- 项目操作记录页面摘要缺少非关键字段，但系统审计完整且不影响业务追溯。
- 有合同权限时 `supplementContractCount` 缺失或展示不稳定，但合同明细权限和核心状态动作不受影响。
- 390px 窄屏可操作但信息密度明显影响查阅效率。

### 一般缺陷

- 非核心文案不统一。
- 列表筛选、排序、默认列宽或状态颜色存在轻微体验问题。
- 成功提示、加载态、空态表达不够清晰但不影响业务操作。
- 非关键截图命名或记录格式不统一。

## 集中整改后的确定测试口径

- 补充合同仅允许在项目 `ACTIVE` 且主合同 `EFFECTIVE` 时创建；激活时再次校验，失败返回稳定业务错误。
- `ACTIVE` 项目允许更新负责人、计划日期、目标收入、目标成本和备注，必须写审计摘要；项目编号和客户锁定；`CLOSED`、`CANCELLED` 不可更新。
- 项目关闭、项目取消、合同关闭、合同终止、合同取消均要求 `reason`，长度 1-200；项目原因缺失使用项目原因错误码，合同原因缺失使用合同错误码。
- 外部纸质合同号可空、可重复，不建唯一约束，可按关键词查询。
- 项目关闭时销售订单终态固定为 `SHIPPED`、`CLOSED`、`CANCELLED`；其他状态均阻止关闭。
- 合同 `CLOSED` 表示正常履约结束；`TERMINATED` 表示提前终止；两者均为终态。
- 项目和合同操作日志基于现有 `sys_audit_log`，只要求 `action`、目标、操作人、时间、`reason` 或变更字段摘要写入 `targetSummary`；不要求结构化 `before/after`，不扩展审计表。
- 销售订单关联或解除写目标为 `SALES_PROJECT` 的审计记录，摘要包含订单号及旧/新关联，使项目 `operations` 可查询。
- 项目查看与合同查看独立；无 `sales:contract:view` 时，项目列表和详情合同字段返回 `null` 或空，并返回 `contractSummaryRestricted=true`，不得泄露编号、金额、状态或数量。
- 无 `sales:order:view` 时，`salesOrderCount` 和订单摘要返回 `null` 或空，`salesOrderSummaryRestricted=true`，前端不得显示为 0。
- 销售订单项目合同选择使用专用最小候选接口，权限由 `sales:order:create` 或 `sales:order:update` 控制，不要求完整项目或合同查看权限。
- 项目负责人使用专用最小用户候选接口，权限由 `sales:project:create` 或 `sales:project:update` 控制，不要求系统用户列表查看权限。
- 更新请求与创建请求拆分，项目和合同 `UpdatePayload` 必须显式包含 `version`；所有状态动作也携带 `version`。
- 合同激活动作统一使用 `activate` 命名：接口 `/activate`、权限 `sales:contract:activate`、审计动作 `SALES_PROJECT_CONTRACT_ACTIVATE`；合同状态仍为 `EFFECTIVE`。
- `projectLinked` 与 `projectId` 或 `contractId` 同时出现时返回 `VALIDATION_ERROR`。
- 合同摘要和详情字段明确区分；列表和项目详情摘要包含状态动作所需 `version`，详情包含完整字段。
- 项目列表在有合同权限时增加 `supplementContractCount`；受限时为 `null`。
- 前端固定路由为 `/sales/projects`、`/sales/projects/create`、`/sales/projects/:id`、`/sales/projects/:id/edit`；`/sales` 根路径进入当前用户第一个可见销售子页面。
- 项目和合同终态操作使用原因确认弹窗，包含 1-200 字文本域、错误区域、确认和取消；390px 下按钮可达。
- 合同日期允许超出项目计划周期，020 不做强制校验。
