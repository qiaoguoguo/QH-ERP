# 测试数据基线

## 文档定位

本文规定自动化测试夹具的通用数据原则。用于本地浏览器验收和新代理接管的 `000` 至 `023` 全量电气制造业演示数据，以 [电气制造业全量演示数据基线与重建验收单](../data/electrical-manufacturing-demo-data.md) 为唯一权威入口；不得用单元测试、集成测试中的直插 SQL 夹具替代真实验收数据生成器。

自动化测试数据与验收演示数据必须隔离：测试夹具可为单个用例构造最小事实并随测试容器销毁，验收演示数据必须通过受控重建入口、真实业务 API/服务和独立验证器生成及验收。

## 基础数据

- 用户与角色。
- 客户与供应商。
- 仓库。
- 单位。
- 物料分类。

## 生产数据

- 原材料。
- 半成品。
- 成品。
- BOM 版本。
- 初始库存。
- 生产工单。
- 领料记录。
- 报工记录。
- 完工入库记录。
- 成本归集记录。

## 数据原则

- 测试数据应能覆盖正常流程和异常流程。
- 核心数据应可重复初始化。
- 变更测试数据必须记录原因。
- 初始化数据必须包含可登录管理员账号、基础角色和基础权限。
- 权限验收至少包含管理员账号、普通业务账号、无权限账号和停用账号。
- 生产主链路数据必须能覆盖物料、BOM、库存、工单、领料、报工、完工入库和成本记录。
- 全量验收演示数据还必须覆盖 `000` 至 `023` 已实现的权限、编码、单位换算、BOM/ECO、采购、质量、批次序列追踪、预留、项目合同、销售、财务往来、反向单据、固定审批、消息、附件、打印、导入导出、公共平均池、项目成本层、价值流水、调拨、权属转换、盘点和估值调整。
- 演示数据不得写入 Flyway，也不得通过 SQL 直接伪造库存余额、价值流水、审批终态、文档任务终态或附件关系。
- 验收库重建前必须备份 PostgreSQL 与 MinIO；重建后必须运行独立验证器，并记录迁移版本、数据规则、对象存储、服务健康和桌面浏览器结论。
- 设计开发仍考虑响应式；演示数据验收与浏览器视觉检验只覆盖桌面端，不保存截图，不测试移动端。
