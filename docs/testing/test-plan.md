# 测试计划

## 目标

保证每个阶段成果符合任务文档、需求说明和阶段验收标准。

## 测试范围

- 功能测试。
- 接口测试。
- 权限测试。
- 异常流程测试。
- 回归测试。
- 部署验收测试。
- 本阶段相关桌面页面的浏览器功能与视觉检验。

## 阶段准入

- 任务文档已明确目标和验收标准。
- 需求、接口、页面或数据规则已可测试。
- 开发自测结果已记录。
- 测试数据、初始化数据和验收账号已明确。
- 本地启动或部署说明已明确。
- 五个 Codex 后台固定角色已完成唯一一轮目标讨论，唯一阶段说明已冻结。

## 阶段准出

- 阻断问题已修复。
- 核心流程已验证。
- 缺陷状态已明确。
- 本地可部署并可浏览器访问。
- 接口、权限、异常流程和必要回归已通过。
- 唯一交付前全量验证窗口已完成全量自动化、迁移、环境、真实桌面浏览器功能和视觉检验。
- 视觉检验范围、发现、处理和结论已写入阶段说明或交接；不截图、不建立视觉目录、不测试或验收移动端。
- 若存在一般缺陷，必须说明影响范围、处理结论和是否影响验收。

## 风险关注

库存、工单、领料、完工入库、成本归集相关缺陷应默认按高风险处理。

权限绕过、关键接口不可用、数据库迁移失败、关键数据错误、浏览器验收路径不可执行和桌面视觉检验缺失，也默认按高风险处理。

开发期间只执行受影响测试、定向接口测试和必要编译检查；全量后端、全量前端、类型检查、生产构建、迁移与浏览器验收只在阶段功能完整、集中审查和差异复审通过后执行一次。完整流程见 `docs/ops/stage-collaboration-delivery-process.md`。
