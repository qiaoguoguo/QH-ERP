## 浏览器复验检查动作

| 检查项 | 操作 | 通过标准 |
| --- | --- | --- |
| 服务可访问 | 访问前端地址和后端健康检查地址 | 前端页面可打开；后端健康检查返回 `UP`。 |
| 登录与会话 | 使用管理员、只读账号、无权限账号分别登录 | 菜单、按钮和路由权限差异符合账号权限。 |
| 主内容滚动 | 在长列表或长表单页面滚动 `.app-main` | `window.scrollY` 保持不变，主内容滚动条变化，侧栏不覆盖内容。 |
| 宽表横向滚动 | 在库存流水、订单列表、报表列表横向滚动表格 | 右侧列、操作列和滚动条可访问，表格文本不互相覆盖。 |
| 日期控件 | 打开列表筛选日期和表单业务日期控件 | 日期弹层显示完整，层级高于卡片和表格，不被侧栏或表格截断。 |
| 确认弹窗 | 触发过账、取消、停用、权限取消等危险动作 | 统一弹窗遮罩、标题、正文、确定和取消按钮可见。 |
| 抽屉 | 打开物料详情或报表追溯抽屉 | 抽屉不被侧栏或顶部栏遮挡，关闭按钮和内容可见。 |
| 空态 | 使用不存在关键词查询 | 空态文案和图标可见，分页和搜索栏布局稳定。 |
| 异常态 | 触发受控业务错误 | 错误提示明确，表单数据保留，按钮恢复可操作。 |
| 移动端 | 使用 390x844 或 375x812 视口访问关键页面 | 侧栏/顶部堆叠后业务内容可触达，表格横向滚动可用，按钮不溢出。 |

## 最终验收输出要求

| 输出物 | 要求 |
| --- | --- |
| 最终截图 | 阻断级 B-01 至 B-25 必须全部存在。建议补充 S 类按实际风险选择，但涉及已发现问题或用户操作手册必需场景时不得省略。 |
| 视觉分析记录 | `notes.md` 必须记录每张最终截图的文件名、视口、页面路径、账号/权限状态、发现问题、处理结果和结论。 |
| 旧截图处理 | 旧 3 张截图可保留为历史过程证据，但最终结论必须引用重截后的当前样式截图。 |
| 缺口记录 | 如果数据前置条件暂时不满足，例如往来冲减无 2 条候选，必须记录为验收数据缺口，并补充数据后重截，不得用空态替代正常态。 |
| 阻断处理 | 发现核心操作不可见、权限提示错误、页面错位、弹窗或抽屉被遮挡、宽表不可横向访问、移动端内容不可触达时，必须先修复并重新截图。 |
