# 页面治理规范

## 适用范围

本规范适用于 QH ERP 后续所有业务页面、管理页面、报表页面和追溯页面。新增页面、重构页面和阶段验收都必须遵守本规范；如业务确需例外，应在任务文档中说明原因、影响范围和验收方式。

本系统是制造业单公司使用的 ERP 工具型界面，不是营销页、门户页或展示型官网。页面设计应服务高频录入、查询、核对、追溯和异常处理，优先保证信息密度、操作可达、业务状态清晰和浏览器验收可验证。

## 整体布局

- 应使用稳定的 ERP 工作台骨架：左侧导航、顶部账号区、主内容区。
- 左侧侧栏使用深色背景，保持固定宽度，不随主内容滚动。
- 左侧侧栏内部菜单必须独立滚动，菜单较多时不得挤压主内容区。
- 主内容区必须独立滚动，页面滚动不应改变侧栏和顶部框架位置。
- 主内容区应使用工作型信息密度，避免大面积装饰留白、营销式首屏、宣传卡片和无业务价值的视觉元素。
- 页面主体宽表、弹窗、抽屉、表单和空态都必须在主内容区滚动模型下可访问。
- 设计和开发仍需考虑移动端、窄屏、触屏和响应式适配。移动端允许侧栏和顶部栏纵向堆叠，但主业务容器不得被固定高度、外层 `overflow: hidden` 或弹层遮罩截断；表格、弹窗、抽屉和分页在窄屏下应保持可触达。
- 页面外层必须控制横向溢出，宽表横向滚动应发生在表格容器内，不得让 `body` 或整个工作台产生不可控横向滚动。

## 导航

- 系统必须展示清晰的 logo 和产品名称，logo 不得只依赖浏览器标签页或登录页。
- 一级主菜单必须有图标，图标应匹配业务语义，例如库存、采购、销售、生产、成本、财务、报表。
- 侧栏菜单只允许同时展开一个一级主菜单，避免菜单过长造成定位困难。
- 菜单激活态必须清晰，当前模块和当前页面应能被识别。
- 跨模块详情跳转必须保留 `returnTo` 或等价返回上下文，返回后应保持来源模块菜单高亮，避免从报表、库存流水、财务追溯跳转后菜单乱跳。
- `returnTo` 必须覆盖报表追溯、库存流水、财务来源、退货反冲追溯和详情返回等跨模块链路；跳转时不得丢失来源查询路径，返回后不得错误高亮目标模块菜单。
- 应用启动和刷新后必须完成登录态补水，再判断菜单、按钮和受保护页面状态；不得在已登录会话下短暂显示“未登录”、错误菜单或错误无权限状态作为最终页面。
- 无权限菜单必须隐藏；只有部分子页面权限时，只显示允许访问的子菜单。

## 页面标题区

- 每个业务页面必须有标题和一句说明，说明应交代该页面的业务对象、用途或当前口径。
- 页面主操作按钮放在标题区右侧；窄屏设计中可换行到标题下方，但必须保持可见。
- 主操作只放页面级动作，例如新增、返回列表、保存权限；行级动作不得放到标题区。
- 标题区不得承载搜索条件、分页、批量状态说明或复杂业务表单。
- 标题、说明和主操作之间必须留有稳定间距，不能因按钮数量变化造成遮挡或重叠。

## 搜索栏

- 列表筛选必须使用统一的标签置顶网格样式。
- 文本框、下拉框、日期选择器在搜索栏中必须同高度、同顶边、同底边。
- 日期范围应使用两个明确的日期选择器，标签和占位文案必须能表达起始和截止含义。
- 搜索栏按钮组放在网格最后一项，和输入控件底边对齐。
- 禁止再新增第二套横向 label 搜索栏；旧页面改造时应迁移到标签置顶网格样式。
- 搜索栏中的控件宽度应由网格和统一样式控制，不应在页面内散落固定宽度。
- 搜索栏日期组件清空时必须回写为空字符串 `''`，不得把 `null`、`undefined`、`Invalid Date` 或组件内部对象提交给筛选状态或接口参数。
- 搜索栏按钮组应作为查询网格的一项参与布局；桌面端与输入控件底边对齐，移动端设计中可换行但查询和重置按钮应保持可触达且不挤压字段。
- 重置必须恢复默认筛选条件，并将分页回到第一页。
- 无分页候选选择池不得因为主列表默认分页规范而缩小业务候选范围。表单内用于选择来源、物料、仓库、客户、供应商、BOM 等业务候选数据时，应按业务需要加载足够候选、支持搜索或给出明确的分页选择机制，不能简单套用主列表 `10` 条分页导致候选缺失。

## 表格

- 所有业务表格必须放入横向滚动容器，宽表不得撑破页面或把右侧操作列挤出屏幕。
- 表格列应使用稳定的最小宽度；数字列右对齐并使用等宽数字表现。
- 行操作列必须固定或在横向滚动后可访问，不能被弹窗边界、表格裁剪或主内容滚动遮挡。
- 表格内长字段必须使用标准 tooltip 或标准省略样式，tooltip 最大宽度必须受视口限制，不能遮挡整行文本或关键操作按钮。
- 默认分页显示 `10` 条，所有主列表必须提供每页条数选择，选项统一为 `10`、`20`、`50`、`100`。
- 分页放在表格下方，与表格属于同一信息组；窄屏设计中分页可换行，但总数、每页条数、上一页、页码、下一页应保持可操作。
- 分页状态必须与查询条件联动；查询和重置后回到第一页，每页条数变化后也必须回到第一页并重新加载当前条件。
- 横向滚动条应在表格区域内可见或可操作；宽表滚动到右侧后，关键金额列、状态列和操作列不得被卡片边界裁剪。
- 空表格必须展示空态，不得只显示空白区域。
- 加载、错误、无权限和只读状态必须能在表格区域或页面提示中被识别。

## 表单

- 所有日期录入必须使用日期选择器，禁止使用普通文本输入框替代日期控件。
- 日期选择器必须统一格式为年-月-日，业务值格式应稳定，不得混用不同显示和提交格式。
- 表单日期组件清空时必须统一落为业务可识别的空字符串或表单默认空值；保存前不得把组件清空事件产生的 `null` 直接透传为非法业务日期。
- 表单字段应按业务分组，常规表单使用标签置顶或项目内已有一致样式；复杂单据表单允许使用固定 label 宽度，但同一页面内必须一致。
- 必填字段必须有明确提示，提交失败时错误信息应指向缺失或不合法的业务字段。
- 错误提示必须出现在用户能看到的位置，不能只写入控制台或仅依赖后端响应。
- 只读态和禁用态必须可辨认，且不能误导用户以为字段可编辑。
- 编辑已过账、已取消、已关闭等不可编辑记录时，应明确展示不可编辑原因，并提供返回详情或返回列表动作。
- 表单保存按钮在提交中必须显示加载或禁用状态，防止重复提交。

## 候选池和选择状态

- 来源候选池、物料候选池、仓库候选池、客户/供应商候选池、BOM 候选池和往来冲减候选池属于业务选择场景，不等同于主列表浏览分页。
- 候选池必须展示当前选中项，选中态应有明确视觉反馈；切换候选后，摘要区、目标单号、可用数量、可冲金额、来源信息和提交校验必须同步更新。
- 候选为空时必须展示明确空态和筛选提示，不得用无权限提示、加载中或空白表格替代。
- 候选数据不足以完成验收时，应记录为数据准备缺口；不得用单测或空态替代多候选切换的真实浏览器检验。

## 弹窗和抽屉

- 确认弹窗必须使用统一封装，按钮文案、危险语义、遮罩交互和关闭行为保持一致。
- 删除、停用、取消、过账、关闭、提交等高风险操作必须有确认。
- 高风险确认不得使用浏览器原生确认框；必须使用系统统一确认弹窗，并保留危险、警告、普通确认的语义差异。
- 确认弹窗、业务弹窗、日期弹层、tooltip 和抽屉必须有稳定层级，不能互相遮挡标题、表单字段、保存按钮、取消按钮或行操作按钮。
- 追溯信息应使用抽屉或弹层承载，不得在列表下方追加很长的追溯表格，避免页面滚动失控。
- 抽屉内宽表也必须有横向滚动容器，字段不得挤压到不可读。
- BOM 明细、销售订单行、采购订单行、库存单据行等宽表弹窗必须保证操作列可见或可通过横向滚动访问。
- 弹窗最大宽度和最大高度必须受视口限制；窄屏设计中不能超出屏幕，也不能遮挡底部保存、取消按钮。
- 弹窗内如果存在表格、表单和错误提示，必须保证弹窗 body 可滚动，页脚操作固定在可访问位置。

## 链接和操作按钮

- 可点击链接必须有明确视觉提示，不能只靠文字颜色的轻微差异表达可点击。
- 表格内操作按钮必须明显，常用行操作应保持一致的按钮样式、间距和悬浮状态。
- 表格内“详情”“查看”“追溯”等链接型操作必须呈现为可识别的链接按钮或操作按钮，不得与普通文本混淆。
- 危险操作、成功操作、普通操作必须保留语义差异；全局样式不得抹平 `danger`、`success`、`warning` 等状态表达。
- 页面主按钮用于关键正向动作；次要按钮用于返回、重置、取消等动作。
- 表格行操作过多时，应优先保证详情、编辑、过账、取消、关闭等核心动作可读可点，不得出现按钮挤压、换行遮挡或固定列宽不足。
- 链接跳转到详情页时应保留返回上下文，尤其是从报表追溯、库存流水、财务来源追溯进入业务详情。

## 经营报表

- 经营报表当前不是完整 BI，但必须满足系统级页面规范，不能因为是报表页就降低视觉和交互要求。
- 报表筛选栏必须使用统一筛选组件，日期、文本、下拉条件高度和对齐方式一致。
- 报表筛选栏不得保留独立的旧横向 label 样式；日期条件清空值、查询按钮对齐和移动端换行设计必须与主列表搜索栏一致。
- 指标区必须使用统一指标样式，指标名称、数值、辅助说明和空值表现保持一致。
- 报表表格必须使用横向滚动容器，宽列和金额列必须可读。
- 报表追溯必须使用抽屉或弹层，不得直接在报表列表下方堆叠长追溯表。
- 报表中来源受限、无权限、空数据、错误数据和异常数据必须有明确状态表达。
- 报表页面不得残留独立旧色值、旧圆角、旧间距或第二套筛选样式；应使用统一设计 token 和全局页面规范。

## 响应式设计与桌面端视觉检验

- 设计和开发必须继续考虑移动端、窄屏、触屏和响应式适配，不得因为测试与验收不覆盖而主动移除或破坏响应式实现。
- 每个阶段只检验本阶段实际新增或修改的相关页面、组件和受影响业务状态，不扩展检查未变化页面。
- 自动化测试、人工测试、集中审查和阶段验收只覆盖桌面端浏览器，不测试或验收移动端、窄屏、触屏及响应式兼容性。
- 视觉检验必须基于当前代码的真实运行页面完成，但默认不采集、不保存截图，不建立视觉截图目录，也不把截图作为验收证据或交付成果。
- 宽表应在浏览器中实际检查横向滚动前后的关键列和操作列。
- 弹窗和抽屉应在浏览器中实际检查打开状态、滚动状态和底部操作可访问状态。
- 与本阶段改动有关的空态、异常态、权限态、只读态和禁用态必须完成真实浏览器检验。
- 验收记录只需说明检查范围、桌面端布局、信息密度、关键操作可见性、文案溢出、控件重叠、业务状态识别、发现问题、处理结果和最终结论。

## 禁止项

- 禁止把 ERP 页面设计成营销页、宣传页或装饰型首屏。
- 禁止左侧菜单跟随主页面滚动。
- 禁止主内容区宽表撑破页面，导致右侧操作不可访问。
- 禁止设计或实现中让移动端主内容、弹窗或抽屉被外层容器截断，导致保存、取消、分页、行操作或错误提示不可触达；该项属于设计开发约束，不纳入后续移动端测试和验收。
- 禁止没有横向滚动容器的业务表格。
- 禁止新增横向 label 搜索栏或与统一查询栏不一致的筛选样式。
- 禁止使用普通文本框录入日期。
- 禁止日期组件清空后向筛选状态或接口提交 `null`、`undefined`、`Invalid Date` 或非 `YYYY-MM-DD` 字符串。
- 禁止表格长字段 tooltip 遮挡整行、遮挡操作按钮或超出视口。
- 禁止为了统一按钮样式而抹掉危险、成功、警告等语义状态。
- 禁止追溯信息在列表下方无限展开成长表。
- 禁止用主列表默认 10 条分页限制业务候选选择池。
- 禁止未完成真实桌面端浏览器视觉检验就声明页面视觉验收通过。
- 禁止仅以单元测试或源码字符串检查替代浏览器视觉验收。

## 验收检查清单

### 页面骨架

- [ ] 左侧深色侧栏固定，主内容滚动时侧栏不移动。
- [ ] 侧栏菜单可独立滚动，菜单数量增加后仍可访问底部菜单。
- [ ] 主内容区独立滚动，顶部账号区和侧栏不被业务内容挤压。
- [ ] 页面没有营销式大留白、装饰卡片或无业务价值内容。

### 导航和返回

- [ ] logo、系统名称和一级菜单图标可见。
- [ ] 只允许展开一个一级主菜单。
- [ ] 当前菜单激活态清晰。
- [ ] 跨模块详情跳转带返回上下文，返回后菜单不乱跳。
- [ ] 登录态补水完成后再展示最终菜单、账号名和权限状态，刷新后不出现错误“未登录”最终态。
- [ ] 无权限菜单和按钮按权限隐藏或禁用。

### 标题和操作

- [ ] 页面标题和说明完整。
- [ ] 主操作按钮位于标题区右侧并保持可见。
- [ ] 标题、说明、操作按钮没有重叠或遮挡。

### 搜索栏

- [ ] 搜索栏使用标签置顶网格布局。
- [ ] 日期、文本、下拉控件高度一致、顶边对齐。
- [ ] 查询和重置按钮与输入控件底边对齐。
- [ ] 日期筛选清空后筛选状态为空字符串，接口参数不出现非法日期值。
- [ ] 重置后筛选条件和分页状态正确。
- [ ] 候选选择池未被主列表分页规范错误限制。
- [ ] 候选池选中态清晰，切换候选后摘要区和提交校验同步更新。

### 表格和分页

- [ ] 表格位于横向滚动容器内。
- [ ] 宽表横向滚动后仍能访问操作列。
- [ ] 长字段省略和 tooltip 表现正常。
- [ ] 默认每页 10 条，并提供 `10`、`20`、`50`、`100` 选择。
- [ ] 查询、重置和每页条数变化后分页回到第一页。
- [ ] 空态、加载态、错误态显示明确。

### 表单

- [ ] 日期字段全部使用日期选择器。
- [ ] 必填、错误、只读、禁用和提交中状态可识别。
- [ ] 保存失败时错误提示可见，表单不丢失用户输入。
- [ ] 不可编辑记录说明原因，并提供可返回路径。

### 弹窗和抽屉

- [ ] 高风险操作使用统一确认弹窗。
- [ ] 未使用浏览器原生确认框，确认弹窗语义和按钮文案统一。
- [ ] 弹窗最大宽度和高度受视口限制。
- [ ] 弹窗内宽表可横向滚动，操作列可访问。
- [ ] 追溯信息使用抽屉或弹层，并验证桌面端可读。

### 报表

- [ ] 报表筛选、指标、表格和追溯抽屉使用统一规范。
- [ ] 报表无权限、来源受限、空态、异常态有明确表现。
- [ ] 报表页没有旧色值、旧圆角、旧间距或第二套筛选样式。

### 视觉检验结论

- [ ] 响应式适配已在设计和开发中考虑，但未作为本次测试或验收覆盖项。
