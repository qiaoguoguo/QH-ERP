# UI 与交互基线

## 设计目标

ERP 是高频工作系统，界面应清晰、稳定、信息密度合理，优先服务业务效率和数据判断。

## 信息架构原则

- 导航应围绕业务模块和生产主链路组织。
- 高频功能入口应清晰可达。
- 跨模块跳转应保留业务上下文。

## 页面模式

- 列表页：支持筛选、排序、分页、批量操作和状态识别。
- 表单页：支持分组、必填标识、校验提示和保存反馈。
- 详情页：展示主信息、明细、流转记录和操作记录。
- 弹窗：只承载轻量确认或局部编辑，不承载复杂流程。

## 账号权限与生产模块共用模式

- 工作台：顶部区域、侧边导航、内容区和用户操作入口保持一致。
- 列表管理：查询区、工具栏、表格、行操作、分页和空状态保持一致。
- 表单录入：字段分组、必填、校验、保存、取消和错误反馈保持一致。
- 详情查看：基础信息、状态信息、关联明细和操作记录保持一致。
- 状态变更：启用、停用、提交、作废、入库、关闭等高风险操作必须有确认。
- 权限表现：菜单不可见、按钮不可见或禁用、字段只读、无权限提示必须有统一规则。

## 视觉基线

后续应统一颜色、字体、间距、按钮、表单、表格、状态标签、图标和提示信息。生产、库存、成本等关键状态需要有一致的视觉表达。

## 设计验收

前端实现后应检查视觉一致性、交互完整性、异常状态、响应式适配和业务语义准确性。

## 浏览器验收关注点

- 页面不应出现营销化大留白或装饰化内容，应保持 ERP 工作台信息密度。
- 常见桌面分辨率下，表格列、操作按钮和状态标签必须可读可操作。
- 加载、空数据、成功、失败、警告、无权限和不可操作状态必须完整。
- 文案不得遮挡、溢出或与操作按钮重叠。
- 主操作、返回路径和危险操作必须清晰。
